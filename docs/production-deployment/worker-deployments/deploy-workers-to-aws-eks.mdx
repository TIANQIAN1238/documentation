在基于 [库伯内斯](https://kubernetes.io) 的部署中运行的 Temporal Worker 可提供规模、弹性和灵活的资源管理。
Amazon EKS（弹性 Kubernetes 服务）提供了运行 Temporal Worker 的最受欢迎的选择之一。
它与 AWS 服务顺利集成，并支持自动扩展和容错功能，这是许多 Temporal 用户的关键功能。

按照本指南在 EKS 中部署和管理 Temporal Worker。
本指南将引导您完成编写 Temporal Worker 代码、将 Worker 容器化并发布到 Amazon Elastic Container Registry (ECR)，以及将 Worker 部署到 Amazon EKS。
本页的示例使用 Temporal 的 Python SDK 和 Temporal Cloud。

对于使用 [Worker 版本控制](/production-deployment/worker-deployments/worker-versioning) 的生产 Kubernetes 部署，
使用[Temporal Worker 控制器](/production-deployment/worker-deployments/kubernetes-controller)这样部署
推出和自动缩放保持附加到每个 Worker 部署版本。

:::tip

本指南适用于为 Temporal OSS 和 Temporal Cloud 运行 Worker。
但是，使用 Temporal OSS 时存在一些差异。
例如，您需要使用 mTLS 证书而不是 API 密钥。
您必须修改 Kubernetes 部署才能处理和挂载适合您的用例的 TLS 证书。
具体细节将根据您的部署而有所不同。

:::

<a id="before-you-begin"></a>
## 开始之前 {#before-you-begin}

要开始将 Worker 部署到 EKS，您需要：

- 您的 Temporal Cloud 帐户，包括：
  - 使用 [API密钥认证](/cloud/api-keys#namespace-authentication) 的 Namespace
  - [服务帐号](/cloud/api-keys#generate-an-api-key-for-a-service-account) 的 API 密钥
- Amazon Web Services (AWS) 帐户，包括：
  - 您的 AWS 账户中部署的 EKS 集群
- [`aws` 命令行](https://aws.amazon.com/cli/) 的安装版本
- [`docker`](https://www.docker.com/get-started/)
- [`kubectl`](https://kubernetes.io/docs/reference/kubectl/) 命令行工具，使用您部署的 EKS 集群进行配置

<a id="write-your-worker-code"></a>
## 编写您的 Worker 代码 {#write-your-worker-code}

在 Temporal 应用程序中，业务逻辑位于主 Workflow 代码中。
您的 Worker 代码单独运行，并负责执行您的 Workflow 和活动。
确保将 Worker 配置为使用环境变量，以便您可以动态地将 Worker 路由到不同的 Temporal 实例、Namespace 和 Task Queue：

```python
TEMPORAL_ADDRESS = os.environ.get("TEMPORAL_ADDRESS", "localhost:7233")
TEMPORAL_NAMESPACE = os.environ.get("TEMPORAL_NAMESPACE", "default")
TEMPORAL_TASK_QUEUE = os.environ.get("TEMPORAL_TASK_QUEUE", "test-task-queue")
TEMPORAL_API_KEY = os.environ.get("TEMPORAL_API_KEY", "")
```

配置完成后，实例化您的 Temporal CLIent：

```
client = await Client.connect(
    TEMPORAL_ADDRESS,
    namespace=TEMPORAL_NAMESPACE,
    rpc_metadata={"temporal-namespace": TEMPORAL_NAMESPACE},
    api_key=TEMPORAL_API_KEY,
    tls=True
)
```

下面是一个完整的 Python 样板，展示了如何在开始执行 Worker 之前实例化客户端并将其传递给 Worker：

```python
import asyncio
import os

from temporalio.worker import Worker
from temporalio.client import Client

from workflows import your_workflow
from activities import your_first_activity, your_second_activity, your_third_activity

TEMPORAL_ADDRESS = os.environ.get("TEMPORAL_ADDRESS", "localhost:7233")
TEMPORAL_NAMESPACE = os.environ.get("TEMPORAL_NAMESPACE", "default")
TEMPORAL_TASK_QUEUE = os.environ.get("TEMPORAL_TASK_QUEUE", "test-task-queue")
TEMPORAL_API_KEY = os.environ.get("TEMPORAL_API_KEY", "your-api-key")

async def main():
  client = await Client.connect(
    TEMPORAL_ADDRESS,
    namespace=TEMPORAL_NAMESPACE,
    rpc_metadata={"temporal-namespace": TEMPORAL_NAMESPACE},
    api_key=TEMPORAL_API_KEY,
    tls=True
  )

  print("Initializing worker...")

  # Run the worker
  worker = Worker(
    client,
    task_queue=TEMPORAL_TASK_QUEUE,
    workflows=[your_workflow],
    activities=[
      your_first_activity,
      your_second_activity,
      your_third_activity
    ]
  )

  print("Starting worker... Waiting for tasks.")
  await worker.run()

if __name__ == "__main__":
  asyncio.run(main())
```

<a id="containerize-the-worker-for-kubernetes"></a>
## 为 Kubernetes 容器化 Worker {#containerize-the-worker-for-kubernetes}

您需要对 Worker 代码进行容器化才能使用 Kubernetes 运行它。
以下是一个示例 Python Dockerfile，其中安装了 Temporal Python SDK：

```docker
# Use Python 3.11 slim image as base
FROM python:3.11-slim

# Set working directory
WORKDIR /app

# Install system dependencies
RUN apt-get update && apt-get install -y \
    gcc \
    && rm -rf /var/lib/apt/lists/*

# Install the Temporal Python SDK dependency
RUN pip install --no-cache-dir temporalio

# Copy application code
COPY . .

# Set Python to run in unbuffered mode
ENV PYTHONUNBUFFERED=1

# Run the worker
CMD ["python", "worker.py"]
```

构建 Docker 映像并针对 `linux/amd64` 架构：

```bash
docker buildx build \
    --platform linux/amd64 \
    -t your-app .
```

<a id="publish-the-worker-image-to-amazon-ecr"></a>
## 将 Worker 映像发布到 Amazon ECR {#publish-the-worker-image-to-amazon-ecr}

构建 Docker 映像后，您就可以将其发布到 Amazon ECR。
确保您已通过 AWS 进行身份验证，并且已设置 `AWS_REGION` 和 `AWS_ACCOUNT_ID` 环境变量：

```bash
export AWS_ACCOUNT_ID=<your_aws_account_id>
export AWS_REGION=<your_aws_region>
```

创建 ECR 存储库并使用 Docker 容器客户端验证 ECR：

```bash
aws ecr create-repository \
    --repository-name your-app
aws ecr get-login-password --region $AWS_REGION | \
    docker login --username AWS --password-stdin \
            $AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com
```

使用 ECR 对 Docker 进行身份验证后，标记您的容器并发布它：

```bash
docker tag your-app $AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com/your-app:latest
docker push $AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com/your-app:latest
```

<a id="deploy-the-workers-to-eks"></a>
## 将 Worker 部署到 EKS {#deploy-the-workers-to-eks}

将 Worker 容器化后，您就可以将其部署到 EKS。在 EKS 集群中创建 Namespace。您将使用 Namespace 来运行 Temporal Worker：

```bash
kubectl create namespace your-namespace
```

创建一个 `ConfigMap` 来保存 Kubernetes 将注入到 Worker 部署中的非敏感值。
这些为实例、Namespace 和 Task Queue 启用动态路由。
要设置这些值，请构建一个 config-map.yaml 文件，如下例所示：

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: temporal-worker-config
  namespace: temporal-system
data:
  TEMPORAL_HOST_URL: “<your-temporal-address>“
  TEMPORAL_NAMESPACE: “<your-temporal-cloud-namespace>”
  TEMPORAL_TASK_QUEUE: “<your-task-queue>”
```

将 `ConfigMap` 应用于您的 Namespace：

```bash
kubectl apply -f config-map.yaml \
    --namespace your-namespace
```

对于敏感值，请使用 Kubernetes Secrets。
创建一个秘密来保存您的 Temporal API 密钥：

```bash
kubectl create secret generic temporal-secret \
    --from-literal=TEMPORAL_API_KEY=$TEMPORAL_API_KEY \
    --namespace your-namespace
```

配置到位后，您可以部署 Worker。
创建一个部署.yaml 文件，配置您的 Worker 映像、资源和机密值。
对于常见部署，请调整您指定的资源，使其与您的生产工作负载相匹配。
请注意，启动的容器从您刚刚创建的 Kubernetes 密钥中读取您的 Temporal API 密钥：

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
   name: your-app
   namespace: your-namespace
   labels:
      app: your-app
spec:
   selector:
      matchLabels:
         app: your-app
   replicas: 1
   template:
      metadata:
         labels:
            app: your-app
      spec:
         serviceAccountName: your-app
         containers:
            - name: your-app
              image: <your-ecr-image-name>
              env:
                - name: TEMPORAL_ADDRESS
                  valueFrom:
                    configMapKeyRef:
                      name: temporal-worker-config
                      key: TEMPORAL_ADDRESS
                - name: TEMPORAL_NAMESPACE
                  valueFrom:
                    configMapKeyRef:
                      name: temporal-worker-config
                      key: TEMPORAL_NAMESPACE
                - name: TEMPORAL_TASK_QUEUE
                  valueFrom:
                    configMapKeyRef:
                      name: temporal-worker-config
                      key: TEMPORAL_TASK_QUEUE
                - name: TEMPORAL_API_KEY
                  valueFrom:
                    secretKeyRef:
                      name: temporal-secret
                      key: TEMPORAL_API_KEY
              resources:
                limits:
                  cpu: "0.5"
                  memory: "512Mi"
                requests:
                  cpu: "0.2"
                  memory: "256Mi"
```

将deployment.yaml文件应用到EKS集群：

```bash
kubectl apply -f deployment.yaml \
    --namespace your-namespace
```

<a id="verify-that-the-workers-are-connected"></a>
## 验证 Worker 是否已连接 {#verify-that-the-workers-are-connected}

将 Worker 部署到 EKS 后，确认它们已连接到 Temporal Cloud。
检索您创建的 Kubernetes/EKS Namespace 的 pod 列表：

```
kubectl get pods -n temporal-system
```

列出 Pod 后，访问 Worker 日志以确认您已正确连接到 Temporal Cloud：

```
kubectl logs <pod-name> -n temporal-system
```

当您看到以下内容时，您确认连接：

```
Initializing worker...
Starting worker... Waiting for tasks.
```

您现已成功将 Temporal Worker 部署到 EKS。
