:::info Looking for Temporal Cloud defaults?

查看 [Temporal Cloud 默认和限制页面](/cloud/limits)

:::

本页详细介绍了编码到 Temporal 平台中的许多默认值，这些默认值可能会产生错误和警告。
错误是硬限制，一旦达到就会失败。
警告是在服务器端生成警告日志的软限制。

:::info

这些限制可能专门适用于每个 Workflow Execution，并不适用于整个 Temporal 平台或单个 Namespace。

:::

- **标识符：** 默认情况下，标识符（例如 Workflow Id、Workflow 类型和 Task Queue 名称）的最大长度为 1000 个字符。
  - 这可以使用 `limit.maxIDLength` 动态配置变量进行配置，在 [这个 SQL 示例](https://github.com/temporalio/samples-server/blob/main/compose/dynamicconfig/development-sql.yaml) 中设置为 255。
  - 字符格式为UTF-8。
- **gRPC：** gRPC 对于 [每条收到的消息](https://github.com/grpc/grpc/blob/v1.36.2/include/grpc/impl/codegen/grpc_types.h#L466) 的限制为 4 MB。
- **事件批量大小：** `DefaultTransactionSizeLimit` 限制为 [4MB](https://github.com/temporalio/temporal/pull/1363)。
  这是事件历史记录持久化所允许的最大事务大小。
- 有效负载的 **Blob 大小限制**（包括 Workflow 上下文以及每个 Workflow 和 Activity 参数和返回值；_[来源](https://github.com/temporalio/temporal/blob/v1.7.0/service/frontend/service.go#L133-L134)_）：
  - Temporal 在 256 KB 处发出警告：`Blob size exceeds limit.`
  - 2 MB 处的 Temporal 错误：`ErrBlobSizeExceedsLimit: Blob data size exceeds limit.`
  - 请参阅 [解决 blob 大小限制错误](/troubleshooting/blob-size-limit-error)。
- **Workflow Execution Update 限制**：
  - 单个 Workflow Execution 最多可拥有 10 个飞行中的 Update，历史上总共 2000 个 Update。
- **历史总大小限制**（导致终止 Workflow Execution）：
  - Temporal 在 10 MB 时发出警告：[历史记录大小超出警告限制](https://github.com/temporalio/temporal/blob/v1.7.0/service/history/workflowExecutionContext.go#L1238)。
  - 50 MB 处的 Temporal 错误：[历史记录大小超出错误限制](https://github.com/temporalio/temporal/blob/v1.7.0/service/history/workflowExecutionContext.go#L1204)。
  - 这可以通过 [HistorySizeLimitError 和 HistorySizeLimitWarn](https://github.com/temporalio/temporal/blob/v1.7.0/service/history/configs/config.go#L380-L381) 进行配置。
- **历史总计数限制**（导致终止 Workflow Execution）：
  - Temporal 在 10,240 个事件后发出警告：[历史记录大小超出警告限制](https://github.com/temporalio/temporal/blob/v1.7.0/service/history/workflowExecutionContext.go#L1238)。
  - 51,200 个事件后出现 Temporal 错误：[历史记录大小超出错误限制](https://github.com/temporalio/temporal/blob/v1.7.0/service/history/workflowExecutionContext.go#L1204)。
  - 这可以通过 [HistoryCountLimitError 和 HistoryCountLimitWarn](https://github.com/temporalio/temporal/blob/v1.7.0/service/history/configs/config.go#L382-L383) 进行配置。
- **并发限制**
  - 以下命令受到限制：
    - `ScheduleActivityTask`
    - `SignalExternalWorkflowExecution`
    - `RequestCancelExternalWorkflowExecution`
    - `StartChildWorkflowExecution`
  - 如果并发挂起计数超过 2,000，这些操作将会失败。
    为了获得最佳性能，请将并发操作限制为 500 或更少。
    这会减少 Workflow 的 Event History 大小并减少 Web UI 中的加载时间。
  - 自 v1.21 起，开源 Temporal 服务的默认限制为 2,000 个待处理活动、Child Workflow、Signal 或 Workflow 取消请求，但您可以使用以下变量覆盖动态配置中的限制：
    - `limit.numPendingActivities.error`
    - `limit.numPendingSignals.error`
    - `limit.numPendingCancelRequests.error`
    - `limit.numPendingChildExecutions.error`
  - 默认情况下，[批量作业](/cli/batch) 一次仅限于一项作业。
- [自定义 Search Attribute 限制](/search-attribute#custom-search-attribute-limits)

有关动态配置键的详细信息，请参阅[动态配置参考](/references/dynamic-configuration)。
