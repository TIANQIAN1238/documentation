:::info Open source Temporal

本页介绍了 **开源 Temporal** 的 Namespace 操作。有关 Namespace 核心概念，请参阅
[Temporal Namespace](/namespaces)。对于 Temporal Cloud，请参见 [Temporal Cloud Namespaces](/cloud/namespaces)。

:::

[Namespace](/namespaces) 是 Temporal 平台内的一个隔离单元。在运行 Workflow 之前，您必须
使用您的 Temporal 服务注册至少一个 Namespace。

<a id="create-a-namespace"></a>
## 创建一个 Namespace {#create-a-namespace}

注册 Namespace 会在 Temporal 服务上创建它。当您注册 Namespace 时，您必须设置
[保留期限](/temporal-service/temporal-server#retention-period) 确定 Workflow 关闭多长时间
执行历史记录被保留。

您可以使用以下命令创建 Namespace：

- **Temporal CLI**（推荐）：[`Temporal operator Namespace create`](/cli/operator#create)
- **Go SDK**：[`RegisterNamespace`](/develop/go/client/namespaces#register-namespace)
- **Java SDK**：[`RegisterNamespace`](/develop/java/client/namespaces#register-namespace)
- **TypeScript SDK**：[Namespace 管理](/develop/typescript/client/namespaces#register-namespace)

<a id="the-default-namespace"></a>
### 默认Namespace {#the-default-namespace}

如果未指定 Namespace，SDK 和 CLI 将使用 `default` Namespace。使用前您必须注册此 Namespace
它。

对于本地开发，[`Temporal Server start-dev`](/cli/server#start-dev) 命令会自动创建
`default` Namespace。

对于所有其他部署方法，请使用 Temporal CLI 手动创建 `default` Namespace：

```bash
temporal operator namespace create --namespace default
```

Namespace 注册最多需要 15 秒才能完成。等待此过程完成后再调用
Namespace。

<a id="manage-namespaces"></a>
## 管理 Namespace {#manage-namespaces}

常见Namespace管理操作：

| 手术 | CLI 命令                                                      | 描述                         |
| --------- | ---------------------------------------------------------------- | ----------------------------------- |
| 列表      | [`Temporal operator Namespace list`](/cli/operator#list)         | 列出所有已注册的 Namespace      |
| 描述  | [`Temporal operator Namespace describe`](/cli/operator#describe) | 获取 Namespace 的详细信息         |
| Update    | [`Temporal operator Namespace Update`](/cli/operator#update)     | Update Namespace 配置      |
| 删除    | [`Temporal operator Namespace delete`](/cli/operator#delete)     | 删除 Namespace 及其所有数据 |

对于基于SDK的Namespace管理：

- [Go SDK Namespace 管理](/develop/go/client/namespaces#manage-namespaces)
- [Java SDK Namespace 管理](/develop/java/client/namespaces#manage-namespaces)
- [TypeScript SDK Namespace 管理](/develop/typescript/client/namespaces#manage-namespaces)

<a id="deprecate-vs-delete"></a>
### 弃用与删除 {#deprecate-vs-delete}

- **弃用**：阻止新的 Workflow Execution 启动，但现有的 Workflow 继续运行。
- **删除**：删除 Namespace 和所有 Workflow Execution。这是不可逆转的。

＃＃ 安全

在前端服务上使用自定义 [授权人](/self-hosted-guide/security#authorizer-plugin) 来控制谁可以
创建 Update 或弃用 Namespace。

如果没有配置授权者，Temporal 使用 `nopAuthority` 授权者无条件允许所有 API 调用。

对于Temporal Cloud、[基于角色的访问控制](/cloud/manage-access/roles-and-permissions#namespace-level-permissions)提供Namespace级别的授权，无需自定义配置。
