欢迎使用自托管 Temporal 服务指南。本指南向您展示如何自行托管开源基础设施
编排持久应用程序的软件。

:::tip Do you need a production Temporal Service?

如果您仍在本地开发和测试应用程序，则可能不需要生产 Temporal 服务。使用
[Temporal CLI开发服务器](/cli/server#start-dev) — 没有外部依赖项的单个二进制文件：

`Temporal Server start-dev`

这将在本地计算机上启动带有 Web UI 的完整 Temporal 服务。我们建议将此用于本地开发
无论您计划在生产中使用 Temporal Cloud 还是自托管。请参阅
[Temporal CLI 服务器](/cli/server) 配置选项页面。

:::

<a id="plan-and-deploy-your-service"></a>
## 规划和部署您的服务 {#plan-and-deploy-your-service}

- [部署](/self-hosted-guide/deployment)：选择部署方法（Docker、Kubernetes 或手动）并设置
  生产就绪 Temporal 服务。
- [嵌入式服务器](/self-hosted-guide/embedded-server)：在进程中运行 Temporal 作为 Go 库以进行本地开发
  和测试场景。
- [默认值](/self-hosted-guide/defaults)：查看可能影响 Workflow 和的平台限制和默认设置
  Activity 行为。
- [生产清单](/self-hosted-guide/production-checklist)：验证规模、可靠性的准备情况，
  操作和长期可维护性。

<a id="operate-your-self-hosted-service"></a>
## 运营您的自托管服务 {#operate-your-self-hosted-service}

- [Namespaces](/self-hosted-guide/namespaces)：创建和管理 Namespace 隔离、保留和相关
  配置。
- [安全](/self-hosted-guide/security)：配置TLS/mTLS、身份验证、授权和相关强化
  控制。
- [监控](/self-hosted-guide/monitoring)：收集和可视化服务和 SDK 指标以进行故障排除和跟踪
  健康。
- [Visibility](/self-hosted-guide/visibility)：配置 Visibility 存储，以便您可以列出、过滤和搜索 Workflow
  处决。
- [升级服务器](/self-hosted-guide/upgrade-server#upgrade-server)：执行安全、顺序的服务器和模式
  升级。

<a id="protect-data-and-enable-advanced-features"></a>
## 保护数据并启用高级功能 {#protect-data-and-enable-advanced-features}

- [数据加密](/production-deployment/data-encryption)：使用有效负载编解码器和 Codec Server 模式来保护
  敏感 Workflow 数据。
- [档案](/self-hosted-guide/archival)：将关闭的事件历史记录和 Visibility 记录移至 Blob 存储以延长时间
  保留。
- [多集群复制](/self-hosted-guide/multi-cluster-replication)：跨集群复制 Workflow 状态
  用于故障转移和灾难恢复。
- [Temporal Nexus](/production-deployment/self-hosted-guide/nexus)：在自托管环境中启用 Nexus 进行连接
  Temporal 跨越边界的应用程序。
