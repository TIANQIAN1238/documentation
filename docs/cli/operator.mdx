本页提供 `Temporal` CLI `operator` 命令的参考。适用于每个子命令的标志显示在子命令标题内的表格中。请参阅 [全球旗帜](#global-flags) 了解可与每个子命令一起使用的标志。

## 簇 {#cluster}

对 Temporal 服务（也称为集群）执行操作员操作。

```
temporal operator cluster [subcommand] [options]
```

例如检查服务/集群运行状况：

```
temporal operator cluster health
```

### 描述 {#describe}

查看有关 Temporal 集群（服务）的信息，包括集群名称、
持久性存储和 Visibility 存储。添加 `--detail` 以获得更多信息：

```
temporal operator cluster describe [--detail]
```

使用以下选项更改此命令的行为。您还可以使用适用于所有子命令的任何 [全局标志](#global-flags)。

| 旗帜 | 必需的 | 描述 |
|------|----------|-------------|
| `--detail` | 不 | **bool** 显示历史分片计数和集群/服务版本信息。 |

### 健康 {#health}

查看有关 Temporal 服务运行状况的信息：

```
temporal operator cluster health
```

使用 [全局标志](#global-flags) 自定义此命令与 Temporal 服务的连接。

### 列表 {#list}

打印注册到本地的远程Temporal集群（服务）列表
服务。报告详细信息包括集群的名称、ID、地址、历史分片
计数、故障转移版本和可用性：

```
temporal operator cluster list [--limit max-count]
```

使用以下选项更改此命令的行为。您还可以使用适用于所有子命令的任何 [全局标志](#global-flags)。

| 旗帜 | 必需的 | 描述 |
|------|----------|-------------|
| `--limit` | 不 | **int** 要显示的最大簇数。 |

### 消除 {#remove}

从本地服务中删除已注册的远程 Temporal 集群（服务）。

```
temporal operator cluster remove \
    --name YourClusterName
```

使用以下选项更改此命令的行为。您还可以使用适用于所有子命令的任何 [全局标志](#global-flags)。

| 旗帜 | 必需的 | 描述 |
|------|----------|-------------|
| `--name` | 是的 | **字符串** 集群/服务名称。 |

### 系统 {#system}

显示 Temporal 集群（服务）的 Temporal Server 信息：服务器
版本、调度支持等等。这些信息有助于诊断
Temporal Server 的问题。

该命令默认为本地服务。否则，请使用
用于指定集群（服务）端点的 `--frontend-address` 选项：

```
temporal operator cluster system \
    --frontend-address "YourRemoteEndpoint:YourRemotePort"
```

使用 [全局标志](#global-flags) 自定义此命令与 Temporal 服务的连接。

## Namespace {#namespace}

管理 Temporal 集群（服务）Namespace：

```
temporal operator namespace [command] [command options]
```

例如：

```
temporal operator namespace create \
    --namespace YourNewNamespaceName
```

### 创造 {#create}

在 Temporal 服务上创建新的 Namespace：

```
temporal operator namespace create \
    --namespace YourNewNamespaceName \
    [options]
```

创建具有多区域数据复制的Namespace：

```
temporal operator namespace create \
    --global \
    --namespace YourNewNamespaceName
```

根据需要配置保留和 Visibility 存档状态等设置。
例如，Visibility 存档可以在单独的 URI 上设置：

```
temporal operator namespace create \
    --retention 5d \
    --visibility-archival-state enabled \
    --visibility-uri YourURI \
    --namespace YourNewNamespaceName
```

注意：存档状态的 URI 值一旦启用就无法更改。

使用以下选项更改此命令的行为。您还可以使用适用于所有子命令的任何 [全局标志](#global-flags)。

| 旗帜 | 必需的 | 描述 |
|------|----------|-------------|
| `--active-cluster` | 不 | **字符串** 活动集群（服务）名称。 |
| `--cluster` | 不 | **字符串[]** 用于 Namespace 创建的集群（服务）名称。可以多次通过。 |
| `--data` | 不 | **字符串[]** Namespace 数据为 `KEY=VALUE` 对。键必须是标识符，值必须是 JSON 值。例如：`YourKey={"your": "value"}` 可以多次传递。 |
| `--description` | 不 | **字符串** Namespace 描述。 |
| `--email` | 不 | **字符串** 所有者电子邮件。 |
| `--global` | 不 | **bool** 启用多区域数据复制。 |
| `--history-archival-state` | 不 | **字符串枚举** 历史档案状态。接受的值：禁用、启用。 |
| `--history-uri` | 不 | **字符串** 将历史记录存档到此 `URI`。一旦启用，就无法更改。 |
| `--retention` | 不 | **持续时间** 删除前保留关闭的 Workflow 的时间。 |
| `--Visibility-archival-state` | 不 | **字符串枚举** Visibility 存档状态。接受的值：禁用、启用。 |
| `--Visibility-uri` | 不 | **字符串** 将 Visibility 数据存档到此 `URI`。一旦启用，就无法更改。 |

Nexus 端点命令遵循以下语法：


#### 创造 {#create}

在服务器上创建 Nexus 端点。

Nexus 端点名称在 Workflow 代码中用于调用 Nexus 操作。
端点目标可以是 Worker，在这种情况下
必须同时提供 `--target-Namespace` 和 `--target-task-queue`，或者
外部 URL，在这种情况下必须提供 `--target-url`。

如果已存在同名端点，则此命令将失败
已注册。

```
temporal operator nexus endpoint create \
  --name your-endpoint \
  --target-namespace your-namespace \
  --target-task-queue your-task-queue \
  --description-file DESCRIPTION.md
```

使用以下选项更改此命令的行为。您还可以使用适用于所有子命令的任何 [全局标志](#global-flags)。

| `--description-file` | 不 | **字符串** Nexus 端点描述文件的路径。描述文件的内容可以使用Markdown格式。 |
| `--name` | 是的 | **字符串** 端点名称。 |
| `--target-Namespace` | 不 | **字符串** Namespace，其中处理程序 Worker 轮询 Nexus 任务。 |

| `--target-url` | 不 | **字符串** 接收转发的 Nexus 请求的外部 Nexus 端点。可用作 `--target-Namespace` 和 `--target-task-queue` 的替代品。 _（实验性）_ |
