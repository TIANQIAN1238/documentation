## 部署 {#deployment}

部署命令对 Worker 部署执行操作：

```
temporal worker deployment [command] [options]
```

例如：

```
temporal worker deployment list
```

列出客户端的 Namespace 中的部署。

参数可以是与以下项关联的 Worker 部署版本
a 部署，使用部署名称和构建 ID 指定。

例如：

```
temporal worker deployment set-current-version \
         --deployment-name YourDeploymentName --build-id YourBuildID
```

设置给定部署的当前部署版本。

### 删除 {#delete}

删除给定部署名称的 Worker 部署。
仅当部署中没有版本时才能删除。

```
temporal worker deployment delete [options]
```

例如，设置删除部署的用户身份：

```
temporal worker deployment delete \
    --name YourDeploymentName \
    --identity YourIdentity

```

使用以下选项更改此命令的行为。您还可以使用适用于所有子命令的任何 [全局标志](#global-flags)。

| 旗帜 | 必需的 | 描述 |
|------|----------|-------------|
| `--name`、`-d` | 是的 | **字符串** Worker 部署的名称。 |

### 删除版本 {#delete-version}

删除 Worker 部署版本（给定其完全限定标识符）。
正常操作期间很少需要此操作
因为未使用的版本最终会被垃圾收集。
只有满足以下所有条件时，客户端才能删除版本
满足：
  - 它不是此部署的当前版本或渐变版本。
  - 它没有活动的轮询器，即，没有 Task Queue
  版本有轮询器。
  - 它没有排水。可以使用选项忽略此要求
`--skip-drainage`。
```
temporal worker deployment delete-version [options]
```

例如，跳过排水限制：

```
temporal worker deployment delete-version \
    --deployment-name YourDeploymentName --build-id YourBuildID \
    --skip-drainage
```

使用以下选项更改此命令的行为。您还可以使用适用于所有子命令的任何 [全局标志](#global-flags)。

| 旗帜 | 必需的 | 描述 |
|------|----------|-------------|
| `--build-id` | 是的 | **字符串** Worker 部署版本的构建 ID。 |
| `--deployment-name` | 是的 | **字符串** Worker 部署的名称。 |
| `--skip-drainage` | 不 | **bool** 忽略不排空的删除要求。 |

### 经理身份 {#manager-identity}

Manager Identity 命令更改 Worker 部署的 `ManagerIdentity`：

```
temporal worker deployment manager-identity [command] [options]
```

如果存在，`ManagerIdentity` 是拥有以下权限的用户的身份： 
对此 Worker 部署进行更改的专有权利。默认为空。
设置后，身份与 `ManagerIdentity` 不匹配的用户将不会
能够更改 Worker 部署。

这在多个用户（例如 CLI
用户和自动控制器）可以与相同的 Worker 部署进行交互。
`ManagerIdentity` 允许不同用户相互交流
谁应该对 Worker 部署进行更改。

当前经理身份以 `describe` 返回：
```
 temporal worker deployment describe \
    --deployment-name YourDeploymentName
```

#### 放 {#set}

根据给定的部署名称设置 Worker 部署的 `ManagerIdentity`。

如果存在，`ManagerIdentity` 是拥有以下权限的用户的身份： 
对此 Worker 部署进行更改的专有权利。默认为空。
设置后，身份与 `ManagerIdentity` 不匹配的用户将不会
能够更改 Worker 部署。

这在多个用户（例如 CLI
用户和自动控制器）可以与相同的 Worker 部署进行交互。
`ManagerIdentity` 允许不同用户相互交流
谁应该对 Worker 部署进行更改。

```
temporal worker deployment manager-identity set [options]
```

例如：

```
temporal worker deployment manager-identity set \
   --deployment-name DeploymentName \
   --self \
   --identity YourUserIdentity # optional, populated by CLI if not provided
```

将 Deployment 的 Manager Identity 设置为用户的身份 
这个请求。如果您没有专门传递身份字段，CLI 将 
为您生成您的身份。

例如：
```
temporal worker deployment manager-identity set \
   --deployment-name DeploymentName \
   --manager-identity NewManagerIdentity
```

将部署的管理器标识设置为任意字符串。

使用以下选项更改此命令的行为。您还可以使用适用于所有子命令的任何 [全局标志](#global-flags)。

| `--manager-identity` | 不 | **字符串** 新经理身份。除非指定 --self ，否则为必需。 |

### 设置当前版本 {#set-current-version}

设置部署的当前版本。
当版本为当前版本时，该部署版本的 Worker 将收到
来自新 Workflow 和现有自动升级 Workflow 的任务
正在此部署上运行。

如果并非所有预期的 Task Queue 都被 Worker 轮询
新版本请求将失败。要覆盖此保护，请使用
`--ignore-missing-task-queues`。请注意，这将忽略 Task Queues
在尚未发现的部署中，导致任务不一致
队列配置。

```
temporal worker deployment set-current-version [options]
```

例如，设置部署的当前版本
`YourDeploymentName`，版本 ID 为 `YourBuildID`，以及
在默认的Namespace中：

```
temporal worker deployment set-current-version \
    --deployment-name YourDeploymentName --build-id YourBuildID
```

set-current-version 的目标也可以是未版本化的 Workers：

```
temporal worker deployment set-current-version \
    --deployment-name YourDeploymentName --unversioned
```

使用以下选项更改此命令的行为。您还可以使用适用于所有子命令的任何 [全局标志](#global-flags)。

| 旗帜 | 必需的 | 描述 |
|------|----------|-------------|
| `--allow-no-pollers` | 不 | **bool** 覆盖保护并将版本设置为当前版本，即使它没有轮询器。 |
| `--build-id` | 不 | **字符串** Worker 部署版本的构建 ID。除非指定 --unversioned，否则是必需的。 |
| `--deployment-name` | 是的 | **字符串** Worker 部署的名称。 |
| `--ignore-missing-task-queues` | 不 | **布尔** 覆盖保护以意外删除 Task Queue。 |
| `--unversioned` | 不 | **bool** 将未版本化的 Workers 设置为目标版本。不能与 --build-id 一起使用。 |
| `--yes`、`-y` | 不 | **bool** 不提示确认设置当前版本。 |

### 设置斜坡版本 {#set-ramping-version}

设置部署的渐变版本和百分比。

可以使用部署名称和构建 ID 设置渐变版本，
或使用 --unversioned 标志设置为未版本化的 Workers。

渐变百分比是一个浮点数，其值在 [0, 100] 范围内。
值 100 不会使渐变版本成为当前版本，请使用
改为 `set-current-version`。

要删除渐变版本，请使用标志 `--delete`。

如果并非所有预期的 Task Queue 都被 Worker 轮询
新的 Ramping 版本请求将失败。要覆盖此保护，请使用
`--ignore-missing-task-queues`。请注意，这将忽略 Task Queues
在尚未发现的部署中，导致任务不一致
队列配置。

```
temporal worker deployment set-ramping-version [options]
```

例如，设置部署的 Ramping 版本
`YourDeploymentName`，具有内部版本 ID `YourBuildID` 的版本，具有
10% 的任务重定向到此版本，并且
使用默认的 Namespace：

```
temporal worker deployment set-ramping-version \
    --deployment-name YourDeploymentName --build-id YourBuildID \
    --percentage 10.0
```

并消除该斜坡：
```
temporal worker deployment set-ramping-version \
    --deployment-name YourDeploymentName --build-id YourBuildID \
    --delete
```

使用以下选项更改此命令的行为。您还可以使用适用于所有子命令的任何 [全局标志](#global-flags)。

| 旗帜 | 必需的 | 描述 |
|------|----------|-------------|
| `--allow-no-pollers` | 不 | **bool** 覆盖保护并将版本设置为斜坡，即使它没有轮询器。 |
| `--build-id` | 不 | **字符串** Worker 部署版本的构建 ID。除非指定 --unversioned，否则是必需的。 |
| `--delete` | 不 | **布尔** 删除斜坡版本。 |
| `--deployment-name` | 是的 | **字符串** Worker 部署的名称。 |
| `--ignore-missing-task-queues` | 不 | **布尔** 覆盖保护以意外删除 Task Queue。 |
| `--percentage` | 不 | **浮动** 重定向到斜坡版本的任务百分比。有效范围 [0,100]。 |
| `--unversioned` | 不 | **bool** 将未版本化的 Workers 设置为目标版本。不能与 --build-id 一起使用。 |
| `--yes`、`-y` | 不 | **bool** 不提示确认设置斜坡版本。 |
