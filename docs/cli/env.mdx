本页提供 `Temporal` CLI `env` 命令的参考。适用于每个子命令的标志显示在子命令标题内的表格中。请参阅 [全球旗帜](#global-flags) 了解可与每个子命令一起使用的标志。

## 删除 {#delete}

完全删除预设环境_或_删除一个键值对
环境。如果您没有指定环境（使用 `--env` 或通过设置
`Temporal_ENV` 变量），此命令 Update 是“默认”环境：

```
temporal env delete \
    --env YourEnvironment
```

或者

```
temporal env delete \
    --env prod \
    --key tls-key-path
```

使用以下选项更改此命令的行为。您还可以使用适用于所有子命令的任何 [全局标志](#global-flags)。

| 旗帜 | 必需的 | 描述 |
|------|----------|-------------|
| `--key`、`-k` | 不 | **字符串** 属性名称。 |

## 得到 {#get}

列出给定环境的属性：

```
temporal env get \
    --env YourEnvironment
```

打印单个属性：

```
temporal env get \
    --env YourEnvironment \
    --key YourPropertyKey
```

如果您没有指定环境（使用 `--env` 或通过设置
`Temporal_ENV` 变量），此命令列出“默认”的属性
环境。

使用以下选项更改此命令的行为。您还可以使用适用于所有子命令的任何 [全局标志](#global-flags)。

| 旗帜 | 必需的 | 描述 |
|------|----------|-------------|
| `--key`、`-k` | 不 | **字符串** 属性名称。 |

## 列表 {#list}

列出您在本地计算机上设置的环境。环境是
存储在“$HOME/.config/temporalio/temporal.yaml”中。

使用 [全局标志](#global-flags) 自定义此命令与 Temporal 服务的连接。

## 放 {#set}

为属性键分配一个值并将其存储到环境中：

```
temporal env set \
    --env environment \
    --key property \
    --value value
```

如果您没有指定环境（使用 `--env` 或通过设置
`Temporal_ENV` 变量），此命令在“默认”中设置属性
环境。

使用 CLI 选项名称存储密钥可以让 CLI 自动设置这些选项
为您提供的选择。这可以减少工作量并有助于避免发布时出现拼写错误
命令。

使用以下选项更改此命令的行为。您还可以使用适用于所有子命令的任何 [全局标志](#global-flags)。

| 旗帜 | 必需的 | 描述 |
|------|----------|-------------|
| `--key`、`-k` | 不 | **字符串** 属性名称（必需）。 |
| `--value`、`-v` | 不 | **字符串** 属性值（必需）。 |
