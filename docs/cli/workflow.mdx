## 取消 {#cancel}

取消正在运行的 Workflow Execution 记录
Event History 中的 `WorkflowExecutionCancelRequested` 事件。服务
Schedule 是一个新的命令任务，Workflow Execution 执行任何清理
其实施所支持的工作。

使用 Workflow ID 取消执行：

```
temporal workflow cancel \
    --workflow-id YourWorkflowId
```

Visibility Query 可让您向 Workflow Execution 发送批量取消订单
匹配结果：

```
temporal workflow cancel \
    --query YourQuery
```

访问 https://docs.Temporal.io/visibility 了解有关 Search Attribute 的更多信息
和 Query 创作。请参阅 `Temporal batch --help` 以获得快速参考。

|------|----------|-------------|
| `--headers` | 不 | **字符串[]** Temporal Workflow 标头采用“KEY=VALUE”格式。键必须是标识符，值必须是 JSON 值。可以多次传递以设置多个 Temporal 标头。注意：这些是 Workflow 标头，而不是 gRPC 标头。 |
| `--Query`、`-q` | 不 | **字符串** 类似 SQL `Query` 列表过滤器的内容。您必须设置 --Workflow-id 或 --Query。 |
| `--reason` | 不 | **字符串** 批量操作的原因。仅与--Query 一起使用。默认为用户名。 |
| `--rps` | 不 | **float** 限制每秒的批次请求数。仅当存在 Query 时才允许。 |
| `--run-id`、`-r` | 不 | **字符串** 运行 ID。仅与 --Workflow-id 一起使用。不能与 --Query 一起使用。 |
| `--Workflow-id`、`-w` | 不 | **字符串** Workflow ID。您必须设置 --Workflow-id 或 --Query。 |

## 执行 {#execute}

建立一个新的 Workflow Execution 并将其进度定向到标准输出。的
命令阻塞并在 Workflow Execution 完成时返回。如果你的

```

    --workflow-id YourWorkflowId \
    --type YourWorkflow \
    --task-queue YourTaskQueue \

使用 `--event-details` 将 Update 中继到 JSON 中的命令行输出
格式。当使用 JSON 输出 (`--output json`) 时，这包括整个

## 执行-Update-with-start {#execute-update-with-start}

向 Workflow Execution 发送消息以调用 Update 处理程序，并等待
Update 来完成。如果 Workflow Execution 未运行，则新的 Workflow
开始执行并发送 Update。

实验性的。

```
temporal workflow execute-update-with-start \
  --update-name YourUpdate \
  --update-input '{"update-key": "update-value"}' \
  --workflow-id YourWorkflowId \
  --type YourWorkflowType \
  --task-queue YourTaskQueue \
  --id-conflict-policy Fail \
  --input '{"wf-key": "wf-value"}'

```

    --workflow-id YourWorkflowId
    --type YourQueryType
    --input '{"YourInputKey": "YourInputValue"}'
```

使用以下选项更改此命令的行为。您还可以使用适用于所有子命令的任何 [全局标志](#global-flags)。

## 终止 {#terminate}

终止 Workflow Execution：

```
temporal workflow terminate \
    --reason YourReasonForTermination \
    --workflow-id YourWorkflowId
```

原因是可选的，默认为当前用户的名称。原因
作为 `WorkflowExecutionTerminated` 的一部分存储在 Event History 中
事件。这成为 Workflow Execution 历史上的结束事件。

可以通过 Visibility Query 列表过滤器批量终止执行：

```
temporal workflow terminate \
    --query YourQuery \
    --reason YourReasonForTermination
```

Workflow 代码无法查看或响应终止。进行清理工作
在您的 Workflow 代码中，使用 `Temporal Workflow cancel` 代替。

访问 https://docs.Temporal.io/visibility 了解有关 Search Attribute 的更多信息
和 Query 创作。请参阅 `Temporal batch --help` 以获得快速参考。

## Update-选项 {#update-options}

修改Workflow Executions的属性：

```
temporal workflow update-options [options]
```

它可以覆盖 Worker 部署配置
Workflow Execution，控制 Worker 版本控制。

例如，要强制当前 Deployment 中的 Workers 执行以下命令
下一个 Workflow 任务将行为更改为 `auto_upgrade`：

```
temporal workflow update-options \
    --workflow-id YourWorkflowId \
    --versioning-override-behavior auto_upgrade
```

或者将 Workflow Execution 固定到 Worker 部署，设置行为
至 `pinned`：

```
temporal workflow update-options \
    --workflow-id YourWorkflowId \
    --versioning-override-behavior pinned \
    --versioning-override-deployment-name YourDeploymentName \
    --versioning-override-build-id YourDeploymentBuildId
```

要删除任何以前的覆盖，请将行为设置为
`unspecified`：

```
temporal workflow update-options \
    --workflow-id YourWorkflowId \
    --versioning-override-behavior unspecified
```

要查看当前覆盖，请使用 `Temporal Workflow describe`
