Temporal CLI 可通过终端直接访问 Temporal 服务。它是一个强大的管理工具，
监视和调试 Temporal 应用程序。您可以用它来启动、停止、检查和操作 Workflow 和
活动，并执行 Namespace、Schedule 和 Task Queue 管理等管理任务。

Temporal CLI 还包括适合在开发和 CI/CD 中使用的嵌入式 Temporal 服务。它包括
[Temporal Server](/temporal-service/temporal-server)、SQLite 持久性和 [Temporal 网页用户界面](/web-ui)。

:::note

从 [tctl](/tctl-v1) 升级到 Temporal CLI 时，请确保 Update 环境变量并使用 Updated
命令。详细信息请参见[CLI 发行说明](https://github.com/temporalio/cli/releases/)。

:::

## 安装 Temporal CLI {#install}

Temporal CLI 可在 macOS、Windows 和 Linux 上使用，或作为 Docker 映像使用。

### macOS {#macos}

选择以下安装方法之一在 macOS 上安装 Temporal CLI：

- 使用 Homebrew 安装 Temporal CLI。

```shell
brew install temporal
```

- 从 CDN 安装 Temporal CLI。

1. 选择所需的平台和架构。

- 下载达尔文 amd64：https://temporal.download/cli/archive/latest?platform=darwin&arch=amd64
  - 下载达尔文arm64：https://temporal.download/cli/archive/latest?platform=darwin&arch=arm64

2. 解压下载的存档。

3. 将 Temporal CLI 二进制文件添加到您的 PATH。

### Linux {#linux}

选择以下安装方法之一在 Linux 上安装 Temporal CLI：

- 从 CDN 安装 Temporal CLI。

1. 选择所需的平台和架构。

- 下载适用于 Linux amd64 的版本：https://temporal.download/cli/archive/latest?platform=linux&arch=amd64
  - 下载适用于 Linux arm64 的版本：https://temporal.download/cli/archive/latest?platform=linux&arch=arm64

2. 解压下载的存档。

3. 将 `Temporal` 二进制文件添加到您的 PATH 中。

<a id="windows"></a>
### 窗口 {#windows}

选择以下方法之一在 Windows 上安装 Temporal CLI：

- 从 CDN 安装 Temporal CLI。

1. 选择所需的平台和架构并下载二进制文件。

- 下载适用于 Windows amd64 的版本：https://temporal.download/cli/archive/latest?platform=windows&arch=amd64
  - 下载适用于 Windows arm64 的版本：https://temporal.download/cli/archive/latest?platform=windows&arch=arm64

2. 解压下载的存档。

3. 将 `Temporal.exe` 二进制文件添加到您的 PATH 中。

<a id="docker"></a>
### 码头工人 {#docker}

Temporal CLI 容器映像可在 [DockerHub](https://hub.docker.com/r/temporalio/temporal) 上使用，并且可以
直接运行：

```shell
docker run --rm temporalio/temporal --help
```

:::note

在 Docker 内运行 Temporal CLI 时，为了能够从主机系统访问开发服务器，
服务器需要配置为侦听外部IP并且需要转发端口：

```shell
docker run --rm -p 7233:7233 -p 8233:8233 temporalio/temporal server start-dev --ip 0.0.0.0
# UI is now accessible from host at http://localhost:8233/
```

:::

## 启动 Temporal 开发服务器 {#start-dev-server}

要启动 Temporal 开发服务器，请运行以下命令：

```bash
temporal server start-dev
```

此命令自动启动 Web UI，创建 `default` [Namespace](/namespaces)，并使用内存中的
数据库。

Temporal Server 将在 `localhost:7233` 上提供，Temporal Web UI 将在
[`http://localhost:8233`](http://localhost:8233/)。

如果停止开发服务器，内存中的 SQLite 数据库将不会保留。使用 `--db-filename` 选项
指定一个数据库文件，持久化应用程序状态。如果您计划停止并重新启动，这会很有帮助
开发服务器。

```shell
temporal server start-dev --db-filename temporal.db
```

:::note

使用 `--db-filename` 创建的本地数据库可能与较新版本的 Temporal CLI 不兼容。的
`Temporal Server start-dev` 命令仅适用于开发环境。

:::

有关开发服务器选项的完整列表，请使用 `--help` 标志：

```shell
temporal server start-dev --help
```

## 启用自动完成 {#enable-auto-completion}

使用以下命令启用自动完成。

<a id="zsh-auto-completion"></a>
### zsh 自动完成 {#zsh-auto-completion}

1. 将以下行添加到 `~/.zshrc` 启动脚本中：

   ```sh
   eval "$(temporal completion zsh)"
   ```

2. 重新启动 shell 或运行：

   ```sh
   source ~/.zshrc
   ```

<a id="bash-auto-completion"></a>
### Bash 自动完成 {#bash-auto-completion}

1. 安装 [bash 完成](https://github.com/scop/bash-completion#installation) 并将软件添加到您的
   `~/.bashrc`。

2. 将以下行添加到 `~/.bashrc` 启动脚本中：

   ```sh
   eval "$(temporal completion bash)"
   ```

3. 重新启动 shell 或运行：

   ```sh
   source ~/.bashrc
   ```

:::note

如果自动完成失败并出现错误：`bash: _get_comp_words_by_ref: command not found`，则说明您没有成功
安装 [bash 完成](https://github.com/scop/bash-completion#installation)。该包必须加载到您的
`Temporal` 自动完成工作的 shell。

:::

<a id="fish-auto-completion"></a>
### Fish 自动完成 {#fish-auto-completion}

1. 创建 Fish 自定义完成目录（如果尚不存在）：

   ```fish
   mkdir -p ~/.config/fish/completions
   ```

2. 配置完成以在需要时加载。注意：文件名必须为`Temporal.fish`，否则补全将
   找不到：

   ```fish
   echo 'eval "$(temporal completion fish)"' >~/.config/fish/completions/temporal.fish
   ```

3. 重新启动 shell 或运行：

   ```fish
   source ~/.config/fish/completions/temporal.fish
   ```

## 配置 {#configuration}

以下信息提供了重要的配置详细信息。

<a id="configuration"></a>
### Namespace 注册 {#namespace-registration}

Namespace 在启动时预先注册以便立即使用。使用以下内容自定义预注册的 Namespace
命令：

```shell
temporal server start-dev --namespace foo --namespace bar
```

使用 `Namespace create` 注册 Namespace：

```shell
temporal operator namespace create --namespace foo
```

<a id="namespace-registration"></a>
### 启用或禁用 Temporal UI {#enable-or-disable-temporal-ui}

默认情况下，使用 Temporal CLI 运行开发服务器时会启用 Temporal UI。要禁用用户界面，
使用 `--headless` 修饰符：

```shell
temporal server start-dev --headless
```

<a id="enable-or-disable-temporal-ui"></a>
### 动态配置 {#dynamic-configuration}

高级 Temporal CLI 配置需要动态配置文件。

要在命令行上设置值，请使用 `--dynamic-config-value KEY=JSON_VALUE`。例如，启用 Search Attribute
缓存：

```bash
temporal server start-dev --dynamic-config-value system.forceSearchAttributesCacheRefreshOnRead=false
```

此设置使创建的 Search Attribute 立即可用。

## 代理支持 {#proxy-support}

Temporal CLI 为在代理后面操作的用户提供支持。此功能可确保无缝
即使在网络受限的环境中也能进行通信。

<a id="proxy-support"></a>
#### 设置代理支持 {#setting-up-proxy-support}

如果您位于代理后面，则需要指示 Temporal CLI 通过该代理路由其请求。你可以
通过设置 `HTTPS_PROXY` 环境变量来实现此目的。

```command
export HTTPS_PROXY=<host>:<port>
```

将 `<host>` 替换为代理的主机名或 IP 地址，将 `<port>` 替换为代理的端口号。

设置后，您可以像平常一样运行 Temporal CLI 命令。

:::note

Temporal CLI 使用 gRPC 库，该库本身支持 HTTP CONNECT 代理。 gRPC 库检查
`HTTPS_PROXY`（及其不区分大小写的变体）环境变量来确定是否应通过路由请求
代理。

:::

除了 `HTTPS_PROXY` 之外，gRPC 还遵循 `NO_PROXY` 环境变量。如果有的话这会很有用
您希望从代理中排除的特定地址或域。

有关详细信息，请参阅 gRPC 中的 [代理人](https://github.com/grpc/grpc-go/blob/master/Documentation/proxy.md)
文档。

### 启动 Workflow {#start-a-workflow}

在另一个终端中，使用以下命令与服务器交互。以下命令启动 Workflow：

```shell
$ temporal workflow start \
  --task-queue hello-world \
  --type MyWorkflow \
  --workflow-id 123 \
  --input 456

  WorkflowId                                   123
  RunId       357074e4-0dd8-4c44-8367-d92536dd0943
  Type        MyWorkflow
  Namespace   default
  TaskQueue   hello-world
  Args        [456]
```

```shell
temporal workflow start --task-queue hello-world --type MyWorkflow --workflow-id 123 --input 456
```

### 自定义您的环境变量 {#customize-your-environment-variables}

要与不同的服务器通信，例如 Temporal Cloud 上的生产 Namespace：

1. 创建一个名为`prod`的环境。
2. 将 `--env prod` 传递给命令，例如 `Temporal Workflow list --env prod`。

要创建新环境并设置其属性：

```shell
temporal env set prod.namespace production.f45a2
temporal env set prod.address production.f45a2.tmprl.cloud:7233
temporal env set prod.tls-cert-path /temporal/certs/prod.pem
temporal env set prod.tls-key-path /temporal/certs/prod.key
```

检查您的设置：

```shell
$ temporal env get prod

  address        production.f45a2.tmprl.cloud:7233
  namespace      production.f45a2
  tls-cert-path  /temporal/certs/prod.pem
  tls-key-path   /temporal/certs/prod.key
```

运行命令来测试连接：

```shell
$ temporal workflow list --env prod
```

有关属性的完整列表，请使用 `Temporal env set -h`。
