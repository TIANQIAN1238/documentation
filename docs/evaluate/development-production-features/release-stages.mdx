本产品发布阶段指南帮助您了解 Temporal 功能的发布方式。它描述并列出了每个发布阶段的标准，以便您可以就每个新功能的采用做出明智的决策。

产品发布指南期望：

|                                 | 预发布                                                        | 公开预览                                                                              | 一般可用性                             |
| ------------------------------- | ------------------------------------------------------------------ | ------------------------------------------------------------------------------------------- | ------------------------------------------------ |
| **功能访问**             | 自托管Temporal用户：所有人； Temporal Cloud：仅限受邀者。 | 每个人。 Temporal Cloud 可能会限制加入的用户数量以确保稳定性。 | 每个人。                                        |
| **功能完整性**        | 功能有限。                                             | 核心功能已经完成。                                                             | 成熟、功能齐全。                     |
| **API稳定性**               | 实验性； API 可能会发生变化。                            | API 重大变更保持在最低限度。                                                 | API稳定。                                   |
| **功能区域可用性** | 地区有限。                                                   | 大多数地区。                                                                               | 全部 [地区](/cloud/regions)。                   |
| **功能支持**             | 社区和工程团队。                                    | [正式支持](/cloud/support#support-ticket)。                                            | [正式支持](/cloud/support#support-ticket)。 |
| **功能推荐使用**   | 实验性的。                                                      | 生产用例。                                                                       | 生产用途。                                |
| **功能云定价**       | 无需额外费用。                                                | 定价变化保持在最低限度。                                                      | 定价稳定。                               |
| **功能互操作性**    | 有限的。                                                           | 除非另有说明，功能是相互兼容的。                           | 功能是相互兼容的。         |

## 预发布 {#pre-release}

**访问：** 大多数预发布功能均在开源 Temporal 软件中发布并公开可用。
但是，一些明确用于托管 Temporal 服务的功能（例如 [API 密钥](/cloud/api-keys)）可能特定于 Temporal Cloud。

在 Temporal Cloud 中，预发布功能仅限邀请：Temporal 将直接与一组现有的 Temporal Cloud 客户合作，参与每个预发布功能的测试。
我们邀请这些客户向 Temporal 团队提供反馈。

**分类：** Pre-release 中的新功能可能尚未完全成熟，可能存在 bug。
用户承认并同意预发布功能是按“原样”提供的，并且不提供任何赔偿、支持、保证或任何形式的陈述。

**反馈：** 强烈鼓励提供反馈，这对于指导 Temporal 功能开发非常重要。
我们鼓励您分享您的经验，以便您可以影响 Temporal 的未来方向。

**可用性：** Temporal 可能会在功能正式可用之前对其进行修改，甚至可能决定删除它们。
这意味着不能保证新功能将普遍可用。
预发布功能可以随时弃用。

预发布功能可能默认被禁用，并且可以通过配置启用。
Temporal Cloud 客户可以联系 Temporal 客户团队或 [Temporal 支持团队](/cloud/support#support-ticket) 以获得预发布访问权限。

## 公开预览 {#public-preview}

**访问：** 公共预览版中的新功能可供所有人使用。

**分类：** 公共预览版中的功能在正式发布之前可能会经过进一步的开发和测试。
这些功能正在完善中，建议用于生产使用。

**反馈：** Temporal 用户受邀通过 [社区闲置](http://t.mp/slack) 分享反馈，通过product@Temporal.io 直接联系 Temporal 团队，或在相关 [GitHub 存储库](https://github.com/temporalio) 中创建问题。
Temporal 还鼓励 Temporal Cloud 用户通过 [支持票](/cloud/support#support-ticket) 提交反馈。
此反馈将有助于指导通用可用性的改进。

**可用性：** 公共预览版中的新功能可能会不断发展。
API 可能会发生变化；然而，Temporal 的目标是保持向后兼容性。

## 一般可用性 {#general-availability}

**访问：** 公开发布的功能可供所有人使用。

**分类：** 该功能现已完全开发、测试并可供使用，无需进一步预期更改。

**反馈：** Temporal 用户受邀通过 [社区闲置](http://t.mp/slack) 分享反馈，通过product@Temporal.io 直接联系 Temporal 团队，或在相关 [GitHub 存储库](https://github.com/temporalio) 中创建问题。

**可用性：** 通用功能通过稳定的 API 发布，并建议在承诺的 SLA 下用于生产使用。

:::info Exceptions

不同的功能可能会有例外，但这是典型的期望。
任何变化都将被记录下来。

:::
