<a id="what-is-a-job-queue"></a>
## 什么是作业队列？ {#what-is-a-job-queue}

作业是在后台异步运行的单个离散工作单元，例如发送电子邮件、处理 Webhook、同步数据或可靠地执行单个函数。

作业队列是管理这些作业的系统：接受工作、将其分派到 Worker、失败时重试以及将 Visibility 提供给正在运行的内容和失败的内容。

**独立活动是 Temporal 的作业队列。**

除了使用与 Workflow 内的步骤相同的活动之外，它们还允许您使用 Temporal 活动作为后台作业。您只需编写一次 Activity，就可以将其作为后台作业或多步骤 Workflow 的一部分运行。

Temporal 提供比传统作业队列更强大的保证、更好的 Visibility 和更多的控制，同时对于大容量用例保持成本效益，并为多步骤 Workflow 编排提供干净的升级路径。

＃＃＃ 概述

独立活动添加了执行任何 Temporal Activity 作为顶级 Activity 执行的能力，以实现持久作业处理。

####统一编程模型&Worker部署

- 使用统一的 Activity 编程模型编写一次 Activity 并在任何地方使用它
- 可选的心跳支持长时间运行作业的检查点
- 部署到 Activity Worker 一次，并独立调用或从 Workflow 内部调用

<a id="overview"></a>
#### 执行生命周期 {#overview}

- 作业作为独立 Activity 执行提交
- 每个作业都以 Temporal 的可靠性持久保留，因此作业不会丢失
- 作业是 Scheduled，具有优先级、公平性、重复数据删除和无队头阻塞
- Worker 轮询 Task Queue 并执行活动（您运行自己的 Worker）
- Temporal 确保重试、超时和指数退避策略得到执行

<a id="unified-programming-model-worker-deployment"></a>
#### 可观察性和生命周期控制 {#unified-programming-model-worker-deployment}

- 完整作业 Visibility（列表、搜索），包含详细的执行状态、重试计数、错误和结果
- OpenMetrics 支持
- 生命周期控制：取消、暂停、取消暂停、重置、终止
- 手动完成外部集成和待命管理
