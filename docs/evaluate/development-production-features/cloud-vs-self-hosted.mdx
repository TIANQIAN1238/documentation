通过部署应用程序代码、Workflow、活动和 Worker 以供操作使用，将 Temporal 应用程序转换为生产就绪系统。
当您的应用程序准备好开始为生产流量提供服务时，我们提供两个 Temporal 服务选项：

- **[选择 Temporal Cloud 作为您的 Temporal 服务](/cloud)**
  让我们处理 Temporal 服务操作，以便您可以专注于您的应用程序。
- **[自托管 Temporal 服务](/self-hosted-guide)**
  部署您自己的生产级别 Temporal 服务来编排您的持久应用程序。

| 特征                            | Temporal Cloud                                                            | 自托管                                      |
| ---------------------------------- | ------------------------------------------------------------------------- | ------------------------------------------------ |
| **多租户**                   | ✅ 最多 100 个 Namespace                                                   | ✅ 无限 Namespace                          |
| **高可用性和故障转移** | ✅ [具有高可用性功能的 Namespace](/cloud/high-availability) | ✅ 全局 Namespace 和多集群复制 |
| **应用程序状态持久化**  | ✅ 30-90 天保留                                                    | ✅ 无限制                                     |
| **长期状态保留**      | ✅ Workflow 历史记录导出                                                | ✅ 档案                                      |
| **社区支持**              | ✅ Slack、论坛                                                           | ✅ Slack、论坛                                  |
| **付费支持**                   | ✅ 优先响应                                                  | ✖️                                                |
