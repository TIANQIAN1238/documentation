与自托管实例相比，Temporal Cloud 提供的功能可显着减少延迟，使您的应用程序更快、性能更高、效率更高。
在现代应用程序领域，低延迟对于确保 Workflow Execution 的延迟最小至关重要。
这种低延迟架构可确保快速的 Workflow Execution 和响应能力，这对于时间敏感的应用程序和高性能系统至关重要。

Temporal Cloud 的自定义持久层包含三个有助于降低延迟的关键组件：

- **更好的分片：** 在多个数据库之间分配负载，防止出现瓶颈。
  支持独立调整大小、提高可扩展性并立即处理高流量事件。
- **预写日志 (WAL)：** 在写入数据库之前聚合 Update，从而减少写入延迟。
  以仅附加格式存储写入，通过在写入数据库之前批处理 Update 来减少延迟和数据库大小。
- **Workflow Event History 的分层存储：** 卸载已完成的 Workflow 事件历史记录，提高数据库效率。

Temporal Cloud 提供较低的延迟，使其适合对延迟敏感、大规模或业务关键型应用程序。
