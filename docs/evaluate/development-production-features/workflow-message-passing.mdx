需要从外部与 Workflow 进行交互吗？考虑如下用例：

- 您的货运跟踪 Workflow 需要知道物品何时离开仓库并装载到卡车上。 **Signal** 当卡车司机扫描条形码时您的 Workflow。
- 贵公司的人员希望跟踪其数据迁移 Workflow 的进度。 **Query** 运行批处理 Workflow 以获取进度条的数据。
- 您的电子商务购物车 Workflow 需要知道何时添加新商品。 **Update** 它用于添加项目并接收当前要渲染的项目。

Temporal 提供 Signal、查询和 Update，以允许与正在运行的 Workflow 进行丰富的交互。

**Signals**：Signal 将消息异步发送到正在运行的 Workflow，实时更改其状态或控制其流程。

**查询**：Query 用于检查 Workflow 的进度或实时调试内部状态。

**Updates**：Update 向您的 Workflow 发送同步请求并实时跟踪它。
