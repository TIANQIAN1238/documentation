**Workflows**、**活动**和 **Workers** 构成 Temporal 应用程序的核心部分。

**Workflows**：Workflow 定义应用程序的整体流程。
您可以使用 Temporal SDK 以您选择的编程语言编写它。
从概念上讲，Workflow 指定一系列步骤并协调活动的执行。

**活动**：Activity 是封装容易失败的业务逻辑的方法或函数（例如，调用可能会宕机的服务）。
系统可以在出现某些故障时自动重试这些活动。
活动执行单一的、明确定义的操作，例如调用其他服务、对媒体文件进行转码或发送电子邮件。

**Workers**：Worker 执行您的 Workflow 和 Activity 代码。

**按照我们的 [开始使用](https://learn.temporal.io/getting_started/) 教程之一学习如何开发 Workflow 和活动并在 Worker 进程中运行它们。**
