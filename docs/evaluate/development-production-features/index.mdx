通过 Temporal SDK，Temporal 提供了广泛的功能，使开发人员能够构建服务于各种用例的应用程序。

- **[核心应用程序原语](/evaluate/development-production-features/core-application)**：使用 Workflow、活动和 Worker 开发并运行您的应用程序。
- **[测试套件](/evaluate/development-production-features/testing-suite)**：每个 Temporal SDK 都附带一个测试套件，使开发人员能够像测试其他应用程序一样测试其应用程序。
- **[Scheduled Workflows](/evaluate/development-production-features/schedules)**：在特定时间或给定时间间隔启动业务流程。
- **[中断 Workflow](/evaluate/development-production-features/interrupt-workflow)**：取消或终止已在进行的业务流程 (Workflow) 并补偿已采取的任何步骤。
- **运行时保障**：防止在运行时执行可避免的错误和问题。
- **[故障检测和缓解](/evaluate/development-production-features/failure-detection)**：检测超时故障并配置自动重试以缓解故障。
- **[Temporal Nexus](/evaluate/nexus)**：在隔离的 Namespace 之间（以及内部）连接 Temporal 应用程序，以改进模块化、安全性、调试和故障隔离。 Nexus 支持跨团队、跨域和多区域用例。
- **[Workflow消息传递](/evaluate/development-production-features/workflow-message-passing)**：构建响应式应用程序，在运行时对事件做出反应，并支持从正在进行的 Workflow 中检索数据。
- **版本控制**：支持长期运行的业务流程的业务逻辑的多个版本。
- **[可观察性](/evaluate/development-production-features/observability)**：列出业务流程、查看其状态并使用指标设置仪表板。
- **[调试](/evaluate/development-production-features/debugging)**：显示错误并逐步执行代码以查找问题。
- **[数据加密](/evaluate/development-production-features/data-encryption)**：转换数据并保护应用程序用户的隐私。
- **[吞吐量可组合性](/evaluate/development-production-features/throughput-composability)**：通过数据流、团队所有权或其他组织因素分解业务流程。
- **[云自动化](/evaluate/development-production-features/cloud-automation)**：通过 Temporal 的云自动化简化云管理并提高安全性。
- **[低延迟](/evaluate/development-production-features/low-latency)**：使您的应用程序更快、性能更强、效率更高。
- **[多租户](/evaluate/development-production-features/multi-tenancy)**：提高效率和成本效益。

有关 Temporal 功能发布阶段和标准的详细信息，请参阅此 [产品发布阶段指南](/evaluate/development-production-features/release-stages)。
