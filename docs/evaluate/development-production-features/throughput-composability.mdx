在 Temporal 中，**Child Workflow** 使应用程序能够在吞吐量方面实现另一个级别的可组合性。

以下示例场景是使用此功能的几个原因：

- 创建可以从多个其他服务或应用程序调用的单独服务。
- 将一个步骤分成更小的块。
- 管理专用资源并保证唯一性。
- 定期执行逻辑，而不会压垮父业务流程。
