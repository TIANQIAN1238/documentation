在 Temporal 中，超时会检测应用程序故障。
然后，系统可以通过重试自动缓解这些故障。
两个主要应用程序功能原语 **Workflows** 和 **Activities** 都具有专用的 **超时配置**，并且可以使用 **Retry Policy** 进行配置。

**按照我们的 [开始使用](https://learn.temporal.io/getting_started/) 教程之一探索超时和重试策略。**

或者直接跳至 ​​Temporal SDK 功能指南。
