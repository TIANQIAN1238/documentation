Temporal Cloud 包括成功入职所需的适当水平的技术支持和指导、服务和培训，协助您高效、大规模地设计和部署应用程序。
我们的团队对 Temporal 拥有丰富的知识和广泛的技能，可以帮助您成功完成任何项目。

Temporal Cloud 提供多个级别的支持，从协助故障/修复场景到问题和服务，到帮助您的应用程序的入职、设计/代码审查以及预生产优化和操作准备。

本页内容仅适用于 Temporal Cloud 客户。

:::

## Temporal Cloud 支持保证 {#guarantees}

Temporal 努力确保您使用 Temporal Cloud 取得成功。
我们提供明确的支持保证。
Temporal Cloud 客户可以通过针对优先问题达成一致的一组 SLA 获得中断/修复支持。
我们使用票务系统来输入、跟踪和关闭这些问题。

如果出现问题，团队还通过专门的 Slack 渠道、论坛和知识库提供支持。

**优先级定义**

- **P0 - 严重**（生产受到影响）
  - Temporal Cloud 服务不可用或降级，造成重大影响。
- **P1 - 高**（生产问题）
  - 与 Temporal Cloud 服务上运行的生产工作负载相关的问题，或重要项目被阻止。
- **P2 - 正常**（一般问题）
  - 一般 Temporal Cloud 服务或其他不影响生产的问题，或者存在缓解影响的解决方法。
- **P3 - 低**（一般指导）
  - Temporal Cloud 服务的疑问或问题不影响系统可用性或功能。

:::note On Page Service

P0：为企业和关键任务帐户提供 24×7（页面服务）。

:::

有关这些支持级别的定价详细信息，请访问我们的 [定价页面](/cloud/pricing)。

参与关键任务计划的客户和（通过选择加入）企业客户可以获得专属支持工程师的帮助。
我们提供：

- 直接联系高级开发专家，他将成为您的 Temporal 客户团队的一员，增加深厚的技术专业知识。
  - 我们的高接触参与模式超越了传统支持，通过实践协作、主动优化、实施设计和运营来提供变革性价值。
  - 在了解您的实施情况的人员的直接帮助下更快地解决问题。
  - 专注于最佳实践和开发配对的咨询，以确保高质量的代码和可扩展性。
  - 通过定期检查和建议进行优化，以提高性能和效率。
- 每月优先获得一位高级工程师最多 20 小时的服务，为一个业务部门或主要群体（特别是在单一区域内）提供专家指导和主动支持。

我们的服务专注于本地时区调整，以确保最佳的响应能力和效率。
可以以 6,000 美元/月/单位的价格购买此服务的附加服务单位以覆盖其他团体或地区。

- 每月最多 20 小时
- 一个主要集团或业务部门

<a id="ticketing"></a>
### 访问 Temporal 支持 {#ticketing}

1. 转到[支持.Temporal.io](https://support.temporal.io/)。
2. 如果出现提示，请使用您通常使用的相同方法（例如 Google、Microsoft、电子邮件密码或其他方法）登录 Temporal Cloud。
3. 您将看到一个屏幕，您可以在其中查看您的 Temporal 帐户的打开和关闭的工单，以及提交新的工单。

要向 Temporal 支持请求帮助，请参阅 [创建票证](#support-ticket)。

<a id="access-temporal-support"></a>
### 创建票证 {#access-temporal-support}

:::info

此过程仅适用于合同包含付费支持的 Temporal Cloud 客户。
如果您需要帮助并且没有付费支持，请在 Slack 中的 [Temporal 社区论坛](https://community.temporal.io) 或 [Temporal 工作区](https://t.mp/slack) 的 `#support-cloud` 频道中发布您的请求。

:::

要在 Temporal 支持门户中创建票证：

1. 转到[支持.Temporal.io](https://support.temporal.io/)。
2. 如果出现提示，请使用您通常使用的相同方法（例如 Google、Microsoft、电子邮件密码或其他方法）登录 Temporal Cloud。
3. 单击右上角的**创建工单**按钮。
4. 在 **提交票证** 页面上，在表单中输入您的请求的详细信息。 **名称**、**主题**和**描述** 为必填项。
5. 在表单底部，选择“**提交**”。
