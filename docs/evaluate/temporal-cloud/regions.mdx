您可以从具有 Internet 连接的任何地方访问 Temporal Cloud，无论您的 Temporal Cloud Namespace 实际位于何处。
您的应用程序可以驻留在您选择的云环境或数据中心中。
考虑到这一点，您_将_通过在靠近您托管 Worker 的区域中创建 Namespace 来减少延迟。

此页面枚举了 Temporal Cloud Namespaces 当前支持的区域。

:::tip Service Availability

请访问 [状态.Temporal.io](https://status.temporal.io) 查看我们支持的区域的状态。
在该页面上，您还可以订阅 Update，以便在 Temporal 创建、Update 或解决事件时接收电子邮件通知。

:::
