## Temporal Cloud 定价模型 {#pricing-model}

本节介绍 Temporal Cloud 定价模型的基础及其工作原理。
每个日历月的发票总额是 Temporal Cloud 消费（[行动](#action) 和 [贮存](#storage)）以及包含 [支持](#base_plans) 的 [Temporal Cloud计划](/cloud/support#support) 的组合。

### 动作 {#action}

**什么是 Temporal 操作？**

操作是 Temporal Cloud 基于消耗的定价的主要单位。
它们跟踪 Temporal Cloud 服务中的计费操作，例如启动 Workflow、记录心跳或发送消息。
**具体的计费操作在 [行动](/cloud/actions) 页面上讨论。**

[伸出手](https://pages.temporal.io/contact-us) 联系我们的团队以获取更多信息或帮助确定您的操作数量。

### 存储 {#storage}

**Workflow 存储的工作原理**

Workflow 的执行可能存在几秒钟、一天、一个月甚至永远。
Temporal 服务存储 Workflow Execution 的 [Event History](/workflow-execution/event#event-history)。
在此框架下，Workflow Execution 只有两种状态，打开（活动存储）或关闭（保留存储）。

- _活动存储_ 测量活动 Workflow 使用的存储量。

- 当Workflow的执行完成时，Temporal Cloud将Event History存储为定义的[保留期限](/temporal-service/temporal-server#retention-period)，这是由用户根据Namespace设置的。
  这是_保留存储_。
  保留存储的典型用途包括合规性、调试、工作负载刷新和业务分析。
  当关闭 Workflow 历史记录需要保留超过 Temporal Cloud 的最长期限 90 天时，我们建议使用我们的 [**出口**](/cloud/export) 功能。

存储成本以千兆字节小时 (GBh) 为单位。

### 定价选项 {#pricing-options}

**如何支付Temporal Cloud**

当您超出基本层中的操作和存储分配后，Temporal Cloud 提供两种付款选项：即用即付和承诺。
两种型号均对三个主要组件进行计量和计费：[行动](#action)、[贮存](#storage) 和 [您的 Temporal Cloud 计划](/cloud/support#support)。

- 使用即用即付方式，我们会根据您的消费情况每个日历月向您开具发票。
  即用即付定价会根据您的操作规模自动应用批量价格。
- 通过承诺，您可以使用 Temporal 积分预购买您的 Temporal Cloud 支出。
  Temporal 积分用于支付您的 Temporal Cloud 消费，包括 Temporal Cloud 计划费用。

## 即用即付 {#payg}

**即用即付定价如何运作？**

即用即付定价基于消费量。
本部分介绍每个日历月的计费方式并提供示例。

### 行动定价 {#payg-action-pricing}

操作定价起价为每百万个操作 50 美元（每个操作 0.00005 美元）。
随着规模的扩大，您将获得渐进的批量折扣。
折扣基于您帐户的总使用量，按每个日历月计量和计费：

| 行动               | 每百万次操作的价格                                                                 |
| --------------------- | ----------------------------------------------------------------------------------------- |
| 前5M              | 50 美元                                                                                   |
| 接下来 5M，最多 10M    | 45 美元                                                                                   |
| 下一个 10M，上一个 20M   | 40 美元                                                                                   |
| 下30M，上50M   | 35 美元                                                                                   |
| 下50M，上100M  | 30 美元                                                                                   |
| 下100M，上200M | 25 美元                                                                                   |
| 超过200M             | 联系[销售量](mailto:sales@temporal.io)获取信息<br />_更多折扣，乐于助人_ |

**例子**

如果您在一个日历月内消耗的 1125 万个操作超出了您的 Temporal Cloud 计划分配，则您的操作账单将为：

```
5M Actions ⨉ $50 Per Million Actions = $250
5M Actions ⨉ $45 Per Million Actions = $225
1.25M Actions ⨉ $40 Per Million Actions = $50
Actions
$250 (First Tier) + $225 (Second Tier) + $50 (Third Tier) = $525
```

### 存储定价 {#payg-storage-pricing}

我们的 Temporal Cloud 计划可以满足大多数帐户的存储需求。
对于一个日历月内的额外存储，您需要支付活动存储和保留存储的费用，如下所示：
| **贮存** | **每 GBh 价格（美元）** |
| ----------- | ----------------------- |
| 保留 | $0.00105 |
| 积极的 | 0.042 美元 |

:::tip Storage costs are also affected by Temporal System Workflows that back features such as:

- [Schedules](https://docs.temporal.io/schedule)：每个 Scheduled Workflow 都有助于存储使用。提供的输入、输出和故障均涉及 Scheduled Workflow 产生的存储使用情况。
- [批量作业](https://docs.temporal.io/cli/batch)：批次 Workflow Execution 也会消耗存储空间。

这些 Workflow Execution 会影响总体活动存储和保留存储消耗。

:::

**例子**

如果您在一个日历月内拥有超过基本层分配的 720 GBh 活动存储和 3,600 GBh 保留存储，则您的账单将为：

```
720 GBh Active Storage ⨉ $0.042 per GBh = $30.24
3,600 GBh Retained Storage ⨉ $0.00105 per GBh = $3.78
Total Storage Bill: $30.24 Active Storage + $3.78 Retained Storage = $34.02
```

## Temporal Cloud 计划定价 {#temporal-cloud-plan-pricing}

您的 Temporal Cloud 计划定价是每月最低价格或消费支出的百分比 (%) 中的较高者：

- Essentials 级别的定价为每月 100 美元或 Temporal Cloud 消费的 5%，以较高者为准。
- 商务层的定价为 500 美元/月或 Temporal Cloud 消费的 10%，以较高者为准。

您的 Temporal Cloud 消耗结合了操作和存储的成本。

**例子**

如果您注册了 Essentials，每月支出为 3,000 美元，您的账单将是：

```
Greater of $100 or 5% ⨉ $3,000 = $150, so $150.
```

## 承诺定价 {#commitment-pricing}

**使用 Temporal 积分的承诺**

Temporal Cloud 提供在给定时间范围内承诺最低支出的选项。
作为这一承诺的交换，您将获得额外的折扣。
主要折扣杠杆包括：

- 账户操作量超过 200M 操作
- 您的承诺期限（1、2 或 3 年）

通过任何 Temporal Cloud 支出履行您的承诺，包括操作、存储和您的 Temporal Cloud 计划。
做出承诺后，Temporal 会根据您的预期数量锁定您的 Actions 价格，并折扣您的 Active Storage 成本。
此价格用于在您的承诺中指定的时间范围内对您帐户中的操作和活动存储进行计费。

承诺必须用 Temporal 积分支付。
Temporal 积分用于支付您的 Temporal Cloud 消费，包括 Temporal Cloud 计划费用。
Temporal 积分相当于 1 美元。
例如，购买 20,000 美元的积分可得到 20,000 个 Temporal 积分。
最低信用购买量需要相当于您承诺的第一年。
对于多年优惠，请联系 [销售量](mailto:sales@temporal.io) 以获取最准确的定价。

### 承诺定价问答 {#commitment-pricing-qa}

**多年承诺如何发挥作用？**

我们的销售团队与您合作，根据您的预期支出匹配年度信用购买。
这使您的付款符合年度条款，而不是一笔预付费用。

**如果我在任期结束前用完基于承诺的 Temporal 积分，会发生什么情况？**

在剩余期限内，您将继续获得协商的折扣价格。
您将根据您最近日历月的支出收到另一笔信用购买的发票。
该金额乘以您任期的年度部分中剩余的月份。
如果您上个月的支出为 5,000 美元，并且您的年度期限还有 10 个月，您将收到 10,000 个 Temporal 积分的发票，以支付剩余两个月的费用。

**如果我在学期结束时还有未使用的 Temporal 积分会怎样？**

承诺可能很难估计。
Temporal Cloud 提供两种方式来延续未使用的积分：

- 当您续订相同或更大金额的承诺时，Temporal Cloud 会将任何未使用的积分转入您的新承诺。
- 如果您需要缩小承诺规模，Temporal Cloud 会将您初始信用购买金额的最多 10% 转入新承诺。

**我如何做出承诺并购买 Temporal 积分？**

请通过 [销售@Temporal.io](mailto:sales@temporal.io) 联系我们的团队或联系您的专属客户经理。
您还可以通过 [AWS 市场](https://aws.amazon.com/marketplace/pp/prodview-xx2x66m6fp2lo) 购买 Temporal Cloud 承诺积分。

## 其他定价 {#other-pricing}

Temporal Cloud 对该平台的其他元素有额外的定价。

### 容量模式 {#capacity-modes-pricing}

**什么是容量模式？**

Temporal 提供按需和预配置容量模式。 
按需容量会根据过去的使用情况自动调整。 
预配置容量模式允许您定义 Workflow 所需的容量，并且对于处理超出标准按需限制的流量非常有用。
有关如何设置容量以及相关限制的详细信息，请参阅 [容量模式](/cloud/capacity-modes)。

:::tip Support, stability, and dependency info

预配置容量当前为 [公开预览](/evaluate/development-production-features/release-stages#public-preview)。

:::

**容量模式的定价如何运作？**

累积的操作数可能会受到容量模式的影响。 
对于按需容量模式，操作会照常累积。 
对于预配置容量，每个日历小时必须为预配置的每个 [Temporal 资源单元 (TRU)](/cloud/capacity-modes) 使用最少数量的操作。
如果一个日历小时内的Action量没有超过最小分配量，子类型`ns_capacity:tru`的Action将被记录到所需的量。 
第一个 TRU 的最低要求是 0 个操作，因为它与 Temporal 上的默认速率限制一致。 
对于每个额外的 TRU，每小时的最低要求为 360,000 次操作。 
每小时 360,000 个操作表示所请求资源的利用率为 20%。 
这可以计算为每个 TRU 500 APS * 3600 秒（1 小时）* 20%= 360,000 个操作。

例如：

如果您有一个 Namespace，请求 4 个 TRU（即 2,000 APS）4 小时，并且使用情况如下：
* 第 1 小时：2,000,000 次操作
* 第 2 小时：5,000,000 次操作
* 第 3 小时：4,000,000 个操作
* 第 4 小时：500,000 次操作
那么预配置容量的成本将计算如下：

请求 4 个 TRU - 包含 1 个默认 TRU = 具有最低使用要求的 3 个 TRU
3 个 TRU * 每小时 360,000 个操作= 每小时最少 1,080,000 个操作
* 第 1 小时：2,000,000 次操作 > 最少 1,080,000 次操作 = 不产生额外操作
* 第 2 小时：5,000,000 次操作 > 最少 1,080,000 次操作 = 不产生额外操作
* 第 3 小时：4,000,000 次操作 > 最少 1,080,000 次操作 = 不产生额外操作
* 第 4 小时：500,000 个操作 < 1,080,000 个最小操作，因此 1,080,000-500,000 个操作 = 580,000 个额外操作将添加到该小时中
4 小时总计：2,000,000+5,000,000+4,000,000+1,080,000=12,080,000 次操作

如果 TRU 在一个小时内多次更改，则该小时内的最高值将用于计算所需的最少操作。

Temporal 的预配置容量定价方法符合我们的目标，即仅按您使用的量收费。 
如果您使用的请求容量超出默认值 20% 或更多，则最低每小时分配不会对您的价格产生影响，并且仅在未使用请求的容量时才会产生额外费用。 
为了避免对预配置容量收取费用，建议将容量“返回”到池中，并在使用情况稳定时切换回按需模式。



### 高可用性功能定价 {#high-availability-features}

**高可用性 (HA) 功能的定价如何运作？**

对于具有严格高可用性要求的工作负载，Temporal Cloud 提供同区域、多区域和多云副本，从而增加了故障转移功能。
为 Namespace 启用 HA 功能会自动将 Workflow Execution 数据和元数据复制到同一区域或不同区域的副本。
这允许在发生事件或中断时实现近乎无缝的故障转移。

高可用性功能的定价与您的工作负载量一致。
Namespace 中的操作和存储会影响您的操作和存储消耗。
要估算此部署模型的成本，请将 2 倍乘数应用于您正在复制的 Namespace 中的操作和存储，并将此扩展包含在您的账户消耗中。
对于添加副本的预先存在的 Namespace，2 倍乘数仅适用于副本处于活动状态后的操作和存储。
在创建副本之前或在创建副本但尚未处于“活动”状态时使用的存储和完成的操作不会产生额外费用。

:::tip Future Pricing Update

未来，对于需要进行大量跨云数据传输的 Namespace，Temporal 将收取额外费用，以覆盖云提供商收取的出口费用。
所有Namespace都将免费获得大量的网络转账，因此绝大多数Namespace将不收取任何费用。

:::

升级现有 Namespace 时，需要考虑以下几点：

- Temporal 不会对升级到具有高可用性功能的 Namespace 之前完成的历史操作收费。
  只有正在进行的（飞行中）和新的 Workflow Execution 才会产生消耗。
- Temporal 从添加副本开始对现有（正在进行的）和新 Workflow 的所有操作进行收费。
- Temporal 从添加新副本开始对保留（历史）、运行（持续）和新 Workflow Execution 的复制存储进行收费。

### 公平定价 {#fairness-pricing}

**公平定价如何运作？**

当在 Namespace 上启用 [公平性](/develop/task-queue-priority-fairness#task-queue-fairness) 时，无论单个 Workflow 或活动是否使用公平密钥，该 Namespace 中的每个操作都会在功能启用的每个小时内收取额外的 `0.1` 操作费用。

下面的示例使用 Namespace、`your-Namespace`，通常每小时生成 10,000 个操作。

**启用公平性，但未使用公平性密钥**

如果启用了公平性，但没有 Workflow 或活动发送 `fairness_key` 元数据，则所有任务将继续像以前一样路由。公平操作仍然适用：计费取决于 Namespace 上是否启用公平性。

| 小时          | Namespace 中的操作 | 启用公平性？ | 公平行动 | 总计费操作数 |
|---------------|----------------------|-------------------|--------------------|----------------------|
| 下午 2:00–3:00  | 10,000               | 是的               | +1,000             | 11,000               |

**带有和不带有公平键的 Workflow 的混合**

假设 `your-Namespace` 有两个 Task Queue：
- `payment-queue`，其中 Workflow 传递 `fairness_key = "customer-id"`
- `notification-queue`，其中 Workflow 没有公平键并且行为与以前完全相同

如果每个队列在给定小时内生成 5,000 个操作（总共 10,000 个），则公平操作适用于全部 10,000 个操作，而不仅仅是 `payment-queue` 中的 5,000 个操作。

| 小时          | 支付队列中的操作 | 通知队列中的操作 | Namespace 操作总数 | 公平性 | 总计费操作数 |
|---------------|----------------------------|---------------------------------|-------------------------|-----------------|----------------------|
| 下午 2:00–3:00  | 5,000                      | 5,000                           | 10,000                  | +1,000          | 11,000               |

### SCIM 和 SSO 通过 SAML 定价 {#sso-and-saml}

**使用 SSO/SAML 会产生哪些成本？**

商业、企业和关键任务计划的所有客户都包含使用 SAML 的单点登录 (SSO) 集成。

**与 SCIM 相关的费用有哪些？**

要启用 SCIM（跨域身份管理系统），您需要企业或关键任务计划或业务计划的附加组件（除业务计划外，每月还需+ 500 美元）。
请注意，您必须通过 SAML 配置 SSO 才能使用 SCIM。

## 计费问题常见问题解答 {#pricing-faq}

**Temporal 接受哪些付款方式？**

您可以使用信用卡、ACH 或电汇付款。
要使用 AWS 账户支付 Temporal Cloud，请在 AWS Marketplace 上注册 [Temporal Cloud 即用即付](https://aws.amazon.com/marketplace/pp/prodview-xx2x66m6fp2lo)。

**我多久需要付费一次？**

Temporal Cloud 开具上个月的使用量和费用的发票。
上个月的发票于当月 3 号开具。
例如，5 月份的发票将于 6 月 3 日午夜（世界标准时间）开具。

**我在哪里可以查看我的使用情况和账单信息？**

账户所有者和财务管理员可以随时查看其详细的账单数据。
访问 Temporal Cloud 中的 [使用情况和计费仪表板](/cloud/billing-and-cost)。

**如何购买 Temporal Cloud 积分？**

您可以通过 [销售@Temporal.io](mailto:sales@temporal.io) 联系我们的团队来购买 Temporal Cloud 积分。

**运行 Temporal Cloud 的最低成本是多少？**

Essentials 计划起价为 100 美元/月。
超出计划分配的消耗按消耗量计费。

**我可以通过 Amazon、Azure 或 Google Cloud Platform Marketplace 购买 Temporal Cloud 吗？**

通过 AWS Marketplace 购买 Temporal Cloud 有两种方法：

- 可用即付即用 [这里](https://aws.amazon.com/marketplace/pp/prodview-xx2x66m6fp2lo)
- 积分：通过私人优惠提供，请通过 [销售@Temporal.io](mailto:sales@temporal.io) 联系我们的团队

要在 Google Cloud Marketplace 上购买 Temporal Cloud，请通过 [销售@Temporal.io](mailto:sales@temporal.io) 联系我们的团队。

**如何查看剩余多少 Temporal Cloud 积分？**

要查看剩余的 Temporal Cloud 积分，账户所有者和财务管理员可以登录 Temporal Cloud 并转至“设置”>“账单”。
您需要适当的管理权限才能访问此部分。

**如果我超出了启动计划等促销活动中的可用积分，会发生什么情况？**

对于从启动计划或促销活动中获得免费积分的客户，一旦其积分余额在该月底用完，就会向其开具发票。

**促销积分会过期吗？**

通过启动计划或优惠获得的积分有一个到期日。
该日期是注册过程的一部分。

**如何获取 Update 我的付款信息？**

账户所有者和财务管理员可以随时在计划选项卡下的 Temporal Cloud [计费](https://cloud.temporal.io/billing) 页面上查看 Update 付款信息。
您需要适当的管理权限才能访问此部分。
选择“管理付款方式”按钮。

有关详细信息，请参阅 [本概述](/cloud/billing-and-cost)。

**如果我的付款失败会怎样？**

Temporal将定期向您发送电子邮件提醒您完成付款。

**如何查看我的发票和账单历史记录？**

发票通过电子邮件发送给帐户所有者或指定的帐单联系人。
帐户所有者和财务管理员可以随时查看他们的 [详细的账单信息](https://cloud.temporal.io/billing)。
有关详细信息，请参阅我们的 [计费和成本](/cloud/billing-and-cost) 页面。
您需要适当的管理权限才能访问此部分。
或者，要查看发票和账单历史记录，请通过 [ar@Temporal.io](mailto: ar@temporal.io) 联系 Temporal 财务部。

**Temporal 收取销售税/增值税吗？**

我们根据要求在美国司法管辖区收取适用的销售税。

**我如何取消我的帐户？**

帐户所有者可以在计费中心的“计划”选项卡中删除其帐户并取消其订阅。
有关如何访问计费中心的详细信息，请参阅 [计费和成本](/cloud/billing-and-cost) 页面。

**如果我取消帐户，我会立即失去访问权限吗？**

一旦 Temporal 完成下线流程，客户将失去对 Temporal Cloud 的访问权限。
计费独立于此过程。

**取消后我可以重新激活我的帐户吗？**

不。
当您的帐户被取消时，您的帐户数据将被删除且无法恢复。
要返回 Temporal Cloud，您必须重新注册。
我们将为您分配一个新的 Temporal 帐户，并将您视为新客户。
