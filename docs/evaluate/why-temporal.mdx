# 为什么选择 Temporal？

Temporal 解决了开发者在构建分布式应用时面临的许多问题。
但大多数问题都围绕以下三个主题：

- 可靠的分布式应用
- 高效的开发范式和代码结构
- 分布式应用状态的可见性

## 可靠的执行

**Temporal 如何使应用更可靠？**

Temporal 使开发者更容易构建和运营可靠、可扩展的应用，同时不牺牲生产力。
系统的设计确保一旦启动，应用的主函数就会执行到完成，无论这需要几分钟、几小时、几天、几周甚至几年。
Temporal 将这种能力称为 _持久执行（Durable Execution）_。

## 代码结构

**Temporal 如何简化软件开发者的应用代码？**

通过将故障处理的负担从应用转移到平台，应用开发者需要编写、测试和维护的代码更少。
Temporal 的编程模型为开发者提供了一种将业务逻辑表达为连贯 _Workflow_ 的方式，这比分布式代码库更容易开发。

选择最适合您首选编程语言的 SDK，开始编写您的业务逻辑。
将您喜爱的 IDE、库和工具集成到您的开发流程中。
Temporal 还支持多语言和惯用编程——这使开发者能够利用各种编程语言的优势，并将 Temporal 集成到现有代码库中。
开发者无需管理队列或复杂的状态机即可实现所有这些目标。

## 状态可见性

**Temporal 如何使查看应用状态变得更加容易？**

Temporal 提供开箱即用的工具，使开发者可以随时查看应用的状态。
Temporal CLI 允许开发者有效地管理、监控和调试 Temporal 应用。
基于浏览器的 Web UI 让您可以快速隔离、调试和解决生产问题。
