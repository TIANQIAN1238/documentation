Temporal Web UI 为用户提供 Workflow Execution 状态和元数据，用于调试目的。它随每个 [Temporal CLI](/cli) 版本一起发布，也可在 [Temporal Cloud](/cloud) 中使用。

您可以配置 Temporal Web UI 以在自己的环境中运行。请参阅
[UI 配置参考](/references/web-ui-configuration)。

Web UI 开源仓库：

- [temporalio/ui](https://github.com/temporalio/ui)
- [temporalio/ui-server](https://github.com/temporalio/ui-server)

## Namespaces

您的自托管 Temporal Service 或 Temporal Cloud 账户中的所有 Namespace 都列在窗口左侧的 **Namespaces** 下。您也可以在 Workflows 视图中，通过窗口右上角的 Namespace 切换器来切换 Namespace。选择一个 Namespace 后，Web UI 会显示该 Namespace 的最近 Workflows 页面。在 Temporal Cloud 中，用户只能访问已被授权访问的 Namespace。详情请参阅 [Namespace 级别权限](/cloud/manage-access/roles-and-permissions#namespace-level-permissions)。

## Workflows

主 Workflows 页面显示保留期内所有 Workflow Execution 的表格。

用户可以按以下任意条件列出 Workflow Execution：

- [状态](/workflow-execution#workflow-execution-status)
- [Workflow ID](/workflow-execution/workflowid-runid#workflow-id)
- [Workflow Type](/workflow-definition#workflow-type)
- 开始时间
- 结束时间
- 使用 [List Filter](/list-filter) 的任何默认或自定义 [Search Attribute](/search-attribute)

对于开始时间和结束时间，用户可以将首选日期和时间格式设置为以下之一：

- UTC
- 本地时间
- 相对时间

选择一个 Workflow Execution 可查看该 Workflow Execution 的历史记录、Worker、关系、待处理的 Activity 和 Nexus 操作、Query 以及元数据。

### 已保存的视图 {#saved-views}

已保存的视图（Saved Views）让您可以在 Temporal Web UI 中保存并重复使用常用的可视化查询。您无需每次都重新创建复杂的过滤条件，只需保存一次即可一键应用。

已保存的视图存储在浏览器本地，每当您在此浏览器中使用 Temporal Web UI 时都可使用。每位用户拥有自己的私有集合。

#### 应用已保存的视图

默认情况下，Workflows 页面有几个默认的已保存视图。您也可以创建自己的已保存视图。

点击列表中某个已保存视图的名称，即可显示与该查询匹配的相应 Workflow。

Workflow 列表页面将刷新并显示已保存视图的结果。

#### 创建已保存的视图

您可以从 Workflows 页面创建新的已保存视图。

1. 使用过滤器 UI 构建查询条件来创建已保存视图，或者使用原始查询编辑器编写自定义查询字符串。
1. 您的新视图将以"New View"的名称出现在自定义视图列表中。点击"Save as New"按钮弹出"Save as View"窗口。为您的已保存视图命名。名称对每位用户必须唯一，最多可包含 255 个字符。
1. 点击保存。您的新视图将出现在自定义视图列表中。

您最多可以创建 20 个已保存的视图。达到此限制后，需要删除一些已保存视图才能保存新的视图。

#### 临时修改已保存视图的查询

您可以临时修改已保存视图而不更改已保存的条件。

1. 选择要更改的已保存视图。
1. 根据需要调整 UI 过滤器。
1. Workflows 页面将以新查询的结果刷新，但不会更改已保存的视图。
1. 如果要保留临时更改，您可以：
   - 点击"保存"，这将用您的修改替换原始的已保存视图。
   - 点击"编辑"，修改名称，然后点击"保存"，这将用您的修改替换原始的已保存视图并更改名称。
   - 点击"编辑"，修改名称，然后点击"Create New"，这将以您的新设置和新名称创建一个新的已保存视图。

#### 重命名已保存的视图查询

您可以从 Workflows 页面重命名现有的已保存视图。

1. 选择要更改的已保存视图。
1. 点击"编辑"。
1. 在"Edit View"对话框中，为已保存视图输入新名称。
1. 点击"保存"以应用更改并重命名现有的已保存视图，或点击"Create Copy"以使用新名称创建新的已保存视图。

#### 删除已保存的视图

您可以从 Workflows 页面删除已保存的视图，因为它不再有用，或为新的已保存视图腾出空间。

1. 选择要删除的已保存视图。您只能删除自己创建的查询；无法删除系统默认的视图。
1. 点击"Edit"，然后点击"Delete this Saved View"。

:::note 删除已保存的视图是永久性的

已删除的查询无法恢复，请确保您不再需要它们。如果意外删除了已保存的列表，您需要重新创建它。

:::

#### 分享已保存的视图

您可以将已保存的视图作为 URL 分享。

1. 选择要分享的已保存视图。
1. 点击"Share"按钮将此已保存视图的 URL 复制到剪贴板。您也可以直接从浏览器复制 URL。

:::note 已保存视图与时间

使用相对时间的已保存视图在分享时将以绝对时间显示。

:::

## Task 失败视图 {#task-failures-view}

Task 失败视图是一个预定义的已保存视图，显示发生 Workflow Task 失败的 Workflow。这些 Workflow 仍在运行，但其中某个 Task 已失败或超时。

Task 失败视图的详情显示 Workflow 的 ID、Run ID 和 Workflow 类型。点击详情中的任意链接将打开该 Workflow 的 Workflow 页面。在此页面上，您可以找到有关失败 Task 和剩余待处理 Task 的更多信息。您也可以点击此页面上的"Request Cancellation"按钮来取消 Workflow。

我们的系统实时监控 Workflow Task 执行模式。当一个 Workflow 连续发生五次 Task 失败或超时时，它会被自动标记。一旦 Workflow 通过成功的 Task 恢复，标记就会清除。这个智能阈值能过滤掉轻微故障，同时显示真正存在问题的 Workflow。

### 激活 Task 失败视图 {#activate-task-failures-view}

对于 Temporal Cloud 用户，此功能默认启用。如果您是自托管 Temporal，您需要更新 `system.numConsecutiveWorkflowTaskProblemsToTriggerSearchAttribute` [动态配置](/references/dynamic-configuration)。

以下是如何为开发服务器进行配置更新的示例：

```command
temporal server start-dev \
 --dynamic-config-value system.numConsecutiveWorkflowTaskProblemsToTriggerSearchAttribute=5
```

`numConsecutiveWorkflowTaskProblemsToTriggerSearchAttribute` 是触发 `TemporalReportedProblems` Search Attribute 所需的连续 Workflow Task 失败次数。默认值为 5。如果添加此 Search Attribute 给可视化系统带来压力，可以考虑增大此数值。

要为某个 Namespace 关闭此功能，请将 `numConsecutiveWorkflowTaskProblemsToTriggerSearchAttribute` 设置为 0。

## 历史记录

Workflow Execution 历史记录是 Workflow Execution 中 [Event](/workflow-execution/event#event) 和 Event 字段的视图。大约有 [40 种不同的 Event](/references/events) 可能出现在 Workflow Execution 的 Event 历史记录中。

页面顶部列出了以下执行元数据：

- 开始时间、关闭时间和持续时间
- [Run Id](/workflow-execution/workflowid-runid#run-id)
- [Workflow Type](/workflow-definition#workflow-type)
- [Task Queue](/task-queue)
- Parent 和 Parent ID
- SDK
- [State Transitions](/workflow-execution#state-transition)
- [可计费操作数量](/cloud/actions#actions-in-workflows)（仅 Temporal Cloud）

"输入和结果"部分显示函数参数和返回值，用于调试目的。在 Workflow 完成之前，结果不可用。

历史记录标签页有以下视图：

- 时间线：按时间正序或倒序排列的 Event 摘要。点击某个 Event 可显示该 Event 的所有详情。
- 全部：查看所有历史 Event。
- 紧凑：Activity、Signal 和 Timer 的逻辑分组。
- JSON：Workflow 的完整 JSON 代码。

### 下载 Event 历史记录

可以从此部分下载整个 Workflow Execution 的 Event 历史记录（JSON 格式）。

### Workflow 操作

Workflow Execution 可以直接从 UI 请求取消、发送 Signal 或 Update，或者进行 Reset 和终止。使用"Start Workflow Like This One"按钮以预填充值启动新的 Workflow Execution。

### 关系

显示 Workflow Execution 的完整层级结构，所有父节点和子节点均以树形显示。

### Worker

显示当前正在 Workflow Task Queue 上轮询的 Worker 及其数量。如果没有 Worker 在轮询，则显示错误。

### 待处理的 Activity

显示最近活跃和/或待处理的 Activity Execution 摘要。点击待处理的 Activity 会将用户引导至"Pending Activities"标签页以查看详情。

### 调用栈

屏幕显示 [\_\_stack_trace](/sending-messages#stack-trace-query) Query 的捕获结果。该 Query 在选择标签页时执行。仅当 Worker 正在运行且可以返回调用栈时才有效。调用栈显示 Workflow 代码正在等待的每个位置。

### Query

列出发送到 Workflow Execution 的所有 Query。

### 元数据

显示用户元数据，包括静态的 Workflow 摘要和详情以及动态的当前详情。列出所有带有用户元数据的 Event，为您提供 Workflow 中正在发生的事情的可读日志。

## Schedule

在 Temporal Cloud 和自托管 Temporal Service Web UI 上，Schedule 页面列出了在所选 Namespace 上创建的所有 [Schedule](/schedule)。

点击某个 Schedule 可查看详情，例如配置的频率、开始和结束时间以及最近和即将到来的运行记录。

:::tip 使用字符串设置 Schedule

Temporal Workflow Schedule Cron 字符串遵循以下格式：

```
┌───────────── minute (0 - 59)
│ ┌───────────── hour (0 - 23)
│ │ ┌───────────── day of the month (1 - 31)
│ │ │ ┌───────────── month (1 - 12)
│ │ │ │ ┌───────────── day of the week (0 - 6) (Sunday to Saturday)
│ │ │ │ │
* * * * *
```

:::

### 设置

在 Temporal Cloud 上，**设置**仅对账户所有者和全局管理员[角色](/cloud/manage-access/roles-and-permissions#account-level-roles)可见。

点击**设置**可以查看和管理账户中的用户列表，并设置集成，例如[可观测性](/cloud/metrics)和[审计日志](/cloud/audit-logs)。

在自托管 Temporal Service 上，在您的[服务器配置](/references/configuration)中管理用户、指标和日志。

### 归档

在自托管 Temporal Service 上，Archive 显示 Namespace 上 Workflow Execution 的[已归档](/temporal-service/archival)数据。

要在自托管 Temporal Service 中查看数据，您必须已[设置并配置归档](/self-hosted-guide/archival)。

有关 Temporal Cloud 中归档功能的信息和详情，请联系您的 Temporal 代表。

### Codec Server

Web UI 可以使用带有自定义 Data Converter 的 [Codec Server](/codec-server) 来解码输入和返回值。详情请参阅[保护您的数据](/production-deployment/data-encryption)。

UI 支持 [Codec Server 端点](/production-deployment/data-encryption#web-ui)。有关设置 Codec Server 端点的详情，请参阅 [Codec Server 设置](/production-deployment/data-encryption#codec-server-setup)。
