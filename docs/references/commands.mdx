[命令](/workflow-execution#command) 是 [Workflow 任务执行](/workers#worker) 完成后由 [Worker](/temporal-service) 向 [Temporal 服务](/tasks#workflow-task-execution) 发出的请求操作。

以下是可能命令的完整列表。

<a id="completeworkflowexecution"></a>
### 完成Workflow执行 {#completeworkflowexecution}

当 Workflow 函数执行返回时，会触发此命令。
它向 Temporal 服务指示 [Workflow Execution](/workflow-execution) 已完成。
该命令对应的 [事件](/workflow-execution/event#event) 是 Workflow Execution [Event History](/workflow-execution/event#event-history) 中最后的少数事件之一。

- 可等待：不，Workflow Execution 不能等待此命令产生的操作。
- 对应事件：[Workflow执行已完成](/references/events#workflowexecutioncompleted)

### ContinueAsNewWorkflowExecution

当从 [Workflow](/workflow-execution/continue-as-new) 内部调用 [Continue-As-New](/workflows) 时，会触发此命令。
该命令的相应事件是 Workflow Execution Event History 中最后一个事件的少数事件之一。

- 可等待：不，Workflow Execution 不能等待此命令产生的操作。
- 对应事件：[Workflow执行继续为新](/references/events#workflowexecutioncontinuedasnew)

<a id="failworkflowexecution"></a>
### 失败Workflow执行 {#failworkflowexecution}

当 Workflow Execution 返回错误或抛出异常时，会触发此命令。

- 可等待：不，Workflow Execution 不能等待此命令产生的操作。
- 对应事件：[Workflow执行失败](/references/events#workflowexecutionfailed)

<a id="cancelworkflowexecution"></a>
### 取消Workflow执行 {#cancelworkflowexecution}

当 Workflow 在收到取消请求（将在 Event History 中显示为 [WorkflowExecutionCancelRequestedEvent](/references/events#workflowexecutioncancelrequested)）后成功清理时，会触发此命令。
此命令的相应事件是 Workflow Execution Event History 中最后一个事件的少数事件之一。

- 可等待：不，Workflow Execution 不能等待此命令产生的操作。
- 对应事件：[Workflow执行取消](/references/events#workflowexecutioncanceled)

<a id="startchildworkflowexecution"></a>
### StartChildWorkflow执行 {#startchildworkflowexecution}

该命令由生成 [儿童 Workflow Execution](/child-workflows) 的调用触发。

- 可等待：是的，Workflow Execution 可以等待此命令产生的操作。
- 对应事件：[子WorkflowExecutionStarted](/references/events#childworkflowexecutionstarted)

默认情况下，待处理的 Child Workflow 不能超过 2,000 个。

<a id="signalexternalworkflowexecution"></a>
### Signal外部Workflow执行 {#signalexternalworkflowexecution}

该命令由调用 [Signal](/sending-messages#sending-signals) 和另一个 Workflow Execution 触发。

- 可等待：是的，Workflow Execution 可以等待此命令产生的操作。
- 对应事件：[Signal外部Workflow执行已启动](/references/events#signalexternalworkflowexecutioninitiated)

默认情况下，与其他 Workflow 的待处理 Signal 不能超过 2,000 个。

### RequestCancelExternalWorkflowExecution

该命令由请求取消另一个 Workflow Execution 的调用触发。

- 可等待：是的，Workflow Execution 可以等待此命令产生的操作。
- 对应事件：[请求取消外部WorkflowExecutionInitiated](/references/events#requestcancelexternalworkflowexecutioninitiated)

默认情况下，与其他 Workflow 的待处理 Signal 不能超过 2,000 个。

<a id="scheduleactivitytask"></a>
### ScheduleActivity任务 {#scheduleactivitytask}

该命令由执行 [Activity](/activities) 的调用触发。

- 可等待：是的，Workflow Execution 可以等待此命令产生的操作。
- 对应事件：[Activity任务Scheduled](/references/events#activitytaskscheduled)

默认情况下，您不能同时执行超过 2,000 个 Schedule 活动。

<a id="requestcancelactivitytask"></a>
### 请求取消Activity任务 {#requestcancelactivitytask}

该命令由请求取消 [Activity 任务](/tasks#activity-task) 的调用触发。

- 可等待：不，Workflow Execution 不能等待此命令产生的操作。
- 对应事件：[Activity任务取消请求](/references/events#activitytaskcancelrequested)

<a id="starttimer"></a>
### 开始Timer {#starttimer}

该命令由启动 Timer 的调用触发。

- 可等待：是的，Workflow Execution 可以等待此命令产生的操作。
- 对应事件：[Timer开始](/references/events#timerstarted)

<a id="canceltimer"></a>
### 取消Timer {#canceltimer}

该命令由取消 Timer 的调用触发。

- 可等待：不，Workflow Execution 不能等待此命令产生的操作。
- 对应事件：[Timer已取消](/references/events#timercanceled)

<a id="recordmarker"></a>
### 记录标记 {#recordmarker}

该命令由SDK触发。

- 可等待：不，Workflow Execution 不能等待此命令产生的操作。
- 对应事件：[标记记录](/references/events#markerrecorded)

### UpsertWorkflowSearchAttributes

该命令由调用“upsert”Workflow [Search Attributes](/search-attribute) 触发。

- 可等待：不，Workflow Execution 不能等待此命令产生的操作。
- 对应事件：[更新插入WorkflowSearchAttributes](/references/events#upsertworkflowsearchattributes)

<a id="protocolmessagecommand"></a>
### 协议消息命令 {#protocolmessagecommand}

此命令有助于保证 Update 等功能的排序约束。

该命令指向创建事件的消息。
因此，仅从命令中，您无法预测生成的事件类型。

<a id="schedulenexusoperation"></a>
### ScheduleNexus操作 {#schedulenexusoperation}

该命令由调用者 Workflow 中执行 Nexus 操作的调用触发。

- 可等待：是的，Workflow Execution 可以等待此命令产生的操作。
- 对应事件：[Nexus操作Scheduled](/references/events#nexusoperationscheduled)

默认情况下，您不能同时执行超过 30 个 Nexus 操作，详细信息请参见 [限制](/workflow-execution/limits#workflow-execution-nexus-operation-limits)。

<a id="cancelnexusoperation"></a>
### 取消Nexus操作 {#cancelnexusoperation}

该命令由请求取消 Nexus 操作的调用触发。

- 可等待：不，Workflow Execution 不能等待此命令产生的操作。
- 对应事件：[Nexus操作取消请求](/references/events#nexusoperationcancelrequested)
