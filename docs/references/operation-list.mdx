＃ 运营

Temporal Cloud [速率限制每秒操作数 (OPS)](/cloud/limits#operations-per-second) 根据 Namespace。操作是指 1. 用户直接执行的任何操作，或 2. Temporal 代表用户在后台执行的、导致 Temporal Server 上加载的任何操作。 Visibility 查询是例外：它们确实会命中服务器（Query 从服务器传递到 Visibility 存储），但主要负载位于 Visibility 系统上。 Visibility 速率限制与 OPS 速率限制是分开的。

以下是操作列表，包括：
- 操作名称
- 描述
- 优先级（前景优先级较高，背景优先级较低）
- 该操作的效果被限制
