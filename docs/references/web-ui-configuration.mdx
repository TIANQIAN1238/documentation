Temporal Web UI 服务器使用配置文件来进行许多 UI 设置。

可以在 [Temporalio/ui-server 存储库](https://github.com/temporalio/ui-server/blob/main/config/development.yaml) 中找到示例development.yaml 文件。

可以创建多个配置文件来配置 UI 的特定区域，例如 Auth 或 TLS。

<a id="auth"></a>
## 授权 {#auth}

配置 Temporal Server 的授权。
设置在启用身份验证时应用。

```yaml
auth:
  enabled: true
  providers:
    label: sso # for internal use; in future may expose as button text
    type: oidc
    providerUrl: https://accounts.google.com
    issuerUrl:
    clientId: xxxxx-xxxx.apps.googleusercontent.com
    clientSecret: xxxxxxxxxxxxxxxxxxxx
    callbackUrl: https://xxxx.com:8080/auth/sso/callback
    scopes:
      - openid
      - profile
      - email
```

<a id="batchactionsdisabled"></a>
## 批量操作禁用 {#batchactionsdisabled}

阻止执行批处理操作。

```yaml
batchActionsDisabled: false
```

<a id="cloudui"></a>
## 云UI {#cloudui}

启用云 UI。

```yaml
cloudUi: false
```

<a id="codec"></a>
## 编解码器 {#codec}

Codec Server 配置。

```yaml
codec:
  endpoint: http://your-codec-server-endpoint
  passAccessToken: false
  includeCredentials: false
  decodeEventHistoryDownload: false
```

<a id="cors"></a>
## 科尔斯 {#cors}

`cors` 字段的名称代表跨源资源共享。
使用此字段提供有权访问 UI 服务器 API 的域列表。

```yaml
cors:
  cookieInsecure: false
  allowOrigins:
    - http://localhost:3000 # used at development by https://github.com/temporalio/ui
```

<a id="defaultnamespace"></a>
## 默认Namespace {#defaultnamespace}

UI 为其加载数据的默认 Namespace。
默认为 `default`。

```yaml
defaultNamespace: default
```

<a id="disablewriteactions"></a>
## 禁用写入操作 {#disablewriteactions}

阻止用户在 Web UI 上执行 Workflow 操作。

此选项影响最近 Workflow 的批量操作以及 Workflow 详细信息页面上的 Workflow 操作。

```yaml
disableWriteActions: false
```

:::note
`disableWriteActions` 会覆盖每个单独的 Workflow 操作的配置值。
将此变量设置为 `true` 将禁用 Web UI 上的所有 Workflow 操作。
:::

<a id="enableui"></a>
## 启用用户界面 {#enableui}

启用浏览器 UI。
可以使用 [Temporal_UI_ENABLED](/references/web-ui-environment-variables#temporal_ui_enabled) 环境变量动态设置此配置。
如果禁用（即设置为 `false`），UI 服务器 API 仍然可用。

```yaml
enableUi: true
```

<a id="feedbackurl"></a>
## 反馈网址 {#feedbackurl}

当用户单击 UI 中的“反馈”按钮时将用户引导至的 URL。
如果未指定，则默认为 UI 的 GitHub Issue 页面。

```yaml
feedbackUrl: https://github.com/temporalio/ui/issues/new/choose
```

<a id="forwardheaders"></a>
## 转发头 {#forwardheaders}

配置转发标头。

```yaml
forwardHeaders:
  -
```

<a id="hidelogs"></a>
## 隐藏日志 {#hidelogs}

如果启用，将禁止将任何服务器日志打印到控制台。

```yaml
hideLogs: true
```

<a id="hideworkflowqueryerrors"></a>
## 隐藏WorkflowQuery错误 {#hideworkflowqueryerrors}

隐藏从 Query 到 Workflow 产生的任何错误。

```yaml
hideWorkflowQueryErrors: false
```

<a id="notifyonnewversion"></a>
## 通知新版本 {#notifyonnewversion}

启用后（即设置为 `true` 时），当 [Temporal Server](/temporal-service/temporal-server) 的较新版本可用时，UI 中会显示一条通知。

```yaml
notifyOnNewVersion: true
```

＃＃ 港口

Temporal Web UI 服务器和任何 API 使用的端口。

```yaml
port: 8080
```

<a id="port"></a>
## 公共路径 {#port}

Temporal Web UI 服务器和任何 API 使用的路径。

```yaml
publicPath: ''
```

<a id="publicpath"></a>
## 刷新间隔 {#publicpath}

配置 UI 服务器读取配置文件中新值的频率。
目前，刷新期间仅传播 [tls](#tls) 配置值。

```yaml
refreshInterval: 1m
```

<a id="refreshinterval"></a>
## 显示Temporal系统Namespace {#refreshinterval}

启用后（即设置为 `true` 时），Temporal 系统 Namespace 在 UI 中变得可见。
Temporal 系统 Namespace 列出了 Temporal 平台使用的 Workflow Execution。

```yaml
showTemporalSystemNamespace: false
```

<a id="showtemporalsystemnamespace"></a>
## TemporalgRPC地址 {#showtemporalsystemnamespace}

Temporal 集群的前端地址。

默认地址是 localhost (127.0.0.1:7233)。

```yaml
temporalGrpcAddress: default
```

<a id="temporalgrpcaddress"></a>
## tls {#temporalgrpcaddress}

Temporal Server 的传输层安全 (TLS) 配置。
启用 TLS 时应用设置。

```yaml
tls:
  caFile: ../ca.cert
  certFile: ../cluster.pem
  keyFile: ../cluster.key
  caData:
  certData:
  keyData:
  enableHostVerification: true
  serverName: tls-server
```

<a id="tls"></a>
## WorkflowCancelDisabled {#tls}

防止用户从 Web UI 取消 Workflow Execution。

```yaml
workflowCancelDisabled: false
```

<a id="workflowcanceldisabled"></a>
## WorkflowResetDisabled {#workflowcanceldisabled}

阻止用户从 Web UI 重置 Workflow。

```yaml
workflowResetDisabled: false
```

<a id="workflowresetdisabled"></a>
## WorkflowSignal禁用 {#workflowresetdisabled}

防止用户从 Web UI 中 Signaling Workflow Execution。

```yaml
workflowSignalDisabled: false
```

<a id="workflowsignaldisabled"></a>
## WorkflowTerminateDisabled {#workflowsignaldisabled}

防止用户从 Web UI 终止 Workflow Execution。

```yaml
workflowTerminateDisabled: false
```
