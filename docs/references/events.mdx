[活动](/workflow-execution/event#event) 由 [Temporal 服务](/temporal-service) 响应外部事件而创建，而 [命令](/workflow-execution#command) 由 [Workflow Execution](/workflow-execution) 生成。
下面列出了 Workflow Execution [Event History](/workflow-execution/event#event-history) 中可能出现的所有事件。

### WorkflowExecutionStarted

这始终是 Workflow Execution Event History 中的第一个 [事件](/workflow-execution/event#event)。
它表明 Temporal 服务收到了生成 Workflow Execution 的请求。

| 场地                              | 描述                                                                                                                                                    |
| ---------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Workflow_type                      | 启动的[Workflow](/workflow-definition#workflow-type)的[姓名](/workflows)。                                                                   |
| 父_Workflow_Namespace          | 父 [Workflow Execution](/namespaces) 的 [Namespace](/workflow-execution)（如果适用）。                                                           |
| 父_Workflow_执行          | 标识父 Workflow 和执行运行。                                                                                                          |
| 父发起的事件 ID          | 该事件对应的 [开始Workflow执行已启动](#startchildworkflowexecutioninitiated) 事件的 ID。                                            |
| 任务队列                         | 该 [Workflow 任务](/task-queue) 已入队的 [Task Queue](/tasks#workflow-task)。                                                                 |
| 输入                              | 由 SDK 反序列化以向 Workflow 提供参数的信息。                                                                              |
| Workflow_execution_timeout         | [Workflow Execution](/workflow-execution) 的总超时时间，包括重试和 Continue-As-New。                                               |
| Workflow_run_timeout               | 单次 Workflow 运行超时。                                                                                                                              |
| Workflow_task_timeout              | 单个 Workflow 任务超时。                                                                                                                             |
| Continuous_execution_run_id         | 前一个 Workflow 的 [运行 ID](/workflow-execution/workflowid-runid#run-id) 继续作为新的，重试或由 Cron 执行到此 Workflow 中。     |
| 发起者                          | 允许 Workflow 继续作为新的 Workflow Execution。                                                                                                   |
| 持续失败                  | 失败的序列化结果。                                                                                                                                |
| 最后完成结果             | 来自先前填写的 [任务](/tasks#task) 的信息（如果适用）。                                                                                  |
| 原始执行运行 ID          | 原来的Workflow的[运行 ID](/workflow-execution/workflowid-runid#run-id)开始了。                                                                    |
| 身份                           | [客户](/self-hosted-guide/security#client-connections) 或请求启动此 Workflow 的父 Workflow [Worker](/workers#worker) 的 ID。 |
| 第一个执行运行 ID             | 第一个 [运行 ID](/workflow-execution/workflowid-runid#run-id)，沿着 [Continue-As-New](/workflow-execution/continue-as-new) 链运行和复位。     |
| 重试策略                       | 由服务的动态配置确定的重试次数。将进行重试，直到达到“Schedule_to_close_timeout”。                  |
| 试图                            | 为完成此任务而尝试的次数。                                                                                              |
| Workflow_execution_expiration_time | Workflow Execution 变为 [暂停](/encyclopedia/detecting-workflow-failures#workflow-execution-timeout) 的绝对时间。                       |
| cron_Schedule                      | 显示 Workflow 的 [克朗 Schedule](/cron-job)（如果适用）。                                                                                             |
| first_Workflow_task_backoff        | 包含 Workflow 的本次迭代为 Scheduled 与下次运行之间的时间量。适用于 Cron 调度。                |
| 备忘录                               | Workflow 中显示的非索引信息。                                                                                                               |
| 搜索属性                  | 提供用于设置 Workflow 的 [Search Attributes](/search-attribute) 的数据。                                                                              |
| 上一个自动重置点             |                                                                                                                                                                |
| 标头                             | [Signal](/sending-messages#sending-signals) 的发送者传递的信息被复制到 [Workflow 任务](/tasks#workflow-task) 中。             |
| 完成回调               | 启动此 Workflow 时附加完成回调。                                                                                                  |

<a id="workflowexecutioncompleted"></a>
### Workflow执行完成 {#workflowexecutioncompleted}

这表明[Workflow Execution](/workflow-execution)已成功完成。 [事件](/workflow-execution/event#event) 包含 Workflow Execution 结果。

| 场地                            | 描述                                                                                                                                  |
| -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------- |
| 结果                           | 已完成的 [Workflow](/workflows) 的序列化结果。                                                                                       |
| Workflow_task_completed_event_id | 报告事件所用的 [Workflow任务已完成](#workflowtaskcompleted) 的 ID。                                              |
| 新的执行运行 ID             | 新 Workflow Execution 的 [运行 ID](/workflow-execution/workflowid-runid#run-id) 是 [克朗 Schedule](/cron-job) 的结果。 |

<a id="workflowexecutionfailed"></a>
### Workflow执行失败 {#workflowexecutionfailed}

此 [事件](/workflow-execution/event#event) 指示 [Workflow Execution](/workflow-execution) 未成功完成并包含 Workflow Execution 错误。

| 场地                            | 描述                                                                                                                                        |
| -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| 失败                          | [Workflow](/workflows) 故障的序列化结果。                                                                                             |
| 重试状态                      | 提供 [任务](/tasks#task) 是否应该重试的原因。                                                            |
| Workflow_task_completed_event_id | 报告事件的 [Workflow任务已完成](/workflow-execution/workflowid-runid#run-id) 的 [运行 ID](#workflowtaskcompleted)。 |
| 新的执行运行 ID             | 由 Cron 或 [重试](/workflow-execution/workflowid-runid#run-id) 启动的新 Workflow 的 [运行 ID](/encyclopedia/retry-policies)。            |

### WorkflowExecutionTimedOut

此 [事件](/workflow-execution/event#event) 类型表示由于 [Workflow](/workflow-execution) 在 [暂停](/temporal-service/temporal-server) 设置内尚未完成，[Workflow Execution](/workflows) 已被 [Temporal Server](/encyclopedia/detecting-workflow-failures#workflow-execution-timeout) 超时。

| 场地                | 描述                                                                                                                             |
| -------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| 重试状态          | 提供 [任务](/tasks#task) 是否应该重试的原因。                                                 |
| 新的执行运行 ID | 由 Cron 或 [重试](/workflow-execution/workflowid-runid#run-id) 启动的新 Workflow 的 [运行 ID](/encyclopedia/retry-policies)。 |

### WorkflowExecutionCancelRequested

此 [事件](/workflow-execution/event#event) 类型表示已发出取消 [Workflow Execution](/workflow-execution) 的请求。

| 场地                       | 描述                                                                                                                                     |
| --------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------- |
| 原因                       | 用户提供的取消请求原因。                                                                                           |
| 外部_发起_事件_id | 请求取消的 [Workflow](/workflow-execution/workflowid-runid#run-id) 中的事件的 [运行 ID](/workflows)（如果适用）。 |
| 外部_Workflow_执行 | 标识外部 Workflow 及其执行的运行。                                                                              |
| 身份                    | 请求取消的 [Worker](/workers#worker) 的 ID。                                                                                 |

### WorkflowExecutionCanceled

这个[事件](/workflow-execution/event#event)类型表示客户端已经确认了取消请求，并且[Workflow Execution](/workflow-execution)已经被取消。

| 场地                            | 描述                                                                                     |
| -------------------------------- | ----------------------------------------------------------------------------------------------- |
| Workflow_task_completed_event_id | 报告事件所用的 [Workflow任务已完成](#workflowtaskcompleted) 的 ID。 |
| 细节                          | 取消时 [Workflow](/workflows) 报告的附加信息。                 |

<a id="workflowexecutionsignaled"></a>
### Workflow执行Signaled {#workflowexecutionsignaled}

此 [事件](/workflow-execution/event#event) 类型表示 [Workflow](/workflows) 已收到 [Signal](/sending-messages#sending-signals) 事件。
事件类型包含 Signal 名称和 Signal 负载。

| 场地       | 描述                                                                                                   |
| ----------- | ------------------------------------------------------------------------------------------------------------- |
| Signal_名称 | 要触发的 Signal 的名称/类型。                                                                          |
| 输入       | 由 SDK 反序列化以向 Workflow 函数提供参数的信息。                    |
| 身份    | 标识 Signal 到 Workflow 的 [Worker](/workers#worker)。                                       |
| 标头      | Signal 的发送者传递的信息被复制到 [Workflow 任务](/tasks#workflow-task) 中。 |

<a id="workflowexecutionterminated"></a>
### WorkflowExecutionTermated {#workflowexecutionterminated}

此 [事件](/workflow-execution/event#event) 类型表示 [Workflow Execution](/workflow-execution) 已被强制终止，并且可能调用了终止 Workflow API。

| 场地    | 描述                                                          |
| -------- | -------------------------------------------------------------------- |
| 原因   | 用户或客户为 Workflow 终止而提供的信息。 |
| 细节  | Workflow 终止时报告的附加信息。    |
| 身份 | 标识请求终止的 Worker。                    |

### WorkflowExecutionContinuedAsNew

此 [事件](/workflow-execution/event#event) 类型表示 Workflow 已成功完成，并且新的 Workflow 已在同一事务中启动。
此事件类型包含最后的 [Workflow Execution](/workflow-execution) 结果以及新的 Workflow Execution 输入。

| 场地                            | 描述                                                                                                          |
| -------------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| 新的执行运行 ID             | 新款Workflow的[运行 ID](/workflow-execution/workflowid-runid#run-id)是由这次Continue-As-New活动开始的。 |
| Workflow_type                    | 由此事件启动的 Workflow 的名称/类型。                                                            |
| 任务队列                       | 该 [Workflow 任务](/task-queue) 已入队的 [Task Queue](/tasks#workflow-task)。                       |
| 输入                            | 由 SDK 反序列化以向 Workflow 提供参数的信息。                                    |
| Workflow_run_timeout             | 单次 Workflow 运行超时。                                                                                    |
| Workflow_task_timeout            | 单个 Workflow 任务超时。                                                                                   |
| Workflow_task_completed_event_id | 报告事件命令所用的 [Workflow任务已完成](#workflowtaskcompleted) 的 ID。              |
| 退避开始间隔           | [续如新](#workflowexecutioncontinuedasnew) Workflow 开始延迟的时间量。        |
| 发起者                        | 允许 Workflow 作为新执行继续。                                                                  |
| 最后完成结果           | 先前完成的任务传递给正在进行的执行的信息。                                        |
| 标头                           | Signal 的发送者传递的信息被复制到 Workflow 任务中。                                |
| 备忘录                             | Workflow 中显示的非索引信息。                                                                     |
| 搜索属性                | 提供用于设置 Workflow 的 [Search Attributes](/search-attribute) 的数据。                                    |

<a id="workflowexecutionoptionsupdated"></a>
### Workflow执行选项Updated {#workflowexecutionoptionsupdated}

此 [事件](/workflow-execution/event#event) 类型表示 Workflow 选项已成为 Updated。
事件类型包含 Updated 选项，例如版本控制覆盖或附加的完成回调。

| 场地                         | 描述                                                                                                            |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| 版本控制_覆盖           | 在此事件中更新插入了版本控制覆盖。如果 nil 或 unset_versioning_override 为 true，则忽略。                    |
| 取消设置版本覆盖     | 在此事件中删除了版本控制覆盖。                                                                             |
| 附加请求 ID           | 请求 ID 附加到正在运行的 Workflow Execution，因此具有相同请求 ID 的后续请求将被重复数据删除。 |
| 附加完成回调 | 附加到正在运行的 Workflow Execution 的完成回调。                                                       |

<a id="workflowtaskscheduled"></a>
### Workflow任务Scheduled {#workflowtaskscheduled}

此 [事件](/workflow-execution/event#event) 类型表示 [Workflow 任务](/tasks#workflow-task) 已成为 Scheduled。
SDK 客户端现在应该能够处理任何新的历史事件。

| 场地                  | 描述                                                                                |
| ---------------------- | ------------------------------------------------------------------------------------------ |
| 任务队列             | 此 Workflow 任务排队的 [Task Queue](/task-queue)。                     |
| 开始到关闭超时 | [Worker](/workers#worker) 收到此任务后处理该任务所花费的时间。 |
| 试图                | 为完成此任务而尝试的次数。                          |

### WorkflowTaskStarted

此 [事件](/workflow-execution/event#event) 类型表示 [Workflow 任务](/tasks#workflow-task) 已启动。
SDK 客户端已获取 Workflow 任务并正在处理新的历史事件。

| 场地              | 描述                                                                                                 |
| ------------------ | ----------------------------------------------------------------------------------------------------------- |
| Scheduled_event_id | 此 Workflow 任务对应的 [Workflow任务Scheduled](#workflowtaskscheduled) 事件的 ID。 |
| 身份           | 标识启动此任务的 [Worker](/workers#worker)。                                            |
| 请求ID         | 标识 Workflow 任务请求。                                                                       |

<a id="workflowtaskcompleted"></a>
### Workflow任务已完成 {#workflowtaskcompleted}

此 [事件](/workflow-execution/event#event) 类型表示 [Workflow 任务](/tasks#workflow-task) 已完成。

| 场地              | 描述                                                                                                 |
| ------------------ | ----------------------------------------------------------------------------------------------------------- |
| Scheduled_event_id | 此 Workflow 任务对应的 [Workflow任务Scheduled](#workflowtaskscheduled) 事件的 ID。 |
| 开始事件 ID   | 该任务对应的 [Workflow任务已开始](#workflowtaskstarted) 事件的 ID。              |
| 身份           | 完成此任务的 [Worker](/workers#worker) 的身份。                                         |
| 二进制校验和    | 完成此任务的 Worker 的二进制 ID。                                                           |

SDK 客户端拾取 Workflow 任务，处理新的历史事件，并且可能会也可能不会要求 [Temporal Server](/temporal-service/temporal-server) 执行其他工作。
以下事件仍有可能发生：

- [Activity任务Scheduled](#activitytaskscheduled)
- [Timer开始](#timerstarted)
- [更新插入WorkflowSearchAttributes](#upsertworkflowsearchattributes)
- [标记记录](#markerrecorded)
- [StartChildWorkflowExecutionInitiated](#startchildworkflowexecutioninitiated)
- [请求取消外部WorkflowExecutionInitiated](#requestcancelexternalworkflowexecutioninitiated)
- [Signal外部Workflow执行已启动](#signalexternalworkflowexecutioninitiated)
- [Workflow执行已完成](#workflowexecutioncompleted)
- [Workflow执行失败](#workflowexecutionfailed)
- [Workflow执行取消](#workflowexecutioncanceled)
- [Workflow执行继续为新](#workflowexecutioncontinuedasnew)

### WorkflowTaskTimedOut

此 [事件](/workflow-execution/event#event) 类型表示 [Workflow 任务](/tasks#workflow-task) 遇到了 [暂停](/encyclopedia/detecting-workflow-failures#workflow-task-timeout)。
要么是带有本地缓存​​的 SDK 客户端当时不可用，要么是 SDK 客户端处理任务的时间太长。

| 场地              | 描述                                                                                                 |
| ------------------ | ----------------------------------------------------------------------------------------------------------- |
| Scheduled_event_id | 此 Workflow 任务对应的 [Workflow任务Scheduled](#workflowtaskscheduled) 事件的 ID。 |
| 开始事件 ID   | 该任务对应的 [Workflow任务已开始](#workflowtaskstarted) 事件的 ID。              |
| 超时类型       | 已发生的超时类型。                                                                      |

<a id="workflowtaskfailed"></a>
### Workflow任务失败 {#workflowtaskfailed}

此 [事件](/workflow-execution/event#event) 类型表示 [Workflow 任务](/tasks#workflow-task) 遇到故障。
通常这意味着 Workflow 是不确定的。
但是，Workflow 重置功能也使用此事件。

| 场地              | 描述                                                                                                                                  |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------- |
| Scheduled_event_id | 此 Workflow 任务对应的 [Workflow任务Scheduled](#workflowtaskscheduled) 事件的 ID。                                  |
| 开始事件 ID   | 此 Workflow 任务对应的 [Workflow任务已开始](#workflowtaskstarted) 事件的 ID。                                      |
| 失败            | Workflow 任务失败的详细信息。                                                                                                     |
| 身份           | 失败此任务的 [Worker](/workers#worker) 的标识。必须显式定义 Worker 才能返回该字段的值。 |
| 基本运行 ID        | Workflow 的原装 [运行 ID](/workflow-execution/workflowid-runid#run-id)。                                                          |
| 新运行ID         | 重置 Workflow 的运行 ID。                                                                                                            |
| fork_event_version | 标识分叉到重置 Workflow 的事件版本。                                                                      |
| 二进制校验和    | 失败此任务的 Worker 的二进制 ID。必须显式定义 Worker 才能返回该字段的值。                   |

<a id="activitytaskscheduled"></a>
### Activity任务Scheduled {#activitytaskscheduled}

此 [事件](/workflow-execution/event#event) 类型表示 [Activity 任务](/tasks#activity-task) 是 Scheduled。
SDK 客户端应选择此 Activity 任务并执行。
此事件类型包含 Activity 输入以及 Activity 超时配置。

| 场地                            | 描述                                                                                                                                        |
| -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| Activity_id                      | 由 [Worker](/workers#worker) 或用户分配给此 Activity 的标识符。                                                                   |
| Activity_type                    | [Activity 类型](/activity-definition#activity-type) 是 Scheduled。                                                                     |
| Namespace                        | [Activity](/activities) 所在的 Workflow 的 Namespace。                                                                             |
| 任务队列                       | 此 Activity 任务排队的 [Task Queue](/task-queue)。                                                                             |
| 标头                           | [Signal](/sending-messages#sending-signals) 的发送者传递的信息被复制到 [Workflow 任务](/tasks#workflow-task) 中。 |
| 输入                            | 由 SDK 反序列化以向 [Workflow](/workflows) 函数提供参数的信息。                                           |
| Schedule_to_close_timeout        | 调用者等待 Activity 完成的时间。限制对此 Activity 尝试重试的时间。    |
| Schedule_to_start_timeout        | 限制 Activity 任务可以在 Task Queue 中停留的时间。此超时无法重试。                                                    |
| 开始到关闭超时           | Activity 被 Worker 选取后允许的最大执行时间。此超时是可重试的。                         |
| 心跳超时                | 成功的 Worker 心跳之间允许的最长时间。                                                                               |
| Workflow_task_completed_event_id | 报告事件所用的 [Workflow任务已完成](#workflowtaskcompleted) 的 ID。                                                    |
| 重试策略                     | 由服务的动态配置确定的重试次数。将进行重试，直到达到 `Schedule_to_close_timeout`。      |

### ActivityTaskStarted

此 [事件](/workflow-execution/event#event) 类型表示 [Activity 任务执行](/tasks#activity-task-execution) 已启动。 SDK Worker 获取 Activity 任务并开始处理 [Activity](/activities) 调用。 `ActivityTaskStarted` 是在任务分派到 Worker 时由服务器生成的，而不是在 Worker 开始执行任务时生成的。

但请注意，直到终端事件（如 [Activity任务已完成](#activitytaskcompleted) 或 [Activity任务失败](#activitytaskfailed)）发生后，此事件才会写入历史记录。

| 场地              | 描述                                                                                                          |
| ------------------ | -------------------------------------------------------------------------------------------------------------------- |
| Scheduled_event_id | 该任务对应的 [Activity任务Scheduled](#activitytaskscheduled) 事件的 ID。                   |
| 身份           | 标识启动任务的 [Worker](/workers#worker)。                                                      |
| 请求ID         | 标识 Activity 任务请求。                                                                                |
| 试图            | 为完成此任务而尝试的次数。                                                    |
| 最后一次失败       | 最近失败事件的详细信息。仅当任务先前失败并重试时才分配值。 |

<a id="activitytaskcompleted"></a>
### Activity任务已完成 {#activitytaskcompleted}

此 [事件](/workflow-execution/event#event) 类型表示 [Activity 任务](/tasks#activity-task) 已完成。
SDK 客户端已拾取并成功完成 Activity 任务。
此事件类型包含 [Activity 执行](/activity-execution) 结果。

| 场地              | 描述                                                                                                    |
| ------------------ | -------------------------------------------------------------------------------------------------------------- |
| 结果             | 已完成的 [Activity](/activities) 的序列化结果。                                                      |
| Scheduled_event_id | 该完成事件对应的[Activity任务Scheduled](#activitytaskscheduled)事件的Id。 |
| 开始事件 ID   | 该任务对应的 [Activity任务已开始](#activitytaskstarted) 事件的 ID。                 |
| 身份           | 完成此任务的 [Worker](/workers#worker) 的身份。                                            |

<a id="activitytaskfailed"></a>
### Activity任务失败 {#activitytaskfailed}

此 [事件](/workflow-execution/event#event) 类型表示 [Activity 任务](/tasks#activity-task) 已发生故障。
SDK 客户端拾取了 Activity 任务，但未成功完成。
此事件类型包含 [Activity 执行](/activity-execution) 错误。

| 场地              | 描述                                                                                                 |
| ------------------ | ----------------------------------------------------------------------------------------------------------- |
| 失败            | [Workflow](/workflows) 故障的序列化结果。                                                      |
| Scheduled_event_id | 该失败事件对应的[Activity任务Scheduled](#activitytaskscheduled)事件的Id。 |
| 开始事件 ID   | 此故障对应的 [Activity任务已开始](#activitytaskstarted) 事件的 ID。           |
| 重试状态        | 提供是否应该重试任务的原因。                                    |

### ActivityTaskTimedOut

此 [事件](/workflow-execution/event#event) 类型指示 Activity 已根据 [Temporal Server](/temporal-service/temporal-server) 超时，原因是以下 [Activity](/activities) 超时之一：[Schedule 关闭超时](/encyclopedia/detecting-activity-failures#schedule-to-close-timeout) 和 [Schedule 启动超时](/encyclopedia/detecting-activity-failures#schedule-to-start-timeout)。

| 场地              | 描述                                                                                                 |
| ------------------ | ----------------------------------------------------------------------------------------------------------- |
| 失败            | [Workflow](/workflows) 故障的序列化结果。                                                      |
| Scheduled_event_id | 该超时事件对应的[Activity任务Scheduled](#activitytaskscheduled)事件的Id。 |
| 开始事件 ID   | 此超时对应的 [Activity任务已开始](#activitytaskstarted) 事件的 ID。           |
| 重试状态        | 提供是否应该重试任务的原因。                                    |
| 超时类型       | 导致此事件的超时类型，例如开始到关闭、Schedule 到关闭、Schedule 到开始。     |

您可以运行包含 Activity 执行的 Workflow，该执行的时间比您设置的开始到关闭超时更长，并使用将 MaxAttempts 设置为 1 的 RetryPolicy，以便它不会无限期地重试。
