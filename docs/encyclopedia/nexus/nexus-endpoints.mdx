:::tip SUPPORT, STABILITY, and DEPENDENCY INFO

[Temporal Cloud](/evaluate/development-production-features/release-stages#general-availability) 和 [自托管部署](/cloud/nexus) 的 Temporal Nexus 现在为 [一般可用](/production-deployment/self-hosted-guide/nexus)。

:::

[Nexus 端点](/glossary#nexus-endpoint) 是 [Nexus 服务](/nexus/services) 的完全托管反向代理。
它将来自调用者 Workflow 的请求路由到目标 Namespace 和 Task Queue。
调用者只需要知道Endpoint名称——目标Namespace、Task Queue，以及内部实现被封装。

Worker 通过注册一项或多项服务并轮询端点的目标 Task Queue 来处理 Nexus 请求。
多个端点可以针对同一 Namespace 中的不同 Task Queue。

端点描述字段支持降价以记录可用的操作、联系信息或架构链接。

<a id="reverse-proxy-for-nexus-services-not-a-general-purpose-proxy"></a>
## Nexus 服务的反向代理，不是通用代理 {#reverse-proxy-for-nexus-services-not-a-general-purpose-proxy}
Nexus 端点充当单个 Nexus 服务的反向代理，将请求路由到一个目标 Namespace 和 Task Queue。

与通用代理不同，它不会路由到多个后端。相反，它提供了到特定上游目标的安全、托管连接，该目标可以位于任何区域或云中。
[端点规范](https://github.com/temporalio/api/blob/2a5b3951e71565e28628edea1b3d88d69ed26607/temporal/api/nexus/v1/message.proto#L170) 支持以下 [目标类型](https://github.com/temporalio/api/blob/2a5b3951e71565e28628edea1b3d88d69ed26607/temporal/api/nexus/v1/message.proto#L185)：

- **Worker**：路由到目标 Namespace 和 Task Queue。

<a id="deploying-a-nexus-endpoint"></a>
## 部署 Nexus 端点 {#deploying-a-nexus-endpoint}

将端点添加到 [Nexus 注册表](/nexus/registry) 会立即部署它。端点一经注册即可在运行时可用。
