:::tip SUPPORT, STABILITY, and DEPENDENCY INFO

Temporal Nexus 现已在 [Temporal Cloud](/cloud/nexus) 和[自托管部署](/production-deployment/self-hosted-guide/nexus)上[正式发布](/evaluate/development-production-features/release-stages#general-availability)。

:::


Nexus 支持跨 caller Workflow、Nexus Operation 和 handler Workflow 的端到端执行调试——甚至跨越跨越多个 Namespace 的[多级调用](/nexus#multi-level-calls)。

## 双向链接

双向链接将 caller Workflow 历史中的 Nexus Operation 事件与 handler Workflow 历史中的对应事件相连接。它们由 SDK 构建函数（如 New-Workflow-Run-Operation）自动建立，支持在 Temporal UI 中跨 Namespace、区域和云进行点击导航。

<CaptionedImage
    src="/img/cloud/nexus/nexus-bi-directional-linking.png"
    title="Bi-directional linking"
    zoom="true"
/>

- **正向**：从 caller 的 Nexus Operation 事件导航到 handler 的 Workflow。
- **反向**：从 handler 的 Workflow 返回到 caller 的 Nexus Operation 事件。

## 待处理的 Operation

待处理的 Nexus Operation 显示在 UI 的 Workflow 详情页面上，也可以通过 CLI 使用 `temporal workflow describe` 命令列出。

从 UI 查看：

<CaptionedImage
    src="/img/cloud/nexus/pending-nexus-operations.png"
    title="Pending Operations"
    zoom="true"
/>

从 CLI 查看：

```
temporal workflow describe

Pending Nexus Operations: 1

  Endpoint                 myendpoint
  Service                  my-hello-service
  Operation                echo
  OperationToken
  State                    BackingOff
  Attempt                  6
  ScheduleToCloseTimeout   0s
  NextAttemptScheduleTime  20 seconds from now
  LastAttemptCompleteTime  11 seconds ago
  LastAttemptFailure       {"message":"handler error (INTERNAL): internal error","applicationFailureInfo":{}}
```

[可重试错误](/nexus/error-handling#errors-in-nexus-handlers)会在待处理 Operation 中显示。不可重试错误会通过 [Failed](/references/events#nexusoperationfailed)、[TimedOut](/references/events#nexusoperationtimedout) 或 [Canceled](/references/events#nexusoperationcanceled) 事件解决该 Operation。

## 待处理的回调

Nexus 完成回调从 handler 的 Namespace 发送到 caller 的 Namespace，用于异步 Operation。可以在 UI 中或通过 CLI 使用 `temporal workflow describe` 命令查看这些回调。

从 UI 查看：

<CaptionedImage
    src="/img/cloud/nexus/nexus-callback.png"
    title="Pending Callbacks"
    zoom="true"
/>

从 CLI 查看：

```
temporal workflow describe

Callbacks: 1

  URL               https://nexus.phil-caller-Namespace.a2dd6.cluster.tmprl.cloud:7243/Namespaces/phil-caller-Namespace.a2dd6/nexus/callback
  Trigger           WorkflowClosed
  State             Succeeded
  Attempt           1
  RegistrationTime  32 minutes ago
```

## 链路追踪

Temporal 与 [OpenTelemetry](https://opentelemetry.io/) 和 [OpenTracing](https://opentracing.io/) 集成，可跨 Activity、Nexus Operation 和 Child Workflow 可视化调用图。通过在 Client 或 Worker 上安装拦截器来启用链路追踪：

- [Go SDK](https://github.com/temporalio/samples-go/tree/main/opentelemetry)
- [Java SDK](https://github.com/temporalio/samples-java/tree/main/core/src/main/java/io/temporal/samples/tracing)
- [Python SDK](https://github.com/temporalio/samples-python/tree/main/open_telemetry)
- [TypeScript](https://github.com/temporalio/samples-typescript/tree/main/interceptors-opentelemetry)
- [.NET SDK](https://github.com/temporalio/samples-dotnet/tree/main/src/OpenTelemetry)
