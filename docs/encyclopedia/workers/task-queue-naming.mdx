# Task Queue 名称 {#task-queue-names}

Temporal 服务维护一组 Task Queue，Worker 轮询以查看这些 Task Queue
需要做什么工作。每个 Task Queue 都由一个名称来标识，该名称是
启动 Workflow Execution 时提供给 Temporal 服务。

<Tabs groupId="start-workflow-configure-worker-by-sdk" queryString>

<TabItem value="python" label="Python">

**用于在 Python 中启动 Workflow 的代码摘录**

```python
client = await Client.connect("localhost:7233", namespace="default")

# Execute a workflow
result = await client.execute_workflow(
    GreetingWorkflow.run,
    name,
    id="my-workflow",
    task_queue="my-task-queue-name",
)
```

**用于在 Python 中配置 Worker 的代码摘录**

```python
worker = Worker(
    client,
    task_queue="my-task-queue-name",
    workflows=[GreetingWorkflow],
    activities=[activities.say_hello],
)
```

</TabItem>

**用于在 Go 中配置 Worker 的代码摘录**

```go
w := worker.New(c, "my-task-queue-name", worker.Options{})
```

由于 Task Queue 是在首次使用时动态创建的，因此不匹配
这两个值之间不会导致错误。反而会产生这样的结果
创建两个不同的 Task Queue。因此，Worker 将
未从 Temporal 服务和 Workflow Execution 接收任何任务
不会进步。因此，我们建议您定义 Task Queue 名称
如果可能的话，在客户端和 Worker 引用的常量中，因为这
将确保它们始终使用相同的值。

<TabItem value="python" label="Python">

**用于在 Python 中定义名称为 Task Queue 的常量的代码摘录（在共享.py 文件中）**

```python
    workflows=[GreetingWorkflow],
```

然而，并不总是可以在常量中定义 Task Queue 名称，例如当客户端使用
启动Workflow正在另一个系统上运行或在一个系统中实现
不同的编程语言。
