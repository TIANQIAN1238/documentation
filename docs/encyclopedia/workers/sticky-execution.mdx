<a id="sticky-execution"></a>
## 什么是粘性执行？ {#sticky-execution}

Worker 缓存它们执行的 Workflow 的状态。
为了使缓存更加有效，Temporal 采用了一种称为“粘性执行”的性能优化，它将 Workflow 任务定向到先前处理特定 Workflow Execution 任务的同一 Worker。

<a id="how-sticky-execution-works"></a>
### 粘性执行的工作原理 {#how-sticky-execution-works}

Workflow Execution 开始后，Temporal 服务 Schedule 会创建一个 Workflow 任务，并将其放入具有您指定名称的 Task Queue 中。
任何轮询 Task Queue 的 Worker 都有资格接受任务并开始执行 Workflow。

获取此 Workflow 任务的 Worker 将继续轮询原始 Task Queue，但也将开始轮询其他 Task Queue，Temporal 服务与该特定 Worker 独家共享该 Task Queue。
该队列具有自动生成的名称，称为“粘性队列”。

Worker 将 Workflow 状态缓存在内存中，从而减少了为每个任务从 Event History 重建 Workflow 的需要，从而提高了性能。
随着 Workflow Execution 的进展，Temporal 服务 Schedule 将附加 Workflow 任务放入此 Worker 特定的粘性队列中。

如果 Worker 在 Scheduled 后不久（默认情况下五秒内）无法启动粘性队列中的 Workflow 任务，则 Temporal 服务会禁用该 Workflow Execution 的粘性。
禁用粘性后，Temporal 服务将重新 Schedule 原始队列中的 Workflow 任务，允许任何 Worker 拾取该任务并继续执行 Workflow Execution。

如果 Workflow 任务失败，Worker 会从其缓存中删除该 Workflow Execution（因为它现在处于未知状态），这会使粘性执行无效。
然后将 Workflow 任务放回原始 Task Queue 中。

<a id="why-sticky-execution"></a>
### 为什么要粘性执行？ {#why-sticky-execution}

粘性执行的主要好处是提高性能。
通过将 Workflow 状态缓存在内存中并将任务定向到相同的 Worker，它减少了为每个任务从 Event History 重建 Workflow 的需要，这对于延迟敏感的 Workflow 特别有用。

粘性执行是 Temporal 平台的默认行为，仅适用于 Workflow 任务。
由于 Event History 与 Workflow 关联，因此粘性执行的概念与 Activity 任务无关。
