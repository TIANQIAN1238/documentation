:::info Open source and Temporal Cloud
本页介绍了适用于开源 Temporal 和 Temporal Cloud 的核心 Namespace 概念。

Temporal Cloud Namespace 包括附加功能，例如 [API密钥](/cloud/api-keys) 和 [mTLS认证](/cloud/certificates)、[内置基于角色的访问控制](/cloud/manage-access/roles-and-permissions#namespace-level-permissions)、[高可用性复制](/cloud/high-availability) 和 [Namespace 的标签](/cloud/namespaces#tag-a-namespace)。

从自托管迁移到云，或者相反，需要零代码更改并导致零停机。
:::

Namespace 是 [Temporal平台](/temporal#temporal-platform) 内的一个隔离单元。

[Task Queues](/task-queue) 和 [Workflow Executions](/workflow-execution) 属于 Namespace。
当 Workflow Execution 生成时，它会在特定的 Namespace 内生成。

＃＃ 用法

- **Workflow ID 唯一性**：Temporal 保证 Namespace 内有唯一的 Workflow Id。
  如果 Workflow Execution 位于不同的 Namespace 中，则它们可能具有相同的 Workflow Id。
- **资源隔离**：来自一台 Namespace 的大流量不会影响在同一 Temporal 服务上运行的其他 Namespace。
- **配置边界**：[保留期限](/temporal-service/temporal-server#retention-period) 和 [档案](/temporal-service/archival) 目标等选项是根据 Namespace 配置的。
- **默认 Namespace**：如果未指定 Namespace，则 Temporal 服务对所有 Temporal SDK 和 Temporal CLI 使用 Namespace“默认”。
  您必须先创建 Namespace，然后才能在客户端中使用它。
- **多租户**：单个 Namespace 仍然是多租户。
  多个应用程序或团队可以共享 Namespace，但必须协调 Workflow ID 和 Task Queue 命名以避免冲突。
