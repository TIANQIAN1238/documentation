## 什么是全局 Namespace？ {#global-namespace}

全局 Namespace 是设置 [多集群复制](/namespaces) 时跨集群存在的 [Namespace](/temporal-service/multi-cluster-replication)。

- [如何注册全球 Namespace](/cli/operator#create)
- [如何更改全局 Namespace 的活动集群](/cli/operator#update)

全局 Namespace 功能使 Workflow Execution 在发生故障转移时能够继续通过另一个集群。

全局 Namespace 可以复制到任意数量的集群，但在任何给定时间仅在一个集群中处于活动状态。

为了使故障转移成功，Worker 进程必须轮询所有集群上全局 Namespace 的任务。

Global Namespace 有故障转移版本。
由于可以从任何集群触发故障转移，因此故障转移版本可以防止在两个集群上同时错误地触发故障转移时发生某些冲突。

仅活动集群调度 [任务](/tasks#task)；然而，某些冲突是可能的。
与为 Activity 执行提供最多一次语义的常规 Namespace 不同，全局 Namespace 只能支持至少一次语义（请参阅 [冲突解决](/temporal-service/multi-cluster-replication#conflict-resolution)）。
Worker 备用集群上的进程处于空闲状态，直到发生故障转移并且其集群变为活动状态。

对非活动集群进行的 Temporal 应用程序 API 调用将被拒绝，并出现 **NamespaceNotActiveError**，其中包含当前活动集群的名称。
Temporal 应用程序负责调用当前活动的集群。
