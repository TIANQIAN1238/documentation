本页提供 Temporal Visibility 的概述。

Temporal 平台中的术语 [Visibility](/visibility) 是指使操作员能够查看、过滤和搜索当前存在于 Temporal 服务中的 Workflow Execution 的子系统和 API。

Temporal 服务中的 [Visibility 商店](/self-hosted-guide/visibility) 存储持久的 Workflow Execution Event History 数据，并设置为 [持久性存储](/temporal-service/persistence) 的一部分，以启用列出和过滤有关 Temporal 服务中存在的 Workflow Execution 的详细信息。

- [如何开设 Visibility 商店](/self-hosted-guide/visibility)

使用 Temporal Server v1.21，您可以设置 [双 Visibility](/dual-visibility) 将 Visibility 存储从一个数据库迁移到另一个数据库。

<a id="advanced-visibility"></a>
## Visibility 功能 {#advanced-visibility}

Visibility 可以通过类似 SQL 的自定义 [列表过滤器](/workflow-execution) 来列出、过滤和排序 [Workflow Executions](/list-filter)。

Visibility 支持 [定制 Search Attribute](/search-attribute#custom-search-attribute) 进行默认系统属性之外的用户定义过滤。

- 在 SQL 数据库（MySQL v8.0.17+、PostgreSQL v12+）上，Visibility 可与 Temporal Server v1.20 及更高版本一起使用。
- 在 Elasticsearch（v7+ 与 Temporal Server v1.7+、v8 与 Temporal Server v1.18+）和 OpenSearch（2+ 与 Temporal Server v1.30.1+）上。
- 在 Temporal Cloud 上，默认情况下为 [所有用户](/cloud/users#invite-users) 启用 Visibility。

有关自托管设置和版本兼容性的详细信息，请参阅 [Visibility 商店设置](/self-hosted-guide/visibility)。

<a id="count-workflow-by-executionstatus"></a>
### 按 ExecutionStatus 计数 Workflow {#count-workflow-by-executionstatus}

计数 API 功能可让您计算与给定 Query 匹配的 Workflow 的数量。例如，命令 `Temporal Workflow count -q "WorkflowType='foo'"` 返回与 `WorkflowType='foo'` 匹配的 Workflow 的数量。

您可以将查询发送到 Count API 以按给定的 Search Attribute 进行分组。例如，`-q "WorkflowType='foo' GROUP BY ExecutionStatus` 返回与按 `ExecutionStatus` 分组的 `WorkflowType='foo'` 匹配的 Workflow 的数量。

`GROUP BY` 子句仅在 Count API 中受支持，并且当前仅支持按 `ExecutionStatus` 进行分组。

:::note

计数 API 返回近似计数。

:::

<a id="standard-visibility"></a>
## 旧版：标准 Visibility {#standard-visibility}

在 Temporal Server v1.20 之前，Temporal 有两种 Visibility 模式：“标准”和“高级”。标准 Visibility 仅支持预定义过滤器，例如 Workflow 类型、Workflow Id、运行 Id 和执行状态，而不支持自定义 Search Attribute。高级 Visibility 需要 Elasticsearch。

从 Temporal Server v1.20 开始，高级 Visibility 在 SQL 数据库上可用。标准 Visibility 在 v1.21 中已弃用，并在 v1.24 中删除。当前的所有部署都使用以前称为高级 Visibility 的内容，现在简称为 Visibility。

如果您运行的 Temporal Server 版本早于 v1.24，请参阅自托管指南中的 [传统标准 Visibility 部分](/self-hosted-guide/visibility#legacy-standard-visibility)。
