## 什么是双 Visibility？ {#dual-visibility}

双 Visibility 功能允许您在 Temporal 服务中设置主存储之外的辅助 Visibility 存储。
设置双 Visibility 是可选的，可用于 [迁移您的 Visibility 数据库](/self-hosted-guide/visibility#migrating-visibility-database) 或创建备份 Visibility 存储。

例如，如果您将 Cassandra 配置为 Visibility 数据库，则可以将受支持的 SQL 数据库设置为辅助 Visibility 存储，并在弃用主存储之前逐步将数据迁移到辅助存储。

双 Visibility 设置需要两个 Visibility 存储配置：

- **主 Visibility：** 默认情况下写入和读取 Visibility 数据的主 Visibility 存储。主 Visibility 存储是使用 Temporal 服务中的 `VisibilityStore` 配置密钥设置的。
- **辅助 Visibility：** Visibility 数据的辅助存储。辅助 Visibility 存储使用 Temporal 服务中的 `secondaryVisibilityStore` 配置密钥进行设置。

有关配置详细信息，请参阅 [双 Visibility 设置](/self-hosted-guide/visibility#dual-visibility)。

双 Visibility 设置中允许使用以下组合。

| 基本的                     | 中学                       |
| --------------------------- | ------------------------------- |
| 标准（Cassandra 或 SQL） | 高级（SQL 或 Elasticsearch） |
| 高级（SQL）              | 高级（SQL）                  |
| 高级（弹性搜索）    | 高级（弹性搜索）        |

使用双 Visibility，您一次只能从一个 Visibility 存储中读取数据，但可以将 Temporal 服务配置为仅写入主要存储、仅写入辅助存储，或者同时写入主要和辅助 Visibility 存储。
从一个 Visibility 商店数据库迁移到另一个 Visibility 商店数据库时，请将要迁移到的数据库设置为辅助 Visibility 商店。

您可以使用特定的动态配置键来规划迁移，以帮助您将读写操作从主 Visibility 存储转移到辅助 Visibility 存储。
有关迁移 Visibility 商店数据库的详细信息，请参阅 [双 Visibility](/self-hosted-guide/visibility#dual-visibility)。
