本页讨论 [Workflow Execution 限制](#workflow-execution-limits)、[Workflow Execution 回调限制](#workflow-execution-callback-limits) 和 [Nexus 操作限制](#workflow-execution-nexus-operation-limits)。

<a id="workflow-execution-limits"></a>
## 限制 {#workflow-execution-limits}

并发 Workflow Execution 的数量没有限制，但您必须遵守 Workflow Execution 的 Event History 限制。

:::caution

作为预防措施，Workflow Execution 的 Event History 仅限于 [51,200 场活动](https://github.com/temporalio/temporal/blob/e3496b1c51bfaaae8142b78e4032cc791de8a76f/service/history/configs/config.go#L382) 或 [50MB](https://github.com/temporalio/temporal/blob/e3496b1c51bfaaae8142b78e4032cc791de8a76f/service/history/configs/config.go#L380)，并将在 10,240 个事件或 10 MB 后向您发出警告。

:::

某些类型的不完整操作的数量也有限制。

每个正在进行的 Activity 都会在 Workflow Execution 的可变状态下生成一个元数据条目。
单个 Workflow Execution 的可变状态中的条目过多会导致持久性不稳定。
为了保护系统，Temporal 强制每个 Workflow Execution 的不完整活动、Child Workflow、Signal 或取消请求的最大数量（默认情况下，每种类型的操作为 2,000）。
一旦达到某种操作类型的限制，如果 Workflow Execution 尝试启动该类型的另一个操作（通过生成 `ScheduleActivityTask`、`StartChildWorkflowExecution`、`SignalExternalWorkflowExecution` 或 `RequestCancelExternalWorkflowExecution` 命令），它将无法启动（Workflow 任务执行将失败并重试）。

这些限制是通过以下 [动态配置键](https://github.com/temporalio/temporal/blob/main/service/history/configs/config.go) 设置的：

- `NumPendingActivitiesLimit`
- `NumPendingChildExecutionsLimit`
- `NumPendingSignalsLimit`
- `NumPendingCancelRequestsLimit`

<a id="workflow-execution-callback-limits"></a>
## Workflow Execution 回调限制 {#workflow-execution-callback-limits}

可附加到单个 Workflow Execution 的 Workflow 回调总数有限制。
连接 [处理程序 Workflow 的多个 Nexus 调用者](/nexus/operations#attaching-multiple-nexus-callers) 可能会超出这些限制。

可以使用以下动态配置键设置这些限制：
- [MaxCallbacksPerWorkflow](https://github.com/temporalio/temporal/blob/3b626075691c483871630d4a4df266e783f86328/common/dynamicconfig/constants.go#L998)
- [MaxCHASMCallbacksPerWorkflow](https://github.com/temporalio/temporal/blob/3b626075691c483871630d4a4df266e783f86328/common/dynamicconfig/constants.go#L1005)

<a id="workflow-execution-nexus-operation-limits"></a>
## Workflow Execution Nexus 操作限制 {#workflow-execution-nexus-operation-limits}

在需要 Continue-As-New 之前，Workflow 中的 Nexus 操作的最大数量有限制。
每个正在进行的 Nexus 操作都会在 Workflow Execution 的可变状态下生成一个元数据条目。
单个 Workflow Execution 的可变状态中的条目过多会导致持久性不稳定。
为了保护系统，Temporal 强制每个 Workflow Execution 的不完整 Nexus 操作请求的最大数量（默认情况下，30 个 Nexus 操作）。
一旦达到某种操作类型的限制，如果 Workflow Execution 尝试启动另一个 Nexus 操作（通过生成 ScheduleNexusOperation），它将无法执行此操作（Workflow 任务执行将失败并重试）。

这些限制是通过以下 [动态配置键](https://github.com/temporalio/temporal/blob/de7c8879e103be666a7b067cc1b247f0ac63c25c/components/nexusoperations/config.go#L38) 设置的：

- 最大并发操作数
