## 什么是动态处理程序？ {#dynamic-handler}

Temporal 支持动态 Workflow、活动、Signal 和查询。

:::note

目前，支持动态处理程序的 Temporal SDK 有：

- [爪哇](/develop/java/workflows/message-passing#dynamic-handler)
- [Python](/develop/python/workflows/message-passing#dynamic-handler)
- [。网](/develop/dotnet/workflows/message-passing#dynamic-handler)
- [去](/develop/go/workflows/dynamic-workflow)
- [红宝石](/develop/ruby/workflows/message-passing#dynamic-handler)

:::

这些是未命名的处理程序，如果不存在具有给定名称的其他静态定义的处理程序，则会调用这些处理程序。

动态处理程序可以灵活地处理运行时未知 Workflow、活动、Signal 或查询的名称的情况。

:::caution

应明智地使用动态处理程序作为后备机制而不是主要方法。
过度使用它们可能会导致可维护性和调试问题。

相反，Workflow、活动、Signal 和查询应尽可能静态定义，并使用表明其用途的清晰名称。
使用静态定义作为构建 Workflow 的主要方式。

保留动态处理程序，用于处理程序名称在编译时未知且需要在运行时动态查找的情况。
它们旨在处理边缘情况并充当包罗万象的角色，而不是作为调用逻辑的主要方式。

:::
