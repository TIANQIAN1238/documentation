## Workflow 简介 {#intro-to-workflows}

从概念上讲，Workflow 定义了一系列步骤。对于 Temporal，这些步骤是通过编写代码来定义的，称为
Workflow Definition，并通过运行该代码来执行，从而生成 Workflow Execution。

在日常对话中，术语 Workflow 可能指 Workflow 类型、Workflow Definition 或 Workflow
执行。

1. **Workflow Definition** 是定义 Workflow 的代码。
2. **Workflow 类型** 是映射到 Workflow Definition 的名称。它是一个标识符，可以
   将 Workflow 的一种类型（例如订单处理）与另一种类型（例如客户引导）区分开来。
3. **Workflow Execution** 是一个正在运行的 Workflow，它是通过将 Workflow Definition 与请求组合而创建的
   执行它。您可以执行 Workflow Definition 任意多次，每次可能提供不同的输入
   时间（即，用于订单处理的 Workflow Definition 可能会在一次执行中处理订单 #123，并在一次执行中处理订单 #567）
   另一次处决）。它是在 Temporal 平台中运行的 Workflow Definition 的实际实例。

您将通过使用通用编程语言（例如 Go、Java、TypeScript、
或Python。您编写的代码与将在运行时执行的代码相同，因此您可以使用您喜欢的工具和
用于开发 Temporal Workflow 的库。

Temporal Workflow 具有弹性。
即使底层基础设施出现故障，它们也可以运行并持续运行多年。
如果应用程序本身崩溃，Temporal 将自动重新创建其失败前的状态，以便它可以从中断处继续。

每个 Workflow Execution 都会执行一系列 **命令** 和 **事件**，这些命令记录在 **事件中
历史**。

Workflow 必须遵循确定性约束以确保一致的重播行为。
