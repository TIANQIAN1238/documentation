每个 Workflow 超时控制 Workflow Execution 不同方面的最大持续时间。 Workflow 超时在启动 Workflow Execution 时设置。

在继续之前，我们要注意的是，我们通常不建议设置 Workflow 超时，因为 Workflow 被设计为长期运行且具有弹性。
相反，设置超时可能会限制其处理意外延迟或长时间运行的进程的能力。
如果您需要在特定时间段后在 Workflow 内执行操作，我们建议使用 Timer。

- [Workflow Execution 超时](#workflow-execution-timeout)
- [Workflow 运行超时](#workflow-run-timeout)
- [Workflow 任务超时](#workflow-task-timeout)

## Workflow Execution 超时 {#workflow-execution-timeout}

**Temporal 中的 Workflow Execution 超时是什么？**

Workflow Execution 超时是 Workflow Execution 可以执行（具有“打开”状态）的最长时间，包括重试和“继续作为新”的任何使用。

**默认值为 ∞（无限）。**
如果达到此超时，Workflow Execution 将更改为超时状态。

## Workflow 运行超时 {#workflow-run-timeout}

**什么是 Temporal 中的 Workflow 运行超时？**

Workflow 运行是特定 Workflow Execution 的实例。

由于 Workflow 重试或继续为新的可能性，Workflow Execution 可能有多次 Workflow 运行。例如，如果指定 Retry Policy 的 Workflow 最初失败，然后在下一次重试尝试期间成功，则有一个 Workflow Execution 跨越两次 Workflow 运行。两次运行将共享相同的 Workflow ID，但有一个唯一的运行 ID 来区分它们。

Workflow 运行超时限制单次 Workflow 运行的最大持续时间。如果达到 Workflow 运行超时，Workflow Execution 将超时。由于此超时仅适用于单个 Workflow 运行，因此不包括重试或 Continue-As-New。

## Workflow 任务超时 {#workflow-task-timeout}

**Temporal 中的 Workflow 任务超时是什么？**

Workflow 任务超时是 Worker 从 [Task Queue](/task-queue) 中提取 Workflow 任务后允许 [Worker](/workers#worker) 执行 [Workflow 任务](/tasks#workflow-task) 的最大时间量。

该超时主要用于识别 Worker 是否已关闭，以便可以在不同的 Worker 上恢复 Workflow Execution。

**默认值为 10 秒。**
该超时主要用于识别 Worker 是否已关闭，以便可以在不同的 Worker 上恢复 Workflow Execution。
增加默认值的主要原因是为了容纳具有广泛 Workflow Execution 历史记录的 Workflow Execution，Worker 加载需要 10 秒以上。
值得一提的是，虽然您可以将超时时间延长至最大值 120 秒，但不建议超出默认值。

## 检测Workflow任务失败

使用 `TemporalReportedProblems` Search Attribute 检测 Workflow 任务失败的 Workflow。
失败的 Workflow 任务不会导致 Workflow 失败。 Workflow 中的某些任务可能会失败。
例如，Workflow 任务可以检查远程数据源中是否有新消息。如果没有，任务将按预期失败。
如果您的任务有处理失败的代码，Workflow 将继续。
但是，如果您的 Workflow 有一个任务失败并且未处理该失败，则 Workflow 将继续运行，但不会完成。
检测此状态下的 Workflow 是常见的故障排除问题。

要识别任务失败的 Workflow，您可以使用 Temporal Web UI。更多详情请参见[任务失败视图](/web-ui/#task-failures-view)。

您还可以通过使用可观察性工具搜索 `TemporalReportedProblems` 搜索属性来检测任务失败的 Workflow。

:::warning Activating Workflow Task Failure
要启用 Namespace 的任务失败视图，您需要更新该 Namespace 的动态配置。请参阅[激活任务失败视图](/web-ui/#activate-task-failures-view)。
:::
