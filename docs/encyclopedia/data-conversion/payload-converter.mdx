## 什么是有效负载转换器？ {#payload-converter}

有效负载转换器序列化数据，将值转换为字节并返回。

当您通过客户端启动 Workflow Execution 并将数据作为输入传递时，输入将使用 Data Converter 进行序列化，该 Data Converter 通过一组有效负载转换器运行它。
当您的 Workflow Execution 启动时，此数据输入将被反序列化并作为输入传递到 Workflow。

### 复合 Data Converters {#composite-data-converters}

复合 Data Converter 用于按指定顺序应用自定义的、特定于类型的有效负载转换器。
复合 Data Converter 可以由您创建的自定义规则组成，它还可以利用 Temporal 中内置的默认 Data Converter。
事实上，默认的 Data Converter 逻辑在 Temporal 源内部作为复合 Data Converter 实现。它按以下顺序定义这些规则：

```go
defaultDataConverter = NewCompositeDataConverter(
    NewNilPayloadConverter(),
    NewByteSlicePayloadConverter(),
    NewProtoJSONPayloadConverter(),
    NewProtoPayloadConverter(),
    NewJSONPayloadConverter(),
)
```

应用有效负载转换器的顺序很重要。
在序列化期间，Data Converter 按特定顺序尝试有效负载转换器，直到有效负载转换器返回非零有效负载。
自定义 PayloadConverter 必须实现以下功能：

- `FromPayload`（对于单个值）或
- `FromPayloads`（用于值列表）转换为有效负载中的值，以及
- `ToPayload`（对于单个值）或
- `ToPayloads`（用于值列表）将值转换为有效负载。

定义新的复合体 Data Converter 并不总是需要实现自定义数据处理。
每个 SDK 都允许您使用自定义负载编解码器覆盖或配置默认转换器。
