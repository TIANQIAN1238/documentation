## 什么是 Codec Server？ {#codec-server}

Codec Server 是一个 HTTP/HTTPS 服务器，通过端点使用[自定义 Payload Codec](/production-deployment/data-encryption) 远程解码数据。

Codec Server 遵循 Temporal [Codec Server 协议](https://github.com/temporalio/samples-go/tree/main/codec-server#codec-server-protocol)。它实现了两个端点：

- `/encode`
- `/decode`

每个端点接收并响应一个 JSON body，其中包含一个带有 [Payload](/dataconversion#payload) 数组的 `payloads` 属性。端点在返回之前通过 [Payload Codec](/payload-codec) 处理 Payload。

大多数 SDK 提供了 Codec Server 实现示例，列举如下：

- [Go](https://github.com/temporalio/samples-go/tree/main/codec-server)
- [Java](https://github.com/temporalio/sdk-java/tree/master/temporal-remote-data-encoder)
- [.NET](https://github.com/temporalio/samples-dotnet/tree/main/src/Encryption)
- [Python](https://github.com/temporalio/samples-python/blob/main/encryption/codec_server.py)
- [TypeScript](https://github.com/temporalio/samples-typescript/blob/main/encryption/src/codec-server.ts)

#### 使用方式

当你对 Workflow 数据应用加密或压缩的自定义编码时，数据会以加密/压缩格式存储在 Temporal Server 上。有关哪些数据被编码的详细信息，请参阅[保护数据安全](/production-deployment/data-encryption)。

要在使用 Temporal CLI 或 Web UI 对 Workflow Execution 执行某些操作时查看解码后的数据，请在 Web UI 和 Temporal CLI 中配置 Codec Server 端点。配置 Codec Server 端点后，Temporal CLI 和 Web UI 将编码后的数据发送到 Codec Server，并显示从 Codec Server 接收到的解码后的数据。

有关创建 Codec Server 的详细信息，请参阅 [Codec Server 设置](/production-deployment/data-encryption#codec-server-setup)。
