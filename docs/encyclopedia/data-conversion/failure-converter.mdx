## 什么是故障转换器？ {#failure-converter}

与输入和输出一样，Temporal 也使用自己的默认转换器逻辑来处理 Workflow 生成的错误。
默认的故障转换器将错误消息和调用堆栈复制为纯文本，然后可以在这些 Workflow Execution 的 `Message` 字段中直接访问此文本输出。

这对于您的应用程序来说可能是不受欢迎的。在某些情况下，错误可能包含特权或敏感信息，您需要防止这些信息泄漏或通过侧通道获取。
默认情况下，失败消息和调用堆栈不会编码为支持编解码器的有效负载；您必须在失败时显式启用对这些公共属性进行编码。

如果您的错误可能包含敏感信息，您可以通过配置默认故障转换器以使用您的编码来加密消息和调用堆栈。
这会将您的 `message` 和 `stack_trace` 字段移动到通过编解码器运行的有效负载。

例如，对于 Temporal Go SDK，您可以通过在调用 `client.Dial()` 时将 `FailureConverter` 参数添加到 `client.Options{}` 数组来实现此目的。
`FailureConverter` 应通过设置 `EncodeCommonAttributes: true` 来覆盖 `DefaultFailureConverterOptions{}`，如下所示：

```go
c, err := client.Dial(client.Options{
	// Set DataConverter here to ensure that workflow inputs and results are
	// encoded as required.
	DataConverter: mycustom.DataConverter,
	FailureConverter: temporal.NewDefaultFailureConverter(temporal.DefaultFailureConverterOptions{
		EncodeCommonAttributes: true,
	}),
})
```

如果由于某种原因您需要为故障指定一组不同的转换器逻辑，您可以使用自定义方法替换 `NewDefaultFailureConverter`。
例如，如果您既处理高度敏感的数据，又使用复杂的日志记录/可观察性实现，则可能需要为每个数据实现不同的加密方法。
