<a id="remote-data-encoding"></a>
## 什么是远程数据编码？ {#remote-data-encoding}

远程数据编码是通过 HTTP 端点公开您的 Payload Codec 以支持远程编码和解码。

远程运行编码允许您将其与 [Temporal CLI](/cli) 一起使用，对包括 `Temporal Workflow show` 在内的多个命令的数据进行编码/解码，并与 Temporal Web UI 一起使用，以解码 Workflow Execution 详细信息视图中的数据。

要远程运行数据编码/解码，请使用 [Codec Server](/codec-server)。 Codec Server 是一个 HTTP 服务器，它使用您的自定义编解码器逻辑来远程解码您的数据。
Codec Server 独立于 Temporal 服务，并通过预定义端点解码您的加密负载。
您可以在自己的环境中创建、操作和管理对 Codec Server 的访问。
Temporal CLI 和 Web UI 反过来提供内置挂钩来调用 Codec Server 以按需解码加密的有效负载。

<a id="encoding-data-on-the-web-ui-and-cli"></a>
### 在 Web UI 和 CLI 上编码数据 {#encoding-data-on-the-web-ui-and-cli}

您可以使用 Temporal CLI 和 Web UI 在 Workflow Execution 上执行一些操作。
例如，您可以从 Temporal CLI 启动或 Signal 活动的 Workflow Execution，或者从 Web UI 取消 Workflow Execution，这可能需要包含敏感数据的输入。

要编码此数据，请使用 [Temporal CLI](/codec-server) 中的 `codec-endpoint` 参数指定 [Codec Server 端点](/cli)，并将 Web UI 配置为使用 Codec Server 端点。

<a id="decoding-data-on-the-web-ui-and-cli"></a>
### 在 Web UI 和 CLI 上解码数据 {#decoding-data-on-the-web-ui-and-cli}

如果您使用自定义编码，则 Temporal 服务处理的有效负载数据将以编码方式存储。由于 Web UI 使用 [Visibility](/temporal-service/visibility) 数据库显示存储在 Temporal Server 上的事件和数据，因此 Web UI 中 Workflow Execution 历史记录中的所有数据都以编码格式显示。

要在使用 Web UI 和 Temporal CLI 时解码输出，请使用 [Codec Server](/codec-server)。

请注意，远程数据编码器是一个独立的系统，可以访问您的加密密钥并公开 API 来编码和解码任何数据。
评估并确保您的远程数据编码器端点是安全的，并且只有授权用户才能访问它们。
