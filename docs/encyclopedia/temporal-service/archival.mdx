## 什么是归档？ {#archival}

使用存档将关闭的 Workflow Execution [事件历史](/workflow-execution/event#event-history) 和 Visibility 记录从 Temporal 服务持久性复制到 Blob 存储。

- [如何创建自定义存档器](/self-hosted-guide/archival#custom-archiver)
- [如何设置存档](/self-hosted-guide/archival#set-up-archival)

当 Workflow Execution 关闭时，Temporal Schedule 会关闭 Visibility 记录和 Event History 归档的处理任务。
然后，归档会在随机延迟后异步运行。
默认情况下，该延迟最长为 5 分钟 (`history.archivalProcessorArchiveDelay`)，由 Namespace [保留期限](/temporal-service/temporal-server#retention-period) 限制。

关闭的执行仍保持 Temporal 持久性，直到运行保留清理。
在一段时间内，相同的封闭执行可以同时存在于持久性存储和归档存储中。
归档使 Workflow Execution 数据能够持久保存，而不会压垮 Temporal 服务持久性存储。

此功能有助于合规性和调试。

Temporal 的存档功能被认为是**实验性**，不受正常 [版本控制和支持政策](/temporal-service/temporal-server#versions-and-support) 的约束。

通过 Docker 运行 Temporal 时不支持存档。
手动安装系统和通过 [舵图](https://github.com/temporalio/helm-charts/blob/main/charts/temporal/templates/server-configmap.yaml) 部署时默认禁用它。
它可以在 [配置](https://github.com/temporalio/temporal/blob/main/config/development.yaml) 中启用。
