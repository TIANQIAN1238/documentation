## Visibility 是什么？ {#visibility}

Temporal 平台中的术语 [Visibility](/visibility) 是指使操作员能够查看、过滤和搜索当前存在于 Temporal 服务中的 Workflow Execution 的子系统和 API。

Temporal 服务中的 [Visibility 商店](/self-hosted-guide/visibility) 存储持久的 Workflow Execution Event History 数据，并设置为 [持久性存储](/temporal-service/persistence) 的一部分，以启用列出和过滤有关 Temporal 服务中存在的 Workflow Execution 的详细信息。

- [如何开设 Visibility 商店](/self-hosted-guide/visibility)

使用 Temporal Server v1.21，您可以设置 [双 Visibility](/dual-visibility) 将 Visibility 存储从一个数据库迁移到另一个数据库。

从 Temporal Server v1.21 开始，将不再支持单独的标准和高级 Visibility 设置。
检查 [支持的数据库](/self-hosted-guide/visibility) 中的 Update。
