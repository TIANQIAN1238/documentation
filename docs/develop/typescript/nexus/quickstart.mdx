<a id="nexus-typescript-quickstart"></a>
# Nexus TypeScript 快速入门 {#nexus-typescript-quickstart}

:::tip SUPPORT, STABILITY, and DEPENDENCY INFO

Temporal TypeScript SDK 对 Nexus 的支持位于 [公开预览](/evaluate/development-production-features/release-stages#public-preview) 中。

:::

[Temporal Nexus](/evaluate/nexus) 使用 Nexus 端点、Nexus 服务合同和 Nexus 操作连接 Namespace 内部和之间的 Temporal 应用程序。构建一个包装现有 Temporal Workflow 的 Nexus 服务，然后从调用者 Workflow 调用它。

:::info NEW TO NEXUS?
此页面将帮助您获得在 TypeScript 中运行的工作示例。
要评估 Nexus 是否适合您的使用案例，请参阅 [评估指南](/evaluate/nexus)，要了解有关 Nexus 功能的更多信息，请单击 [这里](/nexus)。
:::

**先决条件：** 首先完成 [TypeScript SDK 快速入门](/develop/typescript/set-up-your-local-typescript)。
您应该从该指南中获得 `activities.ts`、`Workflows.ts`、`Worker.ts` 和 `client.ts`。

<a id="what-youll-build"></a>
## 你将构建什么 {#what-youll-build}

您有 `example` 在 `default` Namespace 中运行。
在本指南结束时：

1. Nexus 服务将 `example` 公开为操作。
2. 第二个 Namespace 将包含调用该操作的 Workflow。
3. 调用者 Workflow 将返回 `"Hello, Temporal!"` — 相同的结果，但跨越 Namespace。

<a id="1-define-the-nexus-service"></a>
## 1.定义Nexus服务 {#1-define-the-nexus-service}

创建一个名为 `service.ts` 的文件，用于定义 Nexus 服务合同。

创建 Nexus 服务会在您的实现和任何调用者之间建立契约。
它在调用 Nexus 操作时提供类型安全，并确保操作处理程序履行合同。

`Nexus.service()` 声明命名服务，`Nexus.operation<I, O>()` 定义类型化操作。
`example` Workflow 返回 `string`，因此运算输出类型为 `string`。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="typescript">
{`import { randomUUID } from 'crypto';
import * as nexus from 'nexus-rpc';
import * as temporalNexus from '@temporalio/nexus';
import { sayHelloService, MyInput } from './service';
import { example } from './workflows';

export const sayHelloHandler = nexus.serviceHandler(sayHelloService, {
    sayHello: new temporalNexus.WorkflowRunOperationHandler<MyInput, string>(
      async (ctx, input: MyInput) => {
        return await temporalNexus.startWorkflow(ctx, example, {
          args: [input.name],
          workflowId: "say-hello-nexus-" + randomUUID(),

          // Task queue defaults to the task queue this Operation is handled on.
        });
      },
    ),
});`}
</CodeSnippet>

</>
}>

<a id="2-define-the-nexus-operation-handlers"></a>
## 2. 定义 Nexus 操作处理程序 {#2-define-the-nexus-operation-handlers}

创建一个名为 `handler.ts` 的文件，该文件实现 Nexus 操作处理程序。

操作处理程序包含调用者调用 Nexus 操作时运行的逻辑。

`WorkflowRunOperationHandler` 创建启动 Workflow 的异步 Nexus 操作。
该处理程序通过提取 `input.name` 将 Nexus `MyInput` 接口桥接到 `example` Workflow 的 `string` 参数。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="typescript">
{`import { NativeConnection, Worker } from '@temporalio/worker';
import * as activities from './activities';
import { sayHelloHandler } from './handler';

async function run() {
    const connection = await NativeConnection.connect({
      address: 'localhost:7233',
    });
    try {
      const worker = await Worker.create({
        connection,
        namespace: 'default',
        taskQueue: 'hello-world',
        workflowsPath: require.resolve('./workflows'),
        activities,
        nexusServices: [sayHelloHandler],
      });
      await worker.run();
    } finally {
      await connection.close();
    }
}

run().catch((err) => {
    console.error(err);
    process.exit(1);
});`}
</CodeSnippet>

</>
}>

<a id="3-register-the-nexus-service-handler-in-a-worker"></a>
## 3. 在 Worker 中注册 Nexus 服务处理程序 {#3-register-the-nexus-service-handler-in-a-worker}

Update 您现有的 `Worker.ts` 用于注册 Nexus 服务处理程序。

如果注册了 Nexus 服务处理程序，Worker 将仅轮询并处理传入的 Nexus 请求。
这与用于 Workflow 和活动的 Worker 概念相同。

`NexusServices` 参数注册处理程序，以便它可以接收 Nexus 操作请求。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="typescript">
{`import { proxyActivities, createNexusClient } from '@temporalio/workflow';
// Only import the activity types
import type * as activities from './activities';
import { sayHelloService } from './service';

const { greet } = proxyActivities<typeof activities>({
    startToCloseTimeout: '1 minute',
});

/** A workflow that simply calls an activity */
export async function example(name: string): Promise<string> {
    return await greet(name);
}

const NEXUS_ENDPOINT = 'my-nexus-endpoint-name';

export async function callerWorkflow(name: string): Promise<string> {
    const nexusClient = createNexusClient({
      service: sayHelloService,
      endpoint: NEXUS_ENDPOINT,
    });
    return await nexusClient.executeOperation('sayHello', { name }, { scheduleToCloseTimeout: '10s' });
}
`}
</CodeSnippet>

</>
}>

<a id="4-develop-the-caller-workflow"></a>
## 4.开发调用者Workflow {#4-develop-the-caller-workflow}

Update 现有的 `Workflows.ts` 文件，其中包含调用 Nexus 操作的 Workflow。

调用者Workflow演示了Nexus的消费者端。
调用者只依赖于服务合约，而不是直接导入处理程序代码。
这使调用者和处理程序保持解耦，以便他们可以生活在单独的 Namespace、存储库甚至团队中。

`wf.createNexusClient()` 方法创建绑定到 Nexus 服务和端点的客户端。
`executeOperation`开始运算并等待结果。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="bash">
temporal operator namespace create --namespace my-caller-namespace
</CodeSnippet>

<CodeSnippet language="bash">
{`temporal operator nexus endpoint create \\
  --name my-nexus-endpoint-name \\
  --target-namespace default \\
  --target-task-queue hello-world`}
</CodeSnippet>

</>
}>

<a id="5-create-the-caller-namespace-and-nexus-endpoint"></a>
## 5. 创建调用者 Namespace 和 Nexus 端点 {#5-create-the-caller-namespace-and-nexus-endpoint}

在运行应用程序之前，创建一个调用者 Namespace 和一个 Nexus 端点以将请求从调用者路由到处理程序。
该处理程序使用启动开发服务器时创建的 `default` Namespace。

Namespace 在调用方和处理程序方之间提供隔离。
Nexus 端点充当路由层，将调用方 Namespace 连接到处理程序的目标 Namespace 和 Task Queue。
端点名称必须与步骤 4 中 `caller.ts` 中定义的变量匹配。

确保您的本地 Temporal 开发服务器正在运行 (`Temporal Server start-dev`)。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="typescript">
{`import { randomUUID } from 'crypto';
import { Connection, Client } from '@temporalio/client';
import { NativeConnection, Worker } from '@temporalio/worker';
import { callerWorkflow } from './workflows';

const CALLER_TASK_QUEUE = 'my-caller-task-queue';
const NAMESPACE = 'my-caller-namespace';

async function main() {
    const clientConnection = await Connection.connect({
      address: 'localhost:7233',
    });
    const client = new Client({
      connection: clientConnection,
      namespace: NAMESPACE,
    });

    const workerConnection = await NativeConnection.connect({
      address: 'localhost:7233',
    });
    try {
      const worker = await Worker.create({
        connection: workerConnection,
        namespace: NAMESPACE,
        taskQueue: CALLER_TASK_QUEUE,
        workflowsPath: require.resolve('./workflows'),
      });

      await worker.runUntil(async () => {
        const result = await client.workflow.execute(callerWorkflow, {
          args: ['Temporal'],
          workflowId: \`caller-workflow-\${randomUUID()}\`,
          taskQueue: CALLER_TASK_QUEUE,
        });
        console.log('Workflow result:', result);
      });
    } finally {
      await workerConnection.close();
    }
}

main().catch((err) => {
    console.error(err);
    process.exit(1);
});`}
</CodeSnippet>

</>
}>

<a id="6-run-and-verify"></a>
## 6. 运行并验证 {#6-run-and-verify}

创建一个名为 `caller-starter.ts` 的文件来启动调用者 Worker 并执行 Workflow。

此步骤将所有内容整合在一起：调用者 Worker 托管 `callerWorkflow`，后者使用 Nexus 客户端调用处理程序端的 `sayHello`。
完整请求从调用者 Workflow 通过 Nexus 端点流向运行 `example` Workflow 的处理程序 Worker，然后返回调用者。

**运行应用程序：**

1. 在一个终端中启动处理程序 Worker：

```bash
npx ts-node src/worker.ts
```

2. 在另一个终端中运行调用程序：

```bash
npx ts-node src/caller-starter.ts
```

你应该看到：

```
Workflow result: Hello, Temporal!
```

打开[Temporal 网页用户界面](http://localhost:8233)并找到`callerWorkflow`执行。
您应该在 Workflow 历史记录中看到 `NexusOperationScheduled`、`NexusOperationStarted` 和 `NexusOperationCompleted` 事件。

</SetupStep>
</SetupSteps>
