<a id="run-hello-world-test-your-installation"></a>
## 运行 Hello World：测试您的安装 {#run-hello-world-test-your-installation}

现在，让我们通过创建并运行一个完整的 Temporal 应用程序来验证您的设置是否正常工作，其中包含 Workflow 和
Activity。

该测试将确认：

- Temporal TypeScript SDK 已正确安装
- 您本地的 Temporal 服务正在运行
- 您可以成功创建并执行 Workflow 和活动
- 组件之间的通信功能正常

<a id="1-create-the-activity"></a>
### 1.创建Activity {#1-create-the-activity}

创建 Activity 文件（activities.ts）：

```ts
export async function greet(name: string): Promise<string> {
  return `Hello, ${name}!`;
}
```

Activity 是执行单个明确定义的操作（短期或长期运行）的普通函数或方法，
这通常涉及与外界交互，例如发送电子邮件、发出网络请求、向
数据库，或者调用API，这些都容易失败。如果 Activity 失败，Temporal 会根据情况自动重试
根据您的配置。

<a id="2-create-the-workflow"></a>
### 2.创建Workflow {#2-create-the-workflow}

创建 Workflow 文件 (Workflows.ts)：

```ts
import { proxyActivities } from '@temporalio/workflow';
// Only import the activity types
import type * as activities from './activities';

const { greet } = proxyActivities<typeof activities>({
  startToCloseTimeout: '1 minute',
});

/** A workflow that simply calls an activity */
export async function example(name: string): Promise<string> {
  return await greet(name);
}
```

Workflow 编排活动并包含应用程序逻辑。 Temporal Workflow 具有弹性。他们可以奔跑并且
即使底层基础设施出现故障，也能持续运行数年。如果应用程序本身崩溃，Temporal 将
自动重新创建故障前的状态，以便可以从中断处继续。

<a id="3-create-and-run-the-worker"></a>
### 3. 创建并运行 Worker {#3-create-and-run-the-worker}

创建 Worker 文件 (Worker.ts)：

```ts
import { NativeConnection, Worker } from '@temporalio/worker';
import * as activities from './activities';

async function run() {
  // Step 1: Establish a connection with Temporal server.
  //
  // Worker code uses `@temporalio/worker.NativeConnection`.
  // (But in your application code it's `@temporalio/client.Connection`.)
  const connection = await NativeConnection.connect({
    address: 'localhost:7233',
    // TLS and gRPC metadata configuration goes here.
  });
  try {
    // Step 2: Register Workflows and Activities with the Worker.
    const worker = await Worker.create({
      connection,
      namespace: 'default',
      taskQueue: 'hello-world',
      // Workflows are registered using a path as they run in a separate JS context.
      workflowsPath: require.resolve('./workflows'),
      activities,
    });

    // Step 3: Start accepting tasks on the `hello-world` queue
    //
    // The worker runs until it encounters an unexpected error or the process receives a shutdown signal registered on
    // the SDK Runtime object.
    //
    // By default, worker logs are written via the Runtime logger to STDERR at INFO level.
    //
    // See https://typescript.temporal.io/api/classes/worker.Runtime#install to customize these defaults.
    await worker.run();
  } finally {
    // Close the connection once the worker has stopped
    await connection.close();
  }
}

run().catch((err) => {
  console.error(err);
  process.exit(1);
});
```

运行 Worker 并保持此终端运行：

```bash
npm run start
```

定义了 Activity 和 Workflow 后，您需要 Worker 来执行它们。 Worker 轮询 Task Queue，您
将其配置为轮询，寻找要做的工作。一旦 Worker 将 Workflow 或 Activity 任务从 Task Queue 中出列，
然后它执行该任务。

Worker 是 Temporal 应用程序的重要组成部分，因为它们实际执行您的 Temporal 应用程序中定义的任务。
Workflow 和活动。有关 Worker 的更多信息，请参阅
[了解 Temporal](/evaluate/understanding-temporal#workers) 和 [深入研究 Worker](/workers)。

<a id="4-execute-the-workflow"></a>
### 4.执行Workflow {#4-execute-the-workflow}

现在您的 Worker 正在运行，是时候启动 Workflow Execution 了。

最后一步将验证标记为 `client.ts` 的文件是否一切正常。

创建一个名为 `client.ts` 的单独文件。

```ts
import { Client, Connection } from '@temporalio/client';
import { nanoid } from 'nanoid';
import { example } from './workflows';

async function run() {
  // Connect to the default Server location
  const connection = await Connection.connect({ address: 'localhost:7233' });
  // In production, pass options to configure TLS and other settings:
  // {
  //   address: 'foo.bar.tmprl.cloud',
  //   tls: {}
  // }

  const client = new Client({
    connection,
    // namespace: 'foo.bar', // connects to 'default' namespace if not specified
  });

  const handle = await client.workflow.start(example, {
    taskQueue: 'hello-world',
    // type inference works! args: [name: string]
    args: ['Temporal'],
    // in practice, use a meaningful business ID, like customerId or transactionId
    workflowId: 'workflow-' + nanoid(),
  });
  console.log(`Started workflow ${handle.workflowId}`);

  // optional: wait for client result
  console.log(await handle.result()); // Hello, Temporal!
}

run().catch((err) => {
  console.error(err);
  process.exit(1);
});
```

然后运行：

```bash
npm run workflow
```

<a id="verify-success"></a>
### 验证成功 {#verify-success}

如果一切正常，您应该看到：

- Worker 处理 Workflow 和 Activity
- 输出：`Workflow result: Hello, Temporal!`
- [Temporal 网页用户界面](http://localhost:8233) 中的 Workflow Execution 详细信息

<details>
<summary><strong>Additional details about Workflow Execution</strong></summary>

- Temporal CLIents 未明确关闭。
- 要启用 TLS，请将 `tls` 选项设置为 `true` 以进行默认设置，或传递 [`TLSConfig`](https://typescript.temporal.io/api/interfaces/worker.TLSConfig) 以进行自定义配置。
- 调用`client.Workflow.start()`和`client.Workflow.execute()`发送命令到Temporal Server到Schedule一个新的
  Workflow Execution 位于指定的 Task Queue 上。
- 如果您使用`client.Workflow.start()`开始了Workflow，您可以选择随时等待结果
  处理.结果()。
- `client.Workflow.execute()` 无需使用 Workflow 手柄。

</details>
