## 如何安装 Temporal SDK {#install-a-temporal-sdk}

[Temporal SDK](/encyclopedia/temporal-sdks) 为开发 [Temporal 应用](/temporal#temporal-application) 提供了完整框架。

SDK 为您提供以下功能：

- [Temporal Client](/encyclopedia/temporal-sdks#temporal-client) 与 [Temporal 服务](/temporal-service) 通信所需的 API。
- 用于开发 [Workflows](/workflows) 的 API。
- 用于创建和管理 [Worker](/workers#worker) 进程的 API。
- 用于编写 [Activity](/activity-definition) 的 API。

[npm 上的 Temporal 团队包列表](https://www.npmjs.com/search?q=author%3Atemporal-sdk-team)

该项目需要 Node.js 18 或更高版本。

**创建一个项目**

```bash
npx @temporalio/create@latest ./your-app
```

**添加到现有项目**

```bash
npm install @temporalio/client @temporalio/worker @temporalio/workflow @temporalio/activity @temporalio/common
```

:::note

TypeScript SDK 的设计考虑了 TypeScript 优先的开发人员体验，但它同样适用于
JavaScript。

:::

### 如何导入 ECMAScript 模块 {#ecmascript-modules}

JavaScript 生态系统正在迅速朝着发布 ECMAScript 模块 (ESM) 而不是 CommonJS 模块的方向发展。对于
例如，`node-fetch@3` 是 ESM，但 `node-fetch@2` 是 CommonJS。

有关导入纯 ESM 依赖项的更多信息，请参阅我们的
[获取ESM](https://github.com/temporalio/samples-typescript/tree/main/fetch-esm) 必要配置的示例
变化：

- `package.json` 必须包含 `"type": "module"` 属性。
- `tsconfig.json` 应以 `esnext` 格式输出。
- 导入必须包含 `.js` 文件扩展名。

## TypeScript 中的 Linting 和类型 {#linting-and-types}

如果您使用 `@Temporalio/create` 开始您的项目，那么您已经拥有我们推荐的 TypeScript 和 ESLint
配置。

如果您将 Temporal 增量添加到现有应用程序中，我们建议您设置 linting 和类型，因为它们会有所帮助
在将错误交付生产之前就捕获它们，它们可以改善您的开发反馈循环。看看我们的
推荐的 [.eslintrc](https://github.com/temporalio/samples-typescript/blob/main/.shared/.eslintrc.js) 文件和调整
以满足您的需求。
