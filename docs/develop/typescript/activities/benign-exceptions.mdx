当活动抛出预期或不严重的错误时，它们可能会在日志、指标和 OpenTelemetry 跟踪中产生噪音，从而使识别真正问题变得更加困难。
通过将这些错误标记为良性，您可以将它们从可观测性数据中排除，同时仍然在 Workflow 逻辑中处理它们。

要将错误标记为良性，请在创建 [`ApplicationFailure`](https://typescript.temporal.io/api/classes/common.ApplicationFailure) 时将 `category` 字段设置为 `ApplicationFailureCategory.BENIGN`。

良性错误：
- 将 Activity 故障日志降级至 DEBUG 级别
- 不发出 Activity 失败指标
- 不要将 OpenTelemetry 失败状态设置为 ERROR

```typescript
import {
  ApplicationFailure,
  ApplicationFailureCategory,
} from '@temporalio/common';

export async function myActivity(): Promise<string> {
  try {
    return await callExternalService();
  } catch (err) {
    const message = err instanceof Error ? err.message : String(err);
    throw ApplicationFailure.create({
      message,
      // Mark this error as benign since it's expected
      category: ApplicationFailureCategory.BENIGN,
    });
  }
}
```

对正常操作中定期发生的 Activity 错误使用良性异常，例如轮询尚未准备就绪的外部服务，或处理将重试的预期瞬态故障。
