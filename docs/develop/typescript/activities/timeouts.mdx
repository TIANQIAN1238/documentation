<a id="activity-timeouts"></a>
## Activity 超时 {#activity-timeouts}

**如何使用 Temporal TypeScript SDK 设置 Activity 超时**

每个 Activity 超时控制 Activity 执行不同方面的最大持续时间。

Activity 选项中提供以下超时：

- **[Schedule-关闭超时](/encyclopedia/detecting-activity-failures#schedule-to-close-timeout)：** 是整个 [Activity 执行](/activity-execution) 允许的最长时间，从 [Activity 任务](/tasks#activity-task) 最初由 Workflow 变为 Scheduled 时到服务器收到该 Activity 任务的成功完成时
- **[开始到关闭超时](/encyclopedia/detecting-activity-failures#start-to-close-timeout)：** 是单个 [Activity 任务执行](/tasks#activity-task-execution) 允许的最长时间，从 [Worker](/workers#worker) 轮询 Activity 任务执行时到服务器收到该 Activity 任务成功完成时的时间
- **[Schedule-启动超时](/encyclopedia/detecting-activity-failures#schedule-to-start-timeout)：** 是从 [Activity 任务](/tasks#activity-task) 最初由 Workflow 执行 Scheduled 到 [Worker](/workers#worker) 轮询 Activity 任务执行的最大时间量

Activity 执行必须设置“开始到关闭”或“Schedule 到关闭超时”。

使用 [`proxyActivities()`](https://typescript.temporal.io/api/interfaces/common.ActivityOptions) API 创建 Activity 代理函数时，可以在 [`ActivityOptions`](https://typescript.temporal.io/api/namespaces/workflow#proxyactivities) 上设置以下属性：

- [`ScheduleToCloseTimeout`](https://typescript.temporal.io/api/interfaces/common.ActivityOptions/#scheduletoclosetimeout)
- [`startToCloseTimeout`](https://typescript.temporal.io/api/interfaces/common.ActivityOptions/#starttoclosetimeout)
- [`ScheduleToStartTimeout`](https://typescript.temporal.io/api/interfaces/common.ActivityOptions/#scheduletostarttimeout)

```typescript
const { myActivity } = proxyActivities<typeof activities>({
  scheduleToCloseTimeout: '5m',
  // startToCloseTimeout: "30s", // recommended
  // scheduleToStartTimeout: "60s",
});
```

<a id="activity-retries"></a>
## Activity Retry Policy {#activity-retries}

**如何使用 Temporal TypeScript SDK 设置 Activity Retry Policy**

Retry Policy 与超时配合提供精细控制以优化执行体验。

Activity 如果未提供自定义执行，则执行会自动与默认 [Retry Policy](/encyclopedia/retry-policies) 关联。

要在 TypeScript 中设置 Activity 的 Retry Policy，请在使用 [`proxyActivities()`](https://typescript.temporal.io/api/interfaces/common.ActivityOptions#retry) API 创建相应的 Activity 代理函数时分配 [`ActivityOptions.retry`](https://typescript.temporal.io/api/namespaces/workflow#proxyactivities) 属性。

```typescript
const { myActivity } = proxyActivities<typeof activities>({
  // ...
  retry: {
    initialInterval: '10s',
    maximumAttempts: 5,
  },
});
```

<a id="activity-next-retry-delay"></a>
## Activity 下一次重试延迟 {#activity-next-retry-delay}

**如何使用 Temporal TypeScript SDK 覆盖 Activity 失败后的下一次重试延迟**

可重试的 Activity 失败后直到尝试下一次重试的等待时间通常由该 Activity 的 Retry Policy 确定。
但是，当 [`ApplicationFailure`](https://typescript.temporal.io/api/classes/common.ApplicationFailure) 显式失败时，Activity 可以通过设置下一次重试延迟来覆盖该持续时间。

要覆盖 TypeScript 中 Activity 引发的 `ApplicationFailure` 的下一次重试延迟，请在 [`ApplicationFailure.create()`](https://typescript.temporal.io/api/interfaces/common.ApplicationFailureOptions#nextretrydelay) 工厂方法的对象参数上提供 [`nextRetryDelay`](https://typescript.temporal.io/api/classes/common.ApplicationFailure#create) 属性。

```typescript
throw ApplicationFailure.create({
  // ...
  nextRetryDelay: '15s',
});
```

<a id="activity-heartbeats"></a>
## 心跳 Activity {#activity-heartbeats}

**如何使用 Temporal TypeScript SDK 对 Activity 进行检测**

[Activity 心跳](/encyclopedia/detecting-activity-failures#activity-heartbeat) 是从正在执行 Activity 的 [Worker 流程](/workers#worker-process) 到 [Temporal 服务](/temporal-service) 的 ping。
每个心跳都会通知 Temporal 服务 [Activity 执行](/activity-execution) 正在取得进展并且 Worker 尚未崩溃。
如果Temporal服务在[心跳超时](/encyclopedia/detecting-activity-failures#heartbeat-timeout)时间段内没有收到心跳，则Activity将被视为超时，并且根据Retry Policy，另一个[Activity 任务执行](/tasks#activity-task-execution)可能是Scheduled。

Activity 取消会在活动检测到时从 Temporal 服务传递到活动。
没有 Heartbeat 的活动无法收到取消请求的通知。

:::note Handling Activity Cancellation
在 TypeScript SDK 中，Activity 实现必须选择观察取消。

在您的 Activity 中使用以下其中一项即可收到取消通知：
- `await Context.current().cancelled` — 拒绝 `CancelledFailure`。
- `await Context.current().sleep(ms)` — 取消时被拒绝。
- 将 `Context.current().cancellationSignal`（`AbortSignal`）传递给支持中止的库。

> **重要：** 活动在心跳时收到取消通知；如果 Activity 没有检测信号，则取消通知的传送可能会延迟。

有关详细信息，请参阅 [Activity Namespace 参考](https://typescript.temporal.io/api/namespaces/activity#cancellation) 和 [上下文API](https://typescript.temporal.io/api/classes/activity.Context)。
:::

心跳可能并不总是发送到 Temporal 服务 — 它们可能是由 Worker 发送的 [节流](/encyclopedia/detecting-activity-failures#throttling)。
心跳限制可能会导致取消的发送时间晚于预期。

要在 TypeScript 中对 Activity 执行进行检测，请从 Activity 实现中调用 [`heartbeat()`](https://typescript.temporal.io/api/namespaces/activity#heartbeat) 函数。

```typescript
export async function myActivity(): Promise<void> {
  for (let progress = 1; progress <= 1000; ++progress) {
    // Do something that takes time
    await sleep('1s');

    heartbeat();
  }
}
```

通过向 [`heartbeat()`](https://typescript.temporal.io/api/namespaces/activity#heartbeat) 函数提供 `details` 参数，Activity 可以选择检查其进程。
如果 Activity 执行超时并重试，则 Temporal Server 将提供从上次收到的心跳到下一个 Activity 执行的 `details`。
这可用于让 Activity 有效地恢复其工作。

```typescript
export async function myActivity(): Promise<void> {
  // Resume work from latest heartbeat, if there's one, or start from 1 otherwise
  const startingPoint = activityInfo().heartbeatDetails?.progress ?? 1;

  for (let progress = startingPoint; progress <= 1000; ++progress) {
    // Do something that takes time
    await sleep('1s');

    heartbeat({ progress });
  }
}
```

<a id="activity-heartbeat-timeout"></a>
## Activity 心跳超时 {#activity-heartbeat-timeout}

**如何使用 Temporal TypeScript SDK 设置心跳超时**

[心跳超时](/encyclopedia/detecting-activity-failures#heartbeat-timeout) 与 [Activity 心跳](/encyclopedia/detecting-activity-failures#activity-heartbeat) 配合使用。
如果Temporal Server在心跳超时之前没有收到心跳，则Activity被认为超时，并且根据Retry Policy，另一个[Activity 任务执行](/tasks#activity-task-execution)可能是Scheduled。

要在 TypeScript 中设置 Activity 的心跳超时，请在使用 [`proxyActivities()`](https://typescript.temporal.io/api/interfaces/common.ActivityOptions#heartbeattimeout) API 创建相应的 Activity 代理函数时设置 [`ActivityOptions.heartbeatTimeout`](https://typescript.temporal.io/api/namespaces/workflow#proxyactivities) 属性。

```typescript
const { myLongRunningActivity } = proxyActivities<typeof activities>({
  // ...
  heartbeatTimeout: '30s',
});
```
