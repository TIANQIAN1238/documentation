<a id="namespaces"></a>
## 如何创建和管理 Namespace {#namespaces}

您可以使用 Temporal CLI 或 SDK API 创建 Update、弃用或删除 [Namespaces](/namespaces)。

根据您的需要，使用 Namespace 隔离您的 Workflow Execution。
例如，您可以使用 Namespace 通过单独的 `dev` 和 `prod` Namespace 来匹配开发生命周期。
您还可以使用它们来确保不同团队之间的 Workflow Execution 永远不会进行通信 - 例如确保 `teamA` Namespace 永远不会影响 `teamB` Namespace。

在 Temporal Cloud 上，使用 [Temporal Cloud 用户界面](/cloud/namespaces#create-a-namespace) 从 UI 创建和管理 Namespace，或使用 [tcld 命令](https://docs.temporal.io/cloud/tcld/namespace/) 从命令行界面管理 Namespace。

在自托管的 Temporal 服务上，您可以使用 Temporal CLI（推荐）或以编程方式使用 API 来注册和管理您的 Namespace。
请注意，这些 API 和 Temporal CLI 命令不适用于 Temporal Cloud。

在 Temporal 服务中的前端服务上使用自定义 [授权人](/self-hosted-guide/security#authorizer-plugin) 来设置对谁可以创建 Update 或弃用 Namespace 的限制。

您必须先向 Temporal 服务注册 Namespace，然后再在 Temporal CLIent 中设置它。

<a id="register-namespace"></a>
### 如何注册 Namespace {#register-namespace}

注册 Namespace 会在 Temporal 服务或 Temporal Cloud 上创建 Namespace。

在 Temporal Cloud 上，使用 [Temporal Cloud 用户界面](/cloud/namespaces#create-a-namespace) 或 [tcld 命令](https://docs.temporal.io/cloud/tcld/namespace/) 创建 Namespace。

在自托管 Temporal 服务上，您可以使用 Temporal CLI（推荐）或以编程方式使用 API 注册您的 Namespace。
请注意，这些 API 和 Temporal CLI 命令不适用于 Temporal Cloud。

在 Temporal 服务中的前端服务上使用自定义 [授权人](/self-hosted-guide/security#authorizer-plugin) 来设置对谁可以创建 Update 或弃用 Namespace 的限制。

<a id="manage-namespaces"></a>
### 如何管理 Namespaces {#manage-namespaces}

您可以获取 Namespace、Update Namespace 配置的详细信息，并弃用或删除 Namespace。

在Temporal Cloud上，使用[Temporal Cloud 用户界面](/cloud/namespaces#create-a-namespace)或[tcld 命令](https://docs.temporal.io/cloud/tcld/namespace/)来管理Namespace。

在自托管的 Temporal 服务上，您可以使用 Temporal CLI（推荐）或以编程方式使用 API 来管理注册的 Namespace。
请注意，这些 API 和 Temporal CLI 命令不适用于 Temporal Cloud。

在 Temporal 服务中的前端服务上使用自定义 [授权人](/self-hosted-guide/security#authorizer-plugin) 来设置对谁可以创建 Update 或弃用 Namespace 的限制。

您必须先向 Temporal 服务注册 Namespace，然后再在 Temporal CLIent 中设置它。
