## 如何启动子 Workflow Execution {#child-workflows}

[儿童 Workflow Execution](/child-workflows) 是一个 Workflow Execution，它是使用 Child Workflow API 来自另一个 Workflow 的 Scheduled。

使用 Child Workflow API 时，Child Workflow 相关事件（例如 [StartChildWorkflowExecutionInitiated](/references/events#startchildworkflowexecutioninitiated)、[子WorkflowExecutionStarted](/references/events#childworkflowexecutionstarted) 和 [ChildWorkflowExecutionCompleted](/references/events#childworkflowexecutioncompleted)）将记录在子 Workflow Execution 的 Event History 中。

始终阻止进度，直到 [子WorkflowExecutionStarted](/references/events#childworkflowexecutionstarted) 事件记录到 Event History，以确保子 Workflow Execution 已启动。
之后，可以使用 Child Workflow 选项中设置的 `Abandon` [家长关闭政策](/parent-close-policy) 放弃子 Workflow Execution。

要确保 Child Workflow Execution 已启动，请首先在 Child Workflow future 实例上调用 Child Workflow Execution 方法，该方法返回不同的 future。

然后获取一个对象的值，该对象充当最初未知的结果的代理，该结果将等待子 Workflow Execution 生成。

要启动子 Workflow Execution 并向其返回 [处理](https://typescript.temporal.io/api/interfaces/workflow.ChildWorkflowHandle/)，请使用 [启动子进程](https://typescript.temporal.io/api/namespaces/workflow/#startchild)。

```ts
import { startChild } from '@temporalio/workflow';

export async function parentWorkflow(names: string[]) {
  const childHandle = await startChild(childWorkflow, {
    args: [name],
    // workflowId, // add business-meaningful workflow id here
    // // regular workflow options apply here, with two additions (defaults shown):
    // cancellationType: ChildWorkflowCancellationType.WAIT_CANCELLATION_COMPLETED,
    // parentClosePolicy: ParentClosePolicy.PARENT_CLOSE_POLICY_TERMINATE
  });
  // you can use childHandle to signal or get result here
  await childHandle.signal('anySignal');
  const result = childHandle.result();
  // you can use childHandle to signal, query, cancel, terminate, or get result here
}
```

要启动子 Workflow Execution 并等待其完成，请使用 [执行子进程](https://typescript.temporal.io/api/namespaces/workflow/#executechild)。

默认情况下，子项是与父项位于同一 Task Queue 上的 Scheduled。

[子-Workflows/src/workflows.ts](https://github.com/temporalio/samples-typescript/blob/main/child-workflows/src/workflows.ts)
```ts
import { executeChild } from '@temporalio/workflow';

export async function parentWorkflow(...names: string[]): Promise<string> {
  const responseArray = await Promise.all(
    names.map((name) =>
      executeChild(childWorkflow, {
        args: [name],
        // workflowId, // add business-meaningful workflow id here
        // // regular workflow options apply here, with two additions (defaults shown):
        // cancellationType: ChildWorkflowCancellationType.WAIT_CANCELLATION_COMPLETED,
        // parentClosePolicy: ParentClosePolicy.PARENT_CLOSE_POLICY_TERMINATE
      }),
    ),
  );
  return responseArray.join('\n');
}
```

要从 Workflow 内部控制任何正在运行的 Workflow，请使用 [获取外部WorkflowHandle（WorkflowId）](https://typescript.temporal.io/api/namespaces/workflow/#getexternalworkflowhandle)。

```ts
import { getExternalWorkflowHandle, workflowInfo } from '@temporalio/workflow';

export async function terminateWorkflow() {
  const { workflowId } = workflowInfo(); // no await needed
  const handle = getExternalWorkflowHandle(workflowId); // sync function, not async
  await handle.cancel();
}
```

如果未显式设置 Child Workflow 选项，它们将从父 Workflow 选项继承其值。
Child Workflow 独有的两个高级选项：

- [取消类型](https://typescript.temporal.io/api/enums/proto.coresdk.child_workflow.ChildWorkflowCancellationType)：控制取消 Child Workflow 时何时抛出 `CanceledFailure` 异常。
- `parentClosePolicy`：在下一节中解释。

如果您需要取消子项 Workflow Execution，请使用 [取消范围](/develop/typescript/workflows/cancellation-scopes)。
当取消子 Workflow Execution 的包含范围时，子 Workflow Execution 会自动取消。

### 如何设置父关闭策略 {#parent-close-policy}

[家长关闭政策](/parent-close-policy) 确定如果子 Workflow Execution 的父级更改为关闭状态（已完成、失败或超时），子级 Workflow Execution 会发生什么情况。

默认“父关闭策略”选项设置为终止子 Workflow Execution。

要指定 Child Workflow 如何对父 Workflow 达到关闭状态做出反应，请使用 [`parentClosePolicy`](https://typescript.temporal.io/api/interfaces/workflow.ChildWorkflowOptions#parentclosepolicy) 选项。
