在稳定版本发布之前配置可能会发生变化。加入
[#large-payloads Slack 通道](https://temporalio.slack.com/archives/C09VA2DE15Y) 提供反馈或寻求帮助。

:::

默认情况下，Temporal 服务强制执行每个有效负载 2 MB 的限制。此限制可在自托管上配置
部署。当您的 Workflow 或活动处理的数据大于限制时，您可以将有效负载卸载到外部
存储，例如 Amazon S3，并通过 Event History 传递一个小的引用令牌。本页向您展示如何
使用 Amazon S3 设置外部存储以及如何实施自定义存储驱动程序。

有关外部存储及其用例的概念概述，请参阅 [外部存储](/external-storage)。

<a id="store-and-retrieve-large-payloads-with-amazon-s3"></a>
## 使用 Amazon S3 存储和检索大型负载 {#store-and-retrieve-large-payloads-with-amazon-s3}

Go SDK 包含 S3 存储驱动程序。请按照以下步骤进行设置：

<a id="prerequisites"></a>
### 先决条件 {#prerequisites}

- 您具有读写访问权限的 Amazon S3 存储桶。参考[生命周期管理](/external-storage#lifecycle)
  确保您的有效负载在 Workflow 的整个生命周期内保持可用。
- 安装S3驱动模块：`go get go.Temporal.io/sdk/contrib/aws/s3driver`

＃＃＃ 程序

1. 加载您的 AWS 配置并创建 S3 存储驱动程序。驱动程序使用您的标准 AWS
   来自环境的凭证（环境变量、IAM 角色或 AWS 配置文件）：

<!--SNIPSTART go-s3-driver-create-->

   ```go
   import (
       "github.com/aws/aws-sdk-go-v2/config"
       "github.com/aws/aws-sdk-go-v2/service/s3"
       "go.temporal.io/sdk/contrib/aws/s3driver"
       "go.temporal.io/sdk/contrib/aws/s3driver/awssdkv2"
   )

   cfg, err := config.LoadDefaultConfig(context.Background(),
       config.WithRegion("us-east-2"),
   )
   if err != nil {
       log.Fatalf("load AWS config: %v", err)
   }

   driver, err := s3driver.NewDriver(s3driver.Options{
       Client: awssdkv2.NewClient(s3.NewFromConfig(cfg)),
       Bucket: s3driver.StaticBucket("my-temporal-payloads"),
   })
   if err != nil {
       log.Fatalf("create S3 driver: %v", err)
   }
   ```

<!--SNIPEND-->

2. 在 `ExternalStorage` 上配置驱动程序并将其传递到您的客户端选项中：

<!--SNIPSTART go-s3-external-storage-setup-->

   ```go
   import (
       "go.temporal.io/sdk/client"
       "go.temporal.io/sdk/converter"
       "go.temporal.io/sdk/worker"
   )

   c, err := client.Dial(client.Options{
       HostPort: "localhost:7233",
       ExternalStorage: converter.ExternalStorage{
           Drivers: []converter.StorageDriver{driver},
       },
   })
   if err != nil {
       log.Fatalf("connect to Temporal: %v", err)
   }
   defer c.Close()

   w := worker.New(c, "my-task-queue", worker.Options{})
   ```

<!--SNIPEND-->

默认情况下，大于 256 KiB 的有效负载将卸载到外部存储。您可以使用以下命令进行调整
   `PayloadSizeThreshold` 选项，甚至将其设置为 1 以外部化所有有效负载，无论大小如何。参考
   [配置负载大小阈值](#configure-payload-size-threshold) 了解更多信息。

所有在 Worker 上运行的 Workflow 和活动都会自动使用存储驱动程序，而无需更改您的
   业务逻辑。驱动程序同时上传和下载有效负载，并在检索时验证有效负载的完整性。

<a id="procedure"></a>
## 实现自定义存储驱动程序 {#procedure}

如果您需要内置驱动程序允许之外的存储后端，您可以实现自己的存储驱动程序。
持久存储有效负载，以便它们能够在进程崩溃时幸存下来，并在崩溃后仍可用于调试和审核
Workflow 完成。请参阅 [生命周期管理](/external-storage#lifecycle) 了解保留要求。

以下示例显示使用本地磁盘作为后备存储的自定义驱动程序。此示例适用于本地
仅开发和测试。在生产中，使用所有 Worker 都可以访问的耐用存储系统：

<!--SNIPSTART go-custom-storage-driver-->

```go
package main

import (
	"fmt"
	"os"
	"path/filepath"

	"github.com/google/uuid"
	commonpb "go.temporal.io/api/common/v1"
	"go.temporal.io/sdk/converter"
	"google.golang.org/protobuf/proto"
)

type LocalDiskStorageDriver struct {
	storeDir string
}

func NewLocalDiskStorageDriver(storeDir string) converter.StorageDriver {
	return &LocalDiskStorageDriver{storeDir: storeDir}
}

func (d *LocalDiskStorageDriver) Name() string {
	return "my-local-disk"
}

func (d *LocalDiskStorageDriver) Type() string {
	return "local-disk"
}

func (d *LocalDiskStorageDriver) Store(
	ctx converter.StorageDriverStoreContext,
	payloads []*commonpb.Payload,
) ([]converter.StorageDriverClaim, error) {
	dir := d.storeDir
	switch info := ctx.Target.(type) {
	case converter.StorageDriverWorkflowInfo:
		if info.WorkflowID != "" {
			dir = filepath.Join(d.storeDir, info.Namespace, info.WorkflowID)
		}
	case converter.StorageDriverActivityInfo:
		// StorageDriverActivityInfo is only used for standalone (non-workflow-bound)
		// activities. Activities started by a workflow use StorageDriverWorkflowInfo.
		if info.ActivityID != "" {
			dir = filepath.Join(d.storeDir, info.Namespace, info.ActivityID)
		}
	}
	if err := os.MkdirAll(dir, 0o755); err != nil {
		return nil, fmt.Errorf("create store directory: %w", err)
	}

	claims := make([]converter.StorageDriverClaim, len(payloads))
	for i, payload := range payloads {
		key := uuid.NewString() + ".bin"
		filePath := filepath.Join(dir, key)

		data, err := proto.Marshal(payload)
		if err != nil {
			return nil, fmt.Errorf("marshal payload: %w", err)
		}
		if err := os.WriteFile(filePath, data, 0o644); err != nil {
			return nil, fmt.Errorf("write payload: %w", err)
		}

		claims[i] = converter.StorageDriverClaim{
			ClaimData: map[string]string{"path": filePath},
		}
	}
	return claims, nil
}

func (d *LocalDiskStorageDriver) Retrieve(
	ctx converter.StorageDriverRetrieveContext,
	claims []converter.StorageDriverClaim,
) ([]*commonpb.Payload, error) {
	payloads := make([]*commonpb.Payload, len(claims))
	for i, claim := range claims {
		filePath := claim.ClaimData["path"]
		data, err := os.ReadFile(filePath)
		if err != nil {
			return nil, fmt.Errorf("read payload: %w", err)
		}
		payload := &commonpb.Payload{}
		if err := proto.Unmarshal(data, payload); err != nil {
			return nil, fmt.Errorf("unmarshal payload: %w", err)
		}
		payloads[i] = payload
	}
	return payloads, nil
}
```

<!--SNIPEND-->

以下各节将介绍驱动程序实现的关键部分。

<a id="implement-a-custom-storage-driver"></a>
### 1.实现StorageDriver接口 {#implement-a-custom-storage-driver}

自定义驱动程序通过四种方法实现 `converter.StorageDriver` 接口：

- `Name()` 返回标识驱动程序实例的唯一字符串。 SDK 将此名称存储在声明检查中
  引用，以便它可以将检索请求路由到正确的驱动程序。存储有效负载后更改名称
  中断检索。例如，两个 S3 驱动程序可以命名为 `"s3-primary"` 和 `"s3-archive"`。
- `Type()` 返回一个标识驱动程序实现的字符串。与 `Name()` 不同，这必须在所有
  相同驱动程序类型的实例，无论配置如何。两名名为 `"s3-primary"` 和 `"s3-archive"` 的 S3 车手都将回归
  `"aws.s3driver"` 作为其类型，而自定义驱动程序代码示例中的本地磁盘驱动程序返回 `"local-disk"`。
- `Store()` 接收一片有效负载并为每个有效负载返回一个 `StorageDriverClaim`。声明是一组字符串
  驱动程序稍后使用键值对来定位有效负载。
- `Retrieve()` 接收 `Store()` 生成的声明并返回原始有效负载。

<a id="1-implement-the-storagedriver-interface"></a>
### 2. 存储有效负载 {#1-implement-the-storagedriver-interface}

在 `Store()` 中，将每个 Payload Protobuf 消息编组为 `proto.Marshal(payload)` 字节，并将字节写入
你的存储系统。应用数据已被 [有效负载转换器](/develop/go/data-handling/data-conversion) 序列化
和 [有效负载编解码器](/develop/go/data-handling/data-encryption) 在到达驾驶员之前。
有关详细信息，请参阅 [数据转换管道](/external-storage#data-pipeline)。

为每个有效负载返回 `StorageDriverClaim`，其中包含足够的信息以便稍后检索。 `ctx.Target`
根据操作提供身份信息（Namespace、Workflow ID）。使用类型开关
`StorageDriverWorkflowInfo` 和 `StorageDriverActivityInfo` 访问具体值。考虑构建
您的存储密钥包含此信息，以便您可以识别哪个 Workflow 拥有每个有效负载。

<a id="2-store-payloads"></a>
### 3. 检索有效负载 {#2-store-payloads}

在 `Retrieve()` 中，使用声明数据下载字节，然后使用以下命令重建有效负载 Protobuf 消息
`proto.Unmarshal(data, payload)`。 Payload Converter 在驱动程序之后处理应用程序数据的反序列化
返回有效负载。

<a id="3-retrieve-payloads"></a>
### 4. 配置客户端 {#3-retrieve-payloads}

在客户端选项中将 `ExternalStorage` 结构与您的驱动程序一起传递：

```go
c, err := client.Dial(client.Options{
    ExternalStorage: converter.ExternalStorage{
        Drivers: []converter.StorageDriver{NewLocalDiskStorageDriver("/tmp/temporal-payload-store")},
    },
})
```

<a id="4-configure-the-client"></a>
## 配置负载大小阈值 {#4-configure-the-client}

您可以配置触发外部存储的负载大小阈值。默认情况下，有效负载大于 256 KiB
被卸载到外部存储。您可以使用 `PayloadSizeThreshold` 选项调整此值，或将其设置为 1 以
外部化所有有效负载，无论其大小如何。值 0 被解释为默认值 (256 KiB)。

<!--SNIPSTART go-external-storage-threshold-->

```go
import "go.temporal.io/sdk/converter"

c, err := client.Dial(client.Options{
    ExternalStorage: converter.ExternalStorage{
        Drivers:              []converter.StorageDriver{driver},
        PayloadSizeThreshold: 1,
    },
})
```

<!--SNIPEND-->

<a id="configure-payload-size-threshold"></a>
## 使用多个存储驱动程序 {#configure-payload-size-threshold}

当您注册多个驱动程序时，您必须提供实现 `StorageDriverSelector` 的 `DriverSelector`
界面。如果在没有选择器的情况下注册了多个驱动程序，则 SDK 在创建客户端时会返回错误。的
选择器选择哪个驱动程序存储每个有效负载。列表中未选择存储的任何驱动程序仍然是
可用于检索，这在存储后端之间迁移时非常有用。从选择器返回 `nil`
在 Event History 中保留特定的有效负载。

多个驱动程序在以下场景中很有用：

- 驱动程序迁移。您的 Worker 需要检索由使用与该驱动程序不同的驱动程序的客户端创建的有效负载
  你喜欢的一个。注册两个驱动程序并使用选择器始终为新负载选择您首选的驱动程序。
  老司机仍然可以检索现有的索赔。
- 延迟与持久性的权衡。某些 Workflow 类型可能受益于更快的存储后端
  以耐用性为代价，而其他人则需要像 S3 这样的耐用后端。使用选择器根据
  Workflow 上下文。

以下示例注册两个驱动程序，但始终选择 `preferredDriver` 作为新负载。 `legacyDriver`
注册只是为了让 Worker 可以检索之前存储的有效负载：

<!--SNIPSTART go-external-storage-multiple-drivers-->

```go
import (
    commonpb "go.temporal.io/api/common/v1"
    "go.temporal.io/sdk/converter"
)

type PreferredSelector struct {
    preferred converter.StorageDriver
}

func (s *PreferredSelector) SelectDriver(
    ctx converter.StorageDriverStoreContext,
    payload *commonpb.Payload,
) (converter.StorageDriver, error) {
    return s.preferred, nil
}

// Usage:
converter.ExternalStorage{
    Drivers:        []converter.StorageDriver{preferredDriver, legacyDriver},
    DriverSelector: &PreferredSelector{preferred: preferredDriver},
}
```

<!--SNIPEND-->
