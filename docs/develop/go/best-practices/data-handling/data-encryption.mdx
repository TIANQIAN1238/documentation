Temporal 的安全模型是围绕有效负载的客户端加密而设计的。
客户端可以在将有效负载发送到服务器之前对其进行加密，并在从服务器接收到有效负载后对其进行解密。
这提供了高度的保密性，因为 Temporal Server 本身完全不知道实际数据。
它还为实施者提供了更多的权力和更多的自由来决定哪个客户端能够读取哪些数据。实施者可以使用密钥、算法或其他安全措施来控制访问。

Temporal 开发人员通过向其客户端提供自定义有效负载编解码器来添加有效负载的客户端加密。
根据业务需求，有效负载加密的完整实施可能涉及选择适当的加密算法、管理加密密钥、限制部分用户查看有效负载输出，或这些的组合。

服务器本身永远不会在有效负载上添加加密。
因此，除非实现客户端加密，否则有效负载数据将以非加密形式持久保存到数据存储中，并且任何可以向 Temporal Namespace（包括 Temporal UI 和 CLI）发出请求的客户端都将能够读取 Workflow 中包含的有效负载。
在处理敏感数据时，您应该始终实施有效负载加密。

## 使用自定义有效负载编解码器 {#custom-payload-codec}

**第 1 步：创建自定义有效负载编解码器**

创建自定义 [有效负载编解码器](https://pkg.go.dev/go.temporal.io/sdk/converter#PayloadCodec) 实现并在 `Encode` 和 `Decode` 函数中定义加密/压缩和解密/解压缩逻辑。

有效负载编解码器将字节转换为字节。
它必须在包装 Data Converter 的 [编解码器数据转换器](https://pkg.go.dev/go.temporal.io/sdk/converter#CodecDataConverter) 实例中使用，以执行 [有效载荷](/dataconversion#payload) 转换，并将 `PayloadCodec` 中的自定义编码和解码应用于转换后的有效负载。

以下来自 [Data Converter 样品](https://github.com/temporalio/samples-go/blob/main/codec-server/data_converter.go) 的示例展示了如何创建自定义 `NewCodecDataConverter`，该自定义 `NewCodecDataConverter` 使用自定义 `PayloadCodec` 包装 Data Converter 的实例。

```go
// Create an instance of Data Converter with your codec.
var DataConverter = converter.NewCodecDataConverter(
	converter.GetDefaultDataConverter(),
	NewPayloadCodec(),
)
//...
// Create an instance of PayloadCodec.
func NewPayloadCodec() converter.PayloadCodec {
	return &Codec{}
}
```

在自定义 `PayloadCodec` 的 `Encode` 函数中实现加密/压缩逻辑，并在 `Decode` 函数中实现解密/解压缩逻辑，如以下示例所示。

```go
// Codec implements converter.PayloadEncoder for snappy compression.
type Codec struct{}

// Encode implements converter.PayloadCodec.Encode.
func (Codec) Encode(payloads []*commonpb.Payload) ([]*commonpb.Payload, error) {
	result := make([]*commonpb.Payload, len(payloads))
	for i, p := range payloads {
		// Marshal proto
		origBytes, err := p.Marshal()
		if err != nil {
			return payloads, err
		}
		// Compress
		b := snappy.Encode(nil, origBytes)
		result[i] = &commonpb.Payload{
			Metadata: map[string][]byte{converter.MetadataEncoding: []byte("binary/snappy")},
			Data:     b,
		}
	}

	return result, nil
}

// Decode implements converter.PayloadCodec.Decode.
func (Codec) Decode(payloads []*commonpb.Payload) ([]*commonpb.Payload, error) {
	result := make([]*commonpb.Payload, len(payloads))
	for i, p := range payloads {
		// Decode only if it's our encoding
		if string(p.Metadata[converter.MetadataEncoding]) != "binary/snappy" {
			result[i] = p
			continue
		}
		// Uncompress
		b, err := snappy.Decode(nil, p.Data)
		if err != nil {
			return payloads, err
		}
		// Unmarshal proto
		result[i] = &commonpb.Payload{}
		err = result[i].Unmarshal(b)
		if err != nil {
			return payloads, err
		}
	}

	return result, nil
}
```

**步骤 2：设置 Data Converter 以使用自定义有效负载编解码器。**

在用于创建客户端的 `Dial` 客户端选项中，使用 `DataConverter` 的实例设置自定义 `PayloadCodec`。

以下示例演示如何从名为 `mycodecpackage` 的包中设置自定义 Data Converter。

```go
//...
c, err := client.Dial(client.Options{
		// Set DataConverter here to ensure that Workflow inputs and results are
		// encoded as required.
		DataConverter: mycodecpackage.DataConverter,
	})
//...
```

- 数据**编码**由客户端在将输入传递到 Temporal 集群时使用 Temporal 或您的自定义实现提供的转换器和编解码器来执行。例如，纯文本输入通常会序列化为 JSON 对象，然后可以进行压缩或加密。
- 数据**解码**可以根据需要在 Workflow 或活动期间由您的应用程序逻辑执行，但解码的 Workflow 结果永远不会保留回 Temporal 集群。相反，它们以编码形式存储在集群上，并且在使用 [Temporal Workflow 显示](/cli/workflow#show) 命令或浏览 Web UI 以查看输出时，您需要提供附加参数。

作为参考，请参阅 [加密](https://github.com/temporalio/samples-go/tree/main/encryption) 示例。

### 使用 Codec Server {#using-a-codec-server}

Codec Server 是一个 HTTP 服务器，它使用您的自定义编解码器逻辑来远程解码您的数据。
Codec Server 独立于 Temporal 集群，并通过预定义端点解码您的加密负载。
您可以在自己的环境中创建、操作和管理对 Codec Server 的访问。
Temporal CLI 和 Web UI 反过来提供内置挂钩来调用 Codec Server 以按需解码加密的有效负载。
有关如何设计和部署 Codec Server 的信息，请参阅 [Codec Server](/production-deployment/data-encryption) 文档。

作为参考，请参阅 [Codec Server](https://github.com/temporalio/samples-go/tree/main/codec-server) 示例。
