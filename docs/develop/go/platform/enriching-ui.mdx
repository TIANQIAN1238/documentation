Temporal 支持使用元数据向 Workflow 和事件添加上下文。 
这有助于用户识别和了解 Workflow 及其操作。

<a id="adding-summary-and-details-to-workflows"></a>
## 添加摘要和详细信息到 Workflow {#adding-summary-and-details-to-workflows}

<a id="starting-a-workflow"></a>
### 启动 Workflow {#starting-a-workflow}

启动 Workflow 时，您可以提供静态摘要和详细信息以帮助在 UI 中识别 Workflow：

```go
import (
    "context"
    "go.temporal.io/sdk/client"
)

func main() {
    // Create the client
    c, err := client.Dial(client.Options{})
    if err != nil {
        // Handle error
    }
    defer c.Close()

    // Start workflow options with static summary and details
    workflowOptions := client.StartWorkflowOptions{
        ID:        "your-workflow-id",
        TaskQueue: "your-task-queue",
        StaticSummary: "Order processing for customer #12345",
        StaticDetails: "Processing premium order with expedited shipping",
    }

    // Start the workflow
    we, err := c.ExecuteWorkflow(context.Background(), workflowOptions, YourWorkflow, "workflow input")
    if err != nil {
        // Handle error
    }
}
```

`StaticSummary` 是出现在 Workflow 列表视图中的单行描述，限制为 200 字节。
`StaticDetails` 可以是多行的，并提供出现在 Workflow 详细信息视图中的更全面的信息，最大限制为 20K 字节。

输入格式是标准 Markdown，不包括图像、HTML 和脚本。

<a id="inside-the-workflow"></a>
### Workflow 内部 {#inside-the-workflow}

在 Workflow 中，您可以获取并设置_当前 Workflow 详细信息_。 
与 Workflow 启动时设置的静态摘要/详细信息不同，该值在 Workflow 的整个生命周期中可以是 Updated。 
当前的 Workflow 详细信息也采用 Markdown 格式（不包括图像、HTML 和脚本）并且可以跨多行。

```go
import (
    "go.temporal.io/sdk/workflow"
)

func YourWorkflow(ctx workflow.Context, input string) (string, error) {
    // Get the current details
    currentDetails := workflow.GetCurrentDetails(ctx)
    workflow.GetLogger(ctx).Info("Current details", "details", currentDetails)
    
    // Set/update the current details
    workflow.SetCurrentDetails(ctx, "Updated workflow details with new status")
    
    return "Workflow completed", nil
}
```

<a id="adding-summary-to-activities-and-timers"></a>
### 添加摘要到活动和 Timer {#adding-summary-to-activities-and-timers}

当从 Workflow 中启动活动时，您可以将元数据参数 `Summary` 附加到活动：

```go
import (
    "time"
    "go.temporal.io/sdk/workflow"
)

func YourWorkflow(ctx workflow.Context, input string) (string, error) {
    // Activity options with summary
    ao := workflow.ActivityOptions{
        StartToCloseTimeout: 10 * time.Second,
        Summary: "Processing user data",
    }
    ctx = workflow.WithActivityOptions(ctx, ao)

    // Execute the activity
    var result string
    err := workflow.ExecuteActivity(ctx, YourActivity, input).Get(ctx, &result)
    if err != nil {
        return "", err
    }
    
    return result, nil
}
```

同样，您可以将 `Summary` 附加到 Workflow 内的 Timer：

```go
import (
    "time"
    "go.temporal.io/sdk/workflow"
)

func YourWorkflow(ctx workflow.Context, input string) (string, error) {
    // Create a timer with options including summary
    timerFuture := workflow.NewTimerWithOptions(ctx, 5*time.Minute, workflow.TimerOptions{
        Summary: "Waiting for payment confirmation",
    })
    
    // Wait for the timer
    err := timerFuture.Get(ctx, nil)
    if err != nil {
        return "", err
    }
    
    return "Timer completed", nil
}
```

`Summary` 的输入格式是字符串，且限制为 200 个字节。

<a id="viewing-summary-and-details-in-the-ui"></a>
## 在 UI 中查看摘要和详细信息 {#viewing-summary-and-details-in-the-ui}

将摘要和详细信息添加到 Workflow、活动和 Timer 后，您可以在 Temporal Web UI 中查看这些丰富的信息。 
导航到 Workflow 的详细信息页面，查看两个关键位置显示的元数据：

<a id="workflow-overview-section"></a>
### Workflow 概述部分 {#workflow-overview-section}

在 Workflow 详细信息页面的顶部，您将找到 Workflow 级别的元数据：

- **摘要和详细信息** - 显示启动 Workflow 时设置的静态摘要和静态详细信息
- **当前详细信息** - 显示 Workflow Execution 期间可以是 Updated 的动态详细信息

所有 Workflow 详细信息都支持标准 Markdown 格式（不包括图像、HTML 和脚本），允许您创建丰富的结构化信息显示。

### Event History

Workflow 和 Event History 中的各个事件会显示其关联的摘要（如果有）。

Workflow、Activity 和 Timer 摘要以紫色文本显示在其相应事件旁边，提供即时上下文，无需您展开事件详细信息。 
当您展开事件时，摘要也会在详细视图中突出显示。
