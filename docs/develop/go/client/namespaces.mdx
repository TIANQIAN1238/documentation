- [注册 Namespaces](#register-namespace)

您可以使用 Temporal CLI 或 SDK API 创建 Update、弃用或删除 [Namespaces](/namespaces)。

根据您的需要，使用 Namespace 隔离您的 Workflow Execution。
例如，您可以使用 Namespace 通过单独的 `dev` 和 `prod` Namespace 来匹配开发生命周期。
您还可以使用它们来确保不同团队之间的 Workflow Execution 永远不会进行通信 - 例如确保 `teamA` Namespace 永远不会影响 `teamB` Namespace。

在 Temporal Cloud 上，使用 [Temporal Cloud 用户界面](/cloud/namespaces#create-a-namespace) 从 UI 创建和管理 Namespace，或使用 [tcld 命令](https://docs.temporal.io/cloud/tcld/namespace/) 从命令行界面管理 Namespace。

在自托管的 Temporal 服务上，您可以使用 Temporal CLI（推荐）或以编程方式使用 API 来注册和管理您的 Namespace。
请注意，这些 API 和 Temporal CLI 命令不适用于 Temporal Cloud。

在 Temporal 服务中的前端服务上使用自定义 [授权人](/self-hosted-guide/security#authorizer-plugin) 来设置对谁可以创建 Update 或弃用 Namespace 的限制。

您必须先向 Temporal 服务注册 Namespace，然后再在 Temporal CLIent 中设置它。

<a id="register-namespace"></a>
### 如何注册 Namespace {#register-namespace}

注册 Namespace 会在 Temporal 服务或 Temporal Cloud 上创建 Namespace。

在 Temporal Cloud 上，使用 [Temporal Cloud 用户界面](/cloud/namespaces#create-a-namespace) 或 [tcld 命令](https://docs.temporal.io/cloud/tcld/namespace/) 创建 Namespace。

在自托管 Temporal 服务上，您可以使用 Temporal CLI（推荐）或以编程方式使用 API 注册您的 Namespace。
请注意，这些 API 和 Temporal CLI 命令不适用于 Temporal Cloud。

在 Temporal 服务中的前端服务上使用自定义 [授权人](/self-hosted-guide/security#authorizer-plugin) 来设置对谁可以创建 Update 或弃用 Namespace 的限制。

使用[`Register` API](https://pkg.go.dev/go.temporal.io/sdk/client#NamespaceClient)和`NamespaceClient`接口来注册[Namespace](/namespaces)，并为Workflow Execution设置[保留期限](/temporal-service/temporal-server#retention-period)，为Namespace设置Event History。

您也可以[使用 Temporal CLI 命令行工具注册 Namespace](/cli/operator#create)。

```go
client, err := client.NewNamespaceClient(client.Options{HostPort: ts.config.ServiceAddr})
        //...
    err = client.Register(ctx, &workflowservice.RegisterNamespaceRequest{
        Namespace: your-namespace-name,
        WorkflowExecutionRetentionPeriod: &retention,
    })
```

使用 `WorkflowExecutionRetentionPeriod` 的保留期设置是强制性的。
您可以为此时间段设置的最小值为 1 天。

注册后，在 Workflow 客户端中使用 `Dial` 设置 Namespace，以在该 Namespace 中运行 Workflow Execution。
详细信息请参见 [如何在 Go 客户端中设置 Namespace](/develop/go/client/temporal-client#connect-to-temporal-cloud)。

请注意，使用此 API 进行 Namespace 注册最多需要 10 秒才能完成。
确保在针对 Namespace 启动 Workflow Execution 之前等待此注册完成。

对于 Update 和 Namespace，请将 [`Update` API](https://pkg.go.dev/go.temporal.io/sdk/client#NamespaceClient) 与 `NamespaceClient` 一起使用。

要使用 Temporal CLI 的 Update 或 Namespace，请使用 [Temporal 操作员 Namespace Update](/cli/operator#update) 命令。

<a id="manage-namespaces"></a>
### 如何管理 Namespaces {#manage-namespaces}

您可以获取 Namespace、Update Namespace 配置的详细信息，并弃用或删除 Namespace。

在Temporal Cloud上，使用[Temporal Cloud 用户界面](/cloud/namespaces#create-a-namespace)或[tcld 命令](https://docs.temporal.io/cloud/tcld/namespace/)来管理Namespace。

在自托管的 Temporal 服务上，您可以使用 Temporal CLI（推荐）或以编程方式使用 API 来管理注册的 Namespace。
请注意，这些 API 和 Temporal CLI 命令不适用于 Temporal Cloud。

在 Temporal 服务中的前端服务上使用自定义 [授权人](/self-hosted-guide/security#authorizer-plugin) 来设置对谁可以创建 Update 或弃用 Namespace 的限制。

您必须先向 Temporal 服务注册 Namespace，然后再在 Temporal CLIent 中设置它。

在Temporal Cloud上，使用[Temporal Cloud 用户界面](/cloud/namespaces)或[tcld 命令](/cloud/tcld/namespace/)来管理Namespace。

在自托管的 Temporal 服务上，您可以使用 Temporal CLI（推荐）或以编程方式使用 API 来管理注册的 Namespace。
请注意，这些 API 和 Temporal CLI 命令不适用于 Temporal Cloud。

- 在您的 Temporal 服务上注册的 Namespace 的 Update 信息和配置：

- 使用 Temporal CLI：[`Temporal operator Namespace Update`](/cli/operator#update)
    示例
  - 在 Namespace 上使用 [`UpdateNamespace` API](https://pkg.go.dev/go.temporal.io/sdk/client#NamespaceClient) 至 Update 配置。
    例子

    ```go
    //...
      err = client.Update(context.Background(), &workflowservice.UpdateNamespaceRequest{
      Namespace:         "your-namespace-name",
      UpdateInfo:        &namespace.UpdateNamespaceInfo{ //updates info for the namespace "your-namespace-name"
          Description:   "updated namespace description",
          OwnerEmail:    "newowner@mail.com",
          //Data:        nil,
          //State:       0,
      },
      /*other details that you can update:
      Config:            &namespace.NamespaceConfig{ //updates the configuration of the namespace with the following options
          //WorkflowExecutionRetentionTtl: nil,
          //BadBinaries:                   nil,
          //HistoryArchivalState:          0,
          //HistoryArchivalUri:            "",
          //visibilityArchivalState:       0,
          //visibilityArchivalUri:         "",
      },
      ReplicationConfig: &replication.NamespaceReplicationConfig{ //updates the replication configuration for the namespace
          //ActiveClusterName: "",
          //Clusters:          nil,
          //State:             0,
      },
      SecurityToken:     "",
      DeleteBadBinary:   "",
      PromoteNamespace:  false,
      })*/
    //...
    ```

- 获取在 Temporal 服务上注册的 Namespace 的详细信息：

- 使用 Temporal CLI：[`Temporal operator Namespace describe`](/cli/operator#describe)
  - 使用 [`DescribeNamespace` API](https://pkg.go.dev/go.temporal.io/sdk/client#NamespaceClient) 返回已注册 Namespace 的信息和配置详细信息。
    例子

    ```go
    //...
      client, err := client.NewNamespaceClient(client.Options{})
      //...
      client.Describe(context.Background(), "default")
    //...
    ```

- 获取 Temporal 服务上所有已注册 Namespace 的详细信息：

- 使用 Temporal CLI：[`Temporal operator Namespace list`](/cli/operator#list)
  - 使用 [`ListNamespace` API](https://github.com/temporalio/api/blob/f0350f8032ad2f0c60c539b3b61ea37f412f1cf7/temporal/api/operatorservice/v1/service.proto) 返回 Temporal 服务上所有已注册 Namespace 的信息和配置详细信息。
    例子

  ```go
  //...
      namespace.Handler.ListNamespaces(context.Context(), &workflowservice.ListNamespacesRequest{ //lists 1 page (1-100) of namespaces on the active Temporal Service. You can set a large PageSize or loop until NextPageToken is nil
          //PageSize:        0,
          //NextPageToken:   nil,
          //NamespaceFilter: nil,
          })
  //...
  ```

- 删除 Namespace：[`DeleteNamespace` API](https://github.com/temporalio/api/blob/f0350f8032ad2f0c60c539b3b61ea37f412f1cf7/temporal/api/operatorservice/v1/service.proto) 删除 Namespace。删除 Namespace 会删除 Namespace 上所有正在运行和已完成的 Workflow Execution，并将它们从持久性存储和 Visibility 存储中删除。
  例子：

  ```go
  //...
  client.OperatorService().DeleteNamespace(ctx, &operatorservice.DeleteNamespaceRequest{...
  //...
  ```
