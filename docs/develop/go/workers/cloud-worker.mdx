<a id="run-a-temporal-cloud-worker"></a>
## 如何运行 Temporal Cloud Worker {#run-a-temporal-cloud-worker}

要运行使用 [Temporal Cloud](/cloud) 的 Worker，您需要提供其他连接和客户端选项，其中包括以下内容：

- 包含您的 [云 Namespace 名称](/namespaces) 和端口号的地址：`<Namespace>.<ID>.tmprl.cloud:<port>`。
- mTLS CA 证书。
- mTLS 私钥。

有关管理和生成 Temporal Cloud 客户端证书的更多信息，请参阅 [如何在Temporal Cloud中管理证书](/cloud/certificates)。

有关配置 TLS 以保护 Temporal 服务的网络间和网络内通信的更多信息，请参阅 [Temporal 定制样品](https://github.com/temporalio/samples-server)。

要运行与 Temporal Cloud 对话的 Worker，您需要以下内容：

- 已添加到您的 Namespace 的兼容 mTLS CA 证书和 mTLS 私钥。
  参见 [证书要求](/cloud/certificates#certificate-requirements)。
- 您的 [Temporal Cloud Namespace ID](/cloud/namespaces#temporal-cloud-namespace-id)，其中包括您的 [Temporal Cloud Namespace 名称](/cloud/namespaces#temporal-cloud-namespace-name) 以及附加的唯一的五位或六位数字 [Temporal Cloud 帐户 ID](/cloud/namespaces#temporal-cloud-account-id)。
  此信息可以在您的 Namespace 的 URL 中找到；例如，`https://cloud.Temporal.io/namespaces/yourNamespace.a2fx6/`。
  请记住，Namespace ID 必须包含帐户 ID：`yourNamespace.a2fx6`。

有关管理和生成 Temporal Cloud 客户端证书的更多信息，请参阅 [如何在Temporal Cloud中管理证书](/cloud/certificates)。

有关配置 TLS 以保护 Temporal 服务的网络间和网络内通信的更多信息，请参阅 [Temporal 定制样品](https://github.com/temporalio/samples-server)。

```go
package main

import (
    "crypto/tls"
    "log"

    "go.temporal.io/sdk/client"
    "go.temporal.io/sdk/worker"

    "documentation-samples-go/cloud"
)

func main() {
    // Get the key and cert from your env or local machine
    clientKeyPath := "./secrets/yourkey.key"
    clientCertPath := "./secrets/yourcert.pem"
    // Specify the host and port of your Temporal Cloud Namespace
    // Host and port format: namespace.unique_id.tmprl.cloud:port
    hostPort := "<yournamespace>.<id>.tmprl.cloud:7233"
    namespace := "<yournamespace>.<id>"
    // Use the crypto/tls package to create a cert object
    cert, err := tls.LoadX509KeyPair(clientCertPath, clientKeyPath)
    if err != nil {
        log.Fatalln("Unable to load cert and key pair.", err)
    }
    // Add the cert to the tls certificates in the ConnectionOptions of the Client
    temporalClient, err := client.Dial(client.Options{
        HostPort:  hostPort,
        Namespace: namespace,
        ConnectionOptions: client.ConnectionOptions{
            TLS: &tls.Config{Certificates: []tls.Certificate{cert}},
        },
    })
    if err != nil {
        log.Fatalln("Unable to connect to Temporal Cloud.", err)
    }
    defer temporalClient.Close()
    // Create a new Worker.
    yourWorker := worker.New(temporalClient, "cloud-connection-example-go-task-queue", worker.Options{})
// ...
}
```

<a id="register-types"></a>
### 如何注册类型{#register-types}

必须注册所有监听相同 Task Queue 名称的 Worker 才能处理完全相同的 Workflow 类型和 Activity 类型。

如果 Worker 轮询任务以查找它不知道的 Workflow 类型或 Activity 类型，则该任务将失败。
但是，任务失败不会导致关联的 Workflow Execution 失败。

`RegisterWorkflow()` 和 `RegisterActivity()` 调用本质上是在 Worker 进程内创建 Workflow 类型及其实现之间的内存中映射。

**注册 Activity `structs`**

根据 [Activity Definition](/develop/go/activities/basics#activity-definition) 最佳实践，您可能有一个包含多个方法和字段的 Activity 结构。
当您将 `RegisterActivity()` 用于 Activity 结构时，Worker 可以访问所有导出的方法。

**注册多个类型**

要向 Worker 实体注册多个 Activity 类型和/或 Workflow 类型，只需进行多个 Activity 注册调用，但确保每个类型名称是唯一的：

```go
w.RegisterActivity(ActivityA)
w.RegisterActivity(ActivityB)
w.RegisterActivity(ActivityC)
w.RegisterWorkflow(WorkflowA)
w.RegisterWorkflow(WorkflowB)
w.RegisterWorkflow(WorkflowC)
```

<a id="registerworkflowoptions"></a>
### 如何在 Go 中设置 RegisterWorkflowOptions {#registerworkflowoptions}

从 `go.Temporal.io/sdk/Workflow` 包创建 [`RegisterOptions`](https://pkg.go.dev/go.temporal.io/sdk/workflow#RegisterOptions) 的实例，并在使用 Worker 注册 Workflow 类型时将其传递给 [`RegisterWorkflowWithOptions`](https://pkg.go.dev/go.temporal.io/sdk/worker#WorkflowRegistry) 调用。

- 用于设置注册 Workflow 的选项

| 场地                                                             | 必需的 | 类型     |
| ----------------------------------------------------------------- | -------- | -------- |
| [`Name`](#name)                                                   | 不       | `string` |
| [`DisableAlreadyRegisteredCheck`](#disablealreadyregisteredcheck) | 不       | `bool`   |

＃＃＃＃ 姓名

参见 [如何在 Go 中自定义 Workflow 类型](/develop/go/workflows/basics#customize-workflow-type)

<a id="name"></a>
#### 禁用已注册检查 {#name}

禁用检查 Workflow 类型是否已注册。

- 型号：`bool`
- 默认：`false`

```go
// ...
w := worker.New(temporalClient, "your_task_queue_name", worker.Options{})
registerOptions := workflow.RegisterOptions{
  DisableAlreadyRegisteredCheck: `false`,
  // ...
}
w.RegisterWorkflowWithOptions(YourWorkflowDefinition, registerOptions)
// ...
```

<a id="disablealreadyregisteredcheck"></a>
### 如何在 Go 中设置 RegisterActivityOptions {#disablealreadyregisteredcheck}

从 `go.Temporal.io/sdk/Activity` 包创建 [`RegisterOptions`](https://pkg.go.dev/go.temporal.io/sdk/activity#RegisterOptions) 的实例，并在使用 Worker 注册 Activity 类型时将其传递给 [`RegisterActivityWithOptions`](https://pkg.go.dev/go.temporal.io/sdk/worker#ActivityRegistry) 调用。

注册 Activity 的选项

| 场地                                                             | 必需的 | 类型     |
| ----------------------------------------------------------------- | -------- | -------- |
| [`Name`](#name)                                                   | 不       | `string` |
| [`DisableAlreadyRegisteredCheck`](#disablealreadyregisteredcheck) | 不       | `bool`   |
| [`SkipInvalidStructFunctions`](#skipinvalidstructfunctions)       | 不       | `bool`   |

＃＃＃＃ 姓名

参见 [如何在 Go 中自定义 Activity 类型](/develop/go/activities/basics#customize-activity-type)。

<a id="registeractivityoptions"></a>
#### 禁用已注册检查 {#registeractivityoptions}

禁用检查 Activity 是否已注册。

- 型号：`bool`
- 默认：`false`

```go
// ...
w := worker.New(temporalClient, "your_task_queue_name", worker.Options{})
registerOptions := activity.RegisterOptions{
  DisableAlreadyRegisteredCheck: false,
  // ...
}
w.RegisterActivityWithOptions(a.YourActivityDefinition, registerOptions)
// ...
```

<a id="name"></a>
#### 跳过InvalidStructFunctions {#name}

注册具有活动的结构时，跳过无效的函数。
如果为 false，注册会发生混乱。

- 型号：`bool`
- 默认：`false`

```go
// ...
w := worker.New(temporalClient, "your_task_queue_name", worker.Options{})
registerOptions := activity.RegisterOptions{
  SkipInvalidStructFunctions: false,
  // ...
}
w.RegisterActivityWithOptions(a.YourActivityDefinition, registerOptions)
// ...
```
