## 如何在 Go 中开发 Worker {#develop-worker}

通过调用 [`Worker.New()`](https://pkg.go.dev/go.temporal.io/sdk/worker#Worker)（可通过 `go.Temporal.io/sdk/Worker` 包获取）创建 [`Worker`](https://pkg.go.dev/go.temporal.io/sdk/worker#New) 的实例，并向其传递以下参数：

1. Temporal Go SDK `Client` 的实例。
1. 它将轮询的 Task Queue 的名称。
1. `Worker.Options` 的实例，可以为空。

然后，注册 Worker 将能够执行的 Workflow 类型和 Activity 类型。

最后，在 Worker 实例上调用 `Start()` 或 `Run()` 方法。
Run 接受中断通道作为参数，以便可以在终端中停止 Worker。
否则，必须调用 `Stop()` 方法来停止 Worker。

:::tip

如果您安装了 [`gow`](https://github.com/mitranim/gow)，则当您 Update Worker 文件时，Worker 进程会自动“重新加载”：

```bash
go install github.com/mitranim/gow@latest
gow run worker/main.go # automatically reloads when file changes
```

:::

<div className="copycode-notice-container">
  <a href="https://github.com/temporalio/documentation/blob/main/sample-apps/go/yourapp/worker/main_dacx.go">
查看源代码
  </a>{' '}
在应用程序代码的其余部分的上下文中。
</div>

```go
package main

import (
    "log"

    "go.temporal.io/sdk/activity"
    "go.temporal.io/sdk/client"
    "go.temporal.io/sdk/worker"
    "go.temporal.io/sdk/workflow"

    "documentation-samples-go/yourapp"
)

func main() {
    // Create a Temporal Client
    // A Temporal Client is a heavyweight object that should be created just once per process.
    temporalClient, err := client.Dial(client.Options{})
    if err != nil {
        log.Fatalln("Unable to create client", err)
    }
    defer temporalClient.Close()
    // Create a new Worker.
    yourWorker := worker.New(temporalClient, "your-custom-task-queue-name", worker.Options{})
    // Register your Workflow Definitions with the Worker.
    // Use the RegisterWorkflow or RegisterWorkflowWithOptions method for each Workflow registration.
    yourWorker.RegisterWorkflow(yourapp.YourWorkflowDefinition)
// ...
    // Register your Activity Definitions with the Worker.
    // Use this technique for registering all Activities that are part of a struct and set the shared variable values.
    message := "This could be a connection string or endpoint details"
    number := 100
    activities := &yourapp.YourActivityObject{
        Message: &message,
        Number:  &number,
    }
    // Use the RegisterActivity or RegisterActivityWithOptions method for each Activity.
    yourWorker.RegisterActivity(activities)
// ...
    // Run the Worker
    err = yourWorker.Run(worker.InterruptCh())
    if err != nil {
        log.Fatalln("Unable to start Worker", err)
    }
}
// ...
```

<a id="maxconcurrentactivityexecutionsize"></a>
#### 最大并发ActivityExecutionSize {#maxconcurrentactivityexecutionsize}

设置 Worker 的最大并发 Activity 执行。

- 型号：`int`
- 默认：`1000`

`0` 值设置为默认值。

```go
// ...
workerOptions := worker.Options{
  MaxConcurrentActivityExecutionSize: 1000,
  // ...
}
w := worker.New(c, "your_task_queue_name", workerOptions)
// ...
```

#### WorkerActivitiesPerSecond

速率限制 Worker 每秒启动的 Activity 任务执行的数量。

- 型号：`float64`
- 默认：`100000`

`0` 值设置为默认值。

预期用例是限制 Worker 使用的资源。

请注意，值类型是浮点数，因此如果需要，该值可以小于 1。
例如，如果设置为 0.1，Activity 任务执行将每十秒发生一次。
这可用于保护下游服务免受请求泛滥。

```go
// ...
workerOptions := worker.Options{
    WorkerActivitiesPerSecond: 100000,
  // ..
}
w := worker.New(c, "your_task_queue_name", workerOptions)
// ...
```

<a id="taskqueueactivitiespersecond"></a>
#### 任务队列活动每秒 {#taskqueueactivitiespersecond}

速率限制每秒可以启动的 Activity 执行数量。

- 型号：`float64`
- 默认：`100000`

`0` 值设置为默认值。

此速率由 Temporal 服务管理，并限制整个 Task Queue 每秒的 Activity 任务数。这与 [`WorkerActivityTasksPerSecond`](#workeractivitiespersecond) 仅根据 Worker 控制活动形成对比。

请注意，该数字以浮点形式表示，因此您可以根据需要将其设置为小于 1。
例如，将数字设置为 0.1 表示您希望 Activity 每 10 秒执行一次。
这可用于保护下游服务免受洪水影响。

```go
// ...
workerOptions := worker.Options{
    TaskQueueActivitiesPerSecond: 100000,
  // ...
}
w := worker.New(c, "your_task_queue_name", workerOptions)
// ...
```

#### 启用LoggingInReplay {#enablelogginginreplay}

设置为在 Workflow Execution 重放中启用日志记录。

- 型号：`bool`
- 默认：`false`

在Workflow Definitions中，您可以使用[`Workflow.GetLogger(ctx)`](https://pkg.go.dev/go.temporal.io/sdk/workflow#GetLogger)写入日志。
By default, the logger will skip logging during replays, so you do not see duplicate logs.

这仅对于调试目的真正有用。

```go
// ...

    EnableLoggingInReplay: false,
  // ...
}
w := worker.New(c, "your_task_queue_name", workerOptions)
// ...
```

<a id="disablestickyexecution"></a>
#### 禁用StickyExecution {#disablestickyexecution}

:::caution Deprecated

当DisableStickyExecution 为`true` 时，可能会损害性能。
它将很快被删除。
请参阅 [`SetStickyWorkflowCacheSize`](https://pkg.go.dev/go.temporal.io/sdk/worker#SetStickyWorkflowCacheSize)。

:::

设置为禁用粘性执行

- 型号：`bool`
- 默认：`false`

粘性执行在同一主机上运行 Workflow Execution 的 Workflow 任务（可以是不同的 Worker，只要它位于同一主机上）。
这是针对 Workflow Execution 的优化。
当启用粘性执行时，Worker 将 Workflow 状态保留在内存中。
新的 Workflow 任务包含新的历史事件，将被调度到同一个 Worker。
如果此 Worker 崩溃，粘性 Workflow 任务将在 `StickyScheduleToStartTimeout` 之后超时，并且 Temporal 服务将清除该 Workflow Execution 的粘性，并自动重新为 Schedule 重新创建一个新的 Workflow 任务，该任务可供任何 Worker 拾取并恢复进度。

```go
// ...
workerOptions := worker.Options{
    StickyScheduleToStartTimeout: time.Second(5),
  // ...
}
w := worker.New(c, "your_task_queue_name", workerOptions)
// ...
```

#### StickyScheduleToStartTimeout

设置 Workflow 任务的粘性执行 Schedule 启动超时。

- 型号：[`time.Duration`](https://pkg.go.dev/time#Duration)
- 默认值为 `5`

分辨率以秒为单位。

```go
// ...
workerOptions := worker.Options{
    StickyScheduleToStartTimeout: time.Second(5),
  // ...
}
w := worker.New(c, "your_task_queue_name", workerOptions)
// ...
```

<a id="backgroundactivitycontext"></a>
#### 背景ActivityContext {#backgroundactivitycontext}

:::caution Not recommended

不再推荐这种在 Activity 任务执行之间传递依赖关系的方法。

相反，我们建议使用带有包含依赖项的字段的结构体，并将 Activity Definition 开发为结构体方法，然后在结构初始化时传递所有依赖项。

- [如何使用 Go SDK 开发 Activity Definition](/develop/go/activities/basics#activity-definition)

:::

- 型号：[`context.Context`](https://pkg.go.dev/context#Context)

为使用 Worker 注册的所有 Activity 类型设置背景 `context.Context`。

该上下文可用于传递外部依赖项，例如到 Activity 任务执行的数据库连接。

```go
// ...
ctx := context.WithValue(context.Background(), "your-key", "your-value")
workerOptions := worker.Options{
    BackgroundActivityContext: ctx,
  // ...
}
w := worker.New(c, "your_task_queue_name", workerOptions)
// ...
```

#### WorkflowPanicPolicy

设置 Workflow Worker 如何处理非确定性 Workflow Execution 历史事件和来自 Workflow Definition 代码的其他恐慌。

```go
    DisableStickyExecution: internal.BlockWorkflow,
  // ...

// ...
```
