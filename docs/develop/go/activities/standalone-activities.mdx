[预发布](/evaluate/development-production-features/release-stages#pre-release)。

所有 API 都是实验性的，可能会发生向后不兼容的更改。

独立活动是独立运行的 Activity 执行，无需由 Workflow 编排。相反
使用 `Workflow.ExecuteActivity()` 从 Workflow Definition 中启动 Activity 时，您可以启动一个独立的
Activity 直接来自使用 `client.ExecuteActivity()` 的 Temporal CLIent。

Activity Definition和Worker注册与常规活动相同，只有执行路径
不同。

## 开始独立活动 {#get-started}


先决条件：

- 安装 [Temporal CLI](https://github.com/temporalio/cli/releases/tag/v1.6.2-standalone-activity)（独立 Activity 预发行版本）

- 安装 [Temporal Go SDK](https://docs.temporal.io/develop/go/core-application#install-a-temporal-sdk)（v1.41.0 或更高版本）


运行独立 Activity 的第一步涉及启动 Temporal Server：

```
temporal server start-dev
```

此命令自动使用 Web UI 启动 Temporal 开发服务器，并创建 `default` Namespace。
它使用内存数据库，因此不要将其用于实际用例。

:::info Temporal Cloud

此页面上的所有代码示例都使用
[`envconfig.MustLoadDefaultClientOptions()`](https://pkg.go.dev/go.temporal.io/sdk/contrib/envconfig)
配置 Temporal CLIent 连接。它响应 [环境
变量](/references/client-environment-configuration) 和 [TOML配置
文件](/references/client-environment-configuration)，因此相同的代码适用于本地开发
服务器和 Temporal Cloud 没有变化。请参阅下面的 [使用 Temporal 运行独立活动
云](#run-standalone-activities-temporal-cloud)。

:::


Temporal Server 现在应该可用于 `localhost:7233` 上的客户端连接，并且
Temporal Web UI 现在应该可以通过 [http://localhost:8233](http://localhost:8233) 访问。独立式
可以通过页面左上角的导航栏项目进行活动：

<img src="/img/standalone-activities-ui-nav.png" alt="Standalone Activities Web UI nav bar item" height="336" />

克隆 [样品去](https://github.com/temporalio/samples-go) 存储库以继续操作：

git clone https://github.com/temporalio/samples-go.git
cd samples-go

<a id="define-activity"></a>
## 定义你的 Activity {#define-activity}

在共享文件中定义 Activity，以便 Worker 和启动器都可以引用它。

[独立-Activity/helloworld/Activity.go](https://github.com/temporalio/samples-go/blob/main/standalone-activity/helloworld/activity.go)

```go
package helloworld

import (
	"context"
	"go.temporal.io/sdk/activity"
)

func Activity(ctx context.Context, name string) (string, error) {
	logger := activity.GetLogger(ctx)
	logger.Info("Activity", "name", name)
	return "Hello " + name + "!", nil
}
```

<a id="run-worker"></a>
## 使用注册的 Activity 运行 Worker {#run-worker}

为独立活动运行 Worker 与为 Workflow 驱动的活动运行 Worker 相同 - 您创建一个
Worker，注册Activity，并调用`Run()`。 Worker不需要知道Activity是否会被调用
来自 Workflow 或作为独立的 Activity。

有关 Worker 设置和的更多详细信息，请参阅 [如何在 Go 中开发 Worker](/develop/go/workers/run-worker-process#develop-worker)
配置选项。

[独立-Activity/helloworld/Worker/main.go](https://github.com/temporalio/samples-go/blob/main/standalone-activity/helloworld/worker/main.go)

```go
package main

import (
	"github.com/temporalio/samples-go/standalone-activity/helloworld"
	"go.temporal.io/sdk/client"
	"go.temporal.io/sdk/contrib/envconfig"
	"go.temporal.io/sdk/worker"
	"log"
)

func main() {
	c, err := client.Dial(envconfig.MustLoadDefaultClientOptions())
	if err != nil {
		log.Fatalln("Unable to create client", err)
	}
	defer c.Close()

	w := worker.New(c, "standalone-activity-helloworld", worker.Options{})

	w.RegisterActivity(helloworld.Activity)

	err = w.Run(worker.InterruptCh())
	if err != nil {
		log.Fatalln("Unable to start worker", err)
	}
}
```

运行 Worker：

```
go run standalone-activity/helloworld/worker/main.go
```

<a id="execute-activity"></a>
## 执行独立的 Activity {#execute-activity}

使用 [`client.ExecuteActivity()`](https://pkg.go.dev/go.temporal.io/sdk/client#Client) 启动独立的 Activity
执行。这是从应用程序代码（例如启动程序）调用的，而不是从 Workflow Definition 内部调用的。

`ExecuteActivity` 返回您指定的 [`ActivityHandle`](https://pkg.go.dev/go.temporal.io/sdk/client#ActivityHandle)
可用于获取结果、描述、取消或终止 Activity。

以下入门程序演示了如何执行独立 Activity、获取其结果、列出活动以及
计数活动：

[独立-Activity/helloworld/starter/main.go](https://github.com/temporalio/samples-go/blob/main/standalone-activity/helloworld/starter/main.go)

```go
package main

import (
	"context"
	"github.com/temporalio/samples-go/standalone-activity/helloworld"
	"go.temporal.io/sdk/client"
	"go.temporal.io/sdk/contrib/envconfig"
	"log"
	"time"
)

func main() {
	c, err := client.Dial(envconfig.MustLoadDefaultClientOptions())
	if err != nil {
		log.Fatalln("Unable to create client", err)
	}
	defer c.Close()

	activityOptions := client.StartActivityOptions{
		ID:        "standalone_activity_helloworld_ActivityID",
		TaskQueue: "standalone-activity-helloworld",
		ScheduleToCloseTimeout: 10 * time.Second,
	}

	handle, err := c.ExecuteActivity(context.Background(), activityOptions, helloworld.Activity, "Temporal")
	if err != nil {
		log.Fatalln("Unable to execute activity", err)
	}

	log.Println("Started standalone activity", "ActivityID", handle.GetID(), "RunID", handle.GetRunID())

	var result string
	err = handle.Get(context.Background(), &result)
	if err != nil {
		log.Fatalln("Unable get standalone activity result", err)
	}
	log.Println("Activity result:", result)

	resp, err := c.ListActivities(context.Background(), client.ListActivitiesOptions{
		Query: "TaskQueue = 'standalone-activity-helloworld'",
	})
	if err != nil {
		log.Fatalln("Unable to list activities", err)
	}

	log.Println("ListActivity results")
	for info, err := range resp.Results {
		if err != nil {
			log.Fatalln("Error iterating activities", err)
		}
		log.Printf("\tActivityID: %s, Type: %s, Status: %v\n",
			info.ActivityID, info.ActivityType, info.Status)
	}

	resp1, err := c.CountActivities(context.Background(), client.CountActivitiesOptions{
		Query: "TaskQueue = 'standalone-activity-helloworld'",
	})
	if err != nil {
		log.Fatalln("Unable to count activities", err)
	}

	log.Println("Total activities:", resp1.Count)
}
```

您可以将 Activity 作为函数引用或字符串 Activity 类型名称传递：

```go
handle, err := c.ExecuteActivity(ctx, options, helloworld.Activity, "arg1")

// Using a string type name
handle, err := c.ExecuteActivity(ctx, options, "Activity", "arg1")
```

`client.StartActivityOptions` 需要 `ID`、`TaskQueue` 以及 `ScheduleToCloseTimeout` 或 `ScheduleToCloseTimeout` 中的至少一项
`StartToCloseTimeout`。参见 [`StartActivityOptions`](https://pkg.go.dev/go.temporal.io/sdk/client#StartActivityOptions)
在 API 参考中获取完整的选项集。

要运行启动器（在与 Worker 不同的终端中）：

```
go run standalone-activity/helloworld/starter/main.go
```

或者使用 Temporal CLI 执行独立 Activity：

```bash
temporal activity execute \
  --type Activity \
  --activity-id standalone_activity_helloworld_ActivityID \
  --task-queue standalone-activity-helloworld \
  --schedule-to-close-timeout 10s \
  --input '"Temporal"'
```

<a id="get-activity-result"></a>
## 获取独立 Activity 的结果 {#get-activity-result}

使用 `ActivityHandle.Get()` 进行阻止，直到 Activity 完成并检索其结果。这类似于调用
`Get()` 上的 `WorkflowRun`。

```go
var result string
err = handle.Get(context.Background(), &result)
if err != nil {
	log.Fatalln("Activity failed", err)
}
log.Println("Activity result:", result)
```

如果 Activity 成功完成，结果将反序列化到提供的指针中。如果 Activity 失败，
失败作为错误返回。

<a id="get-activity-handle"></a>
## 获取现有独立 Activity 的句柄 {#get-activity-handle}

使用 `client.GetActivityHandle()` 创建先前启动的独立 Activity 的句柄。这类似于
`client.GetWorkflow()` 适用于 Workflow Execution。

`ActivityID` 和 `RunID` 都是必需的。

```go
handle := c.GetActivityHandle(client.GetActivityHandleOptions{
	ActivityID: "my-standalone-activity-id",
	RunID:      "the-run-id",
})

// Use the handle to get the result, describe, cancel, or terminate
var result string
err := handle.Get(context.Background(), &result)
if err != nil {
	log.Fatalln("Unable to get activity result", err)
}
```

## 列出独立活动 {#list-activities}

使用 [`client.ListActivities()`](https://pkg.go.dev/go.temporal.io/sdk/client#Client) 列出独立 Activity
与 [列表过滤器](/list-filter) Query 匹配的执行。结果包含一个迭代器，它产生
[`ActivityExecutionInfo`](https://pkg.go.dev/go.temporal.io/sdk/client#ActivityExecutionInfo) 条目。

```go
resp, err := c.ListActivities(context.Background(), client.ListActivitiesOptions{
	Query: "TaskQueue = 'standalone-activity-helloworld'",
})
if err != nil {
	log.Fatalln("Unable to list activities", err)
}

for info, err := range resp.Results {
	if err != nil {
		log.Fatalln("Error iterating activities", err)
	}
	log.Printf("ActivityID: %s, Type: %s, Status: %v\n",
		info.ActivityID, info.ActivityType, info.Status)
}
```

或者使用 Temporal CLI：

```bash
temporal activity list
```

`Query` 字段接受与 Workflow Visibility 相同的 [列表过滤器](/list-filter) 语法。例如，
`"ActivityType = 'Activity' AND Status = 'Running'"`。


## 计算独立活动 {#count-activities}

使用[`client.CountActivities()`](https://pkg.go.dev/go.temporal.io/sdk/client#Client)来计数独立式Activity
与 [列表过滤器](/list-filter) Query 匹配的执行。

```go
resp, err := c.CountActivities(context.Background(), client.CountActivitiesOptions{
	Query: "TaskQueue = 'standalone-activity-helloworld'",
})
if err != nil {
	log.Fatalln("Unable to count activities", err)
}

log.Println("Total activities:", resp.Count)
```

或者使用 Temporal CLI：

```bash
temporal activity count
```

<a id="run-standalone-activities-temporal-cloud"></a>
## 使用 Temporal Cloud 运行独立活动 {#run-standalone-activities-temporal-cloud}

本页的代码示例使用`envconfig.MustLoadDefaultClientOptions()`，因此相同的代码
适用于 Temporal Cloud — 只需通过环境变量或 TOML 配置连接
简介。无需更改代码。

有关连接到 Temporal Cloud 的完整详细信息，包括 Namespace 创建、证书
生成和身份验证选项，请参阅
[连接到 Temporal Cloud](/develop/go/client/temporal-client#connect-to-temporal-cloud)。

<a id="connect-with-mtls"></a>
### 连接 mTLS {#connect-with-mtls}

```
export TEMPORAL_ADDRESS=<your-namespace>.<your-account-id>.tmprl.cloud:7233
export TEMPORAL_NAMESPACE=<your-namespace>.<your-account-id>
export TEMPORAL_TLS_CLIENT_CERT_PATH='path/to/your/client.pem'
export TEMPORAL_TLS_CLIENT_KEY_PATH='path/to/your/client.key'
```

<a id="connect-with-an-api-key"></a>
### 使用 API 密钥连接 {#connect-with-an-api-key}

```
export TEMPORAL_ADDRESS=<region>.<cloud_provider>.api.temporal.io:7233
export TEMPORAL_NAMESPACE=<your-namespace>.<your-account-id>
export TEMPORAL_API_KEY=<your-api-key>
```

然后运行 ​​Worker 和起始代码，如前面部分所示。
