<a id="activity-definition"></a>
## 如何在 Go 中开发 Activity Definition {#activity-definition}

在 Temporal Go SDK 编程模型中，Activity Definition 是可导出函数或 `struct` 方法。
下面是定义为 Struct 方法的基本 Activity Definition 和 Activity 的示例。
_Activity struct_ 可以有多个方法，每个方法充当单独的 Activity 类型。
编写为结构体方法的活动可以使用共享结构体变量，例如：

- 应用程序级数据库池
- 客户端连接到另一个服务
- 可重复使用的实用程序
- 您只想为每个进程初始化一次的任何其他昂贵资源

由于这是一种常见的需求，因此本指南的其余部分显示了编写为 `struct` 方法的活动。

:::note

虽然可以将结构方法注册为 Workflow，但强烈建议不要这样做。
在某些情况下，Workflow 等结构方法可能会导致非确定性错误。我们推荐
仅对活动使用结构方法。

:::

```go
package yourapp

import (
    "context"

    "go.temporal.io/sdk/activity"
)
// ...

// YourSimpleActivityDefinition is a basic Activity Definition.
func YourSimpleActivityDefinition(ctx context.Context) error {
    return nil
}

// YourActivityObject is the struct that maintains shared state across Activities.
// If the Worker crashes this Activity object loses its state.
type YourActivityObject struct {
    Message *string
    Number  *int
}

// YourActivityDefinition is your custom Activity Definition.
// An Activity Definition is an exportable function.
func (a *YourActivityObject) YourActivityDefinition(ctx context.Context, param YourActivityParam) (*YourActivityResultObject, error) {
// ...
}
```

###如何开发Activity参数{#Activity-参数}

[Activity Definition](/activity-definition) 可以支持的参数总数没有明确的限制。
但是，最终编码到 gRPC 消息有效负载中的数据的总大小是有限制的。

单个参数的最大大小限制为 2 MB。
gRPC 消息的总大小（包括所有参数）限制为最大 4 MB。

另外，请记住，所有有效负载数据都记录在 [Workflow Execution Event History](/workflow-execution/event#event-history) 中，大型事件历史记录可能会影响 Worker 性能。
这是因为整个 Event History 可以转移到带有 [Workflow 任务](/tasks#workflow-task) 的 Worker 进程。

{/* TODO link to gRPC limit section when available */}

某些 SDK 要求您传递上下文对象，而其他 SDK 则不需要。
当涉及到您的应用程序数据（即序列化并编码到有效负载中的数据）时，我们建议您使用单个对象作为包装传递给活动的应用程序数据的参数。
这样您就可以更改传递给 Activity 的数据，而不会破坏函数或方法签名。

Activity Definition 的第一个参数是 `context.Context`。
对于 Activity Definition，此参数是可选的，但建议使用该参数，特别是在 Activity 预计使用其他 Go SDK API 的情况下。

Activity Definition 可以根据需要支持任意数量的其他自定义参数。
但是，所有参数都必须是可序列化的（参数不能是通道、函数、可变参数或不安全指针），并且建议稍后传递可以是 Updated 的单个结构。

  <a href="https://github.com/temporalio/documentation/blob/main/sample-apps/go/yourapp/your_activity_definition_dacx.go">

```go
// YourActivityParam is the struct passed to your Activity.
// Use a struct so that your function signature remains compatible if fields change.
type YourActivityParam struct {
    ActivityParamX string
    ActivityParamY int
}
// ...
func (a *YourActivityObject) YourActivityDefinition(ctx context.Context, param YourActivityParam) (*YourActivityResultObject, error) {
// ...
}
```

<a id="activity-parameters"></a>
### 如何定义 Activity 返回值 {#activity-parameters}

从 Activity 返回的所有数据都必须是可序列化的。

Activity 返回值受 Temporal 中的有效负载大小限制。默认有效负载大小限制为 2MB，并且 Event History 事务 ([请参阅此处的云限制](https://docs.temporal.io/cloud/limits#per-message-grpc-limit)) 中的任何 gRPC 消息大小都有 4MB 的硬限制。请记住，所有返回值都记录在 [Workflow Execution Event History](/workflow-execution/event#event-history) 中。

基于 Go 的 Activity Definition 可以仅返回 `error` 或 `customValue, error` 组合（与 Workflow Definition 相同）。
您可能希望使用 `struct` 类型来保存所有自定义值，但请记住它们必须都是可序列化的。

<div className="copycode-notice-container">
  <a href="https://github.com/temporalio/documentation/blob/main/sample-apps/go/yourapp/your_activity_definition_dacx.go">
查看源代码
  </a>{' '}
在应用程序代码的其余部分的上下文中。
</div>

```go
// YourActivityResultObject is the struct returned from your Activity.
// Use a struct so that you can return multiple values of different types.
// Additionally, your function signature remains compatible if the fields change.
type YourActivityResultObject struct {
    ResultFieldX string
    ResultFieldY int
}
// ...
func (a *YourActivityObject) YourActivityDefinition(ctx context.Context, param YourActivityParam) (*YourActivityResultObject, error) {
// ...
    result := &YourActivityResultObject{
        ResultFieldX: "Success",
        ResultFieldY: 1,
    }
    // Return the results back to the Workflow Execution.
    // The results persist within the Event History of the Workflow Execution.
    return result, nil
}
```

<a id="activity-return-values"></a>
### 如何在 Go 中自定义 Activity 类型 {#activity-return-values}

要自定义 Activity 类型，请在使用 Worker 注册 Activity 时使用 `RegisterOptions` 设置 `Name` 参数。

<div className="copycode-notice-container">
  <a href="https://github.com/temporalio/documentation/blob/main/sample-apps/go/yourapp/worker/main_dacx.go">
查看源代码
  </a>{' '}
在应用程序代码的其余部分的上下文中。
</div>

```go
func main() {
// ...
    yourWorker := worker.New(temporalClient, "your-custom-task-queue-name", worker.Options{})
// ...
    // Use RegisterOptions to change the name of the Activity Type for example.
    registerAOptions := activity.RegisterOptions{
        Name: "JustAnotherActivity",
    }
    yourWorker.RegisterActivityWithOptions(yourapp.YourSimpleActivityDefinition, registerAOptions)
    // Run the Worker
    err = yourWorker.Run(worker.InterruptCh())
// ...
}
// ...
```
