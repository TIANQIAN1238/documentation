Side Effect 用于执行非确定性代码，例如生成 UUID 或随机数，而不影响 Workflow 中的确定性。
这是通过将 Side Effect 的非确定性结果存储到 Workflow [Event History](/workflow-execution/event#event-history) 中来完成的。

Side Effect 在重播期间不会重新执行。相反，它返回 Workflow Execution Event History 的记录结果。

Side Effect 不应失败。从 Side Effect 引发的异常会导致当前 Workflow 任务失败并重试。

也可以使用 Activity 或本地 Activity 来代替 Side Effect，因为其结果也保留在 Workflow Execution 历史记录中。

:::note

您不应修改 Side Effect 函数内的 Workflow 状态，因为它在重播期间不会重新执行。应使用 Side Effect 函数返回值。

:::

使用 `go.Temporal.io/sdk/Workflow` 包中的 [`SideEffect`](https://pkg.go.dev/go.temporal.io/sdk/workflow#SideEffect) 函数直接在 Workflow 中执行 [Side Effect](/workflow-execution/event#side-effect)。

向其传递 `context.Context` 的实例和要执行的函数。

`SideEffect` API 返回一个 Future，即 [`converter.EncodedValue`](https://pkg.go.dev/go.temporal.io/sdk/workflow#SideEffect) 的实例。

在 Future 上使用 `Get` 方法来检索 Side Effect 的结果。

**正确实施**

下面的例子演示了`SideEffect`的正确使用方法：

```go
encodedRandom := workflow.SideEffect(ctx, func(ctx workflow.Context) interface{} {
 return rand.Intn(100)
})

var random int
encodedRandom.Get(&random)
// ...
}
```

**不正确的实施**

以下示例演示了如何不使用 `SideEffect`：

```go
// Warning: This is an incorrect example.
// This code is non-deterministic.
var random int
workflow.SideEffect(func(ctx workflow.Context) interface{} {
      random = rand.Intn(100)
      return nil
})
// random will always be 0 in replay, so this code is non-deterministic.
```

重播时，所提供的函数不会被执行，随机数将始终为 0，并且 Workflow Execution 可能会采取不同的路径，从而破坏确定性。

## 可变 Side Effects {#mutable-side-effects}

可变的 Side Effect 执行提供的函数一次，然后查找具有给定 Workflow ID 的值的历史记录。

- 如果不存在现有值，则它将函数结果记录为历史记录中具有给定 Workflow Id 的值。
- 如果存在现有值，则通过调用 equals 函数来比较历史记录中的现有值是否与新函数结果发生更改。
  - 如果值相等，则返回该值而不记录新的标记事件
  - 如果值不相等，则会在历史记录中记录具有相同 ID 的新值。

:::note

在 Workflow Execution 期间，每个新的 Side Effect 调用都会在 Workflow 历史记录中记录一个新的标记；而可变 Side Effects 仅在 Side Effect ID 的值更改或第一次设置时才在 Workflow 历史记录中记录新标记。

在重放期间，可变 Side Effect 将不会再次执行该函数。相反，它返回与 Workflow Execution 期间返回的值完全相同的值。

:::

要在 Go 中使用 [`MutableSideEffect()`](https://pkg.go.dev/go.temporal.io/sdk/workflow#MutableSideEffect)，请在 Workflow 范围内提供唯一名称。

```go
if err := workflow.MutableSideEffect(ctx, "configureNumber", get, eq).Get(&number); err != nil {
    panic("can't decode number:" + err.Error())
  }
```
