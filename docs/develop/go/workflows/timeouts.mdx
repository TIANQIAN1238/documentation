<a id="workflow-timeouts"></a>
## Workflow 超时 {#workflow-timeouts}

每个 Workflow 超时控制 Workflow Execution 不同方面的最大持续时间。

当 [启动 Workflow Execution](#workflow-timeouts) 时，设置 Workflow 超时。

在继续之前，我们要注意，我们通常不建议设置 Workflow 超时，因为 Workflow 被设计为长期运行且具有弹性。
相反，设置超时可能会限制其处理意外延迟或长时间运行的进程的能力。
如果您需要在特定时间段后在 Workflow 内执行操作，我们建议使用 Timer。

- **[Workflow Execution 超时](/encyclopedia/detecting-workflow-failures#workflow-execution-timeout)** - 限制单个 Workflow Execution 可以执行的最长时间。
- **[Workflow 运行超时](/encyclopedia/detecting-workflow-failures#workflow-run-timeout)：** 限制单个 Workflow 运行可以持续的最长时间。
- **[Workflow 任务超时](/encyclopedia/detecting-workflow-failures#workflow-task-timeout)：** 限制 Worker 可以执行 Workflow 任务的最长时间。

从 `go.Temporal.io/sdk/client` 包创建 [`StartWorkflowOptions`](https://pkg.go.dev/go.temporal.io/sdk/client#StartWorkflowOptions) 的实例，设置超时，并将该实例传递给 `ExecuteWorkflow` 调用。

可用的超时有：

- `WorkflowExecutionTimeout`
- `WorkflowRunTimeout`
- `WorkflowTaskTimeout`

```go
workflowOptions := client.StartWorkflowOptions{
  // ...
  // Set Workflow Timeout duration
  WorkflowExecutionTimeout: 24 * 365 * 10 * time.Hour,
  // WorkflowRunTimeout: 24 * 365 * 10 * time.Hour,
  // WorkflowTaskTimeout: 10 * time.Second,
  // ...
}
workflowRun, err := c.ExecuteWorkflow(context.Background(), workflowOptions, YourWorkflowDefinition)
if err != nil {
  // ...
}
```

<a id="workflow-retries"></a>
## Workflow Retry Policy {#workflow-retries}

Retry Policy 可以与超时配合，提供精细控制以优化执行体验。

如果发生故障，请使用 [Retry Policy](/encyclopedia/retry-policies) 重试 Workflow Execution。

默认情况下，Workflow Execution 不会重试，并且仅在某些情况下才应将重试策略与 Workflow Execution 一起使用。

从 `go.Temporal.io/sdk/temporal` 包创建 [`RetryPolicy`](https://pkg.go.dev/go.temporal.io/sdk/temporal#RetryPolicy) 的实例，并将其作为值提供给 `StartWorkflowOptions` 实例的 `RetryPolicy` 字段。

- 型号：[`RetryPolicy`](https://pkg.go.dev/go.temporal.io/sdk/temporal#RetryPolicy)
- 默认：无

```go
retrypolicy := &temporal.RetryPolicy{
  InitialInterval:    time.Second,
  BackoffCoefficient: 2.0,
  MaximumInterval:    time.Second * 100,
}
workflowOptions := client.StartWorkflowOptions{
  RetryPolicy: retrypolicy,
  // ...
}
workflowRun, err := temporalClient.ExecuteWorkflow(context.Background(), workflowOptions, YourWorkflowDefinition)
if err != nil {
  // ...
}
```
