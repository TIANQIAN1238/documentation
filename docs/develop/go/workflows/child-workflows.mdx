<a id="child-workflows"></a>
## 启动一个子 Workflow Execution {#child-workflows}

[儿童 Workflow Execution](/child-workflows) 是一个 Workflow Execution，它是使用 Child Workflow API 来自另一个 Workflow 的 Scheduled。

使用 Child Workflow API 时，Child Workflow 相关事件（[StartChildWorkflowExecutionInitiated](/references/events#startchildworkflowexecutioninitiated)、[子WorkflowExecutionStarted](/references/events#childworkflowexecutionstarted)、[ChildWorkflowExecutionCompleted](/references/events#childworkflowexecutioncompleted) 等）将记录在 Workflow Execution Event History 中。

始终阻止进度，直到 [子WorkflowExecutionStarted](/references/events#childworkflowexecutionstarted) 事件记录到 Event History，以确保子 Workflow Execution 已启动。
此后，可以使用 Child Workflow 选项中设置的 _Abandon_ [家长关闭政策](/parent-close-policy) 放弃子 Workflow Execution。

要确保 Child Workflow Execution 已启动，请首先在 Child Workflow future 实例上调用 Child Workflow Execution 方法，该方法返回不同的 future。

然后获取一个对象的值，该对象充当最初未知的结果的代理，该结果将等待子 Workflow Execution 生成。

要在 Go 中生成 [儿童 Workflow Execution](/child-workflows)，请使用 [`ExecuteChildWorkflow`](https://pkg.go.dev/go.temporal.io/sdk/workflow#ExecuteChildWorkflow) API，该 API 可从 `go.Temporal.io/sdk/Workflow` 包中获取。

`ExecuteChildWorkflow` 调用需要一个 [`Workflow.Context`](https://pkg.go.dev/go.temporal.io/sdk/workflow#Context) 实例、一个应用于其的 [`Workflow.ChildWorkflowOptions`](https://pkg.go.dev/go.temporal.io/sdk/workflow#ChildWorkflowOptions) 实例、Workflow 类型以及应传递给子 Workflow Execution 的任何参数。

`Workflow.ChildWorkflowOptions` 包含与 `client.StartWorkflowOptions` 相同的字段。
如果未显式设置，Workflow 选项字段会自动从父 Workflow 选项继承其值。
如果未设置自定义 `WorkflowID`，则会在生成子 Workflow Execution 时生成一个。
使用 [`WithChildOptions`](https://pkg.go.dev/go.temporal.io/sdk/workflow#WithChildOptions) API 将 Child Workflow 选项应用到 `Workflow.Context` 的实例。

`ExecuteChildWorkflow` 调用返回 [`ChildWorkflowFuture`](https://pkg.go.dev/go.temporal.io/sdk/workflow#ChildWorkflowFuture) 的实例。

在 `ChildWorkflowFuture` 实例上调用 `.Get()` 方法以等待结果。

```go
func YourWorkflowDefinition(ctx workflow.Context, params ParentParams) (ParentResp, error) {

  childWorkflowOptions := workflow.ChildWorkflowOptions{}
  ctx = workflow.WithChildOptions(ctx, childWorkflowOptions)

  var result ChildResp
  err := workflow.ExecuteChildWorkflow(ctx, YourOtherWorkflowDefinition, ChildParams{}).Get(ctx, &result)
  if err != nil {
    // ...
  }
  // ...
  return resp, nil
}

func YourOtherWorkflowDefinition(ctx workflow.Context, params ChildParams) (ChildResp, error) {
  // ...
  return resp, nil
}
```

<a id="async-child-workflows"></a>
### 异步 Child Workflows {#async-child-workflows}

要异步生成子 Workflow Execution，Child Workflow 必须在 Child Workflow 选项中设置“放弃”父关闭策略。
此外，父 Workflow Execution 必须等待 `ChildWorkflowExecutionStarted` 事件出现在其 Event History 中才能完成。

如果父级进行 `ExecuteChildWorkflow` 调用然后立即完成，则子级 Workflow Execution 不会生成。

要确保子 Workflow Execution 已启动，请首先在 `ChildWorkflowFuture` 实例上调用 `GetChildWorkflowExecution` 方法，该方法将返回不同的 Future。
然后在该 Future 上调用 `Get()` 方法，该方法将等待子 Workflow Execution 生成。

```go
import (
  // ...
  "go.temporal.io/api/enums/v1"
)

func YourWorkflowDefinition(ctx workflow.Context, params ParentParams) (ParentResp, error) {

  childWorkflowOptions := workflow.ChildWorkflowOptions{
    ParentClosePolicy: enums.PARENT_CLOSE_POLICY_ABANDON,
  }
  ctx = workflow.WithChildOptions(ctx, childWorkflowOptions)

  childWorkflowFuture := workflow.ExecuteChildWorkflow(ctx, YourOtherWorkflowDefinition, ChildParams{})
  // Wait for the Child Workflow Execution to spawn
  var childWE workflow.Execution
  if err := childWorkflowFuture.GetChildWorkflowExecution().Get(ctx, &childWE); err != nil {
     return err
  }
  // ...
  return resp, nil
}

func YourOtherWorkflowDefinition(ctx workflow.Context, params ChildParams) (ChildResp, error) {
  // ...
  return resp, nil
}
```

#### 设置父关闭策略 {#parent-close-policy}

[家长关闭政策](/parent-close-policy) 确定如果子 Workflow Execution 的父级更改为关闭状态（已完成、失败或超时），子级 Workflow Execution 会发生什么情况。

默认“父关闭策略”选项设置为终止子 Workflow Execution。

在 Go 中，父关闭策略是在 [`Workflow.ChildWorkflowOptions`](https://pkg.go.dev/go.temporal.io/sdk/workflow#ChildWorkflowOptions) 实例的 `ParentClosePolicy` 字段上设置的。
可能的值可以从 [`go.Temporal.io/api/enums/v1`](https://pkg.go.dev/go.temporal.io/api/enums/v1#ParentClosePolicy) 包中获取。

- `PARENT_CLOSE_POLICY_ABANDON`
- `PARENT_CLOSE_POLICY_TERMINATE`
- `PARENT_CLOSE_POLICY_REQUEST_CANCEL`

然后，通过使用 `WithChildOptions` API 将 Child Workflow 选项应用于 `Workflow.Context` 的实例，然后将其传递给 `ExecuteChildWorkflow()` 调用。

- 型号：[`ParentClosePolicy`](https://pkg.go.dev/go.temporal.io/api/enums/v1#ParentClosePolicy)
- 默认：`PARENT_CLOSE_POLICY_TERMINATE`

```go
import (
  // ...
  "go.temporal.io/api/enums/v1"
)

func YourWorkflowDefinition(ctx workflow.Context, params ParentParams) (ParentResp, error) {
  // ...
  childWorkflowOptions := workflow.ChildWorkflowOptions{
    // ...
    ParentClosePolicy: enums.PARENT_CLOSE_POLICY_ABANDON,
  }
  ctx = workflow.WithChildOptions(ctx, childWorkflowOptions)
  childWorkflowFuture := workflow.ExecuteChildWorkflow(ctx, YourOtherWorkflowDefinition, ChildParams{})
  // ...
}

func YourOtherWorkflowDefinition(ctx workflow.Context, params ChildParams) (ChildResp, error) {
  // ...
  return resp, nil
}
```
