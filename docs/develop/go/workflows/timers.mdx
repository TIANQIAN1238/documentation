Workflow 可以将耐用型 Timer 设置为固定时间段。
在某些 SDK 中，该函数称为 `sleep()`，而在其他 SDK 中，该函数称为 `Timer()`。

Workflow 可以休眠数天、数月甚至数年。
Timer 是持久的，因此即使您的 Worker 或 Temporal 服务在时间段结束时关闭，只要您的 Worker 和 Temporal 服务备份，`sleep()` 调用就会解析，并且您的代码将继续执行。

睡眠是一种轻资源操作：它不会占用进程，并且您可以在单个 Worker 上运行数百万个 Timer。

要在 Go 中设置 Timer，请使用 [`NewTimer()`](https://pkg.go.dev/go.temporal.io/sdk/workflow#NewTimer) 函数并传递您想要在继续​​之前等待的持续时间。

```go
timer := workflow.NewTimer(timerCtx, duration)
```

要在 Go 中设置睡眠持续时间，请使用 [`sleep()`](https://pkg.go.dev/go.temporal.io/sdk/workflow#Sleep) 函数并传递您想要在继续​​之前等待的持续时间。
零或负睡眠持续时间会导致函数立即返回。

```go
sleep = workflow.Sleep(ctx, 10*time.Second)
```

有关详细信息，请参阅 [Go 示例存储库](https://github.com/temporalio/samples-go/tree/main/timer) 中的 [Timer](https://github.com/temporalio/samples-go) 示例。
