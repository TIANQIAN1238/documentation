<a id="schedule-a-workflow"></a>
## Scheduled Workflows {#schedule-a-workflow}

调度 Workflow 是任何自动化流程的一个重要方面，尤其是在处理时间敏感的任务时。通过调度 Workflow，您可以自动执行重复性任务，减少手动干预的需要，并确保及时执行业务流程。

使用以下任一操作来帮助 Schedule 和 Workflow Execution 并控制您的自动化流程。

<a id="create-schedule"></a>
### 创建一个 Schedule {#create-schedule}

Schedule 通过 `create` 调用启动。
用户为每个新的 Schedule 生成唯一的 Schedule ID。

要在 Go 中创建 Schedule，请在 [客户](/encyclopedia/temporal-sdks#temporal-client) 上使用 `Create()`。
Schedule 必须使用 `client.ScheduleOptions{}` 中的 Schedule ID、[规格](/schedule) 和 [行动](/schedule) 进行初始化。

<div className="copycode-notice-container">
  <a href="https://github.com/temporalio/documentation/blob/main/sample-apps/go/features/schedules/create/main.go">
查看源代码
  </a>{' '}
在应用程序代码的其余部分的上下文中。
</div>

```go
func main() {
// ...
	scheduleID := "schedule_id"
	workflowID := "schedule_workflow_id"
	// Create the schedule.
	scheduleHandle, err := temporalClient.ScheduleClient().Create(ctx, client.ScheduleOptions{
		ID:   scheduleID,
		Spec: client.ScheduleSpec{},
		Action: &client.ScheduleWorkflowAction{
			ID:        workflowID,
			Workflow:  schedule.ScheduleWorkflow,
			TaskQueue: "schedule",
		},
	})
// ...
}
// ...
```

:::tip Schedule Auto-Deletion

一旦 Schedule 完成创建其所有 Workflow Execution，Temporal 服务就会将其删除，因为它不会再次触发。
Temporal 服务不保证何时会发生此删除。

:::

<a id="backfill-schedule"></a>
### 回填 Schedule {#backfill-schedule}

回填 Schedule 会在 Schedule 的指定时间范围之前执行 [Workflow 任务](/tasks#workflow-task)。
这对于执行错过或延迟的操作或提前测试 Workflow 非常有用。

要在 Go 中回填 Schedule，请在 `ScheduleHandle` 上使用 `Backfill()`。
指定执行 Workflow 的开始和结束时间以及重叠策略。

<div className="copycode-notice-container">
  <a href="https://github.com/temporalio/documentation/blob/main/sample-apps/go/features/schedules/backfill/main.go">
查看源代码
  </a>{' '}
在应用程序代码的其余部分的上下文中。
</div>

```go
func main() {
// ...
	err = scheduleHandle.Backfill(ctx, client.ScheduleBackfillOptions{
		Backfill: []client.ScheduleBackfill{
			{
				Start:   now.Add(-4 * time.Minute),
				End:     now.Add(-2 * time.Minute),
				Overlap: enums.SCHEDULE_OVERLAP_POLICY_ALLOW_ALL,
			},
			{
				Start:   now.Add(-2 * time.Minute),
				End:     now,
				Overlap: enums.SCHEDULE_OVERLAP_POLICY_ALLOW_ALL,
			},
		},
	})
	if err != nil {
		log.Fatalln("Unable to Backfill Schedule", err)
	}
// ...
}
// ...
```

<a id="delete-schedule"></a>
### 删除 Schedule {#delete-schedule}

删除 Schedule 会擦除 Schedule。
删除不会影响由 Schedule 启动的任何 Workflow。

要删除 Schedule，请在 `ScheduleHandle` 上使用 `Delete()`。

<div className="copycode-notice-container">
  <a href="https://github.com/temporalio/documentation/blob/main/sample-apps/go/features/schedules/delete/main.go">
查看源代码
  </a>{' '}
在应用程序代码的其余部分的上下文中。
</div>

```go
func main() {
// ...
	defer func() {
		log.Println("Deleting schedule", "ScheduleID", scheduleHandle.GetID())
		err = scheduleHandle.Delete(ctx)
		if err != nil {
			log.Fatalln("Unable to delete schedule", err)
		}
	}()
// ...
```

<a id="pause-schedule"></a>
### 暂停 Schedule {#pause-schedule}

`Pause` 和 `Unpause` 启用或停止在给定 Schedule 上运行的所有未来 Workflow。

暂停 Schedule 将停止所有未来的 Workflow 运行。
可以通过将 `State.Paused` 设置为 `true` 或使用 ScheduleHandle 上的 `Pause()` 来启用暂停。

取消暂停 Schedule 允许 Workflow 按计划执行。
要取消暂停 Schedule，请在 `ScheduleHandle` 上使用 `Unpause()`。

<div className="copycode-notice-container">
  <a href="https://github.com/temporalio/documentation/blob/main/sample-apps/go/features/schedules/pause/main.go">
查看源代码
  </a>{' '}
在应用程序代码的其余部分的上下文中。
</div>

```go
func main() {
// ...
	err = scheduleHandle.Pause(ctx, client.SchedulePauseOptions{
		Note: "The Schedule has been paused.",
	})
// ...
	err = scheduleHandle.Unpause(ctx, client.ScheduleUnpauseOptions{
		Note: "The Schedule has been unpaused.",
	})
```

<a id="trigger-schedule"></a>
### 触发 Schedule {#trigger-schedule}

触发 Schedule 会立即执行该 Schedule 中定义的操作。
默认情况下，`trigger` 受重叠策略约束。

要触发 Scheduled Workflow Execution，请在 `ScheduleHandle` 上使用 `trigger()`。

<div className="copycode-notice-container">
  <a href="https://github.com/temporalio/documentation/blob/main/sample-apps/go/features/schedules/trigger/main.go">
查看源代码
  </a>{' '}
在应用程序代码的其余部分的上下文中。
</div>

```go
func main() {
// ...
	for i := 0; i < 5; i++ {
		scheduleHandle.Trigger(ctx, client.ScheduleTriggerOptions{
			Overlap: enums.SCHEDULE_OVERLAP_POLICY_ALLOW_ALL,
		})
		time.Sleep(2 * time.Second)
	}
// ...
```

### Update 和 Schedule {#update-schedule}

更新 Schedule 会更改现有 Schedule 的配置。
这些更改可以对 Workflow 操作、操作参数、备忘录和 Workflow 的取消策略进行。

在 ScheduleHandle 上使用 `Update()` 修改 Schedule。

<div className="copycode-notice-container">
  <a href="https://github.com/temporalio/documentation/blob/main/sample-apps/go/features/schedules/update/main.go">
查看源代码
  </a>{' '}
在应用程序代码的其余部分的上下文中。

```go
func main() {
// ...
	updateSchedule := func(input client.ScheduleUpdateInput) (*client.ScheduleUpdate, error) {
		return &client.ScheduleUpdate{
			Schedule: &input.Description.Schedule,
		}, nil
	}

	_ = scheduleHandle.Update(ctx, client.ScheduleUpdateOptions{
		DoUpdate: updateSchedule,
	})
}
// ...
```

## 启动延迟 {#start-delay}

在特定的一次性未来点而不是在重复出现的 Schedule 上使用 `StartDelay` 至 Schedule 和 Workflow Execution。

从 `go.Temporal.io/sdk/client` 包创建 [`StartWorkflowOptions`](https://pkg.go.dev/go.temporal.io/sdk/client#StartWorkflowOptions) 的实例，设置 `StartDelay` 字段，并将该实例传递给 `ExecuteWorkflow` 调用。

```go
workflowOptions := client.StartWorkflowOptions{
  // ...
  // Start the workflow in 12 hours
  StartDelay: time.Hours * 12,
  // ...
}
workflowRun, err := c.ExecuteWorkflow(context.Background(), workflowOptions, YourWorkflowDefinition)
if err != nil {
  // ...
}
```

<a id="temporal-cron-jobs"></a>
## Temporal Cron 作业 {#temporal-cron-jobs}

:::caution Cron support is not recommended

我们建议使用 [Schedules](https://docs.temporal.io/schedule) 而不是 Cron 作业。
Schedule 旨在提供更好的开发人员体验，包括更多配置选项以及 Update 或暂停运行 Schedule 的能力。

:::

[Temporal 计划任务](/cron-job) 是在调用中提供 Cron Schedule 来生成 Workflow Execution 时发生的一系列 Workflow Execution。

当调用生成 Workflow Execution 时，会提供 Cron Schedule 作为选项。

从 `go.Temporal.io/sdk/client` 包创建 [`StartWorkflowOptions`](https://pkg.go.dev/go.temporal.io/sdk/client#StartWorkflowOptions) 的实例，设置 `CronSchedule` 字段，并将该实例传递给 `ExecuteWorkflow` 调用。

- 型号：`string`
- 默认：无

```go
workflowOptions := client.StartWorkflowOptions{
  CronSchedule: "15 8 * * *",
  // ...
}
workflowRun, err := c.ExecuteWorkflow(context.Background(), workflowOptions, YourWorkflowDefinition)
if err != nil {
  // ...
}
```

Temporal Workflow Schedule Cron 字符串遵循以下格式：

```
┌───────────── minute (0 - 59)
│ ┌───────────── hour (0 - 23)
│ │ ┌───────────── day of the month (1 - 31)
│ │ │ ┌───────────── month (1 - 12)
│ │ │ │ ┌───────────── day of the week (0 - 6) (Sunday to Saturday)
│ │ │ │ │
* * * * *
```
