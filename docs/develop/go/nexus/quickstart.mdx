<a id="nexus-go-quickstart"></a>
# Nexus Go 快速入门 {#nexus-go-quickstart}

:::tip SUPPORT, STABILITY, and DEPENDENCY INFO

Temporal Go SDK 对 Nexus 的支持是 [一般可用](/evaluate/development-production-features/release-stages#general-availability)。

:::

[Temporal Nexus](/evaluate/nexus) 使用 Nexus 端点、Nexus 服务合同和 Nexus 操作连接 Namespace 内部和之间的 Temporal 应用程序。构建一个包装现有 Temporal Workflow 的 Nexus 服务，然后从调用者 Workflow 调用它。

:::info NEW TO NEXUS?

此页面将帮助您获得在 Go 中运行的工作示例。 
要评估 Nexus 是否适合您的使用案例，请参阅 [评估指南](/evaluate/nexus)，要了解有关 Nexus 功能的更多信息，请单击 [这里](/nexus)。

:::

**先决条件：** 首先完成 [Go SDK 快速入门](/develop/go/set-up-your-local-go)。
您应该从该指南中获得 `Activity.go`、`Workflow.go`、`Worker/main.go` 和 `start/main.go`。

<a id="what-youll-build"></a>
## 你将构建什么 {#what-youll-build}

您有 `SayHelloWorkflow` 在 `default` Namespace 中运行。
在本指南结束时：

1. Nexus 服务将 `SayHelloWorkflow` 公开为操作。
2. 第二个 Namespace 将包含调用该操作的 Workflow。
3. 调用者 Workflow 将返回 `"Hello Temporal"` — 相同的结果，但跨越 Namespace。

<a id="1-define-the-nexus-service"></a>
## 1.定义Nexus服务 {#1-define-the-nexus-service}

创建一个名为 `service.go` 的文件，用于定义 Nexus 服务合同。

创建 Nexus 服务会在您的实现和任何调用者之间建立契约。
它在调用 Nexus 操作时提供类型安全，并确保操作处理程序履行合同。

`HelloServiceName` 和 `HelloOperationName` 是唯一标识服务和操作的字符串常量。
`HelloInput` 定义操作的输入类型。
`SayHelloWorkflow` 返回 `string`，因此操作输出类型也是 `string`。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="go" title="nexus_handler.go">
{`package greeting

import (
    "context"

    "github.com/nexus-rpc/sdk-go/nexus"
    "go.temporal.io/sdk/client"
    "go.temporal.io/sdk/temporalnexus"
    "go.temporal.io/sdk/workflow"
)

// HelloNexusWorkflow is the handler Workflow for the Nexus SayHello Operation.
// It bridges the Nexus HelloInput to SayHelloWorkflow's string parameter.
func HelloNexusWorkflow(ctx workflow.Context, input HelloInput) (string, error) {
    return SayHelloWorkflow(ctx, input.Name)
}

var HelloOperation = temporalnexus.NewWorkflowRunOperation(
    HelloOperationName,
    HelloNexusWorkflow,
    func(ctx context.Context, input HelloInput, options nexus.StartOperationOptions) (client.StartWorkflowOptions, error) {
        return client.StartWorkflowOptions{
            // RequestID is stable across retries, making it safe to use as a Workflow ID.
            ID: options.RequestID,
        }, nil
    },
)`}
</CodeSnippet>

</>
}>

<a id="2-define-the-nexus-operation-handlers"></a>
## 2. 定义 Nexus 操作处理程序 {#2-define-the-nexus-operation-handlers}

创建一个名为 `Nexus_handler.go` 的文件，该文件实现 Nexus 操作处理程序。

操作处理程序包含调用者调用 Nexus 操作时运行的逻辑。

`HelloNexusWorkflow` 充当处理程序 Workflow。
它通过提取 `input.Name` 将 Nexus `HelloInput` 桥接到 `SayHelloWorkflow` 的 `string` 参数。

`TemporalNexus.NewWorkflowRunOperation` 创建一个异步 Nexus 操作，该操作在调用时启动 `HelloNexusWorkflow`。
Options 函数返回 `client.StartWorkflowOptions`，包括从 `options.RequestID` 派生的稳定 Workflow ID。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="go" title="worker/main.go">
{`package main

import (
    "log"

    "my-org/greeting"

    "github.com/nexus-rpc/sdk-go/nexus"
    "go.temporal.io/sdk/client"
    "go.temporal.io/sdk/worker"
)

func main() {
    // Empty Options defaults to the "default" Namespace.
    c, err := client.Dial(client.Options{})
    if err != nil {
        log.Fatalln("Unable to create client", err)
    }
    defer c.Close()

    w := worker.New(c, "my-task-queue", worker.Options{})
    w.RegisterWorkflow(greeting.SayHelloWorkflow)
    w.RegisterWorkflow(greeting.HelloNexusWorkflow)
    w.RegisterActivity(greeting.Greet)

    service := nexus.NewService(greeting.HelloServiceName)
    err = service.Register(greeting.HelloOperation)
    if err != nil {
        log.Fatalln("Unable to register operations", err)
    }
    w.RegisterNexusService(service)

    err = w.Run(worker.InterruptCh())
    if err != nil {
        log.Fatalln("Unable to start worker", err)
    }
}`}
</CodeSnippet>

</>
}>

<a id="3-register-the-nexus-service-handler-in-a-worker"></a>
## 3. 在 Worker 中注册 Nexus 服务处理程序 {#3-register-the-nexus-service-handler-in-a-worker}

Update 您现有的 `Worker/main.go` 用于注册 Nexus 服务处理程序。

此 Worker 在 `default` Namespace 中运行 — 与已注册 `SayHelloWorkflow` 的 Namespace 相同。

如果注册了 Nexus 服务处理程序，Worker 将仅轮询并处理传入的 Nexus 请求。
这与用于 Workflow 和活动的 Worker 概念相同。

`Nexus.NewService` 创建一个名为 Nexus 的服务。
`service.Register` 将 `HelloOperation` 添加到服务中。
`w.RegisterNexusService` 向 Worker 注册服务，以便它可以接收 Nexus 操作请求。
`HelloNexusWorkflow` 还必须注册，以便 Worker 可以将其作为处理程序 Workflow 执行。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="go" title="caller_workflow.go">
{`package greeting

import (
    "time"

    "go.temporal.io/sdk/workflow"
)

const (
    CallerTaskQueue = "my-caller-task-queue"
    NexusEndpoint   = "my-nexus-endpoint-name"
)

func CallerWorkflow(ctx workflow.Context, name string) (string, error) {
    c := workflow.NewNexusClient(NexusEndpoint, HelloServiceName)

    fut := c.ExecuteOperation(ctx, HelloOperationName, HelloInput{Name: name}, workflow.NexusOperationOptions{
        ScheduleToCloseTimeout: 10 * time.Second,
    })

    var result string
    if err := fut.Get(ctx, &result); err != nil {
        return "", err
    }
    return result, nil
}`}
</CodeSnippet>

</>
}>

<a id="4-develop-the-caller-workflow"></a>
## 4.开发调用者Workflow {#4-develop-the-caller-workflow}

创建一个名为 `caller_Workflow.go` 的文件，该文件定义调用 Nexus 操作的 Workflow。

调用者Workflow演示了Nexus的消费者端。
调用者只依赖于服务合约，而不是直接导入处理程序代码。
这使调用者和处理程序保持解耦，以便他们可以生活在单独的 Namespace、存储库甚至团队中。

`Workflow.NewNexusClient` 创建一个绑定到您的 Nexus 服务和端点的客户端。
`ExecuteOperation` 开始操作并返回未来。
`fut.Get` 会阻塞，直到操作完成并将结果写入 `result`。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="bash">
temporal operator namespace create --namespace my-caller-namespace
</CodeSnippet>

<CodeSnippet language="bash">
{`temporal operator nexus endpoint create \\
  --name my-nexus-endpoint-name \\
  --target-namespace default \\
  --target-task-queue my-task-queue`}
</CodeSnippet>

</>
}>

<a id="5-create-the-caller-namespace-and-nexus-endpoint"></a>
## 5. 创建调用者 Namespace 和 Nexus 端点 {#5-create-the-caller-namespace-and-nexus-endpoint}

在运行应用程序之前，创建一个调用者 Namespace 和一个 Nexus 端点以将请求从调用者路由到处理程序。
该处理程序使用启动开发服务器时创建的 `default` Namespace。

Namespace 在调用方和处理程序方之间提供隔离。
Nexus 端点充当路由层，将调用方 Namespace 连接到处理程序的目标 Namespace 和 Task Queue。
端点名称必须与步骤 4 中 `caller_Workflow.go` 中定义的 `NexusEndpoint` 常量匹配。

确保您的本地 Temporal 开发服务器正在运行 (`Temporal Server start-dev`)。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="go" title="caller/main.go">
{`package main

import (
    "context"
    "log"

    "my-org/greeting"

    "go.temporal.io/sdk/client"
    "go.temporal.io/sdk/worker"
)

func main() {
    c, err := client.Dial(client.Options{
        Namespace: "my-caller-namespace",
    })
    if err != nil {
        log.Fatalln("Unable to create client", err)
    }
    defer c.Close()

    w := worker.New(c, greeting.CallerTaskQueue, worker.Options{})
    w.RegisterWorkflow(greeting.CallerWorkflow)

    if err := w.Start(); err != nil {
        log.Fatalln("Unable to start worker", err)
    }
    defer w.Stop()

    wr, err := c.ExecuteWorkflow(context.Background(), client.StartWorkflowOptions{
        ID:        "nexus-caller-workflow",
        TaskQueue: greeting.CallerTaskQueue,
    }, greeting.CallerWorkflow, "Temporal")
    if err != nil {
        log.Fatalln("Unable to execute workflow", err)
    }

    var result string
    if err := wr.Get(context.Background(), &result); err != nil {
        log.Fatalln("Unable to get workflow result", err)
    }
    log.Println("Workflow result:", result)
}`}
</CodeSnippet>

</>
}>

<a id="6-run-and-verify"></a>
## 6. 运行并验证 {#6-run-and-verify}

创建`caller/main.go`来启动调用者Worker并执行Workflow。

这将所有内容结合在一起：调用者 Worker 托管 `CallerWorkflow`，它使用 Nexus 客户端在处理程序端调用 `say-hello`。
完整请求从调用者 Workflow 通过 Nexus 端点流向运行 `HelloNexusWorkflow` 的处理程序 Worker（调用 `SayHelloWorkflow`），然后返回调用者。

**确保 `Nexus-rpc` 依赖项已同步：**

在终端中，从您的项目中：

```bash
go mod tidy
```

**运行应用程序：**

1. 在一个终端中启动处理程序 Worker：

```bash
go run worker/main.go
```

2. 在另一个终端中运行调用程序：

```bash
go run caller/main.go
```

你应该看到：

```
Workflow result: Hello Temporal
```

打开 [Temporal 网页用户界面](http://localhost:8233) 并在 Namespace 之间切换以查看两个 Workflow Execution。
在 `my-caller-Namespace` 中，找到 `CallerWorkflow` 执行 — 您应该在其历史记录中看到 `NexusOperationScheduled`、`NexusOperationStarted` 和 `NexusOperationCompleted` 事件。
在 `default` 中，找到由 Nexus 操作启动的 `HelloNexusWorkflow` 执行。
