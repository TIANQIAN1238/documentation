<a id="run-hello-world-test-your-installation"></a>
## 运行 Hello World：测试您的安装 {#run-hello-world-test-your-installation}

现在，让我们通过创建并运行一个完整的 Temporal 应用程序来验证您的设置是否正常工作，其中包含 Workflow 和
Activity。

该测试将确认：

- Temporal Go SDK 已正确安装
- 您本地的 Temporal 服务正在运行
- 您可以成功创建并执行 Workflow 和活动
- 组件之间的通信功能正常

<a id="1-create-the-activity"></a>
### 1.创建Activity {#1-create-the-activity}

Activity 是执行单个明确定义的操作（短期或长期运行）的普通函数或方法
这通常很容易失败。示例包括与外界交互的任何操作，例如发送
电子邮件、发出网络请求、写入数据库或调用 API。如果 Activity 失败，Temporal 自动
根据您的配置重试。

创建 Activity 文件 (Activity.go)：

```go
package greeting

import (
	"context"
	"fmt"
)

func Greet(ctx context.Context, name string) (string, error) {
	return fmt.Sprintf("Hello %s", name), nil
}
```

<a id="2-create-the-workflow"></a>
### 2.创建Workflow {#2-create-the-workflow}

Workflow 编排活动并包含应用程序逻辑。 Temporal Workflow 具有弹性。他们可以奔跑，而且
即使底层基础设施出现故障，也能持续运行数年。如果应用程序本身崩溃，Temporal 将
自动重新创建故障前的状态，以便可以从中断处继续。

创建 Workflow 文件（Workflow.go）：

```go
package greeting

import (
	"time"

	"go.temporal.io/sdk/workflow"
)

func SayHelloWorkflow(ctx workflow.Context, name string) (string, error) {
	ao := workflow.ActivityOptions{
		StartToCloseTimeout: time.Second * 10,
	}
	ctx = workflow.WithActivityOptions(ctx, ao)

	var result string
	err := workflow.ExecuteActivity(ctx, Greet, name).Get(ctx, &result)
	if err != nil {
		return "", err
	}

	return result, nil
}

```

<a id="3-create-and-run-the-worker"></a>
### 3. 创建并运行 Worker {#3-create-and-run-the-worker}

定义了 Activity 和 Workflow 后，您需要 Worker 来执行它们。 Worker 轮询 Task Queue，您
将其配置为轮询，寻找要做的工作。一旦 Worker 将 Workflow 或 Activity 任务从 Task Queue 中出列，
然后它执行该任务。

Worker 是 Temporal 应用程序的重要组成部分，因为它们实际执行您的 Temporal 应用程序中定义的任务。
Workflow 和活动。有关 Worker 的更多信息，请参阅
[了解 Temporal](/evaluate/understanding-temporal#workers) 和 [深入研究 Worker](/workers)。

创建 Worker 文件（Worker/main.go）：

```go
package main

import (
	"log"

	"my-org/greeting"

	"go.temporal.io/sdk/client"
	"go.temporal.io/sdk/worker"
)

func main() {
	c, err := client.Dial(client.Options{})
	if err != nil {
		log.Fatalln("Unable to create client", err)
	}
	defer c.Close()

	w := worker.New(c, "my-task-queue", worker.Options{})

	w.RegisterWorkflow(greeting.SayHelloWorkflow)
	w.RegisterActivity(greeting.Greet)

	err = w.Run(worker.InterruptCh())
	if err != nil {
		log.Fatalln("Unable to start worker", err)
	}
}
```

运行 Worker：

```bash
go run worker/main.go
```

<a id="4-execute-the-workflow"></a>
### 4.执行Workflow {#4-execute-the-workflow}

现在您的 Worker 正在运行，是时候启动 Workflow Execution 了。

创建一个名为 start/main.go 的单独文件：

```go
package main

import (
	"context"
	"log"
	"os"

	greeting "my-org/greeting"

	"go.temporal.io/sdk/client"
)

func main() {
	c, err := client.Dial(client.Options{})
	if err != nil {
		log.Fatalln("Unable to create client", err)
	}
	defer c.Close()

	options := client.StartWorkflowOptions{
		ID:        "greeting-workflow",
		TaskQueue: "my-task-queue",
	}

	we, err := c.ExecuteWorkflow(context.Background(), options, greeting.SayHelloWorkflow, os.Args[1])
	if err != nil {
		log.Fatalln("Unable to execute workflow", err)
	}
	log.Println("Started workflow", "WorkflowID", we.GetID(), "RunID", we.GetRunID())

	var result string
	err = we.Get(context.Background(), &result)
	if err != nil {
		log.Fatalln("Unable get workflow result", err)
	}
	log.Println("Workflow result:", result)
}
```

然后运行：

```bash
go run start/main.go Temporal
```
