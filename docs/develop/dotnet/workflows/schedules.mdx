<a id="schedule-a-workflow"></a>
## Schedule 一个 Workflow {#schedule-a-workflow}

调度 Workflow 是任何自动化流程的一个重要方面，尤其是在处理时间敏感的任务时。通过调度 Workflow，您可以自动执行重复性任务，减少手动干预的需要，并确保及时执行业务流程。

使用以下任一操作来帮助 Schedule 和 Workflow Execution 并控制您的自动化流程。

<a id="create-a-workflow"></a>
### 创建一个 Scheduled Workflow {#create-a-workflow}

创建操作使您能够创建新的 Schedule。创建新的 Schedule 时，会生成唯一的 Schedule ID，您可以使用该 ID 在其他 Schedule 命令中引用 Schedule。

要在 .NET 中创建 Scheduled Workflow Execution，请使用 [创建ScheduleAsync](https://dotnet.temporal.io/api/temporalio.Client.ITemporalClient.html#Temporalio_Client_ITemporalClient_CreateScheduleAsync_System_String_Temporalio_Client_Schedules_Schedule_Temporalio_Client_Schedules_ScheduleOptions_)
客户端上的方法。
然后将 Schedule ID 和 Schedule 对象传递给该方法以创建 Scheduled Workflow Execution。
将 Schedule 的 `Action` 属性设置为 `ScheduleActionStartWorkflow` 的实例，将 Schedule 设为 Workflow Execution。

```csharp
using Temporalio.Client;
using Temporalio.Client.Schedules;

var client = await TemporalClient.ConnectAsync(new("localhost:7233"));

var handle = await client.CreateScheduleAsync(
    "my-schedule-id",
    new(
        Action: ScheduleActionStartWorkflow.Create(
            (MyWorkflow wf) => wf.RunAsync(),
            new(id: "my-workflow-id", taskQueue: "my-task-queue")),
        Spec: new()
        {
            Intervals = new List<ScheduleIntervalSpec> { new(Every: TimeSpan.FromDays(5)) },
        }));
```

:::tip Schedule Auto-Deletion

一旦 Schedule 完成创建其所有 Workflow Execution，Temporal 服务就会将其删除，因为它不会再次触发。
Temporal 服务不保证何时会发生此删除。

:::

<a id="backfill-a-scheduled-workflow"></a>
### 回填 Scheduled Workflow {#backfill-a-scheduled-workflow}

回填操作会在指定时间范围之前执行操作。当您需要执行错过或延迟的操作时，或者当您想要在 Scheduled 时间之前测试 Workflow 时，此命令非常有用。

要在 .NET 中回填 Scheduled Workflow Execution，请使用 [回填异步()](https://dotnet.temporal.io/api/temporalio.Client.Schedules.ScheduleHandle.html#Temporalio_Client_Schedules_ScheduleHandle_BackfillAsync_System_Collections_Generic_IReadOnlyCollection_Temporalio_Client_Schedules_ScheduleBackfill__Temporalio_Client_RpcOptions_)
Schedule 手柄上的方法。

```csharp
using Temporalio.Client;
using Temporalio.Client.Schedules;

var client = await TemporalClient.ConnectAsync(new("localhost:7233"));

var handle = client.GetScheduleHandle("my-schedule-id");
var now = DateTime.Now;
await handle.BackfillAsync(new List<ScheduleBackfill>
{
    new(
        StartAt: now - TimeSpan.FromDays(30),
        EndAt: now - TimeSpan.FromDays(20),
        Overlap: ScheduleOverlapPolicy.AllowAll),
});
```

<a id="delete-a-scheduled-workflow"></a>
### 删除 Scheduled Workflow {#delete-a-scheduled-workflow}

删除操作使您能够删除 Schedule。删除 Schedule 时，不会影响由 Schedule 启动的任何 Workflow。

要在 .NET 中删除 Scheduled Workflow Execution，请在 Schedule 句柄上使用 [删除异步()](https://dotnet.temporal.io/api/temporalio.Client.Schedules.ScheduleHandle.html#Temporalio_Client_Schedules_ScheduleHandle_DeleteAsync_Temporalio_Client_RpcOptions_) 方法。

```csharp
using Temporalio.Client;
using Temporalio.Client.Schedules;

var client = await TemporalClient.ConnectAsync(new("localhost:7233"));

var handle = client.GetScheduleHandle("my-schedule-id");
await handle.DeleteAsync();
```

<a id="describe-a-scheduled-workflow"></a>
### 描述 Scheduled Workflow {#describe-a-scheduled-workflow}

描述操作显示当前的 Schedule 配置，包括有关过去、当前和未来 Workflow 运行的信息。当您想要获取 Schedule 及其关联的 Workflow 运行的详细视图时，此命令非常有用。

要在 .NET 中描述 Scheduled Workflow Execution，请在 Schedule 句柄上使用 [描述异步()](https://dotnet.temporal.io/api/temporalio.Client.Schedules.ScheduleHandle.html#Temporalio_Client_Schedules_ScheduleHandle_DescribeAsync_Temporalio_Client_RpcOptions_) 方法。

```csharp
using Temporalio.Client;
using Temporalio.Client.Schedules;

var client = await TemporalClient.ConnectAsync(new("localhost:7233"));

var handle = client.GetScheduleHandle("my-schedule-id");
var desc = await handle.DescribeAsync();
Console.WriteLine("Schedule info: {0}", desc.Info);
```

<a id="list-a-scheduled-workflow"></a>
### 列出 Scheduled Workflow {#list-a-scheduled-workflow}

列表操作列出了所有可用的 Schedule。当您想要查看所有 Schedule 及其各自的 Schedule ID 的列表时，此命令非常有用。

要列出所有 Schedule，请在客户端上使用 [ListSchedulesAsync()](https://dotnet.temporal.io/api/temporalio.Client.ITemporalClient.html#Temporalio_Client_ITemporalClient_ListSchedulesAsync_Temporalio_Client_Schedules_ScheduleListOptions_) 异步方法。
这将返回一个异步枚举。
如果添加或删除 Schedule，它可能不会立即在列表中可用。

```csharp
using Temporalio.Client;
using Temporalio.Client.Schedules;

var client = await TemporalClient.ConnectAsync(new("localhost:7233"));
await foreach (var desc in client.ListSchedulesAsync())
{
    Console.WriteLine("Schedule info: {0}", desc.Info);
}
```

<a id="pause-a-scheduled-workflow"></a>
### 暂停 Scheduled Workflow {#pause-a-scheduled-workflow}

暂停操作使您能够暂停和取消暂停 Schedule。当您暂停 Schedule 时，与 Schedule 关联的所有未来 Workflow 运行都会暂时停止。当您由于维护或任何其他原因想要暂时停止 Workflow 时，此命令非常有用。

要在 .NET 中暂停 Scheduled Workflow Execution，请在 Schedule 句柄上使用 [暂停异步()](https://dotnet.temporal.io/api/temporalio.Client.Schedules.ScheduleHandle.html#Temporalio_Client_Schedules_ScheduleHandle_PauseAsync_System_String_Temporalio_Client_RpcOptions_) 方法。
您可以向 `PauseAsync()` 方法传递注释以提供暂停 Schedule 的原因。

```csharp
using Temporalio.Client;
using Temporalio.Client.Schedules;

var client = await TemporalClient.ConnectAsync(new("localhost:7233"));

var handle = client.GetScheduleHandle("my-schedule-id");
await handle.PauseAsync("Pausing the schedule for now");
```

<a id="trigger-a-scheduled-workflow"></a>
### 触发 Scheduled Workflow {#trigger-a-scheduled-workflow}

触发操作会使用给定的 Schedule 触发立即操作。默认情况下，此操作受 Schedule 的重叠策略的约束。当您想要在 Scheduled 时间之外执行 Workflow 时，此命令很有用。

要在 .NET 中触发 Scheduled Workflow Execution，请在 Schedule 句柄上使用 [触发异步()](https://dotnet.temporal.io/api/temporalio.Client.Schedules.ScheduleHandle.html#Temporalio_Client_Schedules_ScheduleHandle_TriggerAsync_Temporalio_Client_Schedules_ScheduleTriggerOptions_) 方法。

```csharp
using Temporalio.Client;
using Temporalio.Client.Schedules;

var client = await TemporalClient.ConnectAsync(new("localhost:7233"));

var handle = client.GetScheduleHandle("my-schedule-id");
await handle.TriggerAsync();
```

<a id="update-a-scheduled-workflow"></a>
### Update a Scheduled Workflow {#update-a-scheduled-workflow}

Update 操作使您能够 Update 现有的 Schedule。当您需要修改 Schedule 的配置（例如更改开始时间、结束时间或间隔）时，此命令非常有用。

要在 .NET 中使用 Update 和 Scheduled Workflow Execution，请在 Schedule 句柄上使用 [UpdateAsync()](https://dotnet.temporal.io/api/temporalio.Client.Schedules.ScheduleHandle.html#Temporalio_Client_Schedules_ScheduleHandle_UpdateAsync_System_Func_Temporalio_Client_Schedules_ScheduleUpdateInput_Temporalio_Client_Schedules_ScheduleUpdate__Temporalio_Client_RpcOptions_) 方法。
此方法接受提供当前 Schedule 输入的回调。
可以创建新的 Schedule 并从该回调返回以执行 Update。

```csharp
using Temporalio.Client;
using Temporalio.Client.Schedules;

var client = await TemporalClient.ConnectAsync(new("localhost:7233"));

var handle = client.GetScheduleHandle("my-schedule-id");
await handle.UpdateAsync(input =>
{
    var newAction =  ScheduleActionStartWorkflow.Create(
        (MyWorkflow wf) => wf.RunAsync(),
        new(id: "my-workflow-id", taskQueue: "my-task-queue"));
    return new(input.Description.Schedule with { Action = newAction });
});
```

## 使用开始延迟 {#start-delay}

在特定的一次性未来点而不是重复出现的 Schedule 上使用 `StartDelay` 至 Schedule 和 Workflow Execution。

在客户端的 `StartWorkflowAsync()` 或 `ExecuteWorkflowAsync()` 方法中使用 `WorkflowOptions` 上的 `StartDelay` 选项。

```csharp
var handle = await client.StartWorkflowAsync(
    (MyWorkflow wf) => wf.RunAsync(),
    new(id: "my-workflow-id", taskQueue: "my-task-queue")
    {
        StartDelay = TimeSpan.FromHours(3),
    });
```
