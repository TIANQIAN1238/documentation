<a id="create-a-worker-file-programcs-in-the-worker-project"></a>
### 1. 创建 Activity 和 Workflow {#create-a-worker-file-programcs-in-the-worker-project}

<a id="3-execute-the-workflow"></a>
#### 在 Workflow 项目中创建 Activity 文件（MyActivities.cs）： {#3-execute-the-workflow}

```csharp
namespace MyNamespace;

using Temporalio.Activities;

public class MyActivities
{
    // Activities can be async and/or static too! We just demonstrate instance
    // methods since many will use them that way.
    [Activity]
    public string SayHello(string name) => $"Hello, {name}!";
}
```

Activity 是执行单个、明确定义的操作（短期或长期运行）的普通函数或方法，通常涉及与外界交互，例如发送电子邮件、发出网络请求、写入数据库或调用 API，这些都容易失败。
如果 Activity 失败，Temporal 会根据您的配置自动重试。

<a id="create-a-client-file-programcs-in-the-client-project"></a>
#### 在 Workflow 项目中创建一个 Workflow 文件（SayHelloWorkflow.cs）： {#create-a-client-file-programcs-in-the-client-project}

```csharp
namespace MyNamespace;

using Temporalio.Workflows;

[Workflow]
public class SayHelloWorkflow
{
    [WorkflowRun]
    public async Task<string> RunAsync(string name)
    {
        // This workflow just runs a simple activity to completion.
        // StartActivityAsync could be used to just start and there are many
        // other things that you can do inside a workflow.
        return await Workflow.ExecuteActivityAsync(
            // This is a lambda expression where the instance is typed. If this
            // were static, you wouldn't need a parameter.
            (MyActivities act) => act.SayHello(name),
            new() { StartToCloseTimeout = TimeSpan.FromMinutes(5) }
        );
    }
}
```

Workflow 编排活动并包含应用程序逻辑。
Temporal Workflow 具有弹性。
即使底层基础设施出现故障，它们也可以运行并持续运行数年。
如果应用程序本身崩溃，Temporal 将自动重新创建其失败前的状态，以便它可以从中断处继续。

<a id="verify-success"></a>
### 2.创建Worker {#verify-success}

定义了 Activity 和 Workflow 后，您需要 Worker 来执行它们。

#### 在 Worker 项目中创建 Worker 文件（Program.cs）：

```csharp
using MyNamespace;
using Temporalio.Client;
using Temporalio.Worker;

// Create a client to localhost on "default" namespace
var client = await TemporalClient.ConnectAsync(new("localhost:7233"));

// Cancellation token to shutdown worker on ctrl+c
using var tokenSource = new CancellationTokenSource();
Console.CancelKeyPress += (_, eventArgs) =>
{
    tokenSource.Cancel();
    eventArgs.Cancel = true;
};

// Create an activity instance since we have instance activities. If we had
// all static activities, we could just reference those directly.
var activities = new MyActivities();

// Create worker with the activity and workflow registered
using var worker = new TemporalWorker(
    client,
    new TemporalWorkerOptions("my-task-queue")
        .AddActivity(activities.SayHello)
        .AddWorkflow<SayHelloWorkflow>()
);

// Run worker until cancelled
Console.WriteLine("Running worker");
try
{
    await worker.ExecuteAsync(tokenSource.Token);
}
catch (OperationCanceledException)
{
    Console.WriteLine("Worker cancelled");
}
```

运行 Worker：

```bash
dotnet run --project Worker/Worker.csproj
```
保持此终端运行 - 您应该看到显示 `Running Worker`。

Worker 轮询 Task Queue，您将其配置为轮询，寻找要做的工作。 
一旦 Worker 将 Workflow 或 Activity 任务从 Task Queue 中出列，它就会执行该任务。

Worker 是 Temporal 应用程序的重要组成部分，因为它们实际执行 Workflow 和活动中定义的任务。
有关 Worker 的更多信息，请参阅 [了解 Temporal](/evaluate/understanding-temporal#workers) 和 [深入研究 Worker](/workers)。

### 3.执行Workflow

现在您的 Worker 正在运行，是时候启动 Workflow Execution 了。
最后一步将验证一切是否正常工作。

#### 在Client项目中创建一个Client文件（Program.cs）：

```csharp
using MyNamespace;
using Temporalio.Client;

// Create a client to localhost on "default" namespace
var client = await TemporalClient.ConnectAsync(new("localhost:7233"));

// Run workflow
var result = await client.ExecuteWorkflowAsync(
    (SayHelloWorkflow wf) => wf.RunAsync("Temporal"),
    new(id: $"my-workflow-id-{Guid.NewGuid()}", taskQueue: "my-task-queue")
);

Console.WriteLine("Workflow result: {0}", result);
```

当 Worker 仍在运行时，运行 Workflow：

```bash
dotnet run --project Client/Client.csproj
```

### 验证成功

如果一切正常，您应该看到：

- Worker 处理 Workflow 和 Activity
- 输出：`Workflow result: Hello Temporal`
- [Temporal 网页用户界面](http://localhost:8233) 中的 Workflow Execution 详细信息
