## 启动启用了 Nexus 的 Temporal 开发 Server {#run-the-temporal-nexus-development-server}

前置条件：

- [安装最新的 Temporal CLI](https://learn.temporal.io/getting_started/dotnet/dev_environment/#set-up-a-local-temporal-service-for-development-with-temporal-cli)（推荐 v1.3.0 或更高版本）
- [安装最新的 Temporal .NET SDK](https://learn.temporal.io/getting_started/dotnet/dev_environment/#install-the-temporal-net-sdk)（v1.9.0 或更高版本）

使用 Temporal Nexus 的第一步是启动启用了 Nexus 的 Temporal Server。

```
temporal server start-dev
```

此命令会自动启动带有 Web UI 的 Temporal 开发 Server，并创建 `default` Namespace。它使用内存数据库，因此不要在实际使用场景中使用。

现在应可以通过 [http://localhost:8233](http://localhost:8233) 访问 Temporal Web UI，Temporal Server 也可以在 `localhost:7233` 上接受客户端连接。

## 创建 caller 和 handler Namespace {#create-caller-handler-namespaces}

在设置 Nexus Endpoint 之前，为 caller 和 handler 分别创建独立的 Namespace。

```
temporal operator namespace create --namespace nexus-simple-handler-namespace
temporal operator namespace create --namespace nexus-simple-caller-namespace
```

`nexus-simple-handler-namespace` 将包含 Nexus Operation handler，我们将在 `nexus-simple-caller-namespace` 中使用 Workflow 来调用该 Operation handler。我们使用不同的 Namespace 来演示跨 Namespace 的 Nexus 调用。

## 创建 Nexus Endpoint 将请求从 caller 路由到 handler {#create-nexus-endpoint}

建立 caller 和 handler Namespace 后，下一步是创建 Nexus Endpoint 来路由请求。

```
temporal operator nexus endpoint create \
  --name nexus-simple-endpoint \
  --target-namespace nexus-simple-handler-namespace \
  --target-task-queue nexus-simple-handler-sample
```

也可以使用 Web UI 创建 Namespace 和 Nexus Endpoint。

## 定义 Nexus Service 契约 {#define-nexus-service-contract}

为 Nexus Service 定义清晰的契约对于顺畅通信至关重要。

在此示例中，有一个描述 Service 和 Operation 名称以及输入/输出类型的 service 包，供 caller Workflow 使用 Nexus Endpoint。

每个 [Temporal SDK 都包含并使用默认的 Data Converter](https://docs.temporal.io/dataconversion)。默认 Data Converter 按以下顺序编码 Payload：Null、字节数组、Protobuf JSON 和 JSON。在多语言环境中（即使用多种语言和 SDK 开发 Temporal 解决方案），Protobuf 和 JSON 是常见的选择。此示例使用序列化为 JSON 的 .NET 类。

[NexusSimple/IHelloService.cs](https://github.com/temporalio/samples-dotnet/blob/main/src/NexusSimple/IHelloService.cs)
```csharp
using NexusRpc;

[NexusService]
public interface IHelloService
{
    static readonly string EndpointName = "nexus-simple-endpoint";

    [NexusOperation]
    EchoOutput Echo(EchoInput input);

    [NexusOperation]
    HelloOutput SayHello(HelloInput input);

    public record EchoInput(string Message);

    public record EchoOutput(string Message);

    public record HelloInput(string Name, HelloLanguage Language);

    public record HelloOutput(string Message);

    public enum HelloLanguage
    {
        En,
        Fr,
        De,
        Es,
        Tr,
    }
}
```

## 开发 Nexus Service 和 Operation handler {#develop-nexus-service-operation-handlers}

Nexus Operation handler 通常与其所抽象的底层 Temporal 原语定义在同一 Worker 中。Operation handler 可以决定给定的 Nexus Operation 是同步还是异步的。它们可以使用 Temporal SDK Client 调用底层 Temporal 原语（如 Query、Signal 或 Update），或运行其他可靠代码。Handler 应当可靠，因为[熔断器](/nexus/operations#circuit-breaking)在连续发生 5 次可重试错误后会触发，阻止所有来自 caller 到该 Endpoint 的 Operation。

`Temporalio.Nexus` 命名空间提供了帮助创建 Nexus Operation 的工具：

- `NexusOperationExecutionContext.Current.TemporalClient` \- 获取 Worker 初始化时使用的 Temporal Client，用于由 Temporal 原语（如 Signal 和 Query）支持的同步 handler
- `WorkflowRunOperationHandler.FromHandleFactory` \- 将 Workflow 作为异步 Nexus Operation 运行

此示例从使用 `OperationHandler.Sync` 方法的同步 Operation handler 示例开始，然后展示如何创建使用 `WorkflowRunOperationHandler.FromHandleFactory` 从 Nexus Operation 启动 handler Workflow 的异步 Operation handler。

### 开发同步 Nexus Operation handler

`OperationHandler.Sync` 方法用于暴露简单的 RPC handler。使用 `NexusOperationExecutionContext.Current.TemporalClient` 获取 Temporal Client，用于 Signal、Query 和列出 Workflow。实现也可以进行其他调用，但 handler 应当可靠，以避免触发[熔断器](/nexus/operations#circuit-breaking)。

[NexusSimple/Handler/HelloService.cs](https://github.com/temporalio/samples-dotnet/blob/main/src/NexusSimple/Handler/HelloService.cs)
```csharp
using NexusRpc.Handlers;

[NexusServiceHandler(typeof(IHelloService))]
public class HelloService
{
    [NexusOperationHandler]
    public IOperationHandler<IHelloService.EchoInput, IHelloService.EchoOutput> Echo() =>
        // This Nexus service operation is a simple sync handler
        OperationHandler.Sync<IHelloService.EchoInput, IHelloService.EchoOutput>(
            (ctx, input) => new(input.Message));

    // ...
}
```

### 在同步 handler 中使用 Temporal Client 处理 Signal、Query 和 Update

在同步 handler 中使用 Temporal Client 来对 Workflow 进行 Signal、Query 或 Update 是一种常见模式。你也可以使用 Signal-With-Start 或 Update-With-Start 来确保 Workflow 已启动并向其发送 Signal 或 Update。所有调用必须在 [Nexus 请求超时](/cloud/limits#nexus-operation-request-timeout)内完成。Update 应是短暂的，以保持在此截止时间内。

### 开发用于启动 Workflow 的异步 Nexus Operation handler

使用 `WorkflowRunOperationHandler.FromHandleFactory` 方法，这是将 Workflow 作为 Operation 暴露的最简便方式。

[NexusSimple/Handler/HelloService.cs](https://github.com/temporalio/samples-dotnet/blob/main/src/NexusSimple/Handler/HelloService.cs)
```csharp

using NexusRpc.Handlers;
using Temporalio.Nexus;

[NexusServiceHandler(typeof(IHelloService))]
public class HelloService
{
    // ...

    [NexusOperationHandler]
    public IOperationHandler<IHelloService.HelloInput, IHelloService.HelloOutput> SayHello() =>
        // This Nexus service operation is backed by a workflow run
        WorkflowRunOperationHandler.FromHandleFactory(
            (WorkflowRunOperationContext context, IHelloService.HelloInput input) =>
                context.StartWorkflowAsync(
                    (HelloHandlerWorkflow wf) => wf.RunAsync(input),
                    // Workflow IDs should typically be business meaningful IDs and are used to
                    // dedupe workflow starts. For this example, we're using the request ID
                    // allocated by Temporal when the caller workflow schedules the operation,
                    // this ID is guaranteed to be stable across retries of this operation.
                    new() { Id = context.HandlerContext.RequestId }));
}
```

Workflow ID 通常应是具有业务意义的 ID，用于对 Workflow 启动进行去重。一般情况下，该 ID 应作为 Nexus Service 契约的一部分通过 Operation 输入传递。

:::tip RESOURCES

使用 Use-Existing 的冲突策略[将多个 Nexus caller 附加到 handler Workflow](/nexus/operations#attaching-multiple-nexus-callers)。

:::

#### 将 Nexus Operation 输入映射到多个 Workflow 参数

Nexus Operation 只能接受一个输入参数。如果想让 Nexus Operation 启动一个接受多个参数的 Workflow，只需在 `RunAsync` 中传入不同的参数即可。

[NexusMultiArg/Handler/HelloService.cs](https://github.com/temporalio/samples-dotnet/blob/main/src/NexusMultiArg/Handler/HelloService.cs)
```csharp
[NexusServiceHandler(typeof(IHelloService))]
public class HelloService
{
    [NexusOperationHandler]
    public IOperationHandler<IHelloService.HelloInput, IHelloService.HelloOutput> SayHello() =>
        // This Nexus service operation is backed by a workflow run. For this sample, we are
        // altering the parameters to the workflow (in this case expanding to two parameters).
        WorkflowRunOperationHandler.FromHandleFactory(
            (WorkflowRunOperationContext context, IHelloService.HelloInput input) =>
                context.StartWorkflowAsync(
                    (HelloHandlerWorkflow wf) => wf.RunAsync(input.Language, input.Name),
                    // Workflow IDs should typically be business meaningful IDs and are used to
                    // dedupe workflow starts. For this example, we're using the request ID
                    // allocated by Temporal when the caller workflow schedules the operation,
                    // this ID is guaranteed to be stable across retries of this operation.
                    new() { Id = context.HandlerContext.RequestId }));
}
```

### 在 Worker 中注册 Nexus Service

开发好异步 Nexus Operation handler 以启动 Workflow 后，下一步是在 Worker 中注册 Nexus Service。

[NexusSimple/Program.cs](https://github.com/temporalio/samples-dotnet/blob/main/src/NexusSimple/Program.cs)
```csharp
async Task RunHandlerWorkerAsync()
{
    // Run worker until cancelled
    logger.LogInformation("Running handler worker");
    using var worker = new TemporalWorker(
        await ConnectClientAsync("nexus-simple-handler-namespace"),
        new TemporalWorkerOptions(taskQueue: "nexus-simple-handler-sample").
            AddNexusService(new HelloService()).
            AddWorkflow<HelloHandlerWorkflow>());
    try
    {
        await worker.ExecuteAsync(tokenSource.Token);
    }
    catch (OperationCanceledException)
    {
        logger.LogInformation("Handler worker cancelled");
    }
}
```

## 开发使用 Nexus Service 的 caller Workflow {#develop-caller-workflow-nexus-service}

导入包含必要 Service 和 Operation 名称以及输入/输出类型的 Service API 包，以便从 caller Workflow 执行 Nexus Operation：

[NexusSimple/Caller/EchoCallerWorkflow.workflow.cs](https://github.com/temporalio/samples-dotnet/blob/main/src/NexusSimple/Caller/EchoCallerWorkflow.workflow.cs)
```csharp
using Temporalio.Workflows;

[Workflow]
public class EchoCallerWorkflow
{
    [WorkflowRun]
    public async Task<string> RunAsync(string message)
    {
        var output = await Workflow.CreateNexusWorkflowClient<IHelloService>(IHelloService.EndpointName).
            ExecuteNexusOperationAsync(svc => svc.Echo(new(message)));
        return output.Message;
    }
}
```

[NexusSimple/Caller/HelloCallerWorkflow.workflow.cs](https://github.com/temporalio/samples-dotnet/blob/main/src/NexusSimple/Caller/HelloCallerWorkflow.workflow.cs)
```csharp
using Temporalio.Workflows;

[Workflow]
public class HelloCallerWorkflow
{
    [WorkflowRun]
    public async Task<string> RunAsync(string name, IHelloService.HelloLanguage language)
    {
        var output = await Workflow.CreateNexusWorkflowClient<IHelloService>(IHelloService.EndpointName).
            ExecuteNexusOperationAsync(svc => svc.SayHello(new(name, language)));
        return output.Message;
    }
}
```

### 设置 Nexus Operation 超时

Nexus Operation 支持[三种类型的超时](/nexus/operations#timeouts)，用于控制 caller 在 Operation 生命周期各阶段愿意等待的时长。调用 `ExecuteNexusOperationAsync` 时，在 `NexusWorkflowOperationOptions` 中设置这些超时。

#### Schedule-to-Close 超时

[Schedule-to-Close 超时](/nexus/operations#schedule-to-close-timeout)限制 Operation 从调度到完成的总持续时间。Nexus Machinery 会自动重试失败的请求，直到超过此超时。

```csharp
var output = await Workflow.CreateNexusWorkflowClient<IHelloService>(IHelloService.EndpointName).
    ExecuteNexusOperationAsync(svc => svc.SayHello(new(name, language)), new NexusWorkflowOperationOptions
    {
        ScheduleToCloseTimeout = TimeSpan.FromMinutes(10),
    });
```

#### Schedule-to-Start 超时

[Schedule-to-Start 超时](/nexus/operations#schedule-to-start-timeout)限制 caller 等待 Operation 由 handler 启动的时长。如果未设置，则不强制执行 Schedule-to-Start 超时。

```csharp
var output = await Workflow.CreateNexusWorkflowClient<IHelloService>(IHelloService.EndpointName).
    ExecuteNexusOperationAsync(svc => svc.SayHello(new(name, language)), new NexusWorkflowOperationOptions
    {
        ScheduleToStartTimeout = TimeSpan.FromMinutes(2),
    });
```

#### Start-to-Close 超时

[Start-to-Close 超时](/nexus/operations#start-to-close-timeout)限制 caller 等待异步 Operation 启动后完成的时长。此超时仅适用于异步 Operation。如果未设置，则不强制执行 Start-to-Close 超时。

```csharp
var output = await Workflow.CreateNexusWorkflowClient<IHelloService>(IHelloService.EndpointName).
    ExecuteNexusOperationAsync(svc => svc.SayHello(new(name, language)), new NexusWorkflowOperationOptions
    {
        StartToCloseTimeout = TimeSpan.FromMinutes(5),
    });
```

开发好 caller Workflow 后，下一步是将其注册到 Worker。


### 开发 starter 以启动 caller Workflow

要启动 caller Workflow，需要使用一个 starter 程序。

[NexusSimple/Program.cs](https://github.com/temporalio/samples-dotnet/blob/main/src/NexusSimple/Program.cs)
```csharp
async Task ExecuteCallerWorkflowAsync()
{
    logger.LogInformation("Executing caller echo workflow");
    var client = await ConnectClientAsync("nexus-simple-caller-namespace");
    var result1 = await client.ExecuteWorkflowAsync(
        (EchoCallerWorkflow wf) => wf.RunAsync("Nexus Echo 👋"),
        new(id: "nexus-simple-echo-id", taskQueue: "nexus-simple-caller-sample"));
    logger.LogInformation("Workflow result: {Result}", result1);

    logger.LogInformation("Executing caller hello workflow");
    var result2 = await client.ExecuteWorkflowAsync(
        (HelloCallerWorkflow wf) => wf.RunAsync("Temporal", IHelloService.HelloLanguage.Es),
        new(id: "nexus-simple-hello-id", taskQueue: "nexus-simple-caller-sample"));
    logger.LogInformation("Workflow result: {Result}", result2);
}
```

## 在开发 Server 上进行跨 Namespace 的 Nexus 调用 {#nexus-calls-across-namespaces-dev-server}

按照以下步骤运行 Nexus handler Worker、Nexus caller Worker 和 starter 应用。

### 运行连接到本地开发 Server 的 Worker

运行 Nexus handler Worker：

```bash
dotnet run handler-worker
```

在另一个终端窗口中，运行 Nexus caller Worker：

```bash
dotnet run caller-worker
```

### 启动 caller Workflow

Worker 运行后，本地开发流程的最后一步是启动 caller Workflow。

运行 starter：

```bash
dotnet run caller-workflow
```

这将显示两个启动的 Workflow 及其结果。

### 取消 Nexus Operation {#canceling-a-nexus-operation}

要从 Workflow 内部取消 Nexus Operation，请取消传递给 Operation 调用的取消令牌。在 Nexus 中，只有异步 Operation 可以被取消，因为取消是通过 Operation 令牌发送的。支撑该 Operation 的 Workflow 或其他资源可以选择忽略取消请求。如果被忽略，Operation 可能进入终止状态。

当 Nexus Operation 启动时，caller 可以指定不同的取消类型，控制 caller 对取消的反应：

- `Abandon` - 不请求取消 Operation。
- `TryCancel` - 发起取消请求并立即向 caller 报告取消。注意，如果 caller 在传递完成之前退出，此类型不保证取消被送达 Operation handler。
- `WaitCancellationRequested` - 请求取消 Operation 并等待请求已被接收的确认。不等待实际取消完成。
- `WaitCancellationCompleted` - 等待 Operation 完成。Operation 可能以取消状态完成，也可能不是。

默认值为 `WaitCancellationCompleted`。用户可以在启动 Operation 时，在 `NexusWorkflowOperationOptions` 中为 `CancellationType` 设置不同选项。

caller Workflow 完成后，caller 的 Nexus Machinery 不会再尝试取消仍在运行的 Operation。在某些使用场景中，保持 Operation 运行是可以接受的。为确保取消被送达，请在退出 Workflow 前等待所有待处理的 Operation 完成。

参考示例，请参阅 [Nexus 取消示例](https://github.com/temporalio/samples-dotnet/tree/main/src/NexusCancellation)。

## 在 Temporal Cloud 上进行跨 Namespace 的 Nexus 调用 {#nexus-calls-across-namespaces-temporal-cloud}

本节假设你已熟悉如何将 Worker 连接到 Temporal Cloud。本节将使用 `tcld` CLI 创建 Namespace 和 Nexus Endpoint，并使用 mTLS 客户端证书将 caller 和 handler Worker 安全地连接到各自的 Temporal Cloud Namespace。

### 安装最新版 `tcld` CLI 并生成证书

要安装最新版 `tcld` CLI，在 macOS 上运行以下命令：

```
brew install temporalio/brew/tcld
```

如果还没有证书，可以使用以下命令生成用于 mTLS Worker 身份验证的证书：

```
tcld gen ca --org $YOUR_ORG_NAME --validity-period 1y --ca-cert ca.pem --ca-key ca.key
```

这些证书有效期为一年。

### 创建 caller 和 handler Namespace

在部署到 Temporal Cloud 之前，请确保已为 caller 和 handler 创建了合适的 Namespace。如果已有这些 Namespace，则无需执行此步骤。

```
tcld login

tcld namespace create \
  --namespace <your-caller-namespace> \
  --region us-west-2 \
  --ca-certificate-file 'path/to/your/ca.pem' \
  --retention-days 1

tcld namespace create \
  --namespace <your-target-namespace> \
  --region us-west-2 \
  --ca-certificate-file 'path/to/your/ca.pem' \
  --retention-days 1
```

或者，可以通过 UI 创建 Namespace：[https://cloud.temporal.io/Namespaces](https://cloud.temporal.io/Namespaces)。

### 创建 Nexus Endpoint 将请求从 caller 路由到 handler

要创建 Nexus Endpoint，必须拥有 Developer 或更高级别的账户角色，并对 `--target-namespace` 拥有 NamespaceAdmin 权限。

```
tcld nexus endpoint create \
  --name nexus-simple-endpoint \
  --target-task-queue nexus-simple-handler-sample \
  --target-namespace <your-handler-namespace.account> \
  --allow-namespace <your-caller-namespace.account> \
  --description-file endpoint_description.md
```

`--allow-namespace` 用于构建 Endpoint 允许列表，指定哪些 caller Namespace 可以使用该 Nexus Endpoint，如运行时访问控制所述。

或者，可以通过 UI 创建 Nexus Endpoint：[https://cloud.temporal.io/nexus](https://cloud.temporal.io/nexus)。

### Temporal CLI

使用 `workflow describe` 命令显示 caller Workflow 中待处理的 Nexus Operation 以及 handler Workflow 上附加的任何回调：

```
temporal workflow describe -w <ID>
```

Nexus 事件包含在 caller 的 Workflow 历史中：

```
temporal workflow show -w <ID>
```

对于**异步 Nexus Operation**，以下内容记录在 caller 的历史中：

- `NexusOperationScheduled`
- `NexusOperationStarted`
- `NexusOperationCompleted`

对于**同步 Nexus Operation**，以下内容记录在 caller 的历史中：

- `NexusOperationScheduled`
- `NexusOperationCompleted`

:::note

对于同步 Operation，`NexusOperationStarted` 不会记录在 caller 的历史中。

:::
