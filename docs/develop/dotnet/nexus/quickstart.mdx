<a id="nexus-net-quickstart"></a>
# Nexus .NET 快速入门 {#nexus-net-quickstart}

:::tip SUPPORT, STABILITY, and DEPENDENCY INFO

Temporal .NET SDK 对 Nexus 的支持位于 [公开预览](/evaluate/development-production-features/release-stages#public-preview) 中。

:::

[Temporal Nexus](/evaluate/nexus) 使用 Nexus 端点、Nexus 服务合同和 Nexus 操作连接 Namespace 内部和之间的 Temporal 应用程序。构建一个包装现有 Temporal Workflow 的 Nexus 服务，然后从调用者 Workflow 调用它。

:::info NEW TO NEXUS?

此页面将帮助您获得在 .NET 中运行的工作示例。

要评估 Nexus 是否适合您的使用案例，请参阅 [评估指南](/evaluate/nexus)，要了解有关 Nexus 功能的更多信息，请单击 [这里](/nexus)。

:::

**先决条件：** 首先完成 [.NET SDK 快速入门](/develop/dotnet/set-up-your-local-dotnet)。
您应该拥有该指南中的 `SayHelloWorkflow`、`MyActivities` 和 `Worker` 项目。

<a id="what-youll-build"></a>
## 你将构建什么 {#what-youll-build}

您有 `SayHelloWorkflow` 在 `default` Namespace 中运行。

在本指南结束时：

1. Nexus 服务将 `SayHelloWorkflow` 公开为操作。
2. 第二个 Namespace 将包含调用该操作的 Workflow。
3. 调用者 Workflow 将返回 `"Hello, Temporal!"` — 相同的结果，但跨越 Namespace。

<SetupSteps>

<SetupStep code={
<>

<CodeSnippet language="csharp">
{`namespace MyNamespace;

using NexusRpc;

[NexusService]
public interface ISayHelloNexusService
{
    public static readonly string EndpointName = "my-nexus-endpoint-name";

    [NexusOperation]
    string SayHello(MyInput input);

    public record MyInput(string Name);
}`}
</CodeSnippet>

</>
}>

<a id="1-define-the-nexus-service"></a>
## 1.定义Nexus服务 {#1-define-the-nexus-service}

在 `Workflow` 项目中创建一个名为 `ISayHelloNexusService.cs` 的文件。

接口上的 `[NexusService]` 属性定义 Nexus 服务契约。 `[NexusOperation]` 标记调用者可以调用的每个方法。 `EndpointName` 静态字段在处理程序和调用者之间共享，以将端点名称保留在一处。

`SayHelloWorkflow` 返回 `string`，因此运算输出类型为 `string`。输入是携带 Workflow 参数的 `record`。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="csharp">
{`namespace MyNamespace;

using NexusRpc.Handlers;
using Temporalio.Nexus;

[NexusServiceHandler(typeof(ISayHelloNexusService))]
public class SayHelloNexusServiceHandler
{
    [NexusOperationHandler]
    public IOperationHandler<ISayHelloNexusService.MyInput, string> SayHello() =>
        WorkflowRunOperationHandler.FromHandleFactory(
            (WorkflowRunOperationContext context, ISayHelloNexusService.MyInput input) =>
                context.StartWorkflowAsync(
                    (SayHelloWorkflow wf) => wf.RunAsync(input.Name),
                    new() { Id = context.HandlerContext.RequestId }));
}`}
</CodeSnippet>

</>
}>

<a id="2-define-the-nexus-operation-handler"></a>
## 2. 定义 Nexus 操作处理程序 {#2-define-the-nexus-operation-handler}

在 `Workflow` 项目中创建一个名为 `SayHelloNexusServiceHandler.cs` 的文件。

`[NexusServiceHandler]` 将该类链接到 `ISayHelloNexusService` 合约。每个 `[NexusOperationHandler]` 方法都会返回一个描述操作如何运行的 `IOperationHandler`。

`WorkflowRunOperationHandler.FromHandleFactory` 创建由 Workflow 运行支持的异步操作。 `input.Name` 将 Nexus `MyInput` 记录桥接到 `SayHelloWorkflow` 的 `string` 参数。

使用 `context.HandlerContext.RequestId` 作为 Workflow ID 可确保对重试的 Nexus 操作请求进行重复数据删除。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="csharp">
{`// Worker/Program.cs
var activities = new MyActivities();
using var worker = new TemporalWorker(
    client,
    new TemporalWorkerOptions("my-task-queue")
        .AddActivity(activities.SayHello)
        .AddWorkflow<SayHelloWorkflow>()
        .AddNexusService(new SayHelloNexusServiceHandler()));`}
</CodeSnippet>

</>
}>

<a id="3-register-the-nexus-service-handler-in-a-worker"></a>
## 3. 在 Worker 中注册 Nexus 服务处理程序 {#3-register-the-nexus-service-handler-in-a-worker}

Update `Worker/Program.cs` 与现有的 Workflow 和 Activity 注册一起注册 Nexus 服务处理程序。

如果注册了 Nexus 服务处理程序，Worker 将仅处理传入的 Nexus 请求。与 `.AddActivity()` 一样，`.AddNexusService()` 也采用一个实例 - 两者都注册 Worker 将工作分派到的具体对象。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="csharp">
{`namespace MyNamespace;

using Temporalio.Workflows;

[Workflow]
public class CallerWorkflow
{
    public static readonly string CallerTaskQueue = "my-caller-task-queue";

    [WorkflowRun]
    public async Task<string> RunAsync(string name)
    {
        return await Workflow
            .CreateNexusWorkflowClient<ISayHelloNexusService>(
                ISayHelloNexusService.EndpointName)
            .ExecuteNexusOperationAsync(svc => svc.SayHello(new(name)));
    }
}`}
</CodeSnippet>

</>
}>

<a id="4-develop-the-caller-workflow"></a>
## 4.开发调用者Workflow {#4-develop-the-caller-workflow}

在 `Workflow` 项目中创建一个名为 `CallerWorkflow.cs` 的文件。

调用者 Workflow 使用 `Workflow.CreateNexusWorkflowClient<T>()` 获取绑定到 Nexus 端点的类型化客户端。 `ExecuteNexusOperationAsync`开始运算并等待结果。

调用者仅依赖于服务合约（`ISayHelloNexusService`），而不依赖于处理程序实现。这种解耦允许调用者和处理程序位于单独的 Namespace 甚至单独的代码库中。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="bash">
{`temporal operator namespace create --namespace my-caller-namespace`}
</CodeSnippet>

<CodeSnippet language="bash">
{`temporal operator nexus endpoint create \\
  --name my-nexus-endpoint-name \\
  --target-namespace default \\
  --target-task-queue my-task-queue`}
</CodeSnippet>

</>
}>

<a id="5-create-the-caller-namespace-and-nexus-endpoint"></a>
## 5. 创建调用者 Namespace 和 Nexus 端点 {#5-create-the-caller-namespace-and-nexus-endpoint}

在运行应用程序之前，创建一个调用者 Namespace 和一个 Nexus 端点以将请求从调用者路由到处理程序。该处理程序使用启动开发服务器时创建的 `default` Namespace。

Namespace 在调用方和处理程序方之间提供隔离。 Nexus 端点充当路由层，将调用方 Namespace 连接到处理程序的目标 Namespace 和 Task Queue。端点名称必须与步骤 1 中定义的 `EndpointName` 常量匹配。

确保您的本地 Temporal 开发服务器正在运行 (`Temporal Server start-dev`)。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="bash">
{`dotnet new console -o CallerStarter
dotnet sln TemporalioHelloWorld.sln \\
  add CallerStarter/CallerStarter.csproj
dotnet add CallerStarter/CallerStarter.csproj \\
  reference Workflow/Workflow.csproj
dotnet add CallerStarter/CallerStarter.csproj package Temporalio`}
</CodeSnippet>

</>
}>

<a id="6-add-the-caller-project"></a>
## 6.添加调用者项目 {#6-add-the-caller-project}

创建一个 `CallerStarter` 控制台项目，启动调用者 Worker 并执行 Workflow。

右侧的命令创建一个新的控制台项目，将其添加到解决方案中，引用 Workflow 项目，然后添加 Temporalio 包。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="csharp" title="CallerStarter/Program.cs">
{`using MyNamespace;
using Temporalio.Client;
using Temporalio.Worker;

var client = await TemporalClient.ConnectAsync(
    new("localhost:7233") { Namespace = "my-caller-namespace" });

using var tokenSource = new CancellationTokenSource();
Console.CancelKeyPress += (_, eventArgs) =>
{
    tokenSource.Cancel();
    eventArgs.Cancel = true;
};

using var worker = new TemporalWorker(
    client,
    new TemporalWorkerOptions(CallerWorkflow.CallerTaskQueue)
        .AddWorkflow<CallerWorkflow>());

Console.WriteLine("Running caller worker");
var workerTask = worker.ExecuteAsync(tokenSource.Token);

var result = await client.ExecuteWorkflowAsync(
    (CallerWorkflow wf) => wf.RunAsync("Temporal"),
    new(id: $"caller-workflow-{Guid.NewGuid()}",
        taskQueue: CallerWorkflow.CallerTaskQueue));
Console.WriteLine("Workflow result: {0}", result);

tokenSource.Cancel();
try { await workerTask; } catch (OperationCanceledException) { }`}
</CodeSnippet>

</>
}>

<a id="7-create-the-caller-starter"></a>
## 7. 创建调用者启动程序 {#7-create-the-caller-starter}

使用右侧的代码创建 `CallerStarter/Program.cs` 文件。

这将所有内容结合在一起：调用者 Worker 托管 `CallerWorkflow`，它使用 Nexus 客户端在处理程序端调用 `SayHello`。完整请求从调用者 Workflow 通过 Nexus 端点流向运行 `SayHelloWorkflow` 的处理程序 Worker，然后返回调用者。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="bash" title="Start the handler Worker">
{`dotnet run --project Worker/Worker.csproj`}
</CodeSnippet>

<CodeSnippet language="bash" title="Run the caller">
{`dotnet run --project CallerStarter/CallerStarter.csproj`}
</CodeSnippet>

</>
}>

<a id="8-run-and-verify"></a>
## 8.运行并验证 {#8-run-and-verify}

使用右侧的命令运行应用程序。

你应该看到：

```
Workflow result: Hello, Temporal!
```

打开[Temporal 网页用户界面](http://localhost:8233)，在`my-caller-Namespace` Namespace中找到`CallerWorkflow`执行。您应该在 Workflow 历史记录中看到 `NexusOperationScheduled`、`NexusOperationStarted` 和 `NexusOperationCompleted` 事件。

</SetupStep>

</SetupSteps>
