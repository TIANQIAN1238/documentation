本页介绍如何异步完成 Activity。

[异步 Activity 完成](/activity-execution#asynchronous-activity-completion) 使 Activity 函数能够在 Activity 执行未完成的情况下返回。

需要遵循三个步骤：

1. Activity 向外部系统提供完成 Activity 执行所需的识别信息。
   标识信息可以是 [任务令牌](/activity-execution#task-token)，也可以是 Namespace、Workflow Id 和 Activity Id 的组合。
2. Activity 函数以将其标识为等待外部系统完成的方式完成。
3. Temporal CLIent 用于心跳并完成 Activity。

要将 Activity 标记为异步完成，请在 Activity 内执行以下操作。

```csharp
// Capture token for later completion
capturedToken = ActivityExecutionContext.Current.Info.TaskToken;

// Throw special exception that says an activity will be completed somewhere else
throw new CompleteAsyncException();
```

对于 Update 和 Activity 之外的 Activity，请使用 [GetAsyncActivityHandle()](https://dotnet.temporal.io/api/temporalio.Client.ITemporalClient.html#Temporalio_Client_ITemporalClient_GetAsyncActivityHandle_System_Byte___) 方法获取 Activity 的句柄。

```csharp
var handle = myClient.GetAsyncActivityHandle(capturedToken);
```

然后，在该句柄上，您可以将 Activity、`HeartbeatAsync`、`CompleteAsync`、`FailAsync` 或 `ReportCancellationAsync` 方法的结果调用到 Update 和 Activity。

```csharp
await handle.CompleteAsync("Completion value.");
```
