Temporal 的安全模型围绕 Payload 的客户端加密设计。客户端可以在将 Payload 发送到服务器之前对其进行加密，并在从服务器接收后对其进行解密。这提供了高度的机密性，因为 Temporal Server 本身对实际数据完全不知情。它还为实现者提供了更多关于哪个客户端能够读取哪些数据的权力和自由——他们可以通过密钥、算法或其他安全措施控制访问。

Temporal 开发者通过向其 Client 提供自定义 Payload Codec 来添加 Payload 的客户端加密。根据业务需求，Payload 加密的完整实现可能涉及选择合适的加密算法、管理加密密钥、限制部分用户查看 Payload 输出，或这些措施的组合。

服务器本身从不对 Payload 添加加密。因此，除非实现了客户端加密，否则 Payload 数据将以非加密形式持久化到数据存储中，任何能够向 Temporal Namespace 发出请求的 Client（包括 Temporal UI 和 CLI）都能读取 Workflow 中包含的 Payload。在处理敏感数据时，应始终实现 Payload 加密。

## 自定义 Payload Codec {#custom-payload-codec}

自定义 Data Converter 可以通过添加钩子、将 Payload 发送到外部存储或执行不同的编码步骤来更改默认的 Temporal Data Conversion 行为。如果只需要更改对 Payload 执行的编码——通过添加压缩或加密——可以覆盖默认的 Data Converter 以使用新的 `PayloadCodec`。

`IPayloadCodec` 需要实现 `EncodeAsync()` 和 `DecodeAsync()` 方法。这些方法应使用 `"encoding"` 元数据字段将给定的 Payload 按需转换为新的 Payload。不要修改现有的 Payload。以下是一个使用 base64 进行双向编码的加密 Codec 示例：

```csharp
public class EncryptionCodec : IPayloadCodec
{
    public Task<IReadOnlyCollection<Payload>> EncodeAsync(IReadOnlyCollection<Payload> payloads) =>
        Task.FromResult<IReadOnlyCollection<Payload>>(payloads.Select(p =>
        {
            return new Payload()
            {
                // Set our specific encoding. We may also want to add a key ID in here for use by
                // the decode side
                Metadata = { ["encoding"] = "binary/my-payload-encoding" },
                Data = ByteString.CopyFrom(Encrypt(p.ToByteArray())),
            };
        }).ToList());

    public Task<IReadOnlyCollection<Payload>> DecodeAsync(IReadOnlyCollection<Payload> payloads) =>
        Task.FromResult<IReadOnlyCollection<Payload>>(payloads.Select(p =>
        {
            // Ignore if it doesn't have our expected encoding
            if (p.Metadata.GetValueOrDefault("encoding") != "binary/my-payload-encoding")
            {
                return p;
            }
            // Decrypt
            return Payload.Parser.ParseFrom(Decrypt(p.Data.ToByteArray()));
        }).ToList());

    private byte[] Encrypt(byte[] data) => Encoding.ASCII.GetBytes(Convert.ToBase64String(data));

    private byte[] Decrypt(byte[] data) => Convert.FromBase64String(Encoding.ASCII.GetString(data));
}
```

**设置 Data Converter 以使用自定义 Payload Codec**

创建客户端时，可以使用如下方式通过 Payload Codec 更新默认的 `DataConverter`：

```csharp
var myClient = await TemporalClient.ConnectAsync(new("localhost:7233")
{
    DataConverter = DataConverter.Default with { PayloadCodec = new EncryptionCodec() },
});
```

- 数据**编码**由客户端使用 Temporal 提供的或自定义的转换器和 Codec 在向 Temporal Cluster 传递输入时执行。例如，纯文本输入通常先序列化为 JSON 对象，然后可以进行压缩或加密。
- 数据**解码**可以在 Workflow 或 Activity 中由应用逻辑按需执行，但解码后的 Workflow 结果不会持久化回 Temporal Cluster。相反，它们以编码形式存储在 Cluster 上，你需要在使用 [temporal workflow show](/cli/workflow#show) 命令或在 Web UI 中浏览时提供附加参数来查看输出。

参考示例，请参阅 [Encryption](https://github.com/temporalio/samples-dotnet/tree/main/src/Encryption) 示例。

### 使用 Codec Server

Codec Server 是一个 HTTP 服务器，它使用自定义的 Codec 逻辑远程解码数据。Codec Server 独立于 Temporal Cluster，通过预定义的端点解码加密的 Payload。你在自己的环境中创建、运行和管理对 Codec Server 的访问。Temporal CLI 和 Web UI 提供内置钩子，可按需调用 Codec Server 解码加密的 Payload。有关如何设计和部署 Codec Server 的信息，请参阅 [Codec Server](/production-deployment/data-encryption) 文档。

## Payload 转换

Temporal SDK 提供了一个默认的 [Payload Converter](/payload-converter)，可以自定义以将自定义数据类型转换为 [Payload](/dataconversion#payload) 并再转换回来。

### 转换顺序 {#conversion-sequence}

编码 Payload Converter 的应用顺序取决于提供给 Data Converter 的顺序。可以设置多个编码 Payload Converter 来运行转换。当 Data Converter 接收到需要转换的值时，它会按顺序通过每个 Payload Converter，直到处理该数据类型的转换器完成转换。

### 自定义 Payload Converter {#custom-payload-converter}

Data Converter 用于将原始 Temporal Payload 与实际 .NET 类型相互转换。创建客户端时，可以通过 `DataConverter` 选项设置自定义 Data Converter。Data Converter 是 Payload Converter、Payload Codec 和 Failure Converter 的组合。Payload Converter 将 .NET 值与序列化字节相互转换。Payload Codec 将字节转换为字节（例如用于压缩或加密）。Failure Converter 将异常与序列化的 Failure 相互转换。

Data Converter 位于 `Temporalio.Converters` 命名空间中。默认的 Data Converter 使用默认的 Payload Converter，支持以下类型：

- `null`
- `byte[]`
- `Google.Protobuf.IMessage` 实例
- `System.Text.Json` 支持的任何类型
- `IRawValue` 作为未转换的原始 Payload

可以为所有用途创建自定义转换器。例如，要创建一个将所有 C# 属性名转换为驼峰命名的 Data Converter，可以：

```csharp
using System.Text.Json;
using Temporalio.Client;
using Temporalio.Converters;

public class CamelCasePayloadConverter : DefaultPayloadConverter
{
    public CamelCasePayloadConverter()
      : base(new JsonSerializerOptions { PropertyNamingPolicy = JsonNamingPolicy.CamelCase })
    {
    }
}

var client = await TemporalClient.ConnectAsync(new()
{
    TargetHost = "localhost:7233",
    Namespace = "my-namespace",
    DataConverter = DataConverter.Default with { PayloadConverter = new CamelCasePayloadConverter() },
});
```
