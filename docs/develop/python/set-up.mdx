这条命令会启动本地 Temporal Service。它会启动 Web UI，创建默认 Namespace，并使用内存数据库。

<a id="run-hello-world-test-your-installation"></a>
## 运行 Hello World：验证安装 {#run-hello-world-test-your-installation}

现在，让我们通过创建并运行一个完整的 Temporal 应用程序来验证你的环境是否工作正常。这个示例包含一个 Workflow 和一个 Activity。

该测试将确认：

- Temporal Python SDK 已正确安装
- 本地 Temporal Service 正在运行
- 你可以成功创建并执行 Workflow 和 Activity
- 各组件之间的通信正常

<a id="1-create-the-activity"></a>
### 1. 创建 Activity {#1-create-the-activity}

创建 Activity 文件 `activities.py`：

```python
from temporalio import activity

@activity.defn
async def greet(name: str) -> str:
    return f"Hello {name}"
```

Activity 是执行单个、定义明确操作的普通函数或方法，可以是短时运行，也可以是长时运行。常见示例包括发送邮件、发起网络请求、写入数据库，或调用容易失败的外部 API。如果 Activity 失败，Temporal 会根据你的配置自动重试。

<a id="2-create-the-workflow"></a>
### 2. 创建 Workflow {#2-create-the-workflow}

创建 Workflow 文件 `workflows.py`：

```python
from datetime import timedelta
from temporalio import workflow

with workflow.unsafe.imports_passed_through():
    from activities import greet

@workflow.defn
class SayHelloWorkflow:
    @workflow.run
    async def run(self, name: str) -> str:
        return await workflow.execute_activity(
            greet,
            name,
            schedule_to_close_timeout=timedelta(seconds=10),
        )
```

Workflow 用于编排 Activity 并承载应用逻辑。Temporal Workflow 具备韧性：即使底层基础设施发生故障，它们仍然可以持续运行数年。如果应用本身崩溃，Temporal 会自动恢复崩溃前的状态，使执行能够从中断处继续。

<a id="3-create-the-worker"></a>
### 3. 创建 Worker {#3-create-the-worker}

创建 Worker 文件 `worker.py`：

```python
import asyncio
from temporalio.client import Client
from temporalio.worker import Worker
from temporalio import workflow

with workflow.unsafe.imports_passed_through():
    from workflows import SayHelloWorkflow
    from activities import greet

async def main():
    client = await Client.connect("localhost:7233")
    worker = Worker(
        client,
        task_queue="my-task-queue",
        workflows=[SayHelloWorkflow],
        activities=[greet],
    )
    print("Worker started.")
    await worker.run()

if __name__ == "__main__":
    asyncio.run(main())
```

打开一个新终端，运行 Worker：

```bash
source env/bin/activate
python3 worker.py
```

保持这个终端窗口持续运行。你应该会看到 `Worker started.`。

定义好 Activity 和 Workflow 后，还需要 Worker 来执行它们。Worker 会轮询它所监听的 Task Queue，查找待处理任务。一旦 Worker 从队列中取出 Workflow 或 Activity 任务，就会开始执行。

Worker 是 Temporal 应用的重要组成部分，因为它们实际负责执行你的 Temporal 应用中定义的任务。关于 Worker 的更多信息，请参阅 [了解 Temporal](/evaluate/understanding-temporal#workers) 和 [Workers](/workers)。

<a id="4-execute-the-workflow"></a>
### 4. 执行 Workflow {#4-execute-the-workflow}

现在 Worker 已经运行，可以开始启动 Workflow Execution 了。

最后一步将通过 `starter.py` 这个文件验证整个流程是否正常。

创建一个单独的文件 `starter.py`：

```python
import asyncio
import uuid
from temporalio.client import Client

async def main():
    client = await Client.connect("localhost:7233")
    result = await client.execute_workflow(
        "SayHelloWorkflow",
        "Temporal",
        id=f"say-hello-workflow-{uuid.uuid4()}",
        task_queue="my-task-queue",
    )
    print("Workflow result:", result)

if __name__ == "__main__":
    asyncio.run(main())
```

在 Worker 持续运行的同时，在另一个新终端中运行：

```bash
source env/bin/activate
python3 starter.py
```
