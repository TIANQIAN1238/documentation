## 启动一个子 Workflow Execution {#child-workflows}

**如何使用 Temporal Python SDK 启动子 Workflow Execution。**

[儿童 Workflow Execution](/child-workflows) 是一个 Workflow Execution，它是使用 Child Workflow API 来自另一个 Workflow 的 Scheduled。

使用 Child Workflow API 时，Child Workflow 相关事件（[StartChildWorkflowExecutionInitiated](/references/events#startchildworkflowexecutioninitiated)、[子WorkflowExecutionStarted](/references/events#childworkflowexecutionstarted)、[ChildWorkflowExecutionCompleted](/references/events#childworkflowexecutioncompleted) 等）将记录在 Workflow Execution Event History 中。

始终阻止进度，直到 [子WorkflowExecutionStarted](/references/events#childworkflowexecutionstarted) 事件记录到 Event History，以确保子 Workflow Execution 已启动。
此后，可以使用 Child Workflow 选项中设置的 _Abandon_ [家长关闭政策](/parent-close-policy) 放弃子 Workflow Execution。

要确保 Child Workflow Execution 已启动，请首先在 Child Workflow future 实例上调用 Child Workflow Execution 方法，该方法返回不同的 future。

然后获取一个对象的值，该对象充当最初未知的结果的代理，该结果将等待子 Workflow Execution 生成。

要在 Python 中生成子 Workflow Execution，请使用 [`execute_child_Workflow()`](https://python.temporal.io/temporalio.workflow.html#execute_child_workflow) 函数，该函数启动 Child Workflow 并等待完成或
使用 [`start_child_Workflow()`](https://python.temporal.io/temporalio.workflow.html#start_child_workflow) 函数启动 Child Workflow 并返回其句柄。
如果您想在启动后执行某些操作，或者获取 Workflow/Run ID，或者能够在运行时执行 Signal，则这非常有用。

:::note

`execute_child_Workflow()` 是 `start_child_Workflow()` 和 `await handle` 的辅助函数。

:::

### 设置父关闭策略 {#parent-close-policy}

**如何设置家长关闭政策**

[家长关闭政策](/parent-close-policy) 确定如果子 Workflow Execution 的父级更改为关闭状态（已完成、失败或超时），子级 Workflow Execution 会发生什么情况。

默认“父关闭策略”选项设置为终止子 Workflow Execution。

在 [`start_child_Workflow`](https://python.temporal.io/temporalio.workflow.html#start_child_workflow) 函数或 [`execute_child_Workflow()`](https://python.temporal.io/temporalio.workflow.html#execute_child_workflow) 函数内设置 `parent_close_policy` 参数，以指定父 Workflow 关闭时 Child Workflow 的行为。

```python
from temporalio.workflow import ParentClosePolicy
# ...
# ...
@workflow.defn
class ComposeGreetingWorkflow:
    @workflow.run
    async def run(self, input: ComposeGreetingInput) -> str:
        return f"{input.greeting}, {input.name}!"


@workflow.defn
class GreetingWorkflow:
    @workflow.run
    async def run(self, name: str) -> str:
        return await workflow.execute_child_workflow(
            ComposeGreetingWorkflow.run,
            ComposeGreetingInput("Hello", name),
            id="hello-child-workflow-workflow-child-id",
            parent_close_policy=ParentClosePolicy.ABANDON,
        )
```
