<a id="workflow-timeouts"></a>
## Workflow 超时 {#workflow-timeouts}

每个 Workflow 超时控制 Workflow Execution 不同方面的最大持续时间。

在继续之前，我们要注意，我们通常不建议设置 Workflow 超时，因为 Workflow 被设计为长期运行且具有弹性。
相反，设置超时可能会限制其处理意外延迟或长时间运行的进程的能力。
如果您需要在特定时间段后在 Workflow 内执行操作，我们建议使用 Timer。

当 [启动 Workflow Execution](#workflow-timeouts) 时，设置 Workflow 超时。

- **[Workflow Execution 超时](/encyclopedia/detecting-workflow-failures#workflow-execution-timeout)** - 限制单个 Workflow Execution 可以执行的最长时间。
- **[Workflow 运行超时](/encyclopedia/detecting-workflow-failures#workflow-run-timeout)：** 限制单个 Workflow 运行可以持续的最长时间。
- **[Workflow 任务超时](/encyclopedia/detecting-workflow-failures#workflow-task-timeout)：** 限制 Worker 可以执行 Workflow 任务的最长时间。

将超时设置为 [`start_Workflow()`](https://python.temporal.io/temporalio.client.Client.html#start_workflow) 或 [`execute_Workflow()`](https://python.temporal.io/temporalio.client.Client.html#execute_workflow) 异步方法。

可用的超时有：

- `execution_timeout`
- `run_timeout`
- `task_timeout`

```python
    )
```

<a id="workflow-retries"></a>
## Workflow 重试 {#workflow-retries}

**如何使用 Temporal Python SDK 设置 Workflow Retry Policy**

Retry Policy 可以与超时配合，提供精细控制以优化执行体验。

如果发生故障，请使用 [Retry Policy](/encyclopedia/retry-policies) 重试 Workflow Execution。

默认情况下，Workflow Execution 不会重试，并且仅在某些情况下才应将重试策略与 Workflow Execution 一起使用。

将 Retry Policy 设置为 [`start_Workflow()`](https://python.temporal.io/temporalio.client.Client.html#start_workflow) 或 [`execute_Workflow()`](https://python.temporal.io/temporalio.client.Client.html#execute_workflow) 异步方法。

<div className="copycode-notice-container">
  <a href="https://github.com/temporalio/documentation/blob/main/sample-apps/python/workflow_timeouts_retries/workflows_dacx.py">
查看源代码
  </a>{' '}
在应用程序代码的其余部分的上下文中。
</div>

```python
# ...
    handle = await client.execute_workflow(
        YourWorkflow.run,
        "your retry policy argument",
        id="your-workflow-id",
        task_queue="your-task-queue",
        retry_policy=RetryPolicy(maximum_interval=timedelta(seconds=2)),
    )
```
