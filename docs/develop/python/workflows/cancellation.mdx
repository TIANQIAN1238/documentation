您可以通过以下方式之一中断 Workflow Execution：

- [取消](#cancellation)：取消 Workflow 提供了一种停止 Workflow Execution 的优雅方法。
- [终止](#termination)：终止 Workflow 会强制停止 Workflow Execution。

终止 Workflow 会强制停止 Workflow Execution。此操作类似于终止进程。

- 系统在 Workflow 历史记录中记录 `WorkflowExecutionTerminated` 事件。
- 强制终止并立即停止 Workflow Execution。
- Workflow 代码没有机会处理终止。
- Workflow 任务不会获得 Scheduled。

在大多数情况下，取消是更好的选择，因为它允许 Workflow 优雅地完成。仅在以下情况下终止
Workflow被卡住，无法正常取消。

## 取消 Workflow Execution {#cancellation}

取消 Workflow 提供了一种停止 Workflow Execution 的优雅方法。此操作类似于将 `SIGTERM` 发送到
过程。

- 系统在 Workflow 历史记录中记录 `WorkflowExecutionCancelRequested` 事件。
- Workflow 任务获取 Scheduled 来处理取消。
- Workflow 代码可以处理取消并执行任何清理逻辑。
- 系统不会强制停止Workflow。

要在 Python 中取消 Workflow Execution，请使用
[取消（）](https://python.temporal.io/temporalio.client.WorkflowHandle.html#cancel) 功能在 Workflow 手柄上。

```python
await client.get_workflow_handle("your_workflow_id").cancel()
```

<a id="cancel-activity"></a>
### 从 Workflow 取消 Activity {#cancel-activity}

从 Workflow 内取消 Activity 要求 Activity 执行发送心跳并设置心跳
超时。如果不调用Heartbeat，Activity将无法接收取消请求。当任何非立即
Activity 被执行，Activity 执行应该发送心跳并设置
[心跳超时](/encyclopedia/detecting-activity-failures#heartbeat-timeout) 确保服务器知道它是
仍在工作。

当取消 Activity 时，在下一个可用机会时 Activity 中会引发错误。如果清理逻辑
需要执行，可以在 `finally` 子句中或在捕获的取消错误内完成。然而，对于 Activity
要显示已取消，需要重新引发异常。

:::note

与常规活动不同，[当地活动](/local-activity) 如果不发送心跳，则可以取消。本地
活动在本地处理，处理取消逻辑所需的所有信息都可以在同一个
Worker 过程。

:::

要从 Workflow Execution 取消 Activity，请调用
Activity 句柄上的 [取消（）](https://docs.python.org/3/library/asyncio-task.html#asyncio.Task.cancel) 方法
从 [开始_Activity()](https://python.temporal.io/temporalio.workflow.html#start_activity) 返回。

```python
@activity.defn
async def cancellable_activity(input: ComposeArgsInput) -> NoReturn:
    try:
        while True:
            print("Heartbeating cancel activity")
            await asyncio.sleep(0.5)
            activity.heartbeat("some details")
    except asyncio.CancelledError:
        print("Activity cancelled")
        raise


@activity.defn
async def run_activity(input: ComposeArgsInput):
    print("Executing activity")
    return input.arg1 + input.arg2

@workflow.defn
 class GreetingWorkflow:
     @workflow.run
     async def run(self, input: ComposeArgsInput) -> None:
        activity_handle = workflow.start_activity(
            cancellable_activity,
            ComposeArgsInput(input.arg1, input.arg2),
            start_to_close_timeout=timedelta(minutes=5),
            heartbeat_timeout=timedelta(seconds=30),
        )

        await asyncio.sleep(3)
        activity_handle.cancel()
```

:::note

Activity 句柄是一个 Python 任务。通过调用 `cancel()`，您实际上是在请求取消任务。

:::

## 终止 Workflow Execution {#termination}

终止 Workflow 会强制停止 Workflow Execution。此操作类似于终止进程。

- 系统在 Workflow 历史记录中记录 `WorkflowExecutionTerminated` 事件。
- 强制终止并立即停止 Workflow Execution。
- Workflow 代码没有机会处理终止。
- Workflow 任务不会获得 Scheduled。

要在 Python 中终止 Workflow Execution，请使用
Workflow 上的 [终止（）](https://python.temporal.io/temporalio.client.WorkflowHandle.html#terminate) 功能
手柄。

```python
await client.get_workflow_handle("your_workflow_id").terminate()
```

## 重置 Workflow Execution {#reset}

重置 Workflow Execution 将终止当前的 Workflow Execution 并从
您在其 Event History 中指定的点。当 Workflow 由于非确定性错误或其他原因而被阻止时，请使用重置
阻止其完成的问题。

当您重置 Workflow 时，重置点之前的 Event History 将被复制到新的 Workflow Execution，并且
Workflow 从该点使用当前代码恢复。仅当您解决了根本问题（例如
删除不确定性代码。重置点之后取得的任何进度都将被丢弃。给出一个理由，当
重置，因为它将记录在 Event History 中。

<Tabs>

<TabItem value="web-ui" label="Web UI">

1. 导航至Workflow Execution详细信息页面，
2. 单击右上角下拉菜单中的 **重置** 按钮，
3. 选择要重置的事件 ID，
4. 提供重置原因，
5. 确认重置。

Web UI 显示可用的重置点，并在重置完成后创建指向新 Workflow Execution 的链接。

</TabItem>

<TabItem value="cli" label="Temporal CLI">

使用 `Temporal Workflow reset` 命令重置 Workflow Execution：

```bash
temporal workflow reset \
    --workflow-id <workflow-id> \
    --event-id <event-id> \
    --reason "Reason for reset"
```

例如：

```bash
temporal workflow reset \
    --workflow-id my-background-check \
    --event-id 4 \
    --reason "Fixed non-deterministic code"
```

默认情况下，该命令重置 `default` Namespace 中最新的 Workflow Execution。使用 `--run-id` 重置
具体运行。使用 `--Namespace` 指定不同的 Namespace：

```bash
temporal workflow reset \
    --workflow-id my-background-check \
    --event-id 4 \
    --reason "Fixed non-deterministic code" \
    --namespace my-namespace \
    --tls-cert-path /path/to/cert.pem \
    --tls-key-path /path/to/key.pem
```

重置后监视新的 Workflow Execution 以确保其成功完成。

</TabItem>

</Tabs>
