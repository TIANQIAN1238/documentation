##安装Java JDK

确保您已安装 Java JDK。您可以直接从 Oracle 下载副本或选择 OpenJDK
来自您首选供应商的分销。

您还需要安装 Maven 或 Gradle。

**如果您没有 Maven：** [下载](https://maven.apache.org/download.cgi) 和
来自 Apache.org 的 [安装](https://maven.apache.org/install.html)，或使用 Homebrew：`brew install maven`。

**如果您没有 Gradle：** 来自 Gradle.org 的 [下载](https://gradle.org/install/)，请使用
[智能IDEA](https://www.jetbrains.com/idea/)（捆绑），或使用自制软件：`brew install gradle`。

</SetupStep>

<a id="start-the-development-server"></a>
## 运行 Hello World：测试您的安装 {#start-the-development-server}

现在，让我们通过创建并运行一个完整的 Temporal 应用程序来验证您的设置是否正常工作，其中包含 Workflow 和
Activity。

该测试将确认：

- Temporal Java SDK 已正确安装
- 您本地的 Temporal 服务正在运行
- 您可以成功创建并执行 Workflow 和活动
- 组件之间的通信功能正常

<a id="run-hello-world-test-your-installation"></a>
### 1.创建Activity接口 {#run-hello-world-test-your-installation}

创建Activity接口文件（GreetActivities.java）：

_请注意，本快速入门的所有文件都将在 src/main/java/helloWorkflow 下创建。_

```java
package helloworkflow;

import io.temporal.activity.ActivityInterface;
import io.temporal.activity.ActivityMethod;

@ActivityInterface
public interface GreetActivities {

    @ActivityMethod
    String greet(String name);

}
```

Activity 是一种执行单个明确定义的操作（短期或长期运行）的方法，通常涉及
与外界交互，例如发送电子邮件、发出网络请求、写入数据库或调用
API，很容易出现故障。如果 Activity 失败，Temporal 会根据您的配置自动重试。

您将 Java 中的活动定义为带注释的接口及其实现。

<a id="1-create-the-activity-interface"></a>
### 2. 创建 Activity 实现 {#1-create-the-activity-interface}

创建Activity实现文件（GreetActivitiesImpl.java）：

```java
package helloworkflow;

public class GreetActivitiesImpl implements GreetActivities {

    @Override
    public String greet(String name) {
      return "Hello " + name;
    }

}
```

<a id="2-create-the-activity-implementation"></a>
### 3.创建Workflow {#2-create-the-activity-implementation}

创建 Workflow 文件 (SayHelloWorkflow.java)：

```java
package helloworkflow;

import io.temporal.workflow.WorkflowInterface;
import io.temporal.workflow.WorkflowMethod;

@WorkflowInterface
public interface SayHelloWorkflow {

    @WorkflowMethod
    String sayHello(String name);

}
```

Workflow 编排活动并包含应用程序逻辑。 Temporal Workflow 具有弹性。他们可以奔跑并且
即使底层基础设施出现故障，也能持续运行数年。如果应用程序本身崩溃，Temporal 将
自动重新创建故障前的状态，以便可以从中断处继续。

您在 Java 中将 Workflow 定义为带注释的接口及其实现。

<a id="3-create-the-workflow"></a>
### 4. 创建 Workflow 实现 {#3-create-the-workflow}

创建Workflow实现文件（SayHelloWorkflowImpl.java）：

```java
package helloworkflow;

import io.temporal.activity.ActivityOptions;
import io.temporal.workflow.Workflow;

import java.time.Duration;

public class SayHelloWorkflowImpl implements SayHelloWorkflow {

    private final GreetActivities activities = Workflow.newActivityStub(
      GreetActivities.class,
      ActivityOptions.newBuilder()
        .setStartToCloseTimeout(Duration.ofSeconds(5))
        .build()
    );

    @Override
    public String sayHello(String name) {
      return activities.greet(name);
    }

}
```

<a id="4-create-the-workflow-implementation"></a>
### 5. 创建并运行 Worker {#4-create-the-workflow-implementation}

创建 Worker 文件 (SayHelloWorker.java)：

```java
package helloworkflow;

import io.temporal.client.WorkflowClient;
import io.temporal.serviceclient.WorkflowServiceStubs;
import io.temporal.worker.Worker;
import io.temporal.worker.WorkerFactory;

public class SayHelloWorker {

    public static void main(String[] args) {

      WorkflowServiceStubs service = WorkflowServiceStubs.newLocalServiceStubs();
      WorkflowClient client = WorkflowClient.newInstance(service);
      WorkerFactory factory = WorkerFactory.newInstance(client);

      Worker worker = factory.newWorker("my-task-queue");
      worker.registerWorkflowImplementationTypes(SayHelloWorkflowImpl.class);
      worker.registerActivitiesImplementations(new GreetActivitiesImpl());

      System.out.println("Starting SayHelloWorker for task queue 'my-task-queue'...");

      factory.start();

    }

}
```

定义了 Activity 和 Workflow 后，您需要 Worker 来执行它们。

打开一个新终端并运行 Worker：

<Tabs>
<TabItem value="maven" label="Maven">

```bash
cd temporal-hello-world
mvn compile exec:java -Dexec.mainClass="helloworkflow.SayHelloWorker"
```

</TabItem>
<TabItem value="gradle" label="Gradle">

```bash
./gradlew runWorker
```

</TabItem>
</Tabs>

Worker 轮询 Task Queue，您将其配置为轮询，寻找要做的工作。一旦 Worker 将 Workflow 出列
或来自 Task Queue 的 Activity 任务，然后执行该任务。

Worker 是 Temporal 应用程序的重要组成部分，因为它们实际执行您的 Temporal 应用程序中定义的任务。
Workflow 和活动。有关 Worker 的更多信息，请参阅
[了解 Temporal](/evaluate/understanding-temporal#workers) 和 [深入研究 Worker](/workers)。

<a id="5-create-and-run-the-worker"></a>
### 6.执行Workflow {#5-create-and-run-the-worker}

现在您的 Worker 正在运行，是时候启动 Workflow Execution 了。

最后一步将验证标记为 `Starter.java` 的文件是否一切正常。

创建一个名为 `Starter.java` 的单独文件：

```java
package helloworkflow;

import io.temporal.client.WorkflowClient;
import io.temporal.client.WorkflowOptions;
import io.temporal.serviceclient.WorkflowServiceStubs;

public class Starter {
    public static void main(String[] args) {
        WorkflowServiceStubs service = WorkflowServiceStubs.newLocalServiceStubs();
        WorkflowClient client = WorkflowClient.newInstance(service);

        SayHelloWorkflow workflow = client.newWorkflowStub(
            SayHelloWorkflow.class,
            WorkflowOptions.newBuilder()
                .setTaskQueue("my-task-queue")
                .setWorkflowId("say-hello-workflow-id")
                .build()
        );

        String result = workflow.sayHello("Temporal");
        System.out.println("Workflow result: " + result);

        service.shutdown();
        service.awaitTermination(10, java.util.concurrent.TimeUnit.SECONDS);
    }
}
```

当您的 Worker 仍在运行时，打开一个新终端并运行：

<TabItem value="maven" label="Maven">

```bash
cd temporal-hello-world
mvn compile exec:java -Dexec.mainClass="helloworkflow.Starter"
```

</TabItem>
<TabItem value="gradle" label="Gradle">

```bash
./gradlew runStarter
```

</TabItem>
</Tabs>
