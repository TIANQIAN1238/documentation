<a id="run-the-temporal-nexus-development-server"></a>
## 在启用 Nexus 的情况下运行 Temporal 开发服务器 {#run-the-temporal-nexus-development-server}

先决条件：

- [安装最新的Temporal CLI](https://learn.temporal.io/getting_started/java/dev_environment/#set-up-a-local-temporal-service-for-development-with-temporal-cli)
  （推荐v1.3.0或更高版本）
- [安装最新的 Temporal Java SDK](https://learn.temporal.io/getting_started/java/dev_environment/#add-temporal-java-sdk-dependencies)
  （推荐v1.28.0或更高版本）

使用 Temporal Nexus 的第一步涉及启动 Temporal Server 并启用 Nexus。

```
temporal server start-dev
```

此命令自动使用 Web UI 启动 Temporal 开发服务器，并创建 `default` Namespace。
它使用内存数据库，因此不要将其用于实际用例。

现在应该可以通过 [http://localhost:8233](http://localhost:8233) 访问 Temporal Web UI，并且 Temporal Server
现在应该可用于 `localhost:7233` 上的客户端连接。

<a id="create-caller-handler-namespaces"></a>
## 创建调用者和处理程序 Namespaces {#create-caller-handler-namespaces}

在设置 Nexus 端点之前，请为调用者和处理程序创建单独的 Namespace。

```
temporal operator namespace create --namespace my-target-namespace
temporal operator namespace create --namespace my-caller-namespace
```

`my-target-Namespace` 将包含 Nexus 操作处理程序，我们将在 `my-caller-Namespace` 中使用 Workflow 来
调用该操作处理程序。我们使用不同的 Namespace 来演示跨 Namespace Nexus 调用。

<a id="create-nexus-endpoint"></a>
## 创建一个 Nexus 端点以将请求从调用者路由到处理程序 {#create-nexus-endpoint}

建立调用者和处理程序 Namespace 后，下一步是创建 Nexus 端点来路由请求。

```
temporal operator nexus endpoint create \
  --name my-nexus-endpoint-name \
  --target-namespace my-target-namespace \
  --target-task-queue my-handler-task-queue
```

您还可以使用 Web UI 创建 Namespace 和 Nexus 端点。

<a id="define-nexus-service-contract"></a>
## 定义 Nexus 服务合约 {#define-nexus-service-contract}

为 Nexus 服务定义明确的合同对于顺利沟通至关重要。

在此示例中，有一个服务包描述了服务和操作名称以及输入/输出类型
供调用者 Workflow 使用 Nexus 端点。

每个 [Temporal SDK 包含并使用默认的 Data Converter](https://docs.temporal.io/dataconversion)。默认
Data Converter 按以下顺序对有效负载进行编码：Null、字节数组、Protobuf JSON 和 JSON。在多语言环境中
环境，即使用多种语言和 SDK 来开发 Temporal 解决方案、Protobuf 和
JSON 是常见的选择。此示例使用序列化为 JSON 的 Java 类。

<!--SNIPSTART samples-java-Nexus-service-->

[核心/src/main/java/io/temporal/samples/nexus/service/nexusService.java](https://github.com/temporalio/samples-java/blob/nexus-snip-sync/core/src/main/java/io/temporal/samples/nexus/service/nexusService.java)

```java
@Service
public interface SampleNexusService {
  enum Language {
    EN,
    FR,
    DE,
    ES,
    TR
  }

  class HelloInput {
    private final String name;
    private final Language language;

    @JsonCreator(mode = JsonCreator.Mode.PROPERTIES)
    public HelloInput(
        @JsonProperty("name") String name, @JsonProperty("language") Language language) {
      this.name = name;
      this.language = language;
    }

    @JsonProperty("name")
    public String getName() {
      return name;
    }

    @JsonProperty("language")
    public Language getLanguage() {
      return language;
    }
  }

  class HelloOutput {
    private final String message;

    @JsonCreator(mode = JsonCreator.Mode.PROPERTIES)
    public HelloOutput(@JsonProperty("message") String message) {
      this.message = message;
    }

    @JsonProperty("message")
    public String getMessage() {
      return message;
    }
  }

  class EchoInput {
    private final String message;

    @JsonCreator(mode = JsonCreator.Mode.PROPERTIES)
    public EchoInput(@JsonProperty("message") String message) {
      this.message = message;
    }

    @JsonProperty("message")
    public String getMessage() {
      return message;
    }
  }

  class EchoOutput {
    private final String message;

    @JsonCreator(mode = JsonCreator.Mode.PROPERTIES)
    public EchoOutput(@JsonProperty("message") String message) {
      this.message = message;
    }

    @JsonProperty("message")
    public String getMessage() {
      return message;
    }
  }

  @Operation
  HelloOutput hello(HelloInput input);

  @Operation
  EchoOutput echo(EchoInput input);
}
```

<a id="develop-nexus-service-operation-handlers"></a>
## 开发 Nexus 服务和操作处理程序 {#develop-nexus-service-operation-handlers}

Nexus 操作处理程序通常在与它们抽象的底层 Temporal 原语相同的 Worker 中定义。
操作处理程序可以决定给定的 Nexus 操作是同步还是异步。他们可以调用底层
使用 Temporal SDK 客户端的 Temporal 基元（例如 Query、Signal 或 Update），或运行其他可靠的代码。
处理程序应该可靠，因为 [断路器](/nexus/operations#circuit-breaking) 在连续 5 次后跳闸
可重试的错误，阻止从调用者到该端点的所有操作。

`io.Temporal.Nexus.*` 软件包具有帮助创建 Nexus 操作的实用程序：

- `Nexus.getOperationContext().getWorkflowClient()` \- 获取 Worker 初始化时使用的 Temporal CLIent
  由 Temporal 原语（例如 Signal 和查询）支持的同步处理程序
- `WorkflowRunOperation.fromWorkflowMethod` \- 将 Workflow 作为异步 Nexus 操作运行

此示例从使用 `OperationHandler.sync` 方法的同步操作处理程序示例开始，然后展示如何
创建使用 `WorkflowRunOperation.fromWorkflowMethod` 启动处理程序 Workflow 的异步操作处理程序
来自 Nexus 操作。

<a id="develop-a-synchronous-nexus-operation-handler"></a>
### 开发同步 Nexus 操作处理程序 {#develop-a-synchronous-nexus-operation-handler}

`OperationHandler.sync` 方法用于公开简单的 RPC 处理程序。使用
`Nexus.getOperationContext().getWorkflowClient(ctx)` 获取 Temporal CLIent 用于 Signaling、Querying 和列表
Workflow。实现还可以进行其他调用，但处理程序应该可靠，以避免触发
[断路器](/nexus/operations#circuit-breaking)。

{/* SNIPSTART samples-java-nexus-handler {"selectedLines": ["1-16", "43"]} */}
[核心/src/main/java/io/temporal/samples/nexus/handler/nexusServiceImpl.java](https://github.com/temporalio/samples-java/blob/nexus-snip-sync/core/src/main/java/io/temporal/samples/nexus/handler/nexusServiceImpl.java)

```java
// To create a service implementation, annotate the class with @ServiceImpl and provide the
// interface that the service implements. The service implementation class should have methods that
// return OperationHandler that correspond to the operations defined in the service interface.
@ServiceImpl(service = SampleNexusService.class)
public class SampleNexusServiceImpl {
  @OperationImpl
  public OperationHandler<SampleNexusService.EchoInput, SampleNexusService.EchoOutput> echo() {
    // OperationHandler.sync is a meant for exposing simple RPC handlers.
    return OperationHandler.sync(
        // The method is for making arbitrary short calls to other services or databases, or
        // perform simple computations such as this one. Users can also access a workflow client by
        // calling
        // Nexus.getOperationContext().getWorkflowClient(ctx) to make arbitrary calls such as
        // signaling, querying, or listing workflows.
        (ctx, details, input) -> new SampleNexusService.EchoOutput(input.getMessage()));
  }
// ...
}
```

<a id="use-the-temporal-client-for-signals-queries-and-updates"></a>
### 将 Temporal CLIent 用于 Signal、查询和 Update {#use-the-temporal-client-for-signals-queries-and-updates}

常见模式是在同步处理程序中使用 Temporal CLIent 到 Signal、Query 或 Update 和 Workflow。你
还可以使用 Signal-With-Start 或 Update-With-Start 确保 Workflow 已启动并向其发送 Signal 或 Update。
所有调用必须在 [Nexus 请求超时](/cloud/limits#nexus-operation-request-timeout) 内完成。 Updates
为了保持在这个最后期限内，应该是短暂的。

<a id="develop-an-asynchronous-nexus-operation-handler-to-start-a-workflow"></a>
### 开发异步 Nexus 操作处理程序来启动 Workflow {#develop-an-asynchronous-nexus-operation-handler-to-start-a-workflow}

使用 `WorkflowRunOperation.fromWorkflowMethod` 方法，这是将 Workflow 公开为操作的最简单方法。

<!--SNIPSTART samples-java-Nexus-handler {"selectedLines": ["1-5", "18-40"]}-->

[核心/src/main/java/io/temporal/samples/nexus/handler/nexusServiceImpl.java](https://github.com/temporalio/samples-java/blob/nexus-snip-sync/core/src/main/java/io/temporal/samples/nexus/handler/nexusServiceImpl.java)

```java
// To create a service implementation, annotate the class with @ServiceImpl and provide the
// interface that the service implements. The service implementation class should have methods that
// return OperationHandler that correspond to the operations defined in the service interface.
@ServiceImpl(service = SampleNexusService.class)
public class SampleNexusServiceImpl {
// ...
  @OperationImpl
  public OperationHandler<SampleNexusService.HelloInput, SampleNexusService.HelloOutput> hello() {
    // Use the WorkflowRunOperation.fromWorkflowMethod constructor, which is the easiest
    // way to expose a workflow as an operation. To expose a workflow with a different input
    // parameters then the operation or from an untyped stub, use the
    // WorkflowRunOperation.fromWorkflowHandler constructor and the appropriate constructor method
    // on WorkflowHandle.
    return WorkflowRunOperation.fromWorkflowMethod(
        (ctx, details, input) ->
            Nexus.getOperationContext()
                    .getWorkflowClient()
                    .newWorkflowStub(
                        HelloHandlerWorkflow.class,
                        // Workflow IDs should typically be business meaningful IDs and are used to
                        // dedupe workflow starts.
                        // For this example, we're using the request ID allocated by Temporal when
                        // the
                        // caller workflow schedules
                        // the operation, this ID is guaranteed to be stable across retries of this
                        // operation.
                        //
                        // Task queue defaults to the task queue this operation is handled on.
                        WorkflowOptions.newBuilder().setWorkflowId(details.getRequestId()).build())
```

<!--SNIPEND-->

Workflow ID 通常应该是具有业务意义的 ID，并用于对 Workflow 启动进行重复数据删除。一般来说，身份证
应作为 Nexus 服务合同的一部分在操作输入中传递。

[将多个 Nexus 调用者附加到处理程序 Workflow](/nexus/operations#attaching-multiple-nexus-callers) 带有
现有使用冲突政策。

```java
    // use the WorkflowRunOperation.fromWorkflowHandler constructor and the appropriate constructor
```

### 在 Worker 中注册 Nexus 服务 {#register-a-nexus-service-in-a-worker}

```java
package io.temporal.samples.nexus.handler;

import io.temporal.client.WorkflowClient;
import io.temporal.samples.nexus.options.ClientOptions;

import io.temporal.worker.WorkerFactory;

    worker.registerNexusServiceImplementation(new SampleNexusServiceImpl());

    factory.start();
  }
}
```

[核心/src/main/java/io/temporal/samples/nexus/caller/HelloCallerWorkflowImpl.java](https://github.com/temporalio/samples-java/blob/nexus-snip-sync/core/src/main/java/io/temporal/samples/nexus/caller/HelloCallerWorkflowImpl.java)

```java
package io.temporal.samples.nexus.caller;

import io.temporal.samples.nexus.service.SampleNexusService;
import io.temporal.workflow.NexusOperationHandle;
import io.temporal.workflow.NexusOperationOptions;
import io.temporal.workflow.NexusServiceOptions;
import io.temporal.workflow.Workflow;
import java.time.Duration;

public class HelloCallerWorkflowImpl implements HelloCallerWorkflow {
  SampleNexusService sampleNexusService =
      Workflow.newNexusServiceStub(
          SampleNexusService.class,
          NexusServiceOptions.newBuilder()
              .setOperationOptions(
                  NexusOperationOptions.newBuilder()
                      .setScheduleToCloseTimeout(Duration.ofSeconds(10))
                      .build())
              .build());

  @Override
  public String hello(String message, SampleNexusService.Language language) {
    NexusOperationHandle<SampleNexusService.HelloOutput> handle =
        Workflow.startNexusOperation(
            sampleNexusService::hello, new SampleNexusService.HelloInput(message, language));
    // Optionally wait for the operation to be started. NexusOperationExecution will contain the
    // operation token in case this operation is asynchronous.
    handle.getExecution().get();
    return handle.getResult().get().getMessage();
  }
}
```

```java
package io.temporal.samples.nexus.caller;

import io.temporal.client.WorkflowClient;
import io.temporal.samples.nexus.options.ClientOptions;

        WorkflowImplementationOptions.newBuilder()

                    "SampleNexusService",
```

<a id="develop-a-starter-to-start-the-caller-workflow"></a>
### 开发一个starter来启动调用者Workflow {#develop-a-starter-to-start-the-caller-workflow}

为了启动调用者 Workflow，需要使用启动程序。

<!--SNIPSTART samples-java-Nexus-caller-starter-->

[核心/src/main/java/io/temporal/samples/nexus/caller/CallerStarter.java](https://github.com/temporalio/samples-java/blob/nexus-snip-sync/core/src/main/java/io/temporal/samples/nexus/caller/CallerStarter.java)

```java
package io.temporal.samples.nexus.caller;

import io.temporal.api.common.v1.WorkflowExecution;
import io.temporal.client.WorkflowClient;
import io.temporal.client.WorkflowOptions;
import io.temporal.samples.nexus.options.ClientOptions;
import io.temporal.samples.nexus.service.SampleNexusService;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

public class CallerStarter {
  private static final Logger logger = LoggerFactory.getLogger(CallerStarter.class);

  public static void main(String[] args) {
    WorkflowClient client = ClientOptions.getWorkflowClient(args);

    WorkflowOptions workflowOptions =
        WorkflowOptions.newBuilder().setTaskQueue(CallerWorker.DEFAULT_TASK_QUEUE_NAME).build();
    EchoCallerWorkflow echoWorkflow =
        client.newWorkflowStub(EchoCallerWorkflow.class, workflowOptions);
    WorkflowExecution execution = WorkflowClient.start(echoWorkflow::echo, "Nexus Echo 👋");
    logger.info(
        "Started EchoCallerWorkflow workflowId: {} runId: {}",
        execution.getWorkflowId(),
        execution.getRunId());
    logger.info("Workflow result: {}", echoWorkflow.echo("Nexus Echo 👋"));
    HelloCallerWorkflow helloWorkflow =
        client.newWorkflowStub(HelloCallerWorkflow.class, workflowOptions);
    execution = WorkflowClient.start(helloWorkflow::hello, "Nexus", SampleNexusService.Language.EN);
    logger.info(
        "Started HelloCallerWorkflow workflowId: {} runId: {}",
        execution.getWorkflowId(),
        execution.getRunId());
    logger.info("Workflow result: {}", helloWorkflow.hello("Nexus", SampleNexusService.Language.ES));
  }
}
```

## 使用开发服务器在 Namespace 之间进行 Nexus 调用 {#nexus-calls-across-namespaces-dev-server}

按照以下步骤运行 Nexus 处理程序 Worker、Nexus 调用程序 Worker 和入门应用程序。

<a id="run-workers-connected-to-a-local-development-server"></a>
### 运行连接到本地开发服务器的 Workers {#run-workers-connected-to-a-local-development-server}

运行 Nexus 处理程序 Worker：


<a id="canceling-a-nexus-operation"></a>
### 取消 Nexus 操作 {#canceling-a-nexus-operation}

要从 Workflow 中取消 Nexus 操作，请使用以下命令创建 `CancellationScope`
`Workflow.newCancellationScope` API。 `Workflow.newCancellationScope` 采用 `Runnable`。在此启动的任何 SDK 方法
可运行的操作（例如 Nexus 操作）将与此范围关联。 `Workflow.newCancellationScope` 返回一个新的
当调用 `cancel()` 方法时，取消上下文以及在该范围内启动的任何 SDK 方法。
`Workflow.startNexusOperation`返回的promise在操作完成时解析，无论是否成功，
失败、超时或被取消。

在 Nexus 中只能取消异步操作，因为取消是使用操作令牌发送的。 Workflow 或
支持该操作的其他资源可以选择忽略取消请求。如果忽略，操作可能会进入
终端状态。

当 Nexus 操作启动时，调用者可以指定不同的取消类型，以控制调用者如何
对取消的反应：

- `ABANDON` - 不请求取消操作。
- `TRY_CANCEL` - 发起取消请求并立即向呼叫者报告取消。注意这个类型
  如果调用者在传递之前退出，则不保证取消会传递给操作处理程序
  完成了。
- `WAIT_REQUESTED` - 请求取消操作并等待收到请求的确认。
  不等待实际取消。
- `WAIT_COMPLETED` - 等待操作完成。操作可能会或可能不会按取消的方式完成。

默认值为 `WAIT_COMPLETED`。用户可以通过调用在 `NexusServiceOptions` 上设置不同的选项
`.setCancellationType()` 上 `NexusServiceOptions.Builder`。

一旦调用者 Workflow 完成，调用者的 Nexus 机器将停止尝试取消尚未执行的操作
被取消，让它们运行完成。

在某些用例中让操作保持运行是可以的。为确保取消订单已交付，请等待所有待处理的订单
在退出 Workflow 之前发送取消请求的操作。

请参阅
[Nexus 取消样品](https://github.com/temporalio/samples-java/tree/main/core/src/main/java/io/temporal/samples/nexuscancellation)
供参考。

<a id="nexus-calls-across-namespaces-temporal-cloud"></a>
## 在 Temporal Cloud 中的 Namespace 之间进行 Nexus 调用 {#nexus-calls-across-namespaces-temporal-cloud}

本节假设您已经熟悉
[如何将 Worker 连接到 Temporal Cloud](/develop/java/client/temporal-client#start-workflow-execution)。一样的
本节中使用 [源代码](https://github.com/temporalio/samples-go/tree/main/nexus)，但 `tcld` CLI 将
用于创建 Namespace 和 Nexus 端点，mTLS 客户端证书将用于安全连接
调用者和处理程序 Worker 到其各自的 Temporal Cloud Namespace。

<a id="install-the-latest-tcld-cli-and-generate-certificates"></a>
### 安装最新的`tcld` CLI并生成证书 {#install-the-latest-tcld-cli-and-generate-certificates}

要安装最新版本的 `tcld` CLI，请运行以下命令（在 MacOS 上）：

```
brew install temporalio/brew/tcld
```

如果您还没有证书，可以使用以下命令生成证书以进行 mTLS Worker 身份验证：

```
tcld gen ca --org $YOUR_ORG_NAME --validity-period 1y --ca-cert ca.pem --ca-key ca.key
```

这些证书的有效期为一年。

<a id="create-caller-and-handler-namespaces"></a>
### 创建调用者和处理程序 Namespaces {#create-caller-and-handler-namespaces}

在部署到 Temporal Cloud 之前，请确保为调用方和处理程序创建适当的 Namespace。
如果您已经拥有这些 Namespace，则无需执行此操作。

```
tcld login

tcld namespace create \
	--namespace <your-caller-namespace> \
	--cloud-provider aws \
	--region us-west-2 \
	--ca-certificate-file 'path/to/your/ca.pem' \
	--retention-days 1

tcld namespace create \
	--namespace <your-target-namespace> \
	--cloud-provider aws \
	--region us-west-2 \
	--ca-certificate-file 'path/to/your/ca.pem' \
	--retention-days 1
```

或者，您可以通过 UI 创建 Namespace：
[https://cloud.Temporal.io/namespaces](https://cloud.temporal.io/namespaces)。

<a id="create-a-nexus-endpoint-to-route-requests-from-caller-to-handler"></a>
### 创建一个 Nexus 端点以将请求从调用者路由到处理程序 {#create-a-nexus-endpoint-to-route-requests-from-caller-to-handler}

要创建 Nexus 端点，您必须具有开发人员帐户角色或更高级别，并且具有 NamespaceAdmin 权限
`--target-Namespace`。

```
tcld nexus endpoint create \
  --name <my-nexus-endpoint-name> \
  --target-task-queue my-handler-task-queue \
  --target-namespace <my-target-namespace.account> \
  --allow-namespace <my-caller-namespace.account> \
  --description-file ./core/src/main/java/io/temporal/samples/nexus/service/description.md
```

`--allow-Namespace` 用于构建可以使用 Nexus 端点的调用者 Namespace 的端点白名单，如下所示
运行时访问控制中描述。

或者，您可以通过 UI 创建 Nexus 端点：
[https://cloud.Temporal.io/nexus](https://cloud.temporal.io/nexus)。

### Temporal CLI

使用 `Workflow describe` 命令显示调用者 Workflow 中待处理的 Nexus 操作以及任何附加的回调
在处理程序 Workflow 上：

```
temporal workflow describe -w <ID>
```

Nexus 事件包含在调用者的 Workflow 历史记录中：

```
temporal workflow show -w <ID>
```

对于**异步 Nexus 操作**，调用者历史记录中会报告以下内容：

- `NexusOperationScheduled`
- `NexusOperationStarted`
- `NexusOperationCompleted`

对于**同步 Nexus 操作**，调用者历史记录中会报告以下内容：

- `NexusOperationScheduled`
- `NexusOperationCompleted`

:::note

`NexusOperationStarted` 不会在同步操作的调用者历史记录中报告。

:::
