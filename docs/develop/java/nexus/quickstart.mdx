<a id="nexus-java-quickstart"></a>
# Nexus Java 快速入门 {#nexus-java-quickstart}

:::tip SUPPORT, STABILITY, and DEPENDENCY INFO

Temporal Java SDK 对 Nexus 的支持是 [一般可用](/evaluate/development-production-features/release-stages#general-availability)。

:::

[Temporal Nexus](/evaluate/nexus) 使用 Nexus 端点、Nexus 服务合同和 Nexus 操作连接 Namespace 内部和之间的 Temporal 应用程序。构建一个包装现有 Temporal Workflow 的 Nexus 服务，然后从调用者 Workflow 调用它。

:::info NEW TO NEXUS?

此页面将帮助您获得在 Java 中运行的工作示例。
要评估 Nexus 是否适合您的使用案例，请参阅 [评估指南](/evaluate/nexus)，要了解有关 Nexus 功能的更多信息，请单击 [这里](/nexus)。

:::

**先决条件：** 首先完成 [Java SDK 快速入门](/develop/java/set-up-your-local-java)。
您应该从该指南中获得 `SayHelloWorkflow`、`SayHelloWorkflowImpl`、`GreetActivities`、`GreetActivitiesImpl` 和 `SayHelloWorker`。

<a id="what-youll-build"></a>
## 你将构建什么 {#what-youll-build}

您有 `SayHelloWorkflow` 在 `default` Namespace 中运行。
在本指南结束时：

1. Nexus 服务将 `SayHelloWorkflow` 公开为操作。
2. 第二个 Namespace 将包含调用该操作的 Workflow。
3. 调用者 Workflow 将返回 `"Hello Temporal"` — 相同的结果，但跨越 Namespace。

<SetupSteps>

<SetupStep code={
<>

<CodeSnippet language="java">
{`package helloworkflow;

import io.nexusrpc.Operation;
import io.nexusrpc.Service;

@Service
public interface SayHelloNexusService {

    @Operation
    String sayHello(String name);

}`}
</CodeSnippet>

</>

<a id="1-define-the-nexus-service"></a>
## 1.定义Nexus服务 {#1-define-the-nexus-service}

创建一个名为 `SayHelloNexusService.java` 的文件，用于定义 Nexus 服务合同。

创建 Nexus 服务会在您的实现和任何调用者之间建立契约。
它在调用 Nexus 操作时提供类型安全，并确保操作处理程序履行合同。

`@Service` 注释将其声明为 Nexus 服务。
`@Operation` 注释将 `sayHello` 标记为可调用的 Nexus 操作。
`SayHelloWorkflow`返回`String`，因此运算输出类型也是`String`。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="java">
{`package helloworkflow;

import io.nexusrpc.handler.OperationHandler;
import io.nexusrpc.handler.OperationImpl;
import io.nexusrpc.handler.ServiceImpl;
import io.temporal.client.WorkflowOptions;
import io.temporal.nexus.Nexus;
import io.temporal.nexus.WorkflowRunOperation;

@ServiceImpl(service = SayHelloNexusService.class)
public class SayHelloNexusServiceImpl {

    @OperationImpl
    public OperationHandler<String, String> sayHello() {
        return WorkflowRunOperation.fromWorkflowMethod(
            (ctx, details, name) ->
                Nexus.getOperationContext()
                    .getWorkflowClient()
                    .newWorkflowStub(
                        SayHelloWorkflow.class,
                        WorkflowOptions.newBuilder()
                            .setWorkflowId("say-hello-nexus-" + details.getRequestId())
                            .build())
                    ::sayHello
        );
    }

}`}
</CodeSnippet>

</>
}>

<a id="2-define-the-nexus-operation-handlers"></a>
## 2. 定义 Nexus 操作处理程序 {#2-define-the-nexus-operation-handlers}

创建一个名为 `SayHelloNexusServiceImpl.java` 的文件，该文件实现 Nexus 操作处理程序。

操作处理程序包含调用者调用 Nexus 操作时运行的逻辑。

`@OperationImpl` 注释创建启动 `SayHelloWorkflow` 的异步 Nexus 操作。
该处理程序将 Nexus `String` 输入直接桥接到 `SayHelloWorkflow` 的 `sayHello` 方法。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="java">
{`package helloworkflow;

import io.temporal.client.WorkflowClient;
import io.temporal.serviceclient.WorkflowServiceStubs;
import io.temporal.worker.Worker;
import io.temporal.worker.WorkerFactory;

public class SayHelloWorker {

    public static void main(String[] args) {

        WorkflowServiceStubs service = WorkflowServiceStubs.newLocalServiceStubs();
        WorkflowClient client = WorkflowClient.newInstance(service);
        WorkerFactory factory = WorkerFactory.newInstance(client);

        Worker worker = factory.newWorker("my-task-queue");
        worker.registerWorkflowImplementationTypes(SayHelloWorkflowImpl.class);
        worker.registerActivitiesImplementations(new GreetActivitiesImpl());
        worker.registerNexusServiceImplementation(new SayHelloNexusServiceImpl());

        System.out.println("Starting SayHelloWorker for task queue 'my-task-queue'...");

        factory.start();

    }

}`}
</CodeSnippet>

</>
}>

<a id="3-register-the-nexus-service-handler-in-a-worker"></a>
## 3. 在 Worker 中注册 Nexus 服务处理程序 {#3-register-the-nexus-service-handler-in-a-worker}

Update 您现有的 `SayHelloWorker.java` 用于注册 Nexus 服务处理程序。

如果注册了 Nexus 服务处理程序，Worker 将仅轮询并处理传入的 Nexus 请求。
这与用于 Workflow 和活动的 Worker 概念相同。

registerNexusServiceImplementation 参数注册处理程序，以便它可以接收 Nexus 操作请求。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="java">
{`package helloworkflow;

import io.temporal.workflow.WorkflowInterface;
import io.temporal.workflow.WorkflowMethod;

@WorkflowInterface
public interface NexusCallerWorkflow {

    @WorkflowMethod
    String greetThroughNexus(String name);

}`}
</CodeSnippet>

<CodeSnippet language="java">
{`package helloworkflow;

import io.temporal.workflow.NexusOperationOptions;
import io.temporal.workflow.NexusServiceOptions;
import io.temporal.workflow.Workflow;

import java.time.Duration;

public class NexusCallerWorkflowImpl implements NexusCallerWorkflow {

    private final SayHelloNexusService nexusService = Workflow.newNexusServiceStub(
        SayHelloNexusService.class,
        NexusServiceOptions.newBuilder()
            .setOperationOptions(
                NexusOperationOptions.newBuilder()
                    .setScheduleToCloseTimeout(Duration.ofSeconds(10))
                    .build())
            .build()
    );

    @Override
    public String greetThroughNexus(String name) {
        return nexusService.sayHello(name);
    }

}`}
</CodeSnippet>

</>
}>

<a id="4-develop-the-caller-workflow"></a>
## 4.开发调用者Workflow {#4-develop-the-caller-workflow}

创建两个文件 — `NexusCallerWorkflow.java` 和 `NexusCallerWorkflowImpl.java` — 定义调用 Nexus 操作的 Workflow。

调用者Workflow演示了Nexus的消费者端。
调用者只依赖于服务合约，而不是直接导入处理程序代码。
这使调用者和处理程序保持解耦，以便他们可以生活在单独的 Namespace、存储库甚至团队中。

`Workflow.newNexusServiceStub` 方法创建绑定到 Nexus 服务和端点的客户端。

</SetupStep>

<SetupStep code={
<>

<CodeSnippet language="bash">
temporal operator namespace create --namespace my-caller-namespace
</CodeSnippet>

<CodeSnippet language="bash">
{`temporal operator nexus endpoint create \\
  --name my-nexus-endpoint-name \\
  --target-namespace default \\
  --target-task-queue my-task-queue`}
</CodeSnippet>

</>

<a id="5-create-the-caller-namespace-and-nexus-endpoint"></a>
## 5. 创建调用者 Namespace 和 Nexus 端点 {#5-create-the-caller-namespace-and-nexus-endpoint}

在运行应用程序之前，创建一个调用者 Namespace 和一个 Nexus 端点以将请求从调用者路由到处理程序。
该处理程序使用启动开发服务器时创建的 `default` Namespace。

Namespace 在调用方和处理程序方之间提供隔离。
Nexus 端点充当路由层，将调用方 Namespace 连接到处理程序的目标 Namespace 和 Task Queue。

确保您的本地 Temporal 开发服务器正在运行 (`Temporal Server start-dev`)。

</SetupStep>

<SetupStep wide code={
<>

<CodeSnippet language="java">
{`package helloworkflow;

import io.temporal.client.WorkflowClient;
import io.temporal.client.WorkflowClientOptions;
import io.temporal.client.WorkflowOptions;
import io.temporal.serviceclient.WorkflowServiceStubs;
import io.temporal.worker.Worker;
import io.temporal.worker.WorkerFactory;
import io.temporal.worker.WorkflowImplementationOptions;
import io.temporal.workflow.NexusServiceOptions;

import java.util.Collections;

public class NexusCallerStarter {

    public static void main(String[] args) {

        WorkflowServiceStubs service = WorkflowServiceStubs.newLocalServiceStubs();
        WorkflowClient client = WorkflowClient.newInstance(service,
            WorkflowClientOptions.newBuilder()
                .setNamespace("my-caller-namespace")
                .build()
        );
        WorkerFactory factory = WorkerFactory.newInstance(client);

        Worker worker = factory.newWorker("my-caller-task-queue");
        worker.registerWorkflowImplementationTypes(
            WorkflowImplementationOptions.newBuilder()
                .setNexusServiceOptions(
                    Collections.singletonMap(
                        "SayHelloNexusService",
                        NexusServiceOptions.newBuilder()
                            .setEndpoint("my-nexus-endpoint-name")
                            .build()))
                .build(),
            NexusCallerWorkflowImpl.class
        );
        factory.start();

        NexusCallerWorkflow workflow = client.newWorkflowStub(
            NexusCallerWorkflow.class,
            WorkflowOptions.newBuilder()
                .setTaskQueue("my-caller-task-queue")
                .setWorkflowId("nexus-caller-workflow-id")
                .build()
        );

        String result = workflow.greetThroughNexus("Temporal");
        System.out.println("Workflow result: " + result);

        factory.shutdown();
        System.exit(0);

    }

}`}
</CodeSnippet>

</>
}>

<a id="6-run-and-verify"></a>
## 6. 运行并验证 {#6-run-and-verify}

创建`NexusCallerStarter.java`来启动调用者Worker并执行Workflow。

这将所有内容结合在一起：调用者 Worker 托管 `NexusCallerWorkflow`，它使用 Nexus 存根在处理程序端调用 `sayHello`。
完整请求从调用者 Workflow 通过 Nexus 端点流向运行 `SayHelloWorkflow` 的处理程序 Worker，然后返回调用者。

**运行应用程序：**

1. 在一个终端中启动处理程序 Worker：

```bash
mvn compile exec:java \
  -Dexec.mainClass="helloworkflow.SayHelloWorker"
```

2. 在另一个终端中运行调用程序：

```bash
mvn compile exec:java \
  -Dexec.mainClass="helloworkflow.NexusCallerStarter"
```

你应该看到：

```
Workflow result: Hello Temporal
```

打开 [Temporal 网页用户界面](http://localhost:8233) 并在 Namespace 之间切换以查看两个 Workflow Execution。
在 `my-caller-Namespace` 中，找到 `NexusCallerWorkflow` 执行 — 您应该在其历史记录中看到 `NexusOperationScheduled`、`NexusOperationStarted` 和 `NexusOperationCompleted` 事件。
在 `default` 中，找到由 Nexus 操作启动的 `SayHelloWorkflow` 执行。

</SetupStep>
</SetupSteps>
