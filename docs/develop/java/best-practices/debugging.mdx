除了编写单元和集成测试之外，调试 Workflow 也是一个非常有价值的测试工具。
您可以使用您最喜欢的 Java IDE 提供的调试器来调试 Workflow 代码。

请注意，调试 Workflow 代码时，Temporal Java SDK 包含死锁检测，如果代码阻塞超过一秒而不放弃执行控制，则死锁检测会使 Workflow 任务失败。
因此，在调试期间单步执行 Workflow 代码时，您经常会遇到 `PotentialDeadlockException` 异常。

为了缓解此问题，您可以在调试 Workflow 代码之前将 `Temporal_DEBUG` 环境变量设置为 true。确保仅在调试期间将 `Temporal_DEBUG` 设置为 true。

## 如何在开发环境中调试 {#debug-in-a-development-environment}

除了日志记录和调试器等常规开发工具之外，您还可以使用 [网页用户界面](/web-ui) 或 [Temporal CLI](/cli) 查看 Workflow 中发生的情况。

<a id="debug-in-a-production-environment"></a>
## 如何在生产环境中调试 {#debug-in-a-production-environment}

您可以使用以下方法调试生产 Workflow：

- [网页用户界面](/web-ui)
- [Temporal CLI](/cli)
- [重播](/develop/java/best-practices/testing-suite#replay)
- [追踪](/develop/java/platform/observability#tracing)
- [记录](/develop/java/platform/observability#logging)

您可以使用指标和 [Worker 性能指南](/develop/worker-performance) 来调试和调整 Worker 性能。
有关更多信息，请参阅 [可观察性 ▶️ 指标](/develop/java/platform/observability#metrics) 设置 SDK 指标。

使用 [云指标](/cloud/metrics/) 或 [自托管服务器指标](/self-hosted-guide/production-checklist#scaling-and-metrics) 调试服务器性能。
