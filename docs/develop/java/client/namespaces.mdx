- [注册一个 Namespace](#register-namespace)
- [管理 Namespace](#manage-namespaces)

您可以使用 Temporal CLI 或 SDK API 创建 Update、弃用或删除 [Namespaces](/namespaces)。

根据您的需要，使用 Namespace 隔离您的 Workflow Execution。
例如，您可以使用 Namespace 通过单独的 `dev` 和 `prod` Namespace 来匹配开发生命周期。
您还可以使用它们来确保不同团队之间的 Workflow Execution 永远不会进行通信 - 例如确保 `teamA` Namespace 永远不会影响 `teamB` Namespace。

在 Temporal Cloud 上，使用 [Temporal Cloud 用户界面](/cloud/namespaces#create-a-namespace) 从 UI 创建和管理 Namespace，或使用 [tcld 命令](https://docs.temporal.io/cloud/tcld/namespace/) 从命令行界面管理 Namespace。

在自托管的 Temporal 服务上，您可以使用 Temporal CLI（推荐）或以编程方式使用 API 来注册和管理您的 Namespace。
请注意，这些 API 和 Temporal CLI 命令不适用于 Temporal Cloud。

在 Temporal 服务中的前端服务上使用自定义 [授权人](/self-hosted-guide/security#authorizer-plugin) 来设置对谁可以创建 Update 或弃用 Namespace 的限制。

您必须先向 Temporal 服务注册 Namespace，然后再在 Temporal CLIent 中设置它。

<a id="register-namespace"></a>
## 注册一个 Namespace {#register-namespace}

注册 Namespace 会在 Temporal 服务或 Temporal Cloud 上创建 Namespace。

在 Temporal Cloud 上，使用 [Temporal Cloud 用户界面](/cloud/namespaces#create-a-namespace) 或 [tcld 命令](https://docs.temporal.io/cloud/tcld/namespace/) 创建 Namespace。

在自托管 Temporal 服务上，您可以使用 Temporal CLI（推荐）或以编程方式使用 API 注册您的 Namespace。
请注意，这些 API 和 Temporal CLI 命令不适用于 Temporal Cloud。

在 Temporal 服务中的前端服务上使用自定义 [授权人](/self-hosted-guide/security#authorizer-plugin) 来设置对谁可以创建 Update 或弃用 Namespace 的限制。

使用 [`RegisterNamespace` API](https://github.com/temporalio/api/blob/f0350f8032ad2f0c60c539b3b61ea37f412f1cf7/temporal/api/workflowservice/v1/service.proto) 注册 [Namespace](/namespaces)，并将 [保留期限](/temporal-service/temporal-server#retention-period) 设置为 Workflow Execution，将 Event History 设置为 Namespace。

```java
//...
import com.google.protobuf.util.Durations;
import io.temporal.api.workflowservice.v1.RegisterNamespaceRequest;
//...
public static void createNamespace(String name) {
    RegisterNamespaceRequest req = RegisterNamespaceRequest.newBuilder()
            .setNamespace("your-custom-namespace")
            .setWorkflowExecutionRetentionPeriod(Durations.fromDays(3)) // keeps the Workflow Execution
            //Event History for up to 3 days in the Persistence store. Not setting this value will throw an error.
            .build();
    service.blockingStub().registerNamespace(req);
}
//...
```

使用 `WorkflowExecutionRetentionPeriod` 的保留期设置是强制性的。
您可以为此时间段设置的最小值为 1 天。

注册后，在 Workflow 客户端中使用 `WorkflowClientOptions` 设置 Namespace，以在该 Namespace 中运行 Workflow Execution。
详细信息请参见 [连接到开发 Temporal 服务](/develop/java/client/temporal-client#connect-to-development-service)。

请注意，使用此 API 进行 Namespace 注册最多需要 10 秒才能完成。
确保在针对 Namespace 启动 Workflow Execution 之前等待此注册完成。

对于 Update，您的 Namespace 将 [UpdateNamespace API](#manage-namespaces) 与 NamespaceClient 结合使用。

<a id="manage-namespaces"></a>
## 管理 Namespace {#manage-namespaces}

您可以获取 Namespace、Update Namespace 配置的详细信息，并弃用或删除 Namespace。

在Temporal Cloud上，使用[Temporal Cloud 用户界面](/cloud/namespaces#create-a-namespace)或[tcld 命令](https://docs.temporal.io/cloud/tcld/namespace/)来管理Namespace。

在自托管的 Temporal 服务上，您可以使用 Temporal CLI（推荐）或以编程方式使用 API 来管理注册的 Namespace。
请注意，这些 API 和 Temporal CLI 命令不适用于 Temporal Cloud。

在 Temporal 服务中的前端服务上使用自定义 [授权人](/self-hosted-guide/security#authorizer-plugin) 来设置对谁可以创建 Update 或弃用 Namespace 的限制。

您必须先向 Temporal 服务注册 Namespace，然后再在 Temporal CLIent 中设置它。

在Temporal Cloud上，使用[Temporal Cloud 用户界面](/cloud/namespaces)或[tcld 命令](https://docs.temporal.io/cloud/tcld/namespace/)来管理Namespace。

在自托管的 Temporal 服务上，您可以使用 Temporal CLI（推荐）或以编程方式使用 API 来管理注册的 Namespace。
请注意，这些 API 和 Temporal CLI 命令不适用于 Temporal Cloud。

- 在您的 Temporal 服务上注册的 Namespace 的 Update 信息和配置：

- 使用 Temporal CLI：[`Temporal operator Namespace Update`](/cli/operator#update)
    示例
  - 在 Namespace 上使用 [`UpdateNamespace` API](https://github.com/temporalio/api/blob/e5cf521c6fdc71c69353f3d2ac5506dd6e827af8/temporal/api/workflowservice/v1/service.proto) 至 Update 配置。
    例子

  ```java
  import io.temporal.api.workflowservice.v1.*;
  //...
  UpdateNamespaceRequest updateNamespaceRequest = UpdateNamespaceRequest.newBuilder()
              .setNamespace("your-namespace-name") //the namespace that you want to update
              .setUpdateInfo(UpdateNamespaceInfo.newBuilder() //has options to update namespace info
                      .setDescription("your updated namespace description") //updates description in the namespace info.
                      .build())
              .setConfig(NamespaceConfig.newBuilder() //has options to update namespace configuration
                      .setWorkflowExecutionRetentionTtl(Durations.fromHours(30)) //updates the retention period for the namespace "your-namespace--name" to 30 hrs.
                      .build())
              .build();
      UpdateNamespaceResponse updateNamespaceResponse = namespaceservice.blockingStub().updateNamespace(updateNamespaceRequest);
  //...
  ```

- 获取在 Temporal 服务上注册的 Namespace 的详细信息：

- 使用 Temporal CLI：[`Temporal operator Namespace describe`](/cli/operator#describe)
  - 使用 [`DescribeNamespace` API](https://github.com/temporalio/api/blob/e5cf521c6fdc71c69353f3d2ac5506dd6e827af8/temporal/api/workflowservice/v1/service.proto) 返回已注册 Namespace 的信息和配置详细信息。
    例子

  ```java
  import io.temporal.api.workflowservice.v1.*;
  //...
  DescribeNamespaceRequest descNamespace = DescribeNamespaceRequest.newBuilder()
              .setNamespace("your-namespace-name") //specify the namespace you want details for
              .build();
      DescribeNamespaceResponse describeNamespaceResponse = namespaceservice.blockingStub().describeNamespace(descNamespace);
      System.out.println("Namespace Description: " + describeNamespaceResponse);
  //...
  ```

- 获取 Temporal 服务上所有已注册 Namespace 的详细信息：

- 使用 Temporal CLI：[`Temporal operator Namespace list`](/cli/operator#list)
  - 使用 [`ListNamespace` API](https://github.com/temporalio/api/blob/e5cf521c6fdc71c69353f3d2ac5506dd6e827af8/temporal/api/workflowservice/v1/service.proto) 返回 Temporal 服务上所有已注册 Namespace 的信息和配置详细信息。
    例子

  ```java
  import io.temporal.api.workflowservice.v1.*;
  //...
  ListNamespacesRequest listNamespaces = ListNamespacesRequest.newBuilder().build();
      ListNamespacesResponse listNamespacesResponse = namespaceservice.blockingStub().listNamespaces(listNamespaces); //lists 1-100 namespaces (1 page) in the active Temporal Service. To list all, set the page size or loop until NextPageToken is nil.
  //...
  ```

- 弃用 Namespace：[`DeprecateNamespace` API](https://github.com/temporalio/api/blob/e5cf521c6fdc71c69353f3d2ac5506dd6e827af8/temporal/api/workflowservice/v1/service.proto) Update 将注册的 Namespace 的状态更改为“已弃用”。一旦 Namespace 被弃用，您就无法在其上启动新的 Workflow Execution。已弃用的 Namespace 上的所有现有和正在运行的 Workflow Execution 将继续运行。
  例子：

  ```java
  import io.temporal.api.workflowservice.v1.*;
  //...
  DeprecateNamespaceRequest deprecateNamespace = DeprecateNamespaceRequest.newBuilder()
              .setNamespace("your-namespace-name") //specify the namespace that you want to deprecate
              .build();
      DeprecateNamespaceResponse response = namespaceservice.blockingStub().deprecateNamespace(deprecateNamespace);
  //...
  ```

- 删除 Namespace：[`DeleteNamespace` API](https://github.com/temporalio/api/blob/e5cf521c6fdc71c69353f3d2ac5506dd6e827af8/temporal/api/workflowservice/v1/service.proto) 删除 Namespace。删除 Namespace 会删除 Namespace 上所有正在运行和已完成的 Workflow Execution，并将它们从持久性存储和 Visibility 存储中删除。

例子：

  ```java
  //...
  DeleteNamespaceResponse res =
  OperatorServiceStubs.newServiceStubs(OperatorServiceStubsOptions.newBuilder()
          .setChannel(service.getRawChannel())
          .validateAndBuildWithDefaults())
      .blockingStub()
      .deleteNamespace(DeleteNamespaceRequest.newBuilder().setNamespace("your-namespace-name").build());
  //...
  ```
