<a id="activity-timeouts"></a>
## Activity 超时 {#activity-timeouts}

每个 Activity 超时控制 Activity 执行不同方面的最大持续时间。

Activity 选项中提供以下超时。

- **[Schedule-关闭超时](/encyclopedia/detecting-activity-failures#schedule-to-close-timeout)：** 是整个 [Activity 执行](/activity-execution) 允许的最大时间量。
- **[开始到关闭超时](/encyclopedia/detecting-activity-failures#start-to-close-timeout)：** 是单个 [Activity 任务执行](/tasks#activity-task-execution) 允许的最长时间。
- **[Schedule-启动超时](/encyclopedia/detecting-activity-failures#schedule-to-start-timeout)：** 是从 [Activity 任务](/tasks#activity-task) 为 Scheduled 到 [Worker](/workers#worker) 启动该 Activity 任务所允许的最长时间。

Activity 执行必须设置“开始到关闭”或“Schedule 到关闭超时”。

设置 [`ActivityOptions.Builder`](https://www.javadoc.io/doc/io.temporal/temporal-sdk/latest/io/temporal/activity/ActivityOptions.Builder.html) 类的 Activity 超时。

可用的超时有：

- ScheduleToCloseTimeout()
- ScheduleToStartTimeout()
- 开始到关闭超时()

您可以在 Workflow 实现中使用 `ActivityStub` 设置 Activity 选项，或者在 Worker 中使用 `WorkflowImplementationOptions` 按 Activity 设置 Activity 选项。

以下使用`ActivityStub`。

```java
GreetingActivities activities = Workflow.newActivityStub(GreetingActivities.class,
    ActivityOptions.newBuilder()
        .setScheduleToCloseTimeout(Duration.ofSeconds(5))
        // .setStartToCloseTimeout(Duration.ofSeconds(2)
        // .setScheduletoCloseTimeout(Duration.ofSeconds(20))
        .build());
```

以下使用`WorkflowImplementationOptions`。

```java
WorkflowImplementationOptions options =
    WorkflowImplementationOptions.newBuilder()
        .setActivityOptions(
            ImmutableMap.of(
                "GetCustomerGreeting",
                // Set Activity Execution timeout
                ActivityOptions.newBuilder()
                    .setScheduleToCloseTimeout(Duration.ofSeconds(5))
                    // .setStartToCloseTimeout(Duration.ofSeconds(2))
                    // .setScheduleToStartTimeout(Duration.ofSeconds(5))
                    .build()))
        .build();
```

:::note

如果您使用 `WorkflowImplementationOptions.setActivityOptions()` 定义每个 Activity 类型选项，则在 Workflow 中使用 `ActivityStub` 再次专门设置它们将覆盖此设置。

:::

<a id="activity-retries"></a>
### 自定义 Activity Retry Policy {#activity-retries}

Retry Policy 与超时配合提供精细控制以优化执行体验。

Activity 如果未提供自定义执行，则执行会自动与默认 [Retry Policy](/encyclopedia/retry-policies) 关联。

要设置 Retry Policy（在 Java 中称为 [重试选项](/encyclopedia/retry-policies)），请使用 [`ActivityOptions.newBuilder.setRetryOptions()`](https://www.javadoc.io/doc/io.temporal/temporal-sdk/latest/io/temporal/activity/ActivityOptions.Builder.html)。

- 型号：`RetryOptions`
- 默认值：服务器定义的 Activity Retry Policy。

- 与`ActivityStub`

  ```java
  private final ActivityOptions options =
      ActivityOptions.newBuilder()
          // note that either StartToCloseTimeout or ScheduleToCloseTimeout are
          // required when setting Activity options.
          .setStartToCloseTimeout(Duration.ofSeconds(5))
          .setRetryOptions(
              RetryOptions.newBuilder()
                  .setInitialInterval(Duration.ofSeconds(1))
                  .setMaximumInterval(Duration.ofSeconds(10))
                  .build())
          .build();
  ```

- 与`WorkflowImplementationOptions`

  ```java
  WorkflowImplementationOptions options =
        WorkflowImplementationOptions.newBuilder()
            .setActivityOptions(
                ImmutableMap.of(
                    "EmailCustomerGreeting",
                    ActivityOptions.newBuilder()
                        // note that either StartToCloseTimeout or ScheduleToCloseTimeout are
                        // required when setting Activity options.
                        .setStartToCloseTimeout(Duration.ofSeconds(5))
                        .setRetryOptions(
                            RetryOptions.newBuilder()
                                .setDoNotRetry(NullPointerException.class.getName())
                                .build())
                        .build()))
            .build();
  ```

<a id="activity-next-retry-delay"></a>
## Activity 下一次重试延迟 {#activity-next-retry-delay}

您可以投掷 [`ApplicationFailure`](/references/failures#application-failure) 并设置 `NextRetryDelay` 字段。该值将替换并覆盖 Retry Policy 上的任何重试间隔。

例如，如果在 Activity 中，您希望间隔基于尝试次数，您可以执行以下操作：

```java
int attempt = Activity.getExecutionContext().getInfo().getAttempt();

throw ApplicationFailure.newFailureWithCauseAndDelay(
    "Something bad happened on attempt " + attempt,
    "my_failure_type",
    null,
    3 * Duration.ofSeconds(attempt));
```

<a id="activity-heartbeats"></a>
## 心跳 Activity {#activity-heartbeats}

[Activity 心跳](/encyclopedia/detecting-activity-failures#activity-heartbeat) 是从正在执行 Activity 的 [Worker 流程](/workers#worker-process) 到 [Temporal 服务](/temporal-service) 的 ping。
每个心跳都会通知 Temporal 服务 [Activity 执行](/activity-execution) 正在取得进展并且 Worker 尚未崩溃。
如果Temporal服务在[心跳超时](/encyclopedia/detecting-activity-failures#heartbeat-timeout)时间段内没有收到Heartbeat，则Activity将被视为失败，并且根据Retry Policy，另一个[Activity 任务执行](/tasks#activity-task-execution)可能是Scheduled。

心跳可能并不总是发送到 Temporal 服务 — 它们可能是由 Worker 发送的 [节流](/encyclopedia/detecting-activity-failures#throttling)。

Activity 取消会在活动检测到时从 Temporal 服务传递到活动。没有心跳的活动无法收到取消。
心跳限制可能会导致取消的发送时间晚于预期。

心跳可以包含描述 Activity 当前进度的 `details` 字段。
如果重试 Activity，则 Activity 可以从发送到 Temporal 服务的最后一个心跳访问 `details`。

要在 Java 中对 Activity 执行进行检测，请使用 `Activity.getExecutionContext().heartbeat()` 类方法。

```java
public class YourActivityDefinitionImpl implements YourActivityDefinition {

  @Override
  public String yourActivityMethod(YourActivityMethodParam param) {
    // ...
    Activity.getExecutionContext().heartbeat(details);
    // ...
  }
  // ...
}
```

该方法采用一个可选参数，即上面的 `details` 变量，它表示 Activity 执行的最新进度。
此方法可以采用多种类型，例如异常对象、自定义对象或字符串。

如果Activity执行超时，则最后一个心跳`details`将包含在抛出的`ActivityTimeoutException`中，可以被调用Workflow捕获。
然后，如果需要，Workflow 可以使用 `details` 信息传递到下一个 Activity 调用。

在 Activity 重试的情况下，最后一次心跳的 `details` 可用，并且可以使用 `Activity.getExecutionContext().getHeartbeatDetails(Class<V> detailsClass)` 从上次失败的尝试中提取

### 心跳超时 {#heartbeat-timeout}

[心跳超时](/encyclopedia/detecting-activity-failures#heartbeat-timeout) 与 [Activity 心跳](/encyclopedia/detecting-activity-failures#activity-heartbeat) 配合使用。

要设置 [心跳超时](/encyclopedia/detecting-activity-failures#heartbeat-timeout)，请使用 [`ActivityOptions.newBuilder.setHeartbeatTimeout`](https://www.javadoc.io/doc/io.temporal/temporal-sdk/latest/io/temporal/activity/ActivityOptions.Builder.html)。

- 型号：`Duration`
- 默认：无

您可以在 Workflow 实现中使用 `ActivityStub` 设置 Activity 选项，或者在 Worker 中使用 `WorkflowImplementationOptions` 按 Activity 设置 Activity 选项。
请注意，如果您使用 `WorkflowImplementationOptions.setActivityOptions()` 定义每个 Activity 类型选项，则在 Workflow 中使用 `ActivityStub` 再次专门设置它们将覆盖此设置。

- 与`ActivityStub`

  ```java
  private final GreetingActivities activities =
      Workflow.newActivityStub(
          GreetingActivities.class,
          ActivityOptions.newBuilder()
              // note that either StartToCloseTimeout or ScheduleToCloseTimeout are
              // required when setting Activity options.
              .setStartToCloseTimeout(Duration.ofSeconds(5))
              .setHeartbeatTimeout(Duration.ofSeconds(2))
              .build());
  ```

- 与`WorkflowImplementationOptions`

  ```java
  WorkflowImplementationOptions options =
        WorkflowImplementationOptions.newBuilder()
            .setActivityOptions(
                ImmutableMap.of(
                "EmailCustomerGreeting",
                    ActivityOptions.newBuilder()
                        // note that either StartToCloseTimeout or ScheduleToCloseTimeout are
                        // required when setting Activity options.
                        .setStartToCloseTimeout(Duration.ofSeconds(5))
                        .setHeartbeatTimeout(Duration.ofSeconds(2))
                        .build()))
            .build();
  ```
