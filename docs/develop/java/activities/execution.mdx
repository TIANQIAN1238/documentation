<a id="activity-execution"></a>
## 启动 Activity 执行 {#activity-execution}

对生成 [Activity 执行](/activity-execution) 的调用写入 [Workflow Definition](/workflow-definition) 中。
调用生成 Activity 执行会生成 [ScheduleActivity任务](/references/commands#scheduleactivitytask) 命令。
这会在 Workflow Execution Event History 中产生一组三个 [Activity 任务](/tasks#activity-task) 相关事件（[Activity任务Scheduled](/references/events#activitytaskscheduled)、[Activity任务已开始](/references/events#activitytaskstarted) 和 ActivityTask[Closed]）。

活动实现的单个实例在多个同时的 Activity 调用之间共享。
Activity 实现代码应该是_幂等的_。

通过调用参数传递给活动的值或通过结果值返回的值都记录在执行历史记录中。
当 Workflow 状态需要恢复时，整个执行历史记录将从 Temporal 服务传输到 Workflow Worker。
因此，大量执行历史记录可能会对 Workflow 的性能产生不利影响。

因此，请注意通过 Activity 调用参数或返回值传输的数据量。
否则，Activity 实现不存在其他限制。

活动是远程过程调用，必须使用 `ActivityStub` 从 Workflow 内调用。
活动本身不能执行。您无法自行启动 Activity 执行。

请注意，在调用 Activity 执行之前：

- 必须为 Activity 设置 Activity 选项（需要 [`setStartToCloseTimeout`](/encyclopedia/detecting-activity-failures#start-to-close-timeout) 或 [`ScheduleToCloseTimeout`](/encyclopedia/detecting-activity-failures#schedule-to-close-timeout)）。
  详细信息请参见[如何设置 Activity 超时](/develop/java/activities/timeouts)。
- Activity 必须使用 Worker 注册。
  参见 [Worker 程序](/develop/java/workers/run-worker-process#run-a-dev-worker)
- Activity 代码必须是线程安全的。

活动只能使用 Workflow 中的存根进行实例化。
`ActivityStub` 返回实现 Activity 接口的客户端存根。
您可以使用 `Workflow.newActivityStub`（类型安全）或 `Workflow.newUntypedActivityStub`（无类型）调用活动。

调用 Activity 接口 Schedule 上的方法会通过 Temporal 服务进行 Activity 调用，并生成 [`ActivityTaskScheduled` 事件](/references/events#activitytaskscheduled)。

活动可以同步或异步调用。

<a id="invoking-activities-synchronously"></a>
### 同步调用活动 {#invoking-activities-synchronously}

在以下示例中，我们在“FileProcessingWorkflow”Workflow 实现中使用类型安全的 `Workflow.newActivityStub` 来创建 `FileProcessingActivities` 类的客户端存根。我们还定义了 `ActivityOptions` 并将 `setStartToCloseTimeout` 选项设置为一小时。

```java
public class FileProcessingWorkflowImpl implements FileProcessingWorkflow {

    private final FileProcessingActivities activities;

    public FileProcessingWorkflowImpl() {
        this.activities = Workflow.newActivityStub(
                FileProcessingActivities.class,
                ActivityOptions.newBuilder()
                        .setStartToCloseTimeout(Duration.ofHours(1))
                        .build());
    }

    @Override
    public void processFile(Arguments args) {
        String localName = null;
        String processedName = null;
        try {
            localName = activities.download(args.getSourceBucketName(), args.getSourceFilename());
            processedName = activities.processFile(localName);
            activities.upload(args.getTargetBucketName(), args.getTargetFilename(), processedName);
        } finally {
            if (localName != null) {
                activities.deleteLocalFile(localName);
            }
            if (processedName != null) {
                activities.deleteLocalFile(processedName);
            }
        }
    }
    // ...
}
```

一个 Workflow 可以有多个 Activity 存根。每个 Activity 存根都可以定义自己的 `ActivityOptions`。
以下示例显示了具有两个类型化 Activity 存根的 Workflow 实现。

```java
public FileProcessingWorkflowImpl() {
    ActivityOptions options1 = ActivityOptions.newBuilder()
             .setTaskQueue("taskQueue1")
             .setStartToCloseTimeout(Duration.ofMinutes(10))
             .build();
    this.store1 = Workflow.newActivityStub(FileProcessingActivities.class, options1);

    ActivityOptions options2 = ActivityOptions.newBuilder()
             .setTaskQueue("taskQueue2")
             .setStartToCloseTimeout(Duration.ofMinutes(5))
             .build();
    this.store2 = Workflow.newActivityStub(FileProcessingActivities.class, options2);
}
```

要调用 Workflow 内的活动而不引用其实现的接口，请使用非类型化的 Activity 存根 `Workflow.newUntypedActivityStub`。
当编译时未知 Activity 类型或调用以不同编程语言实现的活动时，这非常有用。

```java
   // Workflow code
    ActivityOptions activityOptions =
        ActivityOptions.newBuilder()
        .setStartToCloseTimeout(Duration.ofSeconds(3))
        .setTaskQueue("simple-queue-node")
        .build();

    ActivityStub activity = Workflow.newUntypedActivityStub(activityOptions);
    activity.execute("ComposeGreeting", String.class, "Hello World", "Spanish");
```

<a id="invoking-activities-asynchronously"></a>
### 异步调用活动 {#invoking-activities-asynchronously}

有时，Workflow 需要并行执行某些操作。
Temporal Java SDK 提供 `Async` 类，其中包括用于异步调用任何 Activity 的静态方法。
这些调用返回 `Promise` 类型的结果，该结果类似于 Java `Future` 和 `CompletionStage`。
调用活动时，对于返回结果的活动使用 `Async.function`，对于返回 void 的活动使用 `Async.procedure`。

在以下异步 Activity 调用中，方法引用传递给 `Async.function`，后跟 Activity 参数。

```java
Promise<String> localNamePromise = Async.function(activities::download, sourceBucket, sourceFile);
```

以下示例演示如何在多个文件上并行调用两个 Activity 方法“下载”和“上传”。

```java
  public void processFile(Arguments args) {
    List<Promise<String>> localNamePromises = new ArrayList<>();
    List<String> processedNames = null;
    try {
      // Download all files in parallel.
      for (String sourceFilename : args.getSourceFilenames()) {
        Promise<String> localName =
            Async.function(activities::download, args.getSourceBucketName(), sourceFilename);
        localNamePromises.add(localName);
      }
      List<String> localNames = new ArrayList<>();
      for (Promise<String> localName : localNamePromises) {
        localNames.add(localName.get());
      }
      processedNames = activities.processFiles(localNames);

      // Upload all results in parallel.
      List<Promise<Void>> uploadedList = new ArrayList<>();
      for (String processedName : processedNames) {
        Promise<Void> uploaded =
            Async.procedure(
                activities::upload,
                args.getTargetBucketName(),
                args.getTargetFilename(),
                processedName);
        uploadedList.add(uploaded);
      }
      // Wait for all uploads to complete.
      Promise.allOf(uploadedList).get();
    } finally {
      for (Promise<String> localNamePromise : localNamePromises) {
        // Skip files that haven't completed downloading.
        if (localNamePromise.isCompleted()) {
          activities.deleteLocalFile(localNamePromise.get());
        }
      }
      if (processedNames != null) {
        for (String processedName : processedNames) {
          activities.deleteLocalFile(processedName);
        }
      }
    }
  }
```

<a id="activity-execution-context"></a>
### Activity 执行上下文 {#activity-execution-context}

`ActivityExecutionContext` 是默认传递给每个 Activity 实现的上下文对象。
您可以通过 `Activity.getExecutionContext()` 在 Activity 实现中访问它。

它提供 getter 来访问有关调用 Activity 的 Workflow 的信息。
请注意，Activity 上下文信息存储在线程局部变量中。
因此，对 `getExecutionContext()` 的调用仅在调用 Activity 函数的线程内成功。

以下是使用 `ActivityExecutionContext` 的示例：

```java
public class FileProcessingActivitiesImpl implements FileProcessingActivities {

  @Override
  public String download(String bucketName, String remoteName, String localName) {

    ActivityExecutionContext ctx = Activity.getExecutionContext();
    ActivityInfo info = ctx.getInfo();

    log.info("namespace=" +  info.getActivityNamespace());
    log.info("workflowId=" + info.getWorkflowId());
    log.info("runId=" + info.getRunId());
    log.info("activityId=" + info.getActivityId());
    log.info("activityTimeout=" + info.getStartToCloseTimeout();

    return downloadFileFromS3(bucketName, remoteName, localDirectory + localName);
  }
    ...
}
```

有关获取 Activity 执行结果的详细信息，请参阅 [Activity 执行结果](#activity-execution-result)。

## 设置所需的 Activity 超时 {#required-timeout}

Activity 执行语义依赖于多个参数。
唯一需要设置的值是 [Schedule-关闭超时](/encyclopedia/detecting-activity-failures#schedule-to-close-timeout) 或 [开始到关闭超时](/encyclopedia/detecting-activity-failures#start-to-close-timeout)。
这些值在 Activity 选项中设置。

设置 [`ActivityOptions.Builder`](https://www.javadoc.io/doc/io.temporal/temporal-sdk/latest/io/temporal/activity/ActivityOptions.Builder.html) 类的 Activity 超时。

可用的超时有：

- ScheduleToCloseTimeout()
- ScheduleToStartTimeout()
- 开始到关闭超时()

您可以在 Workflow 实现中使用 `ActivityStub` 设置 Activity 选项，或者在 Worker 中使用 `WorkflowImplementationOptions` 按 Activity 设置 Activity 选项。

以下使用`ActivityStub`。

```java
GreetingActivities activities = Workflow.newActivityStub(GreetingActivities.class,
                ActivityOptions.newBuilder()
                        .setScheduleToCloseTimeout(Duration.ofSeconds(5))
                        // .setStartToCloseTimeout(Duration.ofSeconds(2)
                        // .setScheduletoCloseTimeout(Duration.ofSeconds(20))
                        .build());
```

以下使用`WorkflowImplementationOptions`。

```java
WorkflowImplementationOptions options =
            WorkflowImplementationOptions.newBuilder()
                    .setActivityOptions(
                            ImmutableMap.of(
                                    "GetCustomerGreeting",
                                    // Set Activity Execution timeout
                                    ActivityOptions.newBuilder()
                                        .setScheduleToCloseTimeout(Duration.ofSeconds(5))
                                        // .setStartToCloseTimeout(Duration.ofSeconds(2))
                                        // .setScheduleToStartTimeout(Duration.ofSeconds(5))
                                        .build()))
                    .build();
```

:::note

如果您使用 `WorkflowImplementationOptions.setActivityOptions()` 定义每个 Activity 类型选项，则在 Workflow 中使用 `ActivityStub` 再次专门设置它们将覆盖此设置。

:::

## Java ActivityOptions 参考 {#activity-options-reference}

使用 [`ActivityOptions`](https://www.javadoc.io/doc/io.temporal/temporal-sdk/latest/io/temporal/activity/ActivityOptions.Builder.html) 配置如何调用 Activity 执行。

您可以在 Workflow 实现中使用 `ActivityStub` 设置 Activity 选项，或者在 Worker 中使用 `WorkflowImplementationOptions` 按 Activity 设置 Activity 选项。
请注意，如果您使用 `WorkflowImplementationOptions.setActivityOptions()` 定义每个 Activity 类型选项，则在 Workflow 中使用 `ActivityStub` 再次专门设置它们将覆盖此设置。

下表列出了可为 Activity 调用配置的所有 `ActivityOptions`。

| 选项                                                 | 必需的                                           | 类型                     |
| ------------------------------------------------------ | -------------------------------------------------- | ------------------------ |
| [`setScheduleToCloseTimeout`](#scheduletoclosetimeout) | 是（如果未指定 `StartToCloseTimeout`）    | 期间                 |
| [`setScheduleToStartTimeout`](#scheduletostarttimeout) | 不                                                 | 期间                 |
| [`setStartToCloseTimeout`](#starttoclosetimeout)       | 是（如果未指定 `ScheduleToCloseTimeout`） | 期间                 |
| [`setHeartbeatTimeout`](#heartbeattimeout)             | 不                                                 | 期间                 |
| [`setTaskQueue`](#taskqueue)                           | 不                                                 | 细绳                   |
| [`setRetryOptions`](#retryoptions)                     | 不                                                 | 重试选项             |
| [`setCancellationType`](#setcancellationtype)          | 不                                                 | Activity取消类型 |

### ScheduleToCloseTimeout

要设置 [Schedule-关闭超时](/encyclopedia/detecting-activity-failures#schedule-to-close-timeout)，请使用 [`ActivityOptions.newBuilder.setScheduleToCloseTimeout​`](https://www.javadoc.io/doc/io.temporal/temporal-sdk/latest/io/temporal/activity/ActivityOptions.Builder.html)。

必须设置此项或 `StartToCloseTimeout`。

- 型号：`Duration`
- 默认：无限制。
  请注意，如果在 Workflow 中定义了 `WorkflowRunTimeout` 和/或 `WorkflowExecutionTimeout`，则当达到其中一个或两个超时时，所有 Activity 重试都将停止。

您可以在 Workflow 实现中使用 `ActivityStub` 设置 Activity 选项，或者在 Worker 中使用 `WorkflowImplementationOptions` 按 Activity 设置 Activity 选项。
请注意，如果您使用 `WorkflowImplementationOptions.setActivityOptions()` 定义每个 Activity 类型选项，则在 Workflow 中使用 `ActivityStub` 再次专门设置它们将覆盖此设置。

- 与`ActivityStub`

  ```java
  GreetingActivities activities = Workflow.newActivityStub(GreetingActivities.class,
                  ActivityOptions.newBuilder()
                          .setScheduleToCloseTimeout(Duration.ofSeconds(5))
                          .build());
  ```

- 与`WorkflowImplementationOptions`

  ```java
  WorkflowImplementationOptions options =
              WorkflowImplementationOptions.newBuilder()
                      .setActivityOptions(
                              ImmutableMap.of(
                                      "GetCustomerGreeting",
                                      ActivityOptions.newBuilder()
                                          .setScheduleToCloseTimeout(Duration.ofSeconds(5))
                                          .build()))
                      .build();
  ```

<a id="activity-execution-result"></a>
## 获取 Activity 执行的结果 {#activity-execution-result}

调用生成 [Activity 执行](/activity-execution) 会生成 [ScheduleActivity任务](/references/commands#scheduleactivitytask) 命令，并为 Workflow 提供 Awaitable。
Workflow Execution 可以阻止进度，直到结果通过 Awaitable 可用为止，或者继续进度，在结果可用时使用它。

要获取异步调用的 Activity 方法的结果，请使用 `Promise` `get` 方法进行阻止，直到 Activity 方法结果可用。

有时，Activity 执行生命周期超出同步方法调用。
例如，可以将请求放入队列中，稍后收到回复并由不同的 Worker 进程获取。
整个请求-回复交互可以建模为单个 Activity。

要指示 Activity 不应在其方法返回时完成，请从原始 Activity 线程调用 `ActivityExecutionContext.doNotCompleteOnReturn()`。

然后，当收到答复时，使用 `ActivityCompletionClient` 完成 Activity。
要将 Activity 调用与完成关联起来，请使用 `TaskToken` 或 Workflow 和 Activity Id。

以下是使用 `ActivityExecutionContext.doNotCompleteOnReturn()` 的示例：

```java
public class FileProcessingActivitiesImpl implements FileProcessingActivities {

  public String download(String bucketName, String remoteName, String localName) {

    ActivityExecutionContext ctx = Activity.getExecutionContext();

    // Used to correlate reply
    byte[] taskToken = ctx.getInfo().getTaskToken();

    asyncDownloadFileFromS3(taskToken, bucketName, remoteName, localDirectory + localName);
    ctx.doNotCompleteOnReturn();

    // Return value is ignored when doNotCompleteOnReturn was called.
    return "ignored";
  }
  ...
}
```

下载完成后，下载服务可能会完成 Activity，或者从不同的进程中失败，例如：

```java
  public <R> void completeActivity(byte[] taskToken, R result) {
    completionClient.complete(taskToken, result);
  }

  public void failActivity(byte[] taskToken, Exception failure) {
    completionClient.completeExceptionally(taskToken, failure);
  }
```
