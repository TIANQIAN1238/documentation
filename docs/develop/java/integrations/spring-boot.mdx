本指南介绍 [Temporal 弹簧靴](https://central.sonatype.com/artifact/io.temporal/temporal-spring-boot-starter?smo=true) 集成。如果您当前是 [春天](https://spring.io/) 用户，Temporal Spring Boot 集成是开始使用 Temporal Java SDK 的最简单方法。

本节包括以下主题：

- [设置依赖关系](#setup-dependency)

- [配置 Worker](#configure-workers)

- [集成](#integrations)
- [测试](#testing)

## 设置依赖项 {#setup-dependency}

要开始使用 Temporal Spring Boot 集成，您需要添加 [`io.Temporal:Temporal-spring-boot-starter`](https://search.maven.org/artifact/io.temporal/temporal-spring-boot-starter)
作为 Spring 项目的依赖项：

:::note
Temporal 的 Spring Boot 集成目前支持 Spring Boot 2.x、3.x 和 4.x
:::

**[阿帕奇Maven](https://maven.apache.org/)：**

```maven
<dependency>
    <groupId>io.temporal</groupId>
    <artifactId>temporal-spring-boot-starter</artifactId>
    <version>1.31.0</version>
</dependency>
```

**[Gradle Groovy DSL](https://gradle.org/)：**

```groovy
implementation ("io.temporal:temporal-spring-boot-starter:1.31.0")
```

## 连接 {#connect}

有关连接到 Temporal 服务的更多信息，请参阅 [Temporal CLIent 文档](/develop/java/client/temporal-client)。

要创建自动配置的 `WorkflowClient`，您需要在 `application.yml` 文件中指定一些连接详细信息，如下一节所述。

<a id="connect-to-your-local-temporal-service"></a>
### 连接到您本地的 Temporal 服务 {#connect-to-your-local-temporal-service}

```yaml
spring.temporal:
  connection:
    target: local # you can specify a host:port here for a remote connection
```

这足以在 Spring Boot 应用程序中自动装配 `WorkflowClient`：

```java
@SpringBootApplication
class App {
  @Autowire
  private WorkflowClient workflowClient;
}
```

<a id="connect-to-a-custom-namespace"></a>
### 连接到自定义 Namespace {#connect-to-a-custom-namespace}

您还可以通过指定 `spring.Temporal.Namespace` 属性连接到自定义 Namespace。

```yaml
spring.temporal:
  connection:
    target: local # you can specify a host:port here for a remote connection
  namespace: <custom namespace> # you can specify a custom namespace that you are using
```

## 连接到 Temporal Cloud {#connect}

您还可以使用 API 密钥或 mTLS 连接到 Temporal Cloud 进行身份验证。

有关连接到 Temporal Cloud 的更多信息，请参阅 [连接到 Temporal Cloud](/develop/java/client/temporal-client#connect-to-temporal-cloud) 部分。

<a id="using-an-api-key"></a>
### 使用 API 密钥 {#using-an-api-key}

```yaml
spring.temporal:
  connection:
    target: <target>
    apiKey: <API key>
  namespace: <namespace>
```

<a id="using-mtls"></a>
### 使用 mTLS {#using-mtls}

```
spring.temporal:
  connection:
    mtls:
      target: <target>
      key-file: /path/to/key.key
      cert-chain-file: /path/to/cert.pem # If you use PKCS12 (.pkcs12, .pfx or .p12), you don't need to set it because the certificates chain is bundled into the key file
  namespace: <namespace>
```

<a id="configure-workers"></a>
## 配置 Workers {#configure-workers}

Temporal 的 Spring Boot 集成支持 Worker 的两种配置方法：显式配置和自动发现。

<a id="explicit-configuration"></a>
### 显式配置 {#explicit-configuration}

```yaml
spring.temporal:
  workers:
    - task-queue: your-task-queue-name
      name: your-worker-name # unique name of the Worker. If not specified, Task Queue is used as the Worker name.
      workflow-classes:
        - your.package.YourWorkflowImpl
      activity-beans:
        - activity-bean-name1
```

<a id="auto-discovery"></a>
### 自动发现 {#auto-discovery}

自动发现允许您通过在 Workflow、Activity 实现和 Nexus 服务上引用 Worker Task Queue 名称或 Worker 名称来跳过在配置中显式指定 Workflow 类、Activity beans 和 Nexus 服务 beans实施。自动发现在显式配置之后和之上应用。

```
spring.temporal:
  workers-auto-discovery:
    packages:
      - your.package # enumerate all the packages that contain your workflow implementations.
```

<a id="what-is-auto-discovered"></a>
#### 自动发现的内容： {#what-is-auto-discovered}

- Workflow 实现类用 `io.Temporal.spring.boot.WorkflowImpl` 注释
- Activity beans呈现Spring上下文，其实现用`io.Temporal.spring.boot.ActivityImpl`注释
- Nexus 存在于 Spring 上下文中的服务 bean，其实现用 `io.Temporal.spring.boot.NexusServiceImpl` 注释
- 如果注释引用了 Task Queue 但未显式配置，则为 Worker。将使用默认配置。

:::note
`io.Temporal.spring.boot.ActivityImpl` 和 `io.Temporal.spring.boot.NexusServiceImpl` 应该应用于 bean，一种方法是使用 `@Component` 注释 Activity 实现类
:::

```
@Component
@ActivityImpl(workers = "myWorker")
public class MyActivityImpl implements MyActivity {
  @Override
  public String execute(String input) {
    return input;
  }
}
```

:::note
自动发现的 Workflow 实现类、Activity bean 和 Nexus 服务 bean 将使用配置的 Worker 进行注册（如果尚未注册）。
:::

## 拦截器 {#interceptors}

要启用拦截器，您可以通过实现 `io.Temporal.common.interceptors.WorkflowClientInterceptor`、`io.Temporal.common.interceptors.ScheduleClientInterceptor` 或 `io.Temporal.common.interceptors.WorkerInterceptor` 接口来创建 Bean。拦截器将按照 `@Order` 注解指定的顺序进行注册。

## 集成 {#integrations}

Temporal Spring Boot 集成还内置了对 Spring 生态系统中各种工具的支持，例如指标和跟踪。

<a id="metrics"></a>
### 指标 {#metrics}

您可以使用 [弹簧启动执行器](https://docs.spring.io/spring-boot/reference/actuator/metrics.html) 设置内置 Spring Boot 指标。 Temporal Spring Boot 集成将获取 `MeterRegistry` bean 并使用它来报告 Temporal 指标。

或者，您可以在应用程序上下文中定义自定义 `io.micrometer.core.instrument.MeterRegistry` bean。

<a id="tracing"></a>
### 追踪 {#tracing}

您可以使用 OpenTelemetry 导出设置 [春云侦探](https://spring.io/projects/spring-cloud-sleuth)。 Temporal Spring Boot 集成将获取 `spring-cloud-sleuth-otel-autoconfigure` 配置的 OpenTelemetry bean 并将其用于 Temporal 跟踪。

或者，您可以在应用程序上下文中为 OpenTelemetry 定义自定义 `io.opentelemetry.api.OpenTelemetry` 或为 OpenTracing bean 定义 `io.opentracing.Tracer`。

## 选项自定义 {#customize-options}

要以编程方式自定义由 Spring Boot 集成创建的各种选项，您可以创建实现 `io.Temporal.spring.boot.TemporalOptionsCustomizer<OptionsBuilderType>` 接口的 bean。这将在应用属性文件中的选项后调用。

其中 `OptionsType` 可能是以下之一：

* `WorkflowServiceStubsOptions.Builder`
    * `WorkflowClientOptions.Builder`
    * `WorkerFactoryOptions.Builder`
    * `WorkerOptions.Builder`
    * `WorkflowImplementationOptions.Builder`
    * `TestEnvironmentOptions.Builder`

如果需要在 Task Queue 或 Worker 名称上自定义 `WorkerOptions`，则可以使用 `io.Temporal.spring.boot.WorkerOptionsCustomizer` 代替 `TemporalOptionsCustomizer<WorkerOptions.Builder>`。

如果需要在 Workflow 类型上定制 `WorkflowImplementationOptions`，则可以使用 `io.Temporal.spring.boot.WorkflowImplementationOptionsCustomizer` 代替 `TemporalOptionsCustomizer<WorkflowImplementationOptions.Builder>`。

## 测试 {#testing}

Temporal Spring Boot 集成还可以轻松支持测试 Temporal 代码。将以下内容添加到 `application.yml` 以重新配置客户端以通过使用内存 Java 测试服务器的 `io.Temporal.testing.TestWorkflowEnvironment` 进行工作：

```
spring.temporal:
  test-server:
    enabled: true
```

添加 `spring.Temporal.test-server.enabled:true` 时，`spring.Temporal.connection` 部分将被忽略。这允许在单元测试中连接 `TestWorkflowEnvironment` bean：

```
@SpringBootTest(classes = Test.Configuration.class)
@TestInstance(TestInstance.Lifecycle.PER_CLASS)
public class Test {
  @Autowired ConfigurableApplicationContext applicationContext;
  @Autowired TestWorkflowEnvironment testWorkflowEnvironment;
  @Autowired WorkflowClient workflowClient;

  @BeforeEach
  void setUp() {
    applicationContext.start();
  }

  @Test
  @Timeout(value = 10)
  public void test() {
    # ...
  }

  @ComponentScan # to discover Activity beans annotated with @Component
  public static class Configuration {}
}
```

有关测试的更多信息，请参阅 [Java SDK 测试框架文档](/develop/java/best-practices/testing-suite#test-frameworks)。
