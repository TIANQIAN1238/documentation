## 如何启动子 Workflow Execution {#child-workflows}

[儿童 Workflow Execution](/child-workflows) 是一个 Workflow Execution，它是使用 Child Workflow API 来自另一个 Workflow 的 Scheduled。

使用 Child Workflow API 时，Child Workflow 相关事件（[StartChildWorkflowExecutionInitiated](/references/events#startchildworkflowexecutioninitiated)、[子WorkflowExecutionStarted](/references/events#childworkflowexecutionstarted)、[ChildWorkflowExecutionCompleted](/references/events#childworkflowexecutioncompleted) 等）将记录在 Workflow Execution Event History 中。

始终阻止进度，直到 [子WorkflowExecutionStarted](/references/events#childworkflowexecutionstarted) 事件记录到 Event History，以确保子 Workflow Execution 已启动。
此后，可以使用 Child Workflow 选项中设置的 _Abandon_ [家长关闭政策](/parent-close-policy) 放弃子 Workflow Execution。

要确保 Child Workflow Execution 已启动，请首先在 Child Workflow future 实例上调用 Child Workflow Execution 方法，该方法返回不同的 future。

然后获取一个对象的值，该对象充当最初未知的结果的代理，该结果将等待子 Workflow Execution 生成。

除了活动之外，Workflow 还可以启动其他 Workflow。

`Workflow::executeChildWorkflow` 和 `Workflow::newChildWorkflowStub` 允许在 Workflow 的实现中调度其他 Workflow。
父 Workflow 能够监视和影响 Child Workflow 的生命周期，类似于它调用的 Activity 的方式。

```php
// Use one stub per child workflow run
$child = Workflow::newChildWorkflowStub(
    ChildWorkflowInterface::class,
    ChildWorkflowOptions::new()
        // Do not specify WorkflowId if you want Temporal to generate a unique Id
        // for the child execution.
        ->withWorkflowId('BID-SIMPLE-CHILD-WORKFLOW')
        ->withExecutionStartToCloseTimeout(DateInterval::createFromDateString('30 minutes'))
);

// This is a non blocking call that returns immediately.
// Use yield $child->workflowMethod(name) to call synchronously.
$promise = $child->workflowMethod('value');

// Do something else here.
try{
    $value = yield $promise;
} catch(TemporalException $e) {
    $logger->error('child workflow failed');
    throw $e;
}
```

让我们看一下这个调用的每个组成部分。

在调用`$child->WorkflowMethod()`之前，您必须配置`ChildWorkflowOptions`以进行调用。
这些选项自定义各种执行超时，并传递到 `Workflow::newChildWorkflowStub` 定义的 Workflow 存根中。
创建存根后，您可以根据属性 `WorkflowMethod` 调用其 Workflow 方法。

该方法调用立即返回并返回 `Promise`。
这允许您执行更多代码，而不必等待 Scheduled Workflow 完成。

当您准备好处理 Workflow 的结果时，请对返回的 Promise 对象调用 `yield $promise` 方法。

当用户取消父 Workflow 时，可以根据可配置的子策略取消或放弃 Child Workflow。

您还可以跳过 Child Workflow 启动的存根部分，直接使用 `Workflow::executeChildWorkflow`：

```php
// Use one stub per child workflow run
$childResult = yield Workflow::executeChildWorkflow(
    'ChildWorkflowName',
    ['args'],
    ChildWorkflowOptions::new()->withWorkflowId('BID-SIMPLE-CHILD-WORKFLOW'),
    Type::TYPE_STRING // optional: defines the return type
);
```

#### 如何设置父关闭策略 {#parent-close-policy}

[家长关闭政策](/parent-close-policy) 确定如果子 Workflow Execution 的父级更改为关闭状态（已完成、失败或超时），子级 Workflow Execution 会发生什么情况。

默认“父关闭策略”选项设置为终止子 Workflow Execution。

在 PHP 中，[家长关闭政策](/parent-close-policy) 是通过 `ChildWorkflowOptions` 对象和 `withParentClosePolicy()` 方法设置的。
可能的值可以从 [`ParentClosePolicy`](https://github.com/temporalio/sdk-php/blob/master/src/Workflow/ParentClosePolicy.php) 类中获取。

- `POLICY_TERMINATE`
- `POLICY_ABANDON`
- `POLICY_REQUEST_CANCEL`

然后使用 `ChildWorkflowOptions` 对象创建一个新的 Child Workflow 对象：

```php
$child = Workflow::newUntypedChildWorkflowStub(
    'child-workflow',
    ChildWorkflowOptions::new()
        ->withParentClosePolicy(ParentClosePolicy::POLICY_ABANDON)
);

yield $child->start();
```

在上面的代码片段中，我们：

1. 使用 `Workflow::newUntypedChildWorkflowStub` 创建一个新的非类型化 Child Workflow 存根。
2. 提供 `ChildWorkflowOptions` 对象，并将父关闭策略设置为 `ParentClosePolicy::POLICY_ABANDON`。
3. 使用 `yield` 和方法 `start()` 异步启动 Child Workflow Execution。

我们在这里需要 `yield` 以确保子 Workflow Execution 在父项关闭之前启动。
