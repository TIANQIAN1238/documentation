Temporal 支持使用元数据向 Workflow 和事件添加上下文。 
这有助于用户识别和了解 Workflow 及其操作。

<a id="adding-summary-and-details-to-workflows"></a>
## 添加摘要和详细信息到 Workflow {#adding-summary-and-details-to-workflows}

<a id="starting-a-workflow"></a>
### 启动 Workflow {#starting-a-workflow}

启动 Workflow 时，您可以提供静态摘要和详细信息以帮助在 UI 中识别 Workflow：

```php
use Temporal\Client\WorkflowClient;
use Temporal\Client\WorkflowOptions;

// Create workflow client
$workflowClient = WorkflowClient::create($serviceClient);

// Start a workflow with static summary and details
$workflow = $workflowClient->newWorkflowStub(
    YourWorkflow::class,
    WorkflowOptions::new()
        ->withWorkflowId('your-workflow-id')
        ->withTaskQueue('your-task-queue')
        ->withStaticSummary('Order processing for customer #12345')
        ->withStaticDetails('Processing premium order with expedited shipping')
);

$result = $workflow->yourWorkflowMethod('workflow input');
```

`withStaticSummary()` 设置出现在 Workflow 列表视图中的单行描述，限制为 200 字节。
`withStaticDetails()` 设置Workflow 详细信息视图中显示的多行综合信息，最大限制为20K 字节。

输入格式是标准 Markdown，不包括图像、HTML 和脚本。

您还可以异步启动 Workflow：

```php
// Start workflow asynchronously
$workflowClient->start($workflow, 'workflow input');
```

<a id="adding-summary-to-activities-and-timers"></a>
### 添加摘要到活动和 Timer {#adding-summary-to-activities-and-timers}

您可以将 `summary` 连接到 Workflow 内的 Timer：

```php
use Temporal\Workflow;
use Temporal\Workflow\TimerOptions;

#[WorkflowInterface]
interface YourWorkflow
{
    #[WorkflowMethod]
    public function yourWorkflowMethod(string $input): string;
}

class YourWorkflowImpl implements YourWorkflow
{
    public function yourWorkflowMethod(string $input): \Generator
    {
        // Create a timer with a summary
        yield Workflow::timer(
            300, // 5 minutes in seconds
            TimerOptions::new()->withSummary('Waiting for payment confirmation')
        );
        
        return 'Timer completed';
    }
}
```

对于活动，您可以使用 Activity 选项设置摘要：

```php
use Temporal\Activity\ActivityOptions;
use Temporal\Workflow;

class YourWorkflowImpl implements YourWorkflow
{
    private YourActivitiesInterface $activities;
    
    public function __construct()
    {
        $this->activities = Workflow::newActivityStub(
            YourActivitiesInterface::class,
            ActivityOptions::new()
                ->withStartToCloseTimeout('10 seconds')
                ->withSummary('Processing user data')
        );
    }
    
    public function yourWorkflowMethod(string $input): \Generator
    {
        // Execute the activity with the summary
        $result = yield $this->activities->yourActivity($input);
        
        return $result;
    }
}
```

`summary` 的输入格式是字符串，且限制为 200 个字节。

<a id="viewing-summary-and-details-in-the-ui"></a>
## 在 UI 中查看摘要和详细信息 {#viewing-summary-and-details-in-the-ui}

将摘要和详细信息添加到 Workflow、活动和 Timer 后，您可以在 Temporal Web UI 中查看这些丰富的信息。 
导航到 Workflow 的详细信息页面，查看两个关键位置显示的元数据：

<a id="workflow-overview-section"></a>
### Workflow 概述部分 {#workflow-overview-section}

在 Workflow 详细信息页面的顶部，您将找到 Workflow 级别的元数据：

- **摘要和详细信息** - 显示启动 Workflow 时设置的静态摘要和静态详细信息
- **当前详细信息** - 显示 Workflow Execution 期间可以是 Updated 的动态详细信息

所有 Workflow 详细信息都支持标准 Markdown 格式（不包括图像、HTML 和脚本），允许您创建丰富的结构化信息显示。

### Event History

Workflow 和 Event History 中的各个事件会显示其关联的摘要（如果有）。

Workflow、Activity 和 Timer 摘要以紫色文本显示在其相应事件旁边，提供即时上下文，无需您展开事件详细信息。 
当您展开事件时，摘要也会在详细视图中突出显示。
