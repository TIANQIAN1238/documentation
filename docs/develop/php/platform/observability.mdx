Temporal 开发人员指南的可观察性部分涵盖了查看 [Temporal 应用](/temporal#temporal-application) 当前状态的多种方法，即查看 [Temporal平台](/workflow-execution) 跟踪哪些 [Workflow Executions](/temporal#temporal-platform) 以及任何指定 Workflow Execution 当前或执行点的状态的方法。

本节介绍与查看应用程序状态相关的功能，包括：

- [从 Workflow 登录](#logging)
- [Visibility](#visibility)

<a id="logging"></a>
## 从 Workflow {#logging} 记录
日志记录使您能够在代码执行期间记录关键信息。
记录器创建审计跟踪并捕获有关 Workflow 操作的信息。
适当的日志记录级别取决于您的具体需求。
在开发或故障排除期间，您可能会使用调试甚至跟踪。
在生产中，您可以使用 info 或 warn 来避免日志量过多。

记录器支持以下日志记录级别：

| 等级   | 使用                                                                                                       |
| ------- | --------------------------------------------------------------------------------------------------------- |
| `TRACE` | 最详细的日志记录级别，用于非常细粒度的信息。                               |
| `DEBUG` | 详细信息，通常可用于调试目的。                                            |
| `INFO`  | 有关应用程序操作的一般信息。                                                    |
| `WARN`  | 指示不会阻止应用程序运行的潜在有害情况或小问题。 |
| `ERROR` | 指示可能仍允许应用程序继续运行的错误情况。                    |

Temporal SDK 核心通常使用 `WARN` 作为其默认日志记录级别。

要在 Workflow 代码中获取 PSR-3 兼容记录器，请使用 [`Workflow::getLogger()`](https://php.temporal.io/classes/temporal-Workflow.html#method_getLogger) 方法。

```php
use Temporal\Workflow;

#[Workflow\WorkflowInterface]
class MyWorkflow
{
    #[Workflow\WorkflowMethod]
    public function execute(string $param): \Generator
    {
        Workflow::getLogger()->info('Workflow started', ['parameter' => $param]);

        // Your workflow implementation

        Workflow::getLogger()->info('Workflow completed');
        return 'Done';
    }
}
```

Workflow 记录器会自动使用当前的 Task Queue 名称来丰富日志上下文。

除非 [`enableLoggingInReplay`](https://php.temporal.io/classes/temporal-Worker-WorkerOptions.html#method_withEnableLoggingInReplay) Worker 选项设置为 true，否则重播模式下的日志将被忽略。

```php
$factory = WorkerFactory::create();
$worker = $factory->newWorker('your-task-queue', WorkerOptions::new()
    ->withEnableLoggingInReplay(true)
);
```

<a id="default-logger"></a>
### 默认记录器 {#default-logger}

默认情况下，PHP SDK 使用 [`StderrLogger`](https://php.temporal.io/classes/temporal-Worker-Logger-StderrLogger.html) 将日志消息输出到标准错误流。
这些消息由 RoadRunner 自动捕获，并以 INFO 级别合并到其日志系统中，确保在开发和生产环境中正确收集日志。
有关 RoadRunner 日志记录功能的更多详细信息，请参阅 [RoadRunner 记录器文档](https://docs.roadrunner.dev/docs/logging-and-observability/logger)。

<a id="custom-logger"></a>
### 如何提供自定义记录器 {#custom-logger}
您可以在创建 Worker 时设置自定义 PSR-3 兼容记录器：

```php
$myLogger = new MyLogger();

$workerFactory = WorkerFactory::create(converter: $converter);
$worker = $workerFactory->newWorker(
    taskQueue: 'my-task-queue',
    logger: $myLogger,
);
```

<a id="visibility"></a>
## Visibility API {#visibility}
Temporal 平台中的术语 Visibility 是指使操作员能够查看当前存在于 Temporal 服务中的 Workflow Execution 的子系统和 API。

<a id="search-attributes"></a>
### 如何使用 Search Attribute {#search-attributes}
检索 Workflow Execution 的典型方法是通过其 Workflow Id。

但是，有时您需要根据另一属性检索一个或多个 Workflow Execution。例如，假设您想要获取某个时间范围内失败的某种类型的所有 Workflow Execution，以便您可以使用相同的参数启动新的 Workflow Execution。

您可以使用 [Search Attributes](/search-attribute) 来完成此操作。

- [默认 Search Attribute](/search-attribute#default-search-attribute) 与 `WorkflowType`、`StartTime` 和 `ExecutionStatus` 一样，会自动添加到 Workflow Execution。
- [定制 Search Attribute](/search-attribute#custom-search-attribute) 可以包含其自己的特定于域的数据（如 `customerId` 或 `numItems`）。

使用自定义 Search Attribute 的步骤是：

- 使用 `Temporal operator search-attribute create` 或云 UI 在 Temporal 服务中创建新的 Search Attribute。
- 设置 Workflow Execution 的 Search Attribute 值：
  - 在客户端上，在启动执行时将其作为选项包含在内。
  - 在 Workflow 中通过调用 `UpsertSearchAttributes`。
- 读取Search Attribute的值：
  - 在客户端上通过调用 `DescribeWorkflow`。
  - 在 Workflow 中查看 `WorkflowInfo`。
- Query Workflow Execution 由 Search Attribute 使用 [列表过滤器](/list-filter) 实现：
  - [在Temporal CLI中](/cli/workflow#list)。
  - 在代码中调用 `ListWorkflowExecutions`。

以下是 Query Workflow Execution 的操作方法：

在客户端上使用 [listWorkflowExecutions()](https://php.temporal.io/classes/temporal-Client-WorkflowClientInterface.html#method_listWorkflowExecutions) 方法并传递 [列表过滤器](/list-filter) 作为参数来过滤列出的 Workflow。
结果是一个可迭代的分页器，因此您可以使用 `foreach` 循环来迭代结果。

```php
$paginator = $workflowClient->listWorkflowExecutions('WorkflowType="GreetingWorkflow"');

foreach ($paginator as $info) {
    echo "Workflow ID: {$info->execution->getID()}\n";
}
```

<a id="custom-search-attributes"></a>
### 如何设置自定义 Search Attribute {#custom-search-attributes}
在 Temporal 服务（使用 `Temporal operator search-attribute create` 或云 UI）中创建自定义 Search Attribute 后，您可以在启动 Workflow 时设置自定义 Search Attribute 的值。

要设置自定义 Search Attribute，请在 `WorkflowOptions` 上对 Workflow 存根使用 `withTypedSearchAttributes` 方法。
类型为 Search Attribute 的是 `TypedSearchAttributes` 系列。

```php
$keyDestinationTime = SearchAttributeKey::forDatetime('DestinationTime');
$keyOrderId = SearchAttributeKey::forKeyword('OrderId');

$workflow = $workflowClient->newWorkflowStub(
    OrderWorkflowInterface::class,
    WorkflowOptions::new()
        ->withWorkflowExecutionTimeout('10 minutes')
        ->withTypedSearchAttributes(
            TypedSearchAttributes::empty()
                ->withValue($keyOrderId, $orderid)
                ->withValue($keyDestinationTime, new \DateTimeImmutable('2028-11-05T00:10:07Z'))
        ),
);
```

<a id="upsert-search-attributes"></a>
### 如何更新插入 Search Attributes {#upsert-search-attributes}
在 Workflow 代码中，您可以使用 [`upsertTypedSearchAttributes`](https://php.temporal.io/classes/temporal-Workflow.html#method_upsertTypedSearchAttributes) 动态添加或 Update Search Attribute。
此方法对于属性需要根据内部逻辑或外部事件更改的 Workflow 特别有用。

```php
#[Workflow\UpdateMethod]
public function postponeDestinationTime(\DateInterval $interval)
{
    // Get the key for the DestinationTime attribute
    $keyDestinationTime = SearchAttributeKey::forDatetime('DestinationTime');

    /** @var DateTimeImmutable $destinationTime */
    $destinationTime = Workflow::getInfo()->typedSearchAttributes->get($keyDestinationTime);

    Workflow::upsertTypedSearchAttributes(
        $keyDestinationTime->valueSet($destinationTime->add($interval)),
    );
}
```

<a id="remove-search-attribute"></a>
### 如何从 Workflow 中删除 Search Attribute {#remove-search-attribute}
要删除之前设置的 Search Attribute，请将其设置为空地图。

```php
#[Workflow\UpdateMethod]
public function unsetDestinationTime()
{
    // Get the key for the DestinationTime attribute
    $keyDestinationTime = SearchAttributeKey::forDatetime('DestinationTime');

    Workflow::upsertTypedSearchAttributes(
        $keyDestinationTime->valueUnset(),
    );
}
```
