<a id="install-php"></a>
## 安装 PHP {#install-php}

确保您已安装 PHP。

**如果你没有PHP：**访问官方网站[下载并安装](https://www.php.net/downloads.php)即可。

<a id="grpc-extension"></a>
### gRPC 扩展 {#grpc-extension}

需要 gRPC 扩展才能与 RoadRunner 应用程序服务器配合使用。

**如果您没有安装`ext-gRPC`：**访问官方网站
[下载并安装](https://docs.cloud.google.com/php/docs/reference/help/grpc) 吧。

:::tip GRPC Installation Tip

在配备 Apple Silicon (M1/M2/M3/M4) 和 PHP 8.3 的 macOS 上，`pecl install gRPC` 可能会出现无限期挂起或安装的情况。
如果发生这种情况，请尝试安装特定版本：

```bash
pecl install channel://pecl.php.net/grpc-1.78.0RC2
```

注意：您可以在 [pecl.php.net/package/gRPC](https://pecl.php.net/package/grpc) 找到最新版本。

:::

</SetupStep>

<a id="run-hello-world-test-your-installation"></a>
## 运行 Hello World：测试您的安装 {#run-hello-world-test-your-installation}

现在，让我们通过创建并运行一个完整的 Temporal 应用程序来验证您的设置是否正常工作，其中包含 Workflow 和
Activity。

该测试将确认：

- Temporal PHP SDK 已正确安装
- 您本地的 Temporal 服务正在运行
- 您可以成功创建并执行 Workflow 和活动
- 组件之间的通信功能正常

<a id="1-create-the-activity"></a>
### 1.创建Activity {#1-create-the-activity}

创建 Activity 文件 (`src/GreetingActivity.php`)：

```php
<?php
declare(strict_types=1);

namespace App;

use Temporal\Activity\ActivityInterface;
use Temporal\Activity\ActivityMethod;

#[ActivityInterface]
class GreetingActivity
{
    #[ActivityMethod]
    public function greet(string $name): string
    {
        return "Hello, $name!";
    }
}
```

Activity 是一种执行单个明确定义的操作（短期或长期运行）的方法，通常涉及
与外界交互，例如发送电子邮件、发出网络请求、写入数据库或调用
API，很容易出现故障。如果 Activity 失败，Temporal 会根据您的配置自动重试。

您可以将 PHP 中的活动定义为用 `Temporal\Activity\ActivityInterface` 属性注释的类。

Workflow 类中使用的每个方法都应使用 `Temporal\Activity\ActivityMethod` 进行注释。

<a id="2-create-the-workflow"></a>
### 2.创建Workflow {#2-create-the-workflow}

创建 Workflow 文件 (`src/SayHelloWorkflow.php`)：

```php
<?php
declare(strict_types=1);

namespace App;

use Temporal\Activity\ActivityOptions;
use Temporal\Workflow;
use Temporal\Workflow\WorkflowInterface;
use Temporal\Workflow\WorkflowMethod;

#[WorkflowInterface]
class SayHelloWorkflow
{
    #[WorkflowMethod]
    public function sayHello(string $name)
    {
        $activity = Workflow::newActivityStub(
            GreetingActivity::class,
            ActivityOptions::new()
                ->withStartToCloseTimeout(5),
        );

        return yield $activity->greet($name);
    }
}
```

<a id="3-create-a-worker-file"></a>
### 3.创建Worker文件 {#3-create-a-worker-file}

创建Worker文件（`Worker.php`，在项目根目录下）：

```php
<?php
declare(strict_types=1);

use Temporal\WorkerFactory;

ini_set('display_errors', 'stderr');
require "vendor/autoload.php";

$factory = WorkerFactory::create();

$worker = $factory->newWorker();

// Register Workflows
$worker->registerWorkflowTypes(\App\SayHelloWorkflow::class);
// Register Activities
$worker->registerActivity(\App\GreetingActivity::class);

$factory->run();
```

<a id="run-the-worker"></a>
#### 运行 Worker {#run-the-worker}

之前，我们创建了一个执行 Workflow 和 Activity 任务的 Worker。

现在，通过打开一个新的终端窗口并运行以下命令来启动 RoadRunner 应用程序服务器来运行 Worker
命令：

```bash
./rr serve
```

Worker 轮询 Task Queue，您将其配置为轮询，寻找要做的工作。一旦 Worker 将 Workflow 出列
或来自 Task Queue 的 Activity 任务，然后执行该任务。

Worker 是 Temporal 应用程序的重要组成部分，因为它们实际执行您的 Temporal 应用程序中定义的任务。
Workflow 和活动。有关 Worker 的更多信息，请参阅
[了解 Temporal](/evaluate/understanding-temporal#workers) 和 [深入研究 Worker](/workers)。

<a id="5-execute-the-workflow"></a>
### 5.执行Workflow {#5-execute-the-workflow}

现在您的 Worker 正在运行，是时候启动 Workflow Execution 了。

最后一步将验证标记为 `client.php` 的文件是否一切正常。

创建一个名为 `client.php` 的单独文件：

```php
<?php
declare(strict_types=1);

use Temporal\Client\GRPC\ServiceClient;
use Temporal\Client\WorkflowClient;

ini_set('display_errors', 'stderr');
require "vendor/autoload.php";

$client = new WorkflowClient(
    ServiceClient::create('localhost:7233'),
);

$workflowStub = $client->newWorkflowStub(\App\SayHelloWorkflow::class);
$result = $workflowStub->sayHello('Temporal');

echo "Result: {$result}\n";
```

当您的 Worker 仍在运行时，打开一个新终端并运行：

```bash
php client.php
```

### 验证成功 {#verify-success}

如果一切正常，您应该看到：

- Worker 处理 Workflow 和 Activity
- 输出：`Result: Hello, Temporal!`
- [Temporal 网页用户界面](http://localhost:8233) 中的 Workflow Execution 详细信息
