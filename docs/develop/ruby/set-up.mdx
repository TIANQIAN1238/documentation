# 快速入门 {#quickstart}

本指南将引导您设置 Temporal Ruby SDK 并运行您的第一个 Workflow。只需几步，
您将安装 SDK 并启动本地开发服务器。验证您的本地环境是否正确
安装后，我们将执行 Workflow ，它将输出“Hello，Temporal”。

＃＃ 安装

此步骤使用 Bundler 设置一个新的 Ruby 项目并安装 Temporal Ruby SDK。

我们建议使用 [捆绑器](https://bundler.io/) 来管理 Ruby 项目依赖项，包括 Temporal SDK。
这些教程假定 Ruby 3.4.3 或更高版本。

按照以下步骤创建目录，使用 `Gemfile` 初始化项目，然后添加 Temporal SDK。

**笔记：**

- 仅支持 macOS ARM/x64 和 Linux ARM/x64。
- 源gem已发布，但**无法直接构建**。
- 不支持 Windows (MinGW)。
- `fibers`/`async` 仅在 Ruby **3.3+** 上受支持。
- 请参阅 [平台支持](#) 了解完整详细信息。

## 启动开发服务器 {#install-temporal-cli}

安装 Temporal CLI 并将其添加到 PATH 后，打开一个新的终端窗口并运行以下命令。

此命令启动本地 Temporal 服务。它启动 Web UI，创建默认的 Namespace，并使用内存中的
数据库。

Temporal 服务将在 localhost:7233 上可用。 Temporal Web UI 将在以下网址提供：
http://localhost:8233.

当您完成教程和其他项目时，让本地 Temporal 服务保持运行。您可以停止 Temporal
按 CTRL+C 可随时提供服务。

安装完所有内容后，您就可以在本地计算机上使用 Temporal 构建应用程序了。

<a id="start-the-development-server"></a>
## 运行 Hello World：测试您的安装 {#start-the-development-server}

现在，让我们通过创建并运行一个完整的 Temporal 应用程序来验证您的设置是否正常工作，其中包含 Workflow 和
Activity。

该测试将确认：

- Temporal Ruby SDK 已正确安装
- 您本地的 Temporal 服务正在运行
- 您可以成功创建并执行 Workflow 和活动
- 组件之间的通信功能正常

<a id="run-hello-world-test-your-installation"></a>
### 1.创建Activity {#run-hello-world-test-your-installation}

创建 Activity 文件 (say_hello_Activity.rb)：

```ruby
require 'temporalio/activity'

# Implementation of a simple activity
class SayHelloActivity < Temporalio::Activity::Definition
  def execute(name)
    "Hello, #{name}!"
  end
end
```

<a id="1-create-the-activity"></a>
### 2.创建Workflow {#1-create-the-activity}

创建 Workflow 文件 (say_hello_Workflow.rb)：

```ruby
require 'temporalio/workflow'
require_relative 'say_hello_activity'

class SayHelloWorkflow < Temporalio::Workflow::Definition
  def execute(name)
    Temporalio::Workflow.execute_activity(
      SayHelloActivity,
      name,
      schedule_to_close_timeout: 300
    )
  end
end
```

<a id="2-create-the-workflow"></a>
### 3. 创建并运行 Worker {#2-create-the-workflow}

定义了 Activity 和 Workflow 后，您需要 Worker 来执行它们。 Worker 是 Temporal 的重要组成部分
应用程序，因为它们实际执行 Workflow 和活动中定义的任务。欲了解更多信息
Workers，参见 [了解 Temporal](/evaluate/understanding-temporal#workers) 和
[深入研究 Worker](/workers)。

创建 Worker 文件 (Worker.rb)：

```ruby
require 'temporalio/client'
require 'temporalio/worker'
require_relative 'say_hello_activity'
require_relative 'say_hello_workflow'

# Create a client
client = Temporalio::Client.connect('localhost:7233', 'default')

# Create a worker with the client, activities, and workflows
worker = Temporalio::Worker.new(
  client:,
  task_queue: 'my-task-queue',
  workflows: [SayHelloWorkflow],
  # There are various forms an activity can take, see "Activities" section for details
  activities: [SayHelloActivity]
)

# Run the worker until SIGINT. This can be done in many ways, see "Workers" section for details.
worker.run(shutdown_signals: ['SIGINT'])
```

运行 Worker：

```bash
ruby worker.rb
```

<a id="3-create-and-run-the-worker"></a>
### 4.执行Workflow {#3-create-and-run-the-worker}

现在您的 Worker 正在运行，是时候启动 Workflow Execution 了。

创建一个名为 starter.rb 的单独文件：

```ruby
require 'temporalio/client'
require_relative 'say_hello_workflow'

# Create a client
client = Temporalio::Client.connect('localhost:7233', 'default')

# Run workflow
result = client.execute_workflow(
  SayHelloWorkflow,
  'Temporal', # This is the input to the workflow
  id: 'my-workflow-id',
  task_queue: 'my-task-queue'
)
puts "Result: #{result}"
```

然后运行：

```bash
ruby starter.rb
```

<a id="4-execute-the-workflow"></a>
### 验证成功 {#4-execute-the-workflow}

如果一切正常，您应该看到：

- Worker 处理 Workflow 和 Activity
- 输出：`Workflow result: Hello, Temporal!`
- [Temporal 网页用户界面](http://localhost:8233) 中的 Workflow Execution 详细信息
