Temporal 支持使用元数据向 Workflow 和事件添加上下文。 
这有助于用户识别和了解 Workflow 及其操作。

<a id="adding-summary-and-details-to-workflows"></a>
## 添加摘要和详细信息到 Workflow {#adding-summary-and-details-to-workflows}

<a id="starting-a-workflow"></a>
### 启动 Workflow {#starting-a-workflow}

启动 Workflow 时，您可以提供静态摘要和详细信息以帮助在 UI 中识别 Workflow：

```ruby
require 'temporalio/client'

# Create client
client = Temporalio::Client.connect('localhost:7233')

# Start a workflow with static summary and details
handle = client.start_workflow(
  'YourWorkflow',
  'workflow input',
  id: 'your-workflow-id',
  task_queue: 'your-task-queue',
  static_summary: 'Order processing for customer #12345',
  static_details: 'Processing premium order with expedited shipping'
)
```

`static_summary:` 是出现在 Workflow 列表视图中的单行描述，限制为 200 字节。
`static_details:` 可以是多行的，并提供出现在 Workflow 详细信息视图中的更全面的信息，最大限制为 20K 字节。

输入格式是标准 Markdown，不包括图像、HTML 和脚本。

您还可以使用 `execute_Workflow` 进行同步执行：

```ruby
# Execute workflow synchronously
result = client.execute_workflow(
  'YourWorkflow',
  'workflow input',
  id: 'your-workflow-id',
  task_queue: 'your-task-queue',
  static_summary: 'Order processing for customer #12345',
  static_details: 'Processing premium order with expedited shipping'
)
```

<a id="inside-the-workflow"></a>
#### Workflow 内部 {#inside-the-workflow}

在 Workflow 中，您可以获取并设置_当前 Workflow 详细信息_。 
与 Workflow 启动时设置的静态摘要/详细信息不同，该值在 Workflow 的整个生命周期中可以是 Updated。 
当前的 Workflow 详细信息也采用 Markdown 格式（不包括图像、HTML 和脚本）并且可以跨多行。

```ruby
require 'temporalio'

class YourWorkflow < Temporalio::Workflow::Definition
  def execute(input)
    # Get the current details
    current_details = Temporalio::Workflow.current_details
    Temporalio::Workflow.logger.info("Current details: #{current_details}")
    
    # Set/update the current details
    Temporalio::Workflow.current_details = 'Updated workflow details with new status'
    
    'Workflow completed'
  end
end
```

<a id="adding-summary-to-activities-and-timers"></a>
#### 添加摘要到活动和 Timer {#adding-summary-to-activities-and-timers}

当从 Workflow 中启动活动时，您可以将 `summary:` 附加到活动：

```ruby
require 'temporalio'

class YourWorkflow < Temporalio::Workflow::Definition
  def execute(input)
    # Execute an activity with a summary
    result = Temporalio::Workflow.execute_activity(
      'YourActivity',
      input,
      start_to_close_timeout: 10,
      summary: 'Processing user data'
    )
    
    result
  end
end
```

同样，您可以将 `summary:` 附加到 Workflow 内的 Timer：

```ruby
require 'temporalio'

class YourWorkflow < Temporalio::Workflow::Definition
  def execute(input)
    # Create a timer with a summary
    Temporalio::Workflow.sleep(300, summary: 'Waiting for payment confirmation')
    
    'Timer completed'
  end
end
```

`summary:` 的输入格式是字符串，且限制为 200 个字节。

<a id="viewing-summary-and-details-in-the-ui"></a>
## 在 UI 中查看摘要和详细信息 {#viewing-summary-and-details-in-the-ui}

将摘要和详细信息添加到 Workflow、活动和 Timer 后，您可以在 Temporal Web UI 中查看这些丰富的信息。 
导航到 Workflow 的详细信息页面，查看两个关键位置显示的元数据：

<a id="workflow-overview-section"></a>
### Workflow 概述部分 {#workflow-overview-section}

在 Workflow 详细信息页面的顶部，您将找到 Workflow 级别的元数据：

- **摘要和详细信息** - 显示启动 Workflow 时设置的静态摘要和静态详细信息
- **当前详细信息** - 显示 Workflow Execution 期间可以是 Updated 的动态详细信息

所有 Workflow 详细信息都支持标准 Markdown 格式（不包括图像、HTML 和脚本），允许您创建丰富的结构化信息显示。

### Event History

Workflow 的 Event History 中的各个事件显示其关联的摘要（如果可用）：

Workflow、Activity 和 Timer 摘要以紫色文本显示在其相应事件旁边，提供即时上下文，无需您展开事件详细信息。当您展开事件时，摘要也会在详细视图中突出显示。
