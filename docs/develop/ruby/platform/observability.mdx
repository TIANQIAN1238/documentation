此页面涵盖与查看应用程序状态相关的功能，包括：

- [指标](#metrics)
- [追踪](#tracing)
- [记录](#logging)
- [Visibility](#visibility)

可观察性指南涵盖了查看 [Temporal 应用](/temporal#temporal-application) 当前状态的多种方法。
这包括查看 [Temporal平台](/workflow-execution) 跟踪的 [Workflow Executions](/temporal#temporal-platform)，以及检查执行期间任意时刻的状态。

<a id="metrics"></a>
## 发出指标 {#metrics}
每个 Temporal SDK 都可以选择从客户端或 Worker 进程发出指标。
指标可以由 Prometheus 等系统抓取，图表可以使用 Grafana 等工具创建。

- 有关 Prometheus 和 Grafana 集成的概述，请参阅 [监控](/self-hosted-guide/monitoring) 指南。
- 有关指标列表，请参阅 [SDK指标参考](/references/sdk-metrics)。

创建全局 `Temporalio::Runtime` 时，Ruby 中的指标是在 `telemetry` 参数的 `metrics` 参数上配置的。该对象应该是全局创建的，并且应该用于所有客户端；因此，您应该在任何其他 Temporal 代码之前配置此代码。

<a id="set-a-prometheus-endpoint"></a>
## 设置 Prometheus 端点 {#set-a-prometheus-endpoint}

以下示例在端口 `9000` 上公开 Prometheus 端点。

```ruby
Temporalio::Runtime.default = Temporalio::Runtime.new(
  telemetry: Temporalio::Runtime::TelemetryOptions.new(
    metrics: Temporalio::Runtime::MetricsOptions.new(
      prometheus: Temporalio::Runtime::PrometheusMetricsOptions.new(
        bind_address: '0.0.0.0:9000'
      )
    )
  )
)
```

<a id="custom-metric-handling"></a>
### 自定义指标处理 {#custom-metric-handling}

可以将 `Temporalio::Runtime::MetricBuffer` 的实例作为 `buffer` 参数提供给 `MetricsOptions`，而不是 Prometheus 或 OpenTelemetry。
然后可以在缓冲区上定期调用 `retrieve_Updates` 以获得指标 Update。

<a id="tracing"></a>
## 设置跟踪 {#tracing}
跟踪可以观察整个应用程序的调用顺序，包括 Workflow 和活动。

可以使用 `Temporalio::Contrib::OpenTelemetry::TracingInterceptor` 启用对客户端、活动和 Workflow 的 OpenTelemetry 跟踪。具体来说，在创建客户端时，设置拦截器如下：

```ruby
require 'opentelemetry/api'
require 'opentelemetry/sdk'
require 'temporalio/client'
require 'temporalio/contrib/open_telemetry'

# ... assumes my_otel_tracer_provider is a tracer provider created by the user
my_tracer = my_otel_tracer_provider.tracer('my-otel-tracer')

my_client = Temporalio::Client.connect(
  'localhost:7233', 'my-namespace',
  interceptors: [Temporalio::Contrib::OpenTelemetry::TracingInterceptor.new(my_tracer)]
)
```

连接客户端后，将为所有客户端调用、活动和 Worker 上的 Workflow 调用创建跨度。
Span 是通过服务器创建和序列化的，以提供 Workflow Execution 的一个跟踪。

<a id="logging"></a>
## 从 Workflow {#logging} 记录
日志记录使您能够捕获并保留 Workflow 和 Activity 代码中的重要执行详细信息。

日志记录级别通常包括：

| 等级   | 使用                                                                                                       |
| ------- | --------------------------------------------------------------------------------------------------------- |
| `DEBUG` | 详细信息，通常可用于调试目的。                                            |
| `INFO`  | 有关应用程序操作的一般信息。                                                    |
| `WARN`  | 指示不会阻止应用程序运行的潜在有害情况或小问题。 |
| `ERROR` | 指示可能仍允许应用程序继续运行的错误情况。                    |

日志记录使用 Ruby 标准日志记录 API。
`logger`可以在连接客户端时设置。
以下示例显示控制台上的日志记录并将级别设置为 `INFO`。

```ruby
require 'logger'
require 'temporalio/client'

my_client = Temporalio::Client.connect(
  'localhost:7233', 'my-namespace',
  logger: Logger.new($stdout, level: Logger::INFO)
)
```

您可以使用 `Temporalio::Workflow.logger` 从 Workflow 登录，这是 Ruby 的 `Logger` 的一个特殊实例，
将 Workflow 详细信息附加到每个日志中，并且在重播期间不记录。

```ruby
Temporalio::Workflow.logger.info("Some log #{some_value}")
```

还有一种用于将 Activity 详细信息附加到每个日志的活动：

```ruby
Temporalio::Activity::Context.current.logger.info("Some log #{some_value}")
```

<a id="visibility"></a>
## 使用 Visibility API {#visibility}
Visibility 指用于列出、过滤和检查 Workflow Execution 的 Temporal 功能。

<a id="search-attributes"></a>
### 使用 Search Attributes {#search-attributes}
- [默认 Search Attribute](/search-attribute#default-search-attribute) 与 `WorkflowType`、`StartTime` 和 `ExecutionStatus` 一样会自动索引。
- [定制 Search Attribute](/search-attribute#custom-search-attribute) 允许您存储 Workflow 的特定于域的元数据。

检索 Workflow Execution 的典型方法是通过其 Workflow Id。

但是，有时您需要根据另一属性检索一个或多个 Workflow Execution。例如，假设您想要获取某个时间范围内失败的某种类型的所有 Workflow Execution，以便您可以使用相同的参数启动新的 Workflow Execution。

您可以使用 [Search Attributes](/search-attribute) 来完成此操作。

- [默认 Search Attribute](/search-attribute#default-search-attribute) 与 `WorkflowType`、`StartTime` 和 `ExecutionStatus` 一样，会自动添加到 Workflow Execution。
- [定制 Search Attribute](/search-attribute#custom-search-attribute) 可以包含其自己的特定于域的数据（如 `customerId` 或 `numItems`）。

使用自定义 Search Attribute 的步骤是：

- 在 CLI 或 Web UI 中的 Temporal 服务中创建新的 Search Attribute。
  - 例如：`Temporal operator search-attribute create --name CustomKeywordField --type Text`
    - 将 `CustomKeywordField` 替换为您的 Search Attribute 的名称。
    - 将 `Text` 替换为与 Search Attribute 关联的类型值：`Text` | `Keyword` | `Int` | `Double` | `Bool` | `Datetime` | `KeywordList`
- 设置 Workflow Execution 的 Search Attribute 值：
  - 在客户端上，在启动执行时将其作为参数包含在内。
  - 在 Workflow 中通过调用 `Temporalio::Workflow.upsert_search_attributes`。
- 读取Search Attribute的值：
  - 在客户端上，通过在 `WorkflowHandle` 上调用 `describe`。
  - 在 Workflow 中查看 `Temporalio::Workflow.search_attributes`。
- Query Workflow Execution 由 Search Attribute 使用 [列表过滤器](/list-filter) 实现：
  - [在Temporal CLI中](/cli/operator#list-2)
  - 在代码中调用 `list_Workflows`。

<a id="list-workflow-executions"></a>
### 列出 Workflow Execution {#list-workflow-executions}
在客户端上使用 [list_Workflows](https://ruby.temporal.io/temporalio/Client.html#list_workflows-instance_method) 方法并传递 [列表过滤器](/list-filter) 作为参数来过滤列出的 Workflow。
结果是一个惰性枚举器/可枚举。

```ruby
my_client.list_workflows("WorkflowType='GreetingWorkflow'").each do |wf|
  puts "Workflow: #{wf.id}"
end
```

<a id="custom-search-attributes"></a>
### 设置自定义 Search Attributes {#custom-search-attributes}
在 Temporal 服务（使用 `Temporal operator search-attribute create` 或云 UI）中创建自定义 Search Attribute 后，您可以在启动 Workflow 时设置自定义 Search Attribute 的值。

要设置自定义 Search Attribute，请使用 `start_Workflow` 或 `execute_Workflow` 的 `search_attributes` 参数。
应预定义密钥以供重用。

```ruby
# Predefined search attribute key, usually a global somewhere
MY_KEYWORD_KEY = Temporalio::SearchAttributes::Key.new(
  'my-keyword',
  Temporalio::SearchAttributes::IndexedValueType::KEYWORD
)

# ...

# Start workflow with the search attribute set
handle = my_client.start_workflow(
  MyWorkflow, 'some-input',
  id: 'my-workflow-id', task_queue: 'my-task-queue',
  search_attributes: Temporalio::SearchAttributes.new({ MY_KEYWORD_KEY => 'some-value' })
)
```

<a id="upsert-search-attributes"></a>
### 更新插入 Search Attributes {#upsert-search-attributes}
您可以更新插入 Search Attribute 以在 Workflow 代码中添加 Update 或删除 Search Attribute。

要更新插入自定义 Search Attribute，请将 [`upsert_search_attributes`](https://ruby.temporal.io/temporalio/Workflow.html#upsert_search_attributes-class_method) 方法与一组 Update 结合使用。
应预定义密钥以供重用。

```ruby
# Predefined search attribute key, usually a global somewhere
MY_KEYWORD_KEY = Temporalio::SearchAttributes::Key.new(
  'my-keyword',
  Temporalio::SearchAttributes::IndexedValueType::KEYWORD
)

# ...

class MyWorkflow < Temporalio::Workflow::Definition
  def execute
    # ...

    Temporalio::Workflow.upsert_search_attributes(MY_KEYWORD_KEY.value_set('some-new-value'))

    # ...
  end
end
```
