<a id="schedule-a-workflow"></a>
## Schedule 一个 Workflow {#schedule-a-workflow}

调度 Workflow 是自动化的一个重要方面。
通过调度 Workflow，您可以自动执行重复性任务，减少人工干预并确保及时执行。

使用以下操作来管理 Scheduled Workflow。

<a id="create-a-workflow"></a>
### 创建一个 Scheduled Workflow {#create-a-workflow}

创建操作使您能够创建新的 Schedule。创建新的 Schedule 时，会生成唯一的 Schedule ID，您可以使用该 ID 在其他 Schedule 命令中引用 Schedule。

要在 Ruby 中创建 Scheduled Workflow Execution，请使用 [创建_Schedule](https://ruby.temporal.io/temporalio/Client.html#create_schedule-instance_method)
客户端上的方法。
然后将 Schedule ID 和 Schedule 对象传递给该方法以创建 Scheduled Workflow Execution。
将 Schedule 的 `action` 成员设置为 `Temporalio::Client::Schedule::Action::StartWorkflow` 的实例，将 Schedule 设为 Workflow Execution。

```ruby
handle = my_client.create_schedule(
  'my_schedule_id',
  Temporalio::Client::Schedule.new(
    action: Temporalio::Client::Schedule::Action::StartWorkflow.new(
      MyWorkflow, 'some-input',
      id: 'my-workflow-id', task_queue: 'my-task-queue'
    ),
    spec: Temporalio::Client::Schedule::Spec.new(
      intervals: [
        Temporalio::Client::Schedule::Spec::Interval.new(
          every: 5 * 24 * 60 * 60.0, # 5 days
        )
      ]
    )
  )
)
```

:::tip Schedule Auto-Deletion

一旦 Schedule 完成创建其所有 Workflow Execution，Temporal 服务就会将其删除，因为它不会再次触发。
Temporal 服务不保证何时会发生此删除。

:::

<a id="backfill-a-scheduled-workflow"></a>
### 回填 Scheduled Workflow {#backfill-a-scheduled-workflow}

回填操作会在指定时间范围之前执行操作。当您需要执行错过或延迟的操作时，或者当您想要在 Scheduled 时间之前测试 Workflow 时，此命令非常有用。

要在 Ruby 中回填 Scheduled Workflow Execution，请使用 [回填](https://ruby.temporal.io/temporalio/Client/scheduleHandle.html#backfill-instance_method)
Schedule 手柄上的方法。

```ruby
handle = my_client.schedule_handle('my-schedule-id')
now = Time.now(in: 'UTC')
handle.backfill(
  Temporalio::Client::Schedule::Backfill.new(
    start_at: now - (4 * 60),
    end_at: now - (2 * 60),
    overlap: Temporalio::Client::Schedule::OverlapPolicy::ALLOW_ALL
  )
)
```

<a id="delete-a-scheduled-workflow"></a>
### 删除 Scheduled Workflow {#delete-a-scheduled-workflow}

删除操作使您能够删除 Schedule。删除 Schedule 时，不会影响由 Schedule 启动的任何 Workflow。

要在 Ruby 中删除 Scheduled Workflow Execution，请在 Schedule 句柄上使用 [删除](https://ruby.temporal.io/temporalio/Client/scheduleHandle.html#delete-instance_method) 方法。

```ruby
handle = my_client.schedule_handle('my-schedule-id')
handle.delete
```

<a id="describe-a-scheduled-workflow"></a>
### 描述 Scheduled Workflow {#describe-a-scheduled-workflow}

描述操作显示当前的 Schedule 配置，包括有关过去、当前和未来 Workflow 运行的信息。当您想要获取 Schedule 及其关联的 Workflow 运行的详细视图时，此命令非常有用。

要在 Ruby 中描述 Scheduled Workflow Execution，请在 Schedule 句柄上使用 [描述](https://ruby.temporal.io/temporalio/Client/scheduleHandle.html#describe-instance_method) 方法。

```ruby
handle = my_client.schedule_handle('my-schedule-id')
desc = handle.describe
puts "Schedule info: #{desc.info}"
```

<a id="list-a-scheduled-workflow"></a>
### 列出 Scheduled Workflow {#list-a-scheduled-workflow}

列表操作列出了所有可用的 Schedule。当您想要查看所有 Schedule 及其各自的 Schedule ID 的列表时，此命令非常有用。

要列出所有 Schedule，请在客户端上使用 [list_Schedules](https://ruby.temporal.io/temporalio/Client.html#list_schedules-instance_method) 异步方法。
这将返回一个枚举器/可枚举值。
如果添加或删除 Schedule，它可能不会立即在列表中可用。

```ruby
my_client.list_schedules.each do |sched|
  puts "Schedule info: #{sched}"
end
```

<a id="pause-a-scheduled-workflow"></a>
### 暂停 Scheduled Workflow {#pause-a-scheduled-workflow}

暂停操作使您能够暂停和取消暂停 Schedule。当您暂停 Schedule 时，与 Schedule 关联的所有未来 Workflow 运行都会暂时停止。当您由于维护或任何其他原因想要暂时停止 Workflow 时，此命令非常有用。

要在 Ruby 中暂停 Scheduled Workflow Execution，请在 Schedule 句柄上使用 [暂停](https://ruby.temporal.io/temporalio/Client/scheduleHandle.html#pause-instance_method) 方法。
您可以向 `pause` 方法传递注释以提供暂停 Schedule 的原因。

```ruby
handle = my_client.schedule_handle('my-schedule-id')
handle.pause(note: 'Pausing the schedule for now')
```

<a id="trigger-a-scheduled-workflow"></a>
### 触发 Scheduled Workflow {#trigger-a-scheduled-workflow}

触发操作会使用给定的 Schedule 触发立即操作。默认情况下，此操作受 Schedule 的重叠策略的约束。当您想要在 Scheduled 时间之外执行 Workflow 时，此命令很有用。

要在 Ruby 中触发 Scheduled Workflow Execution，请在 Schedule 句柄上使用 [扳机](https://ruby.temporal.io/temporalio/Client/scheduleHandle.html#trigger-instance_method) 方法。

```ruby
handle = my_client.schedule_handle('my-schedule-id')
handle.trigger
```

<a id="update-a-scheduled-workflow"></a>
### Update a Scheduled Workflow {#update-a-scheduled-workflow}

Update 操作使您能够 Update 现有的 Schedule。当您需要修改 Schedule 的配置（例如更改开始时间、结束时间或间隔）时，此命令非常有用。

要在 Ruby 中使用 Update 和 Scheduled Workflow Execution，请在 Schedule 句柄上使用 [Update](https://ruby.temporal.io/temporalio/Client/scheduleHandle.html#update-instance_method) 方法。
此方法接受一个块，该块本身接受一个 Update 输入对象，并预计返回一个带有新值的 Update
Schedule 至 Update，或 `nil` 至非 Update。

```ruby
handle = my_client.schedule_handle('my-schedule-id')
handle.update do |input|
  # Return a new schedule with the action updated
  Temporalio::Client::Schedule::Update.new(
    schedule: input.description.schedule.with(
      # Update the action
      action: Temporalio::Client::Schedule::Action::StartWorkflow.new(
        MyNewWorkflow, 'some-new-input',
        id: 'my-workflow-id', task_queue: 'my-task-queue'
      )
    )
  )
end
```

## 使用开始延迟 {#start-delay}

在特定的一次性未来点而不是重复出现的 Schedule 上使用 `start_delay` 至 Schedule 和 Workflow Execution。

在客户端中的 `start_Workflow` 或 `execute_Workflow` 方法上使用 `start_delay` 参数。

```ruby
handle = my_client.start_workflow(
  MyWorkflow, 'some-input',
  id: 'my-workflow-id', task_queue: 'my-task-queue',
  start_delay: 3 * 60 * 60 # 3 hours
)
```
