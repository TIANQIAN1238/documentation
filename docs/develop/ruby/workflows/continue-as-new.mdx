本页介绍如何使用 Temporal Ruby SDK 进行 Continue-As-New。

如果 Event History 中的事件数量变得太大，[Continue-As-New](/workflow-execution/continue-as-new) 使 Workflow Execution 能够成功关闭并在单个原子操作中创建新的 Workflow Execution。
使用 Continue-As-New 生成的 Workflow Execution 具有相同的 Workflow Id、新的运行 Id 和新的 Event History，并传递所有适当的参数。

:::caution

作为预防措施，Workflow Execution 的 Event History 仅限于 [51,200 场活动](https://github.com/temporalio/temporal/blob/e3496b1c51bfaaae8142b78e4032cc791de8a76f/service/history/configs/config.go#L382) 或 [50MB](https://github.com/temporalio/temporal/blob/e3496b1c51bfaaae8142b78e4032cc791de8a76f/service/history/configs/config.go#L380)，并将在 10,240 个事件或 10 MB 后向您发出警告。

:::

要防止 Workflow Execution Event History 超出此限制并失败，请使用 Continue-As-New 以新的 Event History 启动新的 Workflow Execution。

非常大的 Event History 会对 Workflow Execution 的性能产生不利影响。
例如，如果 Workflow Worker 发生故障，则必须从 Temporal 服务中提取完整的 Event History，并通过 Workflow 任务将其提供给另一个 Worker。
如果 Event History 非常大，则加载可能需要一些时间。

Continue-As-New 功能使开发人员能够自动完成当前的 Workflow Execution 并启动新的 Workflow Execution。

新的 Workflow Execution 具有相同的 Workflow Id，但运行 Id 不同，并具有自己的 Event History。

<a id="continue-as-new"></a>
## Ruby 中的 Continue-As-New {#continue-as-new}

对于 Ruby 中的 Continue-As-New，从 Workflow 内部引发 `Temporalio::Workflow::ContinueAsNewError`，这将立即停止 Workflow 和 Continue-As-New。

```ruby
raise Temporalio::Workflow::ContinueAsNewError.new('my-new-arg')
```

:::warning Using Continue-as-New and Updates

- Temporal_不_支持 Update 处理程序中的 Continue-As-New 功能。
- 使用 Continue-As-New_之前_完成所有处理程序。
- 从您的主要 Workflow Definition 方法中使用 Continue-As-New，就像完成或失败 Workflow Execution 一样。

:::
