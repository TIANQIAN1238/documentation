本页介绍如何使用 Temporal Ruby SDK 设置耐用 Timer。

[耐用 Timer](/workflow-execution/timers-delays) 用于暂停指定的 Workflow 的执行
持续时间。 Workflow 可以休眠数天甚至数月。 Timer 是持久的，因此即使您的 Worker 或 Temporal 服务
当时间段结束时，一旦您的 Worker 和 Temporal 服务恢复，持久 Timer 呼叫
将解决并且您的代码将继续执行。

睡眠是一种资源消耗较少的操作：它不会占用进程，并且您可以在单个进程上运行数百万个 Timer
Worker。

要在 Workflow 中添加 Timer，请使用 `Temporalio::Workflow.sleep`。 _技术上_`Kernel#sleep` 可以工作，但 Workflow
表单允许设置要在 UI 中查看的摘要。

```ruby
# Sleep for 72 hours
Temporalio::Workflow.sleep(72 * 60 * 60, summary: 'my timer')
```

还有一个 `Temporalio::Workflow.timeout` 方法，它接受一个块并且像标准 Ruby 一样工作
`Timeout.timeout` 如果需要使一组代码超时的能力。
