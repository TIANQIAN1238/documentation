:::info tctl is deprecated

tctl 命令行实用程序已被弃用，并且不再主动支持。
我们建议过渡到 [Temporal CLI](/cli) 以继续使用和访问新功能。

感谢您成为 Temporal 社区的重要组成部分。

:::

**如何运行 tctl 批处理命令。**

`tctl batch` 命令使您能够使用单个命令影响多个现有的 [Workflow Executions](/workflow-execution)。
批处理作业在后台运行，一次影响一个 Workflow Execution。

使用 [tctl批量启动](#start) 启动批处理作业。

:::note

`tctl-v1` 可以运行 `batch` 和 `batch-v2` 命令。

:::

启动批处理作业时，您必须提供 [列表过滤器](/list-filter) 以及应发生的批处理作业的类型。
批处理作业在后台运行，一次影响一个 Workflow Execution。

列表过滤器标识受批处理作业影响的 Workflow Execution 集合。
`tctl batch start` 命令显示有多少个 Workflow Execution 将受到批处理作业的影响，并要求您在继续之前进行确认。

批次类型决定了您必须提供的其他参数以及受影响的参数。
批处理作业分为三种类型：

- Signal：将 Signal 发送到列表过滤器指定的 Workflow Execution 集合。
- 取消：取消列表过滤器指定的Workflow Execution 集合。
- 终止：终止列表过滤器指定的 Workflow Execution 集合。

成功启动的批处理作业会返回作业 ID。
您可以在 `tctl batch describe` 命令中使用此作业 ID，它描述了特定批处理作业的进度。

您还可以使用作业 ID 来终止批处理作业本身。
终止批处理作业不会回滚批处理作业已执行的操作。

## 开始 {#start}

`tctl batch start` 命令启动批处理作业。

`tctl batch start --Query <value> <modifiers>`

以下修饰符控制命令的行为。

<a id="start"></a>
### `--Query` {#query}

_所需修饰符_

指定该批处理作业应运行的 [Workflow Executions](/workflow-execution)。

[Search Attributes](/search-attribute) 的类似 SQL 的 Query 与 `tctl Workflow list --Query` 命令使用的相同。

别名：`-q`

**例子**

```bash
tctl batch start --query <value>
```

<a id="query"></a>
### `--reason` {#reason}

指定运行此批处理作业的原因。

**例子**

```bash
tctl batch start --query <value> --reason <string>
```

<a id="reason"></a>
### `--batch_type` {#batchtype}

指定该批处理作业执行的操作。支持的操作有 `Signal`、`cancel` 和 `terminate`。

**例子**

```bash
tctl batch start --query <value> --batch_type <operation>
```

<a id="batchtype"></a>
### `--Signal_name` {#signalname}

指定 [Signal](/sending-messages#sending-signals) 的名称。当 `--batch_type` 为 `Signal` 时需要此修饰符。

**例子**

```bash
tctl batch start --query <value> --batch_type signal --signal_name <name>
```

<a id="signalname"></a>
### `--input` {#input}

传递 [Signal](/sending-messages#sending-signals) 的输入。输入必须为 JSON 格式。

别名：`-i`

**例子**

```bash
tctl batch start --query <value> --input <json>
```

<a id="input"></a>
### `--rps` {#rps}

指定处理的RPS。默认值为 50。

**例子**

```bash
tctl batch start --query <value> --rps <value>
```

<a id="rps"></a>
### `--yes` {#yes}

禁用确认提示。

别名：`y`

**例子**

```bash
tctl batch start --query <value> --yes
```

## 列表 {#list}

`tctl batch list` 命令列出所有批处理作业。

`tctl batch list <modifiers>`

:::note

`tctl-v1` 可以运行 `batch` 和 `batch-v2` 命令。

:::

以下修饰符控制命令的行为。

<a id="yes"></a>
### --页面大小 {#pagesize}

指定要在页面上列出的批处理作业的最大数量。默认值为 30。

**例子**

```bash
tctl batch list --pagesize <value>
```

## 描述 {#describe}

`tctl batch describe` 命令描述批处理作业的进度。

`tctl batch describe --job_id <id>`

:::note

`tctl` 可以运行 `batch` 和 `batch-v2` 命令。

:::

以下修饰符控制命令的行为。

<a id="list"></a>
### --job_id {#jobid}

_所需修饰符_

指定批处理作业的作业 ID。

**例子**

```bash
tctl batch describe --job_id <id>
```

<a id="pagesize"></a>
## 终止 {#terminate}

`tctl batch terminate` 命令终止批处理作业。

`tctl batch terminate --job_id <id> <modifiers>`

:::note

`tctl-v1` 可以运行 `batch` 和 `batch-v2` 命令。

:::

以下修饰符控制命令的行为。

<a id="describe"></a>
### `--job_id` {#jobid}

_所需修饰符_

指定批处理作业的作业 ID。

**例子**

```bash
tctl batch terminate --job_id <id>
```

<a id="jobid"></a>
### `--reason` {#reason}

指定终止该批处理作业的原因。

**例子**

```bash
tctl batch terminate --job_id <id> --reason <string>
```
