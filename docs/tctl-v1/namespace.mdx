tctl 命令行实用程序已被弃用，并且不再主动支持。
我们建议过渡到 [Temporal CLI](/cli) 以继续使用和访问新功能。

## 描述 {#describe}

`tctl Namespace describe` 命令描述 [Namespace](/namespaces)。

`tctl Namespace describe`

以下修饰符控制命令的行为。

<a id="describe"></a>
### --Namespace_id {#namespaceid}

指定Namespace的ID进行描述。

除非指定了全局 `--Namespace` 修饰符 (`tctl --Namespace <name> describe`)，否则需要此修饰符。

**例子**

```bash
tctl namespace describe --namespace_id <id>
```

[全球 Namespace](/global-namespace) 的示例结果

```bash
$ tctl --ns canary-namespace n desc
Name: canary-namespace
Description: testing namespace
OwnerEmail: dev@yourtech.io
NamespaceData:
Status: REGISTERED
RetentionInDays: 7
EmitMetrics: true
ActiveClusterName: dc1
Clusters: dc1, dc2
```

## 列表 {#list}

`tctl Namespace list` 命令列出所有 [Namespaces](/namespaces)。

`tctl Namespace list`

该命令没有修饰符。

## 登记 {#register}

`tctl Namespace register` 命令注册 [Namespace](/namespaces)。

`tctl Namespace register`

默认情况下，Temporal 使用“默认”Namespace。
使用以下命令创建并注册新的 Namespace：

```bash
tctl --namespace your-namespace namespace register
# OR using short alias
tctl --ns your-namespace n re
```

以下修饰符控制命令的行为。

<a id="namespaceid"></a>
### --active_cluster {#activecluster}

注册 [Namespace](/temporal-service) 时指定活动 [Temporal集群](/namespaces) 的名称。
当发生故障转移时，全局 Namespace 的该值会发生变化。

**例子**

```bash
tctl namespace register --active_cluster <name>
```

<a id="list"></a>
### --集群 {#clusters}

注册 [Namespace](/temporal-service) 时指定 [Temporal 集群](/namespaces) 列表。

该列表包含 Namespace 可以故障转移到的集群名称（以空格分隔）。
确保包含到当前活动的集群中。
这是只读设置，无法更改。

仅当 `--global_Namespace` 修饰符设置为 true 时，此修饰符才有效。

**例子**

```bash
tctl namespace register --clusters <names>
```

### - 描述 {#description}

注册 [Namespace](/namespaces) 时指定说明。

**例子**

```bash
tctl namespace register --description <value>
```

<a id="register"></a>
### --global_Namespace {#globalnamespace}

指定 [Namespace](/namespaces) 是否为 [全球 Namespace](/global-namespace)。
启用后，它控制 Update 上复制任务的创建，从而允许跨集群复制状态。
这是只读设置，无法更改。

**例子**

```bash
tctl namespace register --global_namespace <boolean>
```

<a id="activecluster"></a>
### --history_archival_state {#historyarchivalstate}

设置[档案](/temporal-service/archival)的状态。
有效值为 `disabled` 和 `enabled`。

**例子**

```bash
tctl namespace register --history_archival_state <value>
```

<a id="clusters"></a>
### --history_uri {#historyuri}

指定 [档案](/temporal-service/archival) 的 URI。
首次启用存档后，无法更改 URI。

**例子**

```bash
tctl namespace register --history_uri <uri>
```

<a id="description"></a>
### --Namespace_data {#namespacedata}

以键值对的形式指定 [Namespace](/namespaces) 的数据（例如 `k1:v1,k2:v2,k3:v3`）。

**例子**

```bash
tctl namespace register --namespace_data <data>
```

<a id="globalnamespace"></a>
### --所有者电子邮件 {#owneremail}

指定 [Namespace](/namespaces) 所有者的电子邮件地址。

**例子**

```bash
tctl namespace register --owner_email <value>
```

### - 保留 {#retention}

将 [保留期限](/temporal-service/temporal-server#retention-period) 设置为 [Namespace](/namespaces)。

保留期适用于已关闭的 [Workflow Executions](/workflow-execution)。

**例子**

```bash
tctl namespace register --retention <value>
```

<a id="historyarchivalstate"></a>
### --Visibility_archival_state {#visibilityarchivalstate}

设置 [档案](/temporal-service/archival) 的 Visibility 状态。
有效值为 `disabled` 和 `enabled`。

**例子**

```bash

```

<a id="historyuri"></a>
### --Visibility_uri {#visibilityuri}

指定 [档案](/temporal-service/archival) 的 Visibility URI。
首次启用存档后，无法更改 URI。

**例子**

```bash

```

## Update {#update}

`tctl Namespace Update` 命令 Update 是 [Namespace](/namespaces)。

`tctl Namespace Update`

以下修饰符控制命令的行为。

<a id="owneremail"></a>
### --active_cluster {#activecluster}

更新 [Namespace](/temporal-service) 时指定活动 [Temporal集群](/namespaces) 的名称。

**例子**

```bash
tctl namespace update --active_cluster <name>
```

<a id="retention"></a>
### --add_bad_binary {#addbadbinary}

添加二进制校验和以在重置 [Workflow Execution](/workflow-execution) 时使用。
Temporal 不会将任何 [命令](/workflow-execution#command) 分派给给定的二进制文件。

另请参见 [`--remove_bad_binary`](#--remove_bad_binary)。

**例子**

```bash
tctl namespace update --add_bad_binary <value>
```

<a id="visibilityarchivalstate"></a>
### --集群 {#clusters}

该列表包含 Namespace 可以故障转移到的集群名称（以空格分隔）。

仅当 `--global_Namespace` 修饰符设置为 true 时，此修饰符才有效。

**例子**

```bash
tctl namespace update --clusters <names>
```
