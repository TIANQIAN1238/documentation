### 添加搜索属性 {#addsearchattributes}

`tctl admin cluster add-search-attributes` 命令允许将 Search Attribute 添加到集群。
自定义 Search Attribute 可用于使集群更易于识别。

:::note
由于 Elasticsearch 限制，您只能添加新的自定义 Search Attribute。现有的 Search Attribute 无法重命名或从 Elasticsearch 索引中删除。
:::

使用此命令将自定义 Search Attribute 添加到 Temporal 集群：

```bash
tctl admin cluster add-search-attributes --name <SearchAttributeName> --type <SearchAttributeValueType>
```

:::note
如果要将自定义 Search Attribute 添加到从 [Temporalio/docker-compose](https://github.com/temporalio/docker-compose) 存储库中的 `docker-compose-es.yml` 文件运行的集群，请确保将 Docker 内存增加到 6 GB 以上。
:::

<a id="cluster"></a>
#### --skip_schema_Update {#skipschemaupdate}

允许用户跳过 Elasticsearch 索引架构 Update。

:::note
这只会在元数据中注册。
:::

#### - 姓名 {#name}

要添加的 Search Attribute 的名称。名称可以有多个值。

Search Attribute 名称区分大小写。

#### - 类型 {#type}

要添加的 Search Attribute 类型。
可以一次添加多个值。

值：文本、关键字、Int、Double、Bool、日期时间

### 删除搜索属性 {#removesearchattributes}

> Temporal tctl 文档涵盖 Temporal CLI 版本 1.17。

`tctl admin cluster remove-search-attributes` 命令从集群中删除自定义 Search Attribute 元数据。
此操作对 Elasticsearch 索引架构没有影响。

使用以下命令从集群的元数据中删除 [Search Attribute](/search-attribute)：

```bash
tctl admin cluster remove-search-attributes --name <SearchAttributeKey>
```

只能从集群的元数据中删除自定义 Search Attribute。
默认的 Search Attribute 无法删除。

删除 Search Attribute 会将其从集群的元数据中删除，但不会将其从 Elasticsearch 索引中删除。
这意味着 Search Attribute 可以稍后作为相同类型添加回来。
Search Attribute 添加到 Elasticsearch 索引后就无法更改。

以下修饰符更改操作的行为：

#### - 姓名 {#name}

要删除的 Search Attribute 的名称。
