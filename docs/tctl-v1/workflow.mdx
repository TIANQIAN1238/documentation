## 取消 {#cancel}

`tctl Workflow cancel --Query` 命令取消 [Workflow Execution](/workflow-execution)。

取消正在运行的 Workflow Execution 会在历史记录中记录 `WorkflowExecutionCancelRequested` 事件。
新的 [Workflow 任务](/tasks#Workflow-task) 将是 Scheduled。
取消后，Workflow Execution 可以执行清理工作。

另请参见 [`tctl Workflow terminate --Query`](#terminate)。

`tctl Workflow cancel --Query <Query> <modifiers>`

以下修饰符控制命令的行为。

<a id="cancel"></a>
### --Workflow_id {#workflowid}

指定 [Workflow ID](/workflow-execution/workflowid-runid#Workflow-id)。

别名：`-w`

**例子**

```bash
tctl workflow cancel --workflow_id <id>
```

<a id="workflowid"></a>
### --run_id {#runid}

指定 [运行 ID](/workflow-execution/workflowid-runid#run-id)。

别名：`-r`

**例子**

```bash
tctl workflow cancel --run_id <id>
```

## 计数 {#count}

`tctl Workflow count` 命令对 [Workflow Executions](/workflow-execution) 进行计数。
此命令需要启用 Elasticsearch。

`tctl Workflow count <modifiers>`

以下修饰符控制命令的行为。

<a id="runid"></a>
### --Query {#query}

_所需修饰符_

指定 [Search Attributes](/search-attribute) 的类似 SQL 的 Query。

别名：`-q`

**例子**

要计算所有打开的 [Workflow Executions](/workflow-execution)：

```bash
tctl workflow count --query 'ExecutionStatus="Running"'
```

## 描述 {#describe}

`tctl Workflow describe` 命令显示有关 [Workflow Execution](/workflow-execution) 的信息。
例如，此信息可用于定位失败的 Workflow Execution。

要查找具有给定运行 ID 的 Workflow，请参阅 [`tctl Workflow describeid`](#describeid)。

`tctl Workflow describe <modifiers>`

以下修饰符控制命令的行为。
执行此命令时始终包含所需的修饰符。

<a id="count"></a>
### --Workflow_id {#workflowid}

**这是必需的修饰符。**

指定 [Workflow ID](/workflow-execution/workflowid-runid#Workflow-id)。

别名：`-w`

**例子**

```bash
tctl workflow describe --workflow_id <id>
```

<a id="query"></a>
### --run_id {#runid}

指定 [运行 ID](/workflow-execution/workflowid-runid#run-id)。
如果未提供运行 ID，该命令将显示该 Workflow Id 的最新 Workflow Execution。

别名：`-r`

**例子**

```bash
tctl workflow describe --run_id <id>
```

<a id="describe"></a>
### --print_raw {#printraw}

完全按照存储的方式打印属性。

**例子**

```bash
tctl workflow describe --print_raw
```

<a id="workflowid"></a>
### --仅重置点 {#resetpointsonly}

仅显示符合重置条件的事件。
如果成功，该命令将返回所有部署的运行 ID 以及创建事件的时间。

**例子**

```bash
tctl workflow describe --reset_points_only
```

<a id="printmemo"></a>
### --最早时间 {#earliesttime}

指定列出的最早开始时间。
支持的格式如下：

- `<year>-<month>-<day>T<hour>:<minute>:<second><+|-><offsethours>:<offsetminutes>`
- 原始 Unix 纪元时间（自 1970 年 1 月 1 日 0000 UTC 以来的毫秒数）。
- `<n><duration`，其中 `<n>` 是 0 到 1000000 之间的值，`<duration>` 是以下之一：
  - `second` 或 `s`
  - `minute` 或 `m`
  - `hour` 或 `h`
  - `day` 或 `d`
  - `week` 或 `w`
  - `month` 或 `M`
  - `year` 或 `y`

**示例**

要指定 2022 年 1 月 2 日印度标准时间下午 3:04:05：

```bash
tctl workflow list --earliest-time '2022-01-02T15:04:05+05:30'
```

要指定当前时间之前 15 分钟：

```bash
tctl workflow list --earliest-time '15minute'
```

### --Query {#query}

**如何使用 tctl 列出和过滤带有 [列表过滤器](/list-filter) 的 Workflow Execution。**

仅当使用集群配置 [高级Visibility](/visibility#advanced-Visibility) 时，才支持 `--Query` 标志。

使用 `--Query` 选项会导致 tctl 忽略所有其他筛选器选项，包括 `open`、`earliest_time`、`latest_time`、`Workflow_id` 和 `Workflow_type`。

别名：`-q`

**例子**

```bashbash
tctl workflow list --query "WorkflowId=<your-workflow-id>"
```

更多示例：

```bashbash
tctl workflow list \
  --query "WorkflowType='main.SampleParentWorkflow' AND ExecutionStatus='Running'"
```

```bashbash
tctl workflow list \
  --query '(CustomKeywordField = "keyword1" and CustomIntField >= 5) or CustomKeywordField = "keyword2"' \
  --print_search_attr
```

```bashbash
tctl workflow list \
  --query 'CustomKeywordField in ("keyword2", "keyword1") and CustomIntField >= 5 and CloseTime between "2018-06-07T16:16:36-08:00" and "2019-06-07T16:46:34-08:00" order by CustomDatetimeField desc' \
  --print_search_attr
```

```bashbash
tctl workflow list \
  --query 'WorkflowType = "main.Workflow" and (WorkflowId = "1645a588-4772-4dab-b276-5f9db108b3a8" or RunId = "be66519b-5f09-40cd-b2e8-20e4106244dc")'
```

```bashbash
tctl workflow list \
  --query 'WorkflowType = "main.Workflow" StartTime > "2019-06-07T16:46:34-08:00" and ExecutionStatus = "Running"'
```

别名：`q`

`tctl Workflow Query` 命令将 [Query](/sending-messages#sending-queries) 发送到 [Workflow Execution](/workflow-execution)。

查询可用于检索具有给定参数的全部或部分 Workflow 状态。

```bash
$ tctl workflow query --workflow_id "HelloQuery" --query_type "getCount"
Query result as JSON:
3
```

查询也可用于已完成的 Workflow。
让我们通过更新问候语来完成 Workflow，然后更新 Query 现已完成的 Workflow。

```bash
$ tctl workflow signal --workflow_id "HelloQuery" --name "updateGreeting" --input \"Bye\"
Signal workflow succeeded.
$ tctl workflow query --workflow_id "HelloQuery" --query_type "getCount"
Query result as JSON:
4
```

查询写法如下：

`tctl Workflow Query --Workflow_id [modifiers]`

以下修饰符控制命令的行为。
执行此命令时始终包含所需的修饰符。

## Signal {#signal}

`tctl Workflow Signal` 命令 [Signals](/sending-messages#sending-Signals) 和 [Workflow Execution](/workflow-execution)。

Workflow 通过 Signal 名称侦听 Signal，并且可以使其侦听一个或多个 Signal 名称。
Workflow 还可以侦听 SQL 查询。

下面的 Workflow 侦听“HelloSignal”的实例：

```bash
tctl workflow start  --workflow_id "HelloSignal" --taskqueue HelloWorldTaskQueue --workflow_type HelloWorld --execution_timeout 3600 --input \"World\"
```

Worker 将在收到 Signal 后返回此输出：

```text
13:57:44.258 [workflow-method] INFO  c.t.s.javaquickstart.GettingStarted - 1: Hello World!
```

Signal 还可用于更改变量值。

```bash
tctl workflow signal --workflow_id "HelloSignal" --name "updateGreeting" --input \"Hi\"
```

输出将从收到的第一个 Signal 开始发生变化。

```text
13:57:44.258 [workflow-method] INFO  c.t.s.javaquickstart.GettingStarted - 1: Hello World!
13:58:22.352 [workflow-method] INFO  c.t.s.javaquickstart.GettingStarted - 2: Hi World!
```

当发送 Signal 时，会产生一个等待条件来阻止任何包含相同输入值的 Signal。
但是，在我们的示例中更改问候语可以解除阻止：

```bash
tctl workflow signal --workflow_id "HelloSignal" --name "updateGreeting" --input \"Welcome\"
```

Worker 输出：

```text
13:57:44.258 [workflow-method] INFO  c.t.s.javaquickstart.GettingStarted - 1: Hello World!
13:58:22.352 [workflow-method] INFO  c.t.s.javaquickstart.GettingStarted - 2: Hi World!
13:59:29.097 [workflow-method] INFO  c.t.s.javaquickstart.GettingStarted - 3: Welcome World!
```

发送 Signal 不需要运行 Worker。

```bash
tctl workflow signal --workflow_id "HelloSignal" --name "updateGreeting" --input \"Welcome\"
```

CLI 输出：

```text
Signal workflow succeeded.
```

Signal 请求在 Temporal Server 内排队，直到 Worker 重新启动。
如果给定的 Signal 包含与之前相同的输入，则排队的 Signal 将被忽略。

通过发送带有“再见”问候语的 Signal 来完成 Workflow：

```bash
tctl workflow signal --workflow_id "HelloSignal" --name "updateGreeting" --input \"Bye\"
```

检查 Workflow Execution 是否已完成。

```bash
tctl workflow showid HelloSignal
```

Signals的写法如下：

```bash
tctl workflow signal --workflow_id <id> <modifiers>
```

或者

```bash
tctl workflow signal --query <query> <modifiers>
```

`tctl Workflow start` 命令启动一个新的 [Workflow Execution](/workflow-execution)。

`tctl Workflow start <modifiers>`

## 终止 {#terminate}

`tctl Workflow terminate` 命令终止 [Workflow Execution](/workflow-execution)。

终止正在运行的 Workflow Execution 将 `WorkflowExecutionTerminated` 事件记录为历史记录中的关闭事件。
[Workflow 任务](/tasks#Workflow-task) 将不再是 Scheduled。

另请参见 [`tctl Workflow cancel`](#cancel)。

`tctl Workflow terminate --Query <modifiers>`

以下修饰符控制命令的行为。

<a id="name"></a>
### --Workflow_id {#workflowid}

_所需修饰符_

指定 [Workflow ID](/workflow-execution/workflowid-runid#Workflow-id)。

别名：`-w`

**例子**

```bash
tctl workflow terminate --workflow_id <id>
```

<a id="input"></a>
### --run_id {#runid}

指定 [运行 ID](/workflow-execution/workflowid-runid#run-id)。

如果未指定 `run_id`，则 `tctl` 终止指定 `Workflow_id` 的最后一个 Workflow Execution。

别名：`-r`

**例子**

```bash
tctl workflow terminate --run_id <id>
```

### - 原因 {#reason}

指定终止 [Workflow Execution](/workflow-execution) 的原因。

**例子**

```bash
tctl workflow terminate --workflow_id --reason <string>
```
