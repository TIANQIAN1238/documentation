`tctl Activity` 命令启用 [Activity 执行](/activity-execution) 操作。

- [tctl Activity 完整](#complete)
- [tctl Activity 失败](#fail)

## 完全的 {#complete}

`tctl Activity complete` 命令完成 [Activity 执行](/activity-execution)。

`tctl Activity complete <modifiers>`

以下修饰符控制命令的行为。

<a id="complete"></a>
### --Workflow_id {#workflowid}

指定要完成的 [Activity 执行](/workflow-execution/workflowid-runid#workflow-id) 的 [Workflow ID](/activity-execution)。

别名：`-w`

**例子**

```bash
tctl activity complete --workflow_id <id>
```

<a id="workflowid"></a>
### --run_id {#runid}

指定要完成的 [Activity 执行](/workflow-execution/workflowid-runid#run-id) 的 [运行 ID](/activity-execution)。

别名：`-r`

**例子**

```bash
tctl activity complete --run_id <id>
```

<a id="runid"></a>
### --Activity_id {#activityid}

指定要完成的 [Activity 执行](/activity-execution#activity-id) 的 [Activity ID](/activity-execution)。

**例子**

```bash
tctl activity complete --activity_id <id>
```

### - 结果 {#result}

指定使用 tctl 完成 Activity 执行时 [Activity 执行](/activity-execution) 的结果。

**例子**

```bash
tctl activity complete --result <value>
```

### - 身份 {#identity}

使用 tctl 完成 [Activity 执行](/activity-execution) 时指定操作员的身份。

**例子**

```bash
tctl activity complete --identity <value>
```

## 失败 {#fail}

`tctl Activity fail` 命令失败 [Activity 执行](/activity-execution)。

`tctl Activity fail [<modifiers>]`

以下修饰符控制命令的行为。

<a id="activityid"></a>
### --Workflow_id {#workflowid}

指定 [Activity 执行](/workflow-execution/workflowid-runid#workflow-id) 的 [Workflow ID](/activity-execution) 失败。

别名：`-w`

**例子**

```bash
tctl activity fail --workflow_id <id>
```

<a id="result"></a>
### --run_id {#runid}

指定 [Activity 执行](/workflow-execution/workflowid-runid#run-id) 的 [运行 ID](/activity-execution) 失败。

别名：`-r`

**例子**

```bash
tctl activity fail --run_id <id>
```

<a id="identity"></a>
### --Activity_id {#activityid}

指定 [Activity 执行](/activity-execution#activity-id) 的 [Activity ID](/activity-execution) 失败。

**例子**

```bash
tctl activity fail --activity_id <id>
```

### - 原因 {#reason}

指定 [Activity 执行](/activity-execution) 失败的原因。

**例子**

```bash
tctl activity fail --reason <value>
```

### - 细节 {#detail}

指定 [Activity 执行](/activity-execution) 失败原因的详细信息。

**例子**

```bash
tctl activity fail --detail <value>
```

### - 身份 {#identity}

使用 tctl 使 [Activity 执行](/activity-execution) 失败时指定操作员的身份。

**例子**

```bash
tctl activity complete --identity <value>
```
