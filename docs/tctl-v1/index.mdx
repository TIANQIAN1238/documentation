tctl 命令行实用程序已被弃用，并且不再主动支持。
我们建议过渡到 [Temporal CLI](/cli) 以继续使用和访问新功能。

感谢您成为 Temporal 社区的重要组成部分。

:::

:::note

本文档反映 tctl 版本 1.17。

:::

Temporal CLI (tctl) 是一个命令行工具，可用于与 Temporal 集群交互。
它可以执行 [Namespace](/namespaces) 操作（如寄存器、Update 和描述）和 [Workflow](/workflows) 操作（如启动
Workflow，显示 Workflow 历史记录和 Signal Workflow）。

## 如何安装 tctl {#install}

> Temporal tctl 文档涵盖 Temporal CLI 版本 1.17。

您可以通过以下方式安装[tctl](/tctl-v1)。

- 使用 [自制](https://brew.sh/) 进行本地安装：`brew install tctl`
- 与 [Docker 组合](https://github.com/temporalio/docker-compose) 中的 Temporal Server 一起在本地运行：`docker exec Temporal-admin-tools tctl YOUR COMMANDS HERE`
  - 要像本地安装一样调用 [tctl](/tctl-v1)（例如 `tctl Namespace describe`），请设置别名：`alias tctl="docker exec Temporal-admin-tools tctl"`
- 运行 [Temporal-管理工具](https://hub.docker.com/r/temporalio/admin-tools) Docker 映像：
  - 在 Linux 上：`docker run --rm -it --entrypoint tctl --network host --env Temporal_CLI_ADDRESS=localhost:7233 Temporalio/admin-tools:1.14.0`
  - 在 macOS 或 Windows 上：`docker run --rm -it --entrypoint tctl --env Temporal_CLI_ADDRESS=host.docker.internal:7233 Temporalio/admin-tools:1.14.0`
  - 如果您的 Temporal Server 在远程主机上运行，请更改 `Temporal_CLI_ADDRESS` 的值。
  - 要简化命令行，请创建 `tctl` 别名。
- 在 `GOPATH` 中安装最新版本的 tctl：`go install github.com/temporalio/tctl/cmd/tctl@latest`

**注意：** 要使用 [tctl](/tctl-v1)，您必须运行 Temporal Server。

要查看 [tctl](/tctl-v1) 命令的帮助，请输入以下命令。

| 命令             | 描述                                            |
| ------------------- | ------------------------------------------------------ |
| `tctl -h`           | 显示顶级命令和全局选项的帮助 |
| `tctl Namespace -h` | 显示 [Namespace](/namespaces) 操作的帮助   |
| `tctl Workflow -h`  | 显示 [Workflow](/workflows) 操作的帮助     |
| `tctl taskqueue -h` | 显示 [Task Queue](/task-queue) 操作的帮助  |

## 全局修饰符 {#global-modifiers}

您可以通过设置 [环境变量](#environment-variables) 来为其中许多修饰符提供值，而不是在 tctl 命令中包含修饰符。

### - 地址 {#address}

指定前端服务的主机和端口。
默认值为 `127.0.0.1:7233`。

<a id="address"></a>
### --自动确认 {#autoconfirm}

自动确认所有提示。

<a id="autoconfirm"></a>
### --context_timeout {#contexttimeout}

指定 RPC 调用上下文的超时（以秒为单位）。
默认值为 5。

<a id="contexttimeout"></a>
### --data_converter_plugin {#dataconverterplugin}

指定自定义 Data Converter 插件的可执行文件的名称。

<a id="dataconverterplugin"></a>
### --headers_provider_plugin {#headersproviderplugin}

指定标头提供程序插件的可执行文件的名称。

### - 帮助 {#help}

在 CLI 中显示 tctl 的帮助。

别名：`-h`

<a id="headersproviderplugin"></a>
### --Namespace {#namespace}

指定 Namespace。
通过使用此修饰符，您无需为子命令指定 `--Namespace` 修饰符。
默认 Namespace 为 `default`。

别名：`--n`

<a id="help"></a>
### --tls_ca_path {#tlscapath}

指定服务器证书颁发机构 (CA) 证书文件的路径。

<a id="namespace"></a>
### --tls_cert_path {#tlscertpath}

指定用于相互 TLS 身份验证的公共 X.509 证书文件的路径。
如果使用此修饰符，则还必须使用 `--tls_key_path` 修饰符。

<a id="tlscapath"></a>
### --tls_disable_host_verification {#tlsdisablehostverification}

禁用服务器证书的验证（以及主机验证）。

<a id="tlscertpath"></a>
### --tls_key_path {#tlskeypath}

指定用于双向 TLS 身份验证的私钥文件的路径。
如果使用此修饰符，则还必须使用 `--tls_cert_path` 修饰符。

<a id="tlsdisablehostverification"></a>
### --tls_服务器_名称 {#tlsservername}

指定用于 TLS 主机验证的目标服务器名称的替代。
该名称必须是服务器 TLS 证书中列出的 DNS 名称之一。
指定此修饰符还可以启用主机验证。

### - 版本 {#version}

在 CLI 中显示 tctl 的版本。

<a id="tlskeypath"></a>
### --codec_端点 {#codecendpoint}

Codec Server 的 URL 和端口号。

<a id="tlsservername"></a>
