Temporal Cloud 为帐户所有者和全局管理员 [角色](/cloud/manage-access/roles-and-permissions#account-level-roles) 提供创建名为服务帐户的计算机身份的选项。

服务帐户是 Temporal Cloud 中的一种身份类型。
Temporal Cloud 支持用户身份作为使用 Temporal Cloud 的人类用户的表示。
服务帐户使 Temporal Cloud 帐户所有者和全局管理员 [角色](/cloud/manage-access/roles-and-permissions#account-level-roles) 能够创建用于机器身份验证的身份，该身份与人类用户无关。

添加服务帐户后，Temporal Cloud 现在支持 2 种身份类型：

- 用户（与人相关，通过电子邮件地址或 ID 进行识别）
- 服务帐户（不与人绑定，电子邮件地址可选，通过名称或 ID 标识）

服务帐户使用 API 密钥作为连接到 Temporal Cloud 的身份验证机制。
在向 Temporal Cloud 进行操作自动化验证或向 Temporal SDK 和 Temporal CLI 进行 Workflow Execution 和管理验证时，您应该使用服务帐户来代表非人类身份。

:::tip

Namespace 管理员现在可以管理和创建 [Namespace 范围的服务帐户](/cloud/service-accounts#scoped)，无论其帐户角色如何。

:::

<a id="manage-service-accounts"></a>
## 管理服务帐户 {#manage-service-accounts}

帐户所有者和全局管理员 [角色](/cloud/manage-access/roles-and-permissions#account-level-roles) 可以使用以下工具通过创建、查看、更新、删除服务帐户来管理服务帐户：

- Temporal Cloud 用户界面
- Temporal Cloud CLI (tcld)
  - 使用 `tcld service-account --help` 获取所有服务帐户命令的列表

帐户所有者和全局管理员 [角色](/cloud/manage-access/roles-and-permissions#account-level-roles) 还能够管理服务帐户的 API 密钥。

<a id="prerequisites"></a>
### 先决条件 {#prerequisites}

- 具有帐户所有者或全局管理员 [角色](/cloud/manage-access/roles-and-permissions#account-level-roles) 权限的云用户帐户
- 访问 Temporal Cloud UI 或 Temporal Cloud CLI (tcld)
- 允许访问您帐户的 API 密钥
- 要使用 Temporal Cloud CLI (tcld) 管理服务帐户，请使用 `brew upgrade tcld` 升级到最新版本的 tcld（v0.18.0 或更高版本）。
  - 如果使用低于 v0.31.0 的 tcld 版本，请使用 `tcld feature toggle-service-account` 启用服务帐户命令。

<a id="create-a-service-account"></a>
### 创建服务帐户 {#create-a-service-account}

使用 Temporal Cloud UI 或 tcld 创建服务帐户。
当用户身份被邀请到 Temporal Cloud 时，服务帐户是在 Temporal Cloud 中创建的。

<Tabs groupId="service-account">
  <TabItem value="cloud-ui" label="Using the Cloud UI">

1. 前往[设置 → 身份](https://cloud.temporal.io/settings/identities)
2. 单击位于 `Identities` 页面顶部附近的 `Create Service Account` 按钮
3. 提供以下信息：
   - **姓名**（必填）
   - **描述**（可选）
   - **帐户级别角色**（必填）
   - **Namespace 权限**（可选）
     - 使用“创建服务帐户”页面的此部分向服务帐户授予对各个 Namespace 的访问权限
4.点击页面底部的`Create Service Account`
   - 状态消息显示在屏幕右下角和下一个屏幕上
   - 系统将提示您为服务帐户创建 API 密钥（可选）
5.（可选）创建API密钥
   - 建议您在创建服务帐户后立即为服务帐户创建 API 密钥，尽管您可以随时创建/管理服务帐户的 API 密钥
   - 有关创建和管理 API 密钥的更多信息，请参阅 API 密钥 [文档](/cloud/api-keys)

  </TabItem>
  <TabItem value="tcld" label="Using tcld">

要使用 tcld 创建服务帐户，请使用 `tcld service-account create` 命令：

```
tcld service-account create -n "sa_test" -d "this is a test SA" --ar "Read"
```

此示例创建一个名为 `"sa_test"`、描述为 `"this is a test SA"` 的服务帐户和一个 `Read` 帐户角色。

创建服务帐户需要以下属性：`name` 和 `account-role`（如上所述）。
您还可以使用 `—-np` 标志为服务帐户提供 Namespace 权限。
创建服务帐户会返回 `ServiceAccountId`，用于检索、Update 或删除服务帐户。

  </TabItem>
</Tabs>

<a id="view-service-accounts"></a>
### 查看服务帐户 {#view-service-accounts}

使用 Temporal Cloud UI 或 tcld 查看单个或所有服务帐户。

<Tabs groupId="service-account">
  <TabItem value="cloud-ui" label="Using the Cloud UI">

服务帐户与用户一起列在 `Settings` 页面的 `Identities` 部分。
要找到服务帐户：

1. 前往[设置 → 身份](https://cloud.temporal.io/settings/identities)
2. 选择`Service Accounts`过滤器

  </TabItem>
  <TabItem value="tcld" label="Using tcld">

要使用 tcld 查看帐户中的所有服务帐户，请使用 `tcld service-account list` 命令：

```
tcld service-account list
```

  </TabItem>
</Tabs>

<a id="delete-a-service-account"></a>
### 删除服务帐户 {#delete-a-service-account}

使用 Temporal Cloud UI 或 tcld 删除服务帐户。当您删除服务帐户时，所有关联的 API 密钥也会自动删除。
因此，删除服务帐户后无需手动删除 API 密钥。

<Tabs groupId="service-account">
  <TabItem value="cloud-ui" label="Using the Cloud UI">

1. 前往[设置 → 身份](https://cloud.temporal.io/settings/identities)
2.找到相关的服务帐号
3. 选择“服务帐户”行中的垂直省略号菜单
4. 选择`Delete`
5. 出现提示时确认删除操作

  </TabItem>
  <TabItem value="tcld" label="Using tcld">

要使用 tcld 删除服务帐户，请使用 `tcld service-account delete` 命令：

```
tcld service-account delete --service-account-id "e9d87418221548"
```

使用 tcld 服务帐户列表命令验证服务帐户是否已从帐户中删除。
当服务帐户在 list 命令的输出中不再可见时，该服务帐户将被删除。

  </TabItem>
</Tabs>

<a id="update"></a>
### Update 服务帐户 {#update}

Update 使用 Temporal Cloud UI 或 tcld 的服务帐户描述。

<Tabs groupId="service-account">
  <TabItem value="cloud-ui" label="Using the Cloud UI">

1. 前往[设置 → 身份](https://cloud.temporal.io/settings/identities)
2.找到相关的服务帐号
3. 选择“服务帐户”行中的垂直省略号菜单
4. 选择`Edit`
5. 更改服务帐户
   - 您可以更改服务帐户的名称、描述、帐户级别角色和 Namespace 权限
6. 单击屏幕左下角的 `Save` 按钮
   - 状态消息显示在屏幕右下角

  </TabItem>
  <TabItem value="tcld" label="Using tcld">

存在三个不同的命令来帮助用户 Update 使用 tcld 服务帐户：

- `tcld service-account Update`：到 Update 服务帐户的名称或描述字段
- `tcld service-account set-account-role`：到 Update 服务帐户的帐户角色
- `tcld service-account set-Namespace-permissions`：到 Update 服务帐户的 Namespace 权限

例子：

```
tcld service-account update --id "2f68507677904e09b9bcdbf93380bb95" -d "new description"
```

  </TabItem>
</Tabs>


## Namespace 范围的服务帐户 {#scoped}

有一种特殊类型的服务帐户，称为 Namespace 范围的服务帐户，它共享
与上述服务帐户的功能相同，但仅限（或范围）为单个 Namespace。

特别是，Namespace 范围的服务帐户必须_始终_具有：

- `Read` 帐户角色
- 单一 Namespace 权限

请注意，Namespace 范围的服务帐户在创建后无法重新分配给不同的 Namespace，但可以修改其 Namespace 权限（例如从 `Read` 到 `Write`）。

当您需要限制客户端对单个 Namespace 的访问时，Namespace 范围的服务帐户非常有用。

您可以使用与上述相同的流程和命令来检索 Update 和删除 Namespace 范围的服务帐户，但创建过程略有不同。

<a id="permissions"></a>
### 权限 {#permissions}
与需要全局管理员或帐户所有者角色的常规服务帐户不同，Namespace 范围的服务帐户可以由 Namespace 管理员创建和管理。
例如，具有 `test_ns` 的 Namespace 管理员的帐户开发人员可以创建范围为 `test_ns` 的服务帐户。

全局管理员和帐户所有者还可以创建 Namespace 范围的服务帐户，因为他们隐式拥有所有 Namespace 的 Namespace 管理员权限。

<a id="create-a-namespace-scoped-service-account"></a>
### 创建 Namespace 范围的服务帐户 {#create-a-namespace-scoped-service-account}

与常规服务帐户一样，可以使用 Temporal Cloud UI 或 tcld 创建 Namespace 范围的服务帐户。

#### 使用云 UI {#scoped-ui}

目前，从 Temporal Cloud UI 创建 Namespace 范围的服务帐户发生在单独的 [Namespace](/cloud/namespaces#manage-namespaces) 页面上。
如果当前 Namespace 启用了 API 密钥身份验证，则 Namespace 页面顶部或 `Authentication` 部分中将有一个 `Generate API Key` 按钮作为横幅。

通过单击 `Generate API Key` 按钮，将为给定的 Namespace（如果尚不存在）自动创建一个 Namespace 范围的服务帐户，并且将显示关联的 API 密钥。该密钥的最长过期时间为 2 年。

生成的 Namespace 范围的服务帐户将被命名为 `<Namespace>-service-account`，并且默认情况下将具有 `Admin` Namespace 权限。

<a id="using-tcld"></a>
#### 使用 tcld {#using-tcld}

要使用 tcld 创建 Namespace 范围的服务帐户，请使用 `tcld service-account create-scoped` 命令：

```
tcld service-account create-scoped -n "test-scoped-sa" --np "test-ns=Admin"
```

此示例为 Namespace `test-ns` 创建一个 Namespace 范围的服务帐户，名为 `test-scoped-sa`，具有 `Admin` Namespace 权限。
请注意，帐户角色被省略，因为 Namespace 范围的服务帐户始终具有 `Read` 帐户角色。

<a id="lifecycle"></a>
### 生命周期 {#lifecycle}

删除 Namespace 时，所有关联的 Namespace 范围的服务帐户及其关联的 API 密钥也会自动删除。
因此，在删除 Namespace 后，您无需手动删除 Namespace 范围的服务帐户及其 API 密钥。
