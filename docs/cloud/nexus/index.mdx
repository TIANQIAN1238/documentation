:::tip SUPPORT, STABILITY, and DEPENDENCY INFO

Temporal Nexus 现已[正式发布](/evaluate/development-production-features/release-stages#general-availability)。在[评估指南](/evaluate/nexus)中了解为何应使用 Nexus。

:::

Temporal Cloud 在[核心 Nexus 体验](/nexus)之上构建了以下能力：

- **全局 [Nexus Registry](/nexus/registry)** - 跨账户下所有 Namespace 的统一注册表。任何 Namespace 中的 Worker 均可托管 Nexus Service 供其他人使用。
- **内置[访问控制](/nexus/registry#configure-runtime-access-controls)** - 在运行时限制哪些 caller Namespace 可以使用 Nexus Endpoint。
- **[审计日志](/cloud/audit-logs)** - 将 Nexus Registry 操作（创建、更新、删除 Endpoint）流式传输到你的审计日志集成。
- **多区域连接** - Nexus 请求通过全局 mTLS 加密的 Envoy 网格在 AWS 和 GCP 内及跨云的 Namespace 之间路由。兼容将[高可用](/cloud/high-availability) Namespace 作为 Endpoint 目标。
- **[Terraform 支持](/cloud/terraform-provider#manage-temporal-cloud-nexus-endpoints-with-terraform)** - 使用 Temporal Cloud Terraform provider 管理 Nexus Endpoint。
