`tcld generate-certificates` 命令为 Temporal Cloud 生成证书颁发机构 (CA) 和最终实体 TLS 证书。

别名：`gen`

- [tcld 生成证书 证书颁发机构证书](#certificate-authority-certificate)
- [tcld 生成证书最终实体证书](#end-entity-certificate)

<a id="certificate-authority-certificate"></a>
## tcld 生成证书 证书颁发机构证书 {#certificate-authority-certificate}

`tcld generate-certificates certificate-authority-certificate` 命令为 Temporal Cloud 生成证书颁发机构 (CA) 证书。

`tcld generate-certificates certificate-authority-certificate <modifiers>`

别名：`ca`

以下修饰符控制命令的行为。

#### - 组织 {#organization}

指定证书生成的组织名称。

别名：`--org`

**例子**

```bash
tcld generate-certificates certificate-authority-certificate --organization <value>
```

<a id="organization"></a>
#### --有效期 {#validity-period}

指定证书的有效期限。
将值的格式设置为 d/h（例如，`30d10h` 表示持续 30 天 10 小时的证书）。

别名：`-d`

**例子**

```bash
tcld generate-certificates certificate-authority-certificate --validity-period <value>
```

<a id="validity-period"></a>
#### --ca 证书文件 {#ca-certificate-file}

指定将存储生成的 X.509 证书文件的 `.pem` 文件的路径。

别名：`--ca-cert`

**例子**

```bash
tcld generate-certificates certificate-authority-certificate --ca-certificate-file <path>
```

<a id="ca-certificate-file"></a>
#### --ca-密钥文件 {#ca-key-file}

指定将存储证书私钥的 `.key` 文件的路径。

别名：`--ca-key`

**例子**

```bash
tcld generate-certificates certificate-authority-certificate --ca-key-file <path>
```

<a id="ca-key-file"></a>
#### --rsa 算法 {#rsa-algorithm}

启用后，将为证书生成 4096 位 RSA 密钥对，而不是 ECDSA P-384 密钥对。
由于 ECDSA P-384 密钥对是建议的默认值，因此禁用此选项。

别名：`--rsa`

**例子**

```bash
tcld generate-certificates certificate-authority-certificate --rsa-algorithm <boolean>
```

<a id="rsa-algorithm"></a>
## tcld 生成证书最终实体证书 {#end-entity-certificate}

`tcld generate-certificates end-entity-certificate` 命令为 Temporal Cloud 生成最终实体（叶）证书。

`tcld generate-certificates end-entity-certificate <modifiers>`

别名：`leaf`

以下修饰符控制命令的行为。

#### - 组织 {#organization}

指定证书生成的组织名称。

别名：`--org`

**例子**

```bash
tcld generate-certificates end-entity-certificate --organization <value>
```

<a id="end-entity-certificate"></a>
#### --组织单位 {#organization-unit}

可选：指定组织单位的名称。

**例子**

```bash
tcld generate-certificates end-entity-certificate --organization-unit <value>
```

<a id="organization"></a>
#### --有效期 {#validity-period}

指定证书的有效期限。
将值的格式设置为 d/h（例如，`30d10h` 表示持续 30 天 10 小时的证书）。

别名：`-d`

**例子**

```bash
tcld generate-certificates end-entity-certificate --validity-period <value>
```

<a id="organization-unit"></a>
#### --ca 证书文件 {#ca-certificate-file}

为证书颁发机构指定 `.pem` 文件中 X.509 CA 证书的路径。

别名：`--ca-cert`

**例子**

```bash
tcld generate-certificates end-entity-certificate --ca-certificate-file <path>
```

<a id="validity-period"></a>
#### --ca-密钥文件 {#ca-key-file}

指定证书颁发机构的 `.key` 文件中私钥的路径。

别名：`--ca-key`

**例子**

```bash
tcld generate-certificates end-entity-certificate --ca-key-file <path>
```

<a id="ca-certificate-file"></a>
#### --证书文件 {#certificate-file}

指定将存储生成的 X.509 叶证书文件的 `.pem` 文件的路径。

别名：`--cert`

**例子**

```bash
tcld generate-certificates end-entity-certificate --certificate-file <path>
```

<a id="ca-key-file"></a>
#### --密钥文件 {#key-file}

指定将存储叶证书私钥的 `.key` 文件的路径。

别名：`--key`

**例子**

```bash
tcld generate-certificates end-entity-certificate --key-file <path>
```
