## 添加区域 {#add-region}

使用 `tcld Namespace add-region` 将 <ToolTipTerm term="replica" /> 区域添加到现有的 Temporal Cloud [Namespace](/namespaces)，将其升级以支持 [高可用性](/cloud/high-availability)。

请参阅 [地区](/cloud/regions) 了解可用区域及其支持的复制选项。

以下修饰符控制命令的行为。

<a id="request-id"></a>
#### --请求 ID {#request-id}

用于异步操作的请求标识符。如果未设置，服务器会分配一个标识符。

别名：`-r`

#### --Namespace {#namespace}

**必需。** 指定在 Temporal Cloud 上托管的 Namespace。如果未指定，则使用环境变量 `$Temporal_CLOUD_Namespace` 的值。

别名：`-n`

#### - 地区 {#region}

**必需。** 要添加到现有 Namespace 的区域。有关受支持区域的列表，请参阅 [地区](/cloud/regions)。

:::tip Choosing Replica Regions

请参阅 [地区](/cloud/regions) 了解可用区域及其支持的复制选项。请参阅 [高可用性](/cloud/high-availability) 了解复制和故障转移的工作原理。

:::

别名：`--re`

**例子**

```bash
tcld namespace add-region \
    --namespace <namespace_id> \
    --region <replica_region_name>
```

指定要在其中创建副本的区域的区域名称（例如，`us-east-1`）作为 `--region` 标志的参数。
有关同区域、多区域和多云复制选项的详细信息，请参阅 [高可用性](/cloud/high-availability)。

一旦您的 Namespace 可供使用，Temporal Cloud 就会发送电子邮件提醒。

<a id="region"></a>
#### --云提供商 {#cloud-provider}

该地区的云提供商。 [`aws`、`gcp`] 之一。

默认值：`aws`

## 创造 {#create}

`tcld Namespace create` 命令在 Temporal Cloud 中创建 Temporal [Namespace](/namespaces)。

别名：`c`

以下修饰符控制命令的行为。

<a id="cloud-provider"></a>
#### --Namespace {#namespace}

**必需。** 新 Namespace 的名称。这成为 Namespace ID (`<Namespace_name>.<account_suffix>`) 的一部分。

别名：`-n`

#### - 地区 {#region}

**必需。** 用于创建 Namespace 的云提供商区域。为标准 Namespace 提供一个 `--region`，或为带有 [高可用性](/cloud/high-availability) 的 Namespace 提供两个。

请参阅 [地区](/cloud/regions) 了解可用区域及其支持的复制选项。

别名：`--re`

<a id="create"></a>
#### --auth-方法 {#auth-method}

Namespace 的身份验证方法。 [`mTLS`、`api_key`] 之一。

- `mTLS`（默认）：需要 `--ca-certificate` 或 `--ca-certificate-file`
- `api_key`：没有其他修饰符

**例子**

```bash
tcld namespace create \
    --namespace test-namespace.a1b2c \
    --region us-east-1 \
    --auth-method api_key
```

<a id="namespace"></a>
#### --ca 证书 {#ca-certificate}

Base64 编码的 [CA证书](/cloud/certificates)。如果同时指定了 `--ca-certificate` 和 `--ca-certificate-file`，则仅使用 `--ca-certificate`。

别名：`-c`

<a id="region"></a>
#### --ca 证书文件 {#ca-certificate-file}

[CA证书](/cloud/certificates) PEM 文件的路径。如果指定了两个选项，则仅使用 `--ca-certificate`。

别名：`--cf`

<a id="auth-method"></a>
#### --证书过滤器文件 {#certificate-filter-file}

JSON 文件的路径，该文件定义要应用于 Namespace 的 [证书过滤器](/cloud/certificates#manage-certificate-filters)。

示例 JSON：`{ "filters": [ { "commonName": "test1" } ] }`

如果同时指定了 `--certificate-filter-file` 和 `--certificate-filter-input`，该命令将返回错误。

别名：`--cff`

<a id="ca-certificate"></a>
#### --证书过滤器输入 {#certificate-filter-input}

定义要应用于 Namespace 的 [证书过滤器](/cloud/certificates#manage-certificate-filters) 的 JSON 字符串。

示例 JSON：`{ "filters": [ { "commonName": "test1" } ] }`

如果同时指定了 `--certificate-filter-input` 和 `--certificate-filter-file`，该命令将返回错误。

别名：`--cfi`

<a id="ca-certificate-file"></a>
#### --云提供商 {#cloud-provider}

该地区的云提供商。 [`aws`、`gcp`] 之一。

默认值：`aws`

别名：`--cp`

<a id="certificate-filter-file"></a>
#### --连接规则 ID {#connectivity-rule-ids}

应用于 Namespace 的 [连接规则](/cloud/connectivity#connectivity-rules) ID 列表。可以指定多次。

别名：`--ids`

**例子**

```bash
tcld namespace create \
    --namespace test-namespace.a1b2c \
    --region us-east-1 \
    --auth-method api_key \
    --connectivity-rule-ids <rule_id1> \
    --connectivity-rule-ids <rule_id2>
```

<a id="certificate-filter-input"></a>
#### --启用删除保护 {#enable-delete-protection}

在 Namespace 上启用 [删除保护](/cloud/namespaces#delete-protection)。

默认值：`false`

别名：`--edp`

<a id="cloud-provider"></a>
#### --端点 {#endpoint}

[Codec Server](/production-deployment/data-encryption) 端点，用于为与此 Namespace 交互的所有用户解码有效负载。必须是 HTTPS。

别名：`-e`

<a id="connectivity-rule-ids"></a>
#### --包含凭证 {#include-credentials}

调用 [Codec Server](/production-deployment/data-encryption) 时包括跨域凭据。

默认值：`false`

别名：`--ic`

<a id="enable-delete-protection"></a>
#### --pass-access-token {#pass-access-token}

将用户访问令牌传递到 [Codec Server](/production-deployment/data-encryption) 端点。

默认值：`false`

别名：`--pat`

<a id="endpoint"></a>
#### --请求 ID {#request-id}

用于异步操作的请求标识符。如果未设置，服务器会分配一个标识符。

别名：`-r`

<a id="include-credentials"></a>
#### --保留天数 {#retention-days}

关闭的 Workflow Execution 的 [保留期限](/temporal-service/temporal-server#retention-period) 天数。

默认值：`30`

别名：`--rd`

<a id="pass-access-token"></a>
#### --搜索属性 {#search-attribute}

格式为“_name_=_type_”的自定义 [Search Attribute](/search-attribute)。可以指定多次。

_type_ 的有效值：`Bool` | `Datetime` | `Double` | `Int` | `Keyword` | `Text`

别名：`--sa`

**例子**

```bash
tcld namespace create \
    --namespace test-namespace.a1b2c \
    --region us-east-1 \
    --auth-method api_key \
    --search-attribute "customer_id=Int" \
    --search-attribute "customer_name=Text"
```

#### - 标签 {#tag}

形式为“_key_=_value_”的 [标签](/cloud/namespaces#tag-a-namespace)。可以指定多次。

参见 [标签结构和限制](/cloud/namespaces#tag-structure-and-limits)。

别名：`--t`

**例子**

```bash
tcld namespace create \
    --namespace test-namespace.a1b2c \
    --region us-east-1 \
    --auth-method api_key \
    --tag "key=value" \
    --tag "key2=value2"
```

<a id="request-id"></a>
#### --用户-Namespace-权限 {#user-namespace-permission}

用户的 [Namespace级权限](/cloud/manage-access/roles-and-permissions#namespace-level-permissions)，格式为“_email_=_permission_”。可以指定多次。

_permission_ 的有效值：`Admin` | `Write` | `Read`

别名：`-p`

**例子**

```bash
tcld namespace create \
    --namespace test-namespace.a1b2c \
    --region us-east-1 \
    --auth-method api_key \
    --user-namespace-permission "user@example.com=Admin" \
    --user-namespace-permission "user2@example.com=Write"
```

## 删除区域 {#delete-region}

使用 `tcld Namespace delete-region` 删除现有 Temporal Cloud 的 <ToolTipTerm term="replica" />
[Namespace](/namespaces)。删除副本会禁用 [高可用性功能](/cloud/high-availability) 和结果
您必须经过 7 天的等待期，然后才能在同一位置重新启用高可用性功能。参考
[启用高可用性](/cloud/high-availability/enable) 了解更多信息。

以下修饰符控制命令的行为。

<a id="delete"></a>
#### --请求 ID {#request-id}

用于异步操作的请求标识符。如果未设置，服务器会分配一个标识符。

别名：`-r`

#### --Namespace {#namespace}

**必需。** 指定在 Temporal Cloud 上托管的 Namespace。如果未指定，则使用环境变量 `$Temporal_CLOUD_Namespace` 的值。

别名：`-n`

#### - 地区 {#region}

**必需。** 要从 Namespace 中删除的区域。指定区域名称，例如 `us-east-1`。删除后，Temporal 停止复制，Namespace 成为标准 Namespace。删除 Namespace 区域后 7 天内无法重新添加区域或添加新区域。

别名：`--re`

**例子**

```bash
tcld namespace delete-region \
    --namespace <namespace_id> \
    --region <region_name>
```

使用 API 密钥身份验证时，请在按 Enter 之前添加您的 API 凭据：

```bash
tcld --api-key <your_api_key> \
    delete-region \
    --namespace <namespace_id> \
    --region <region_name>
```

<a id="request-id"></a>
#### --云提供商 {#cloud-provider}

要故障转移到的区域的云提供商。 [aws、gcp] 之一。

默认值：aws（默认值：“aws”）

从主 Namespace 到其副本的 Namespace 区域。

**例子**

```bash

    --namespace <namespace_id> \
    --region <target_region>
```

使用 API 密钥身份验证时，请在按 Enter 之前添加您的 API 凭据：

```bash
tcld --api-key <your_api_key> \
    namespace failover \
    --namespace <namespace_id> \
    --region <target_region>
```

别名：`--re`

### 放 {#set}

别名：`s`

{/* How to rollover accepted client CA certificates in Temporal Cloud using tcld */}

更新 CA 证书时，遵循滚动更新过程非常重要。这样做可以使您的 Namespace 能够服务
两个 CA 证书都会保留一段时间，直到旧 CA 证书的流量停止。

1. 创建一个包含新旧 CA 证书 PEM 块的单个文件。只需连接 PEM 块即可
   在相邻的线上。

   ```
   -----BEGIN CERTIFICATE-----
   ... old CA cert ...
   -----END CERTIFICATE-----
   -----BEGIN CERTIFICATE-----
   ... new CA cert ...
   -----END CERTIFICATE-----
   ```

1. 使用 CA 证书捆绑文件运行 `tcld Namespace accepted-client-ca set` 命令。

   ```bash
   tcld namespace accepted-client-ca set \
       --ca-certificate-file <path>
   ```

1. 监控旧证书的流量直至其停止。

1. 创建另一个仅包含新 CA 证书的文件。

1. 使用 Updated CA 证书捆绑文件再次运行 `tcld Namespace accepted-client-ca set` 命令。

以“_name_=_type_”形式指定自定义 Search Attribute。 _type_ 的有效值如下：

- 布尔
- 日期时间
- 双人间
- 智力
- 关键字
- 文字

别名：`--sa`

**例子**

```bash
tcld namespace search-attributes add \
    --search-attribute "YourSearchAttribute1=Text" \
    --search-attribute "YourSearchAttribute2=Double"
```
