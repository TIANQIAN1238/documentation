## 删除 {#delete}

`tcld user delete` 命令删除 Temporal Cloud 中指定的用户。
您必须设置 `--user-email` 或 `--user-id` 来指定要删除的用户。

别名：`d`

以下修饰符控制命令的行为。

<a id="delete"></a>
#### --用户电子邮件 {#user-email}

指定要删除的用户的电子邮件地址。

**例子**

```command
tcld user delete --user-email <test@example.com>
```

#### - 用户身份 {#user-id}

指定要删除的用户的用户标识符。

**例子**

```command
tcld user delete --user-id <test-user-id>
```

<a id="user-email"></a>
#### --请求 ID {#request-id}

用于异步操作的请求标识符。

如果未设置，服务器会分配一个标识符。

别名：`-r`

<a id="user-id"></a>
#### --资源版本 {#resource-version}

将资源版本 (ETag) 指定为 Update。
如果未指定，则使用最新版本。

别名：`-v`

## 邀请 {#invite}

`tcld Namespace invite` 命令邀请指定用户加入 Temporal Cloud。

别名：`i`

以下修饰符控制命令的行为。

<a id="resource-version"></a>
#### --用户电子邮件 {#user-email}

_所需修饰符_

指定要邀请的用户的电子邮件地址。
您可以多次提供此修饰符以在单个请求中邀请多个用户。

别名：`-e`

<a id="get"></a>
#### --账户角色 {#account-role}

_所需修饰符_

为受邀用户指定 [账户级角色](/cloud/manage-access/roles-and-permissions#account-level-roles)。

可用帐户角色：`admin` | `developer` | `read`。

别名：`--ar`

<a id="user-email"></a>
#### --Namespace-权限 {#namespace-permission}

为受邀用户指定 [Namespace级权限](/cloud/manage-access/roles-and-permissions#namespace-level-permissions)。
您可以多次提供此修饰符以在单个请求中设置多个 Namespace 权限。

每个值的格式必须为 `Namespace=permission-type`。

可用的 Namespace 权限：`Admin` | `Write` | `Read`。

别名：`-p`

<a id="user-id"></a>
#### --请求 ID {#request-id}

用于异步操作的请求标识符。

如果未设置，服务器会分配一个标识符。

别名：`-r`

```command
tcld user invite --user-email <test@example.com> --account-role developer --namespace-permission ns1=Admin --namespace-permission ns2=Write --request-id <123456>
```

## 列表 {#list}

`tcld user list` 命令返回 Temporal Cloud 中的分页用户列表。

别名：`l`

**例子**

```command
tcld user list
```

以下修饰符控制命令的行为。

<a id="invite"></a>
#### --Namespace {#namespace}

列出拥有 Namespace 权限的用户。

别名：`-n`

**例子**

```command
tcld user list --namespace <namespace_id>
```

<a id="user-email"></a>
#### --页面令牌 {#page-token}

用于分页列表用户请求的页面令牌。

别名：`-p`

<a id="account-role"></a>
#### --页面大小 {#page-size}

分页列表用户请求的页面大小。

默认为 10。

别名：`-s`


#### - 用户身份 {#user-id}

指定要重新发送邀请的用户的用户标识符。

**例子**

