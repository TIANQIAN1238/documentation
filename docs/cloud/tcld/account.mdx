### 接受-客户端-ca {#accepted-client-ca}

`tcld account metrics accepted-client-ca` 命令管理当前登录的 Temporal Cloud 帐户的指标端点的最终实体证书。

:::info

指标端点的最终实体证书必须链接到用于帐户的 CA 证书。有关详细信息，请参阅 [证书要求](/cloud/certificates#certificate-requirements)。

:::

别名：`ca`

- [tcld 帐户指标已接受-客户-ca 添加](#add)
- [tcld 帐户指标已接受-客户-ca 列表](#list)
- [tcld 帐户指标已接受-客户-ca 集](#set)
- [tcld 帐户指标已接受-客户-ca 删除](#remove)

#### 添加 {#add}

`tcld account metrics accepted-client-ca add` 命令将终端实体证书添加到 Temporal Cloud 帐户的指标端点。

:::info

指标端点的最终实体证书必须链接到用于帐户的 CA 证书。有关详细信息，请参阅 [证书要求](/cloud/certificates#certificate-requirements)。

:::

`tcld account metrics accepted-client-ca add --ca-certificate <value>`

别名：`a`

以下修饰符控制命令的行为。

<a id="resource-version"></a>
##### --请求 ID {#request-id}

指定用于异步操作的请求标识符。如果未指定，服务器会分配一个请求标识符。

别名：`-r`

**例子**

```bash
tcld account metrics accepted-client-ca add --request-id <request_id> --ca-certificate <encoded_certificate>
```

<a id="service-account-email"></a>
##### --资源版本 {#resource-version}

将资源版本 (ETag) 指定为 Update。如果未指定，则使用最新版本。

别名：`-v`

**例子**

```bash
tcld account metrics accepted-client-ca add --resource-version <etag> --ca-certificate <encoded_certificate>
```

<a id="sink-name"></a>
##### --ca 证书 {#ca-certificate}

_必需的修饰符，除非指定了 `--ca-certificate-file`_

指定 CA 证书 PEM 文件的 Base64 编码字符串。

如果同时指定了 `--ca-certificate` 和 `--ca-certificate-file`，则仅使用 `--ca-certificate`。

别名：`-c`

**例子**

```bash
tcld account metrics accepted-client-ca add --ca-certificate <encoded_certificate>
```

<a id="topic-name"></a>
##### --ca 证书文件 {#ca-certificate-file}

_必需的修饰符，除非指定了 `--ca-certificate`_

指定 CA 证书 PEM 文件的路径。

如果同时指定了 `--ca-certificate` 和 `--ca-certificate-file`，则仅使用 `--ca-certificate`。

别名：`-f`

**例子**

```bash
tcld account metrics accepted-client-ca add --ca-certificate-file <path>
```

#### 列表 {#list}

`tcld account metrics accepted-client-ca list`

别名：`r`

<a id="validate"></a>
##### --请求 ID {#request-id}

指定用于异步操作的请求标识符。如果未指定，服务器会分配一个请求标识符。

别名：`-r`


<a id="service-account-email"></a>

```bash

```

<a id="sink-name"></a>
##### --ca 证书 {#ca-certificate}

_必需的修饰符，除非指定了 `--ca-certificate-fingerprint` 或 `--ca-certificate-file`_

指定 CA 证书 PEM 文件的 Base64 编码字符串。

如果还指定了 `--ca-certificate-fingerprint`，则 `--ca-certificate` 和 `--ca-certificate-file` 都将被忽略。

如果还指定了 `--ca-certificate-file` 但未指定 `--ca-certificate-fingerprint`，则仅使用 `--ca-certificate`。

别名：`-c`

**例子**

```bash
tcld account metrics accepted-client-ca remove --ca-certificate <encoded_certificate>
```

<a id="topic-name"></a>
##### --ca 证书文件 {#ca-certificate-file}

_必需的修饰符，除非指定了 `--ca-certificate-fingerprint` 或 `--ca-certificate`_

指定 CA 证书 PEM 文件的路径。

如果还指定了 `--ca-certificate-fingerprint`，则 `--ca-certificate-file` 和 `--ca-certificate` 都将被忽略。

如果还指定了 `--ca-certificate` 但未指定 `--ca-certificate-fingerprint`，则仅使用 `--ca-certificate`。

别名：`-f`

**例子**

```bash
tcld account metrics accepted-client-ca remove --ca-certificate-file <path>
```

<a id="get"></a>
##### --ca-证书指纹 {#ca-certificate-fingerprint}

_必需的修饰符，除非指定了 `--ca-certificate` 或 `--ca-certificate-file`_

指定CA证书的指纹。

如果同时指定了 `--ca-certificate`、`--ca-certificate-file` 或两者，则它们将被忽略。

别名：`--fp`

**例子**

```bash
tcld account metrics accepted-client-ca remove --ca-certificate-fingerprint <fingerprint>
```

### 使能够 {#enable}

`tcld account metrics enable` 命令为当前登录的 Temporal Cloud 帐户启用指标端点。

:::info

在启用端点之前，必须配置指标端点的最终实体。请参阅 [tcld 帐户指标已接受-客户-ca](#accepted-client-ca) 命令。

:::

`tcld account metrics enable`

该命令没有修饰符。
