`tcld apikey` 命令管理 Temporal Cloud 中的 API 密钥。

别名：`ak`

## 创造 {#create}

`tcld apikey create` 命令在 Temporal Cloud 中创建 API 密钥。

`tcld apikey create --name <name> --description <description> --duration <duration> --expiry <expiry> --request-id <request_id>`

以下选项控制命令的行为。

#### - 姓名 {#name}

_所需修饰符_

指定 API 密钥的显示名称。

别名：`-n`

**例子**

```bash
tcld apikey create --name <name>
```

#### - 描述 {#description}

指定 API 密钥的描述。

别名：`-desc`

**例子**

```bash
tcld apikey create --name <name> --description "Your API Key"
```

#### - 期间 {#duration}

指定 API 密钥从现在开始过期的持续时间。
如果设置了过期标志，这将被忽略。

示例格式：`24h`（默认值：0s）。

别名：`-d`

**例子**

```bash
tcld apikey create --name <name> --duration 24h
```

<a id="create"></a>
#### --过期 {#expiry}

指定 API 密钥到期时的绝对时间戳 (RFC3339)。

示例：`2023-11-28T09:23:24-08:00`。

别名：`-e`

**例子**

```bash
tcld apikey create --name <name> --expiry '2023-11-28T09:23:24-08:00'
```

<a id="name"></a>
#### --请求 ID {#request-id}

指定异步操作的 request-id。
如果没有设置，服务器会分配一个。

别名：`-r`

**例子**

```bash
tcld apikey create --name <name> --request-id <request_id>
```

## 得到 {#get}

`tcld apikey get` 命令检索 Temporal Cloud 中指定 API 密钥的详细信息。

`tcld apikey get --id <id>`

以下选项控制命令的行为。

#### - ID {#id}

_所需修饰符_

指定要检索的 API 密钥的 ID。

别名：`-i`

**例子**

```bash
tcld apikey get --id <apikey_id>
```

## 列表 {#list}

`tcld apikey list` 命令列出了 Temporal Cloud 中的所有 API 密钥。

`tcld apikey list`

该命令不需要任何特定选项。

别名：`l`

**例子**

```bash
tcld apikey list
```

## 删除 {#delete}

`tcld apikey delete` 命令删除 Temporal Cloud 中的 API 密钥。

`tcld apikey delete --id <id> [--resource-version <version>] [--request-id <request_id>]`

以下选项控制命令的行为。

#### - ID {#id}

_所需修饰符_

指定要删除的 API 密钥的 ID。

别名：`-i`

**例子**

```bash
tcld apikey delete --id <apikey_id>
```

<a id="description"></a>
#### --资源版本 {#resource-version}

将资源版本 (etag) 指定为 Update。
如果未设置，CLI 将使用最新的。

别名：`-v`

**例子**

```bash
tcld apikey delete --id <apikey_id> --resource-version <version>
```

<a id="duration"></a>
#### --请求 ID {#request-id}

指定异步操作的 request-id。
如果没有设置，服务器会分配一个。

别名：`-r`

**例子**

```bash
tcld apikey delete --id <apikey_id> --request-id <request_id>
```
