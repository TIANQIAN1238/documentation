## 添加用户 {#add-users}

`tcld user-group add-users` 命令将用户添加到 Temporal Cloud 中指定的用户组。
您必须设置 `--group-id` 来指定要将用户添加到的组。

别名：`au`

以下标志控制命令的行为。

<a id="group-id--id"></a>
#### --组 ID (-id) {#group-id--id}

指定要添加用户的组的 ID。

<a id="user-email--e"></a>
#### --用户电子邮件 (-e) {#user-email--e}

指定要添加的用户的电子邮件。该标志可以多次指定以添加
一个命令中有多个用户


## 创造 {#create}

创建用户组。

别名：`c`

以下标志控制命令的行为。

<a id="create"></a>
#### --显示名称 {#display-name}

组的显示名称。

<a id="display-name"></a>
#### --账户角色 {#account-role}

该组应具有的帐户角色。 `admin`、`read`、`developer`、`owner`、`financeadmin`、`none` 之一。

<a id="account-role"></a>
#### --Namespace-角色 (-nr) {#namespace-role--nr}

指定该组应具有的 Namespace 角色。可以重复多次添加
组中有多个 Namespace 角色。值的形式为 `<Namespaceid>-<role>`，其中 Namespace ID 是 Namespace 的完整 ID，角色为 `admin`、`read` 或 `write` 之一。示例：`myNamespace.abc123-read` 为 `myNamespace.abc123` Namespace 添加读取角色。

## 设置访问 {#set-access}

此命令设置组的访问角色。它遵循与 [创造](#create) 命令相同的约定，指定一个可选帐户角色和 0 个或多个 Namespace 角色。

别名：`sa`

<a id="remove-users"></a>
#### --组 ID (-id) {#group-id--id}

指定要设置访问权限的组 ID。

<a id="group-id--id"></a>
#### --账户角色 {#account-role}

该组应具有的帐户角色。 `admin`、`read`、`developer`、`owner`、`financeadmin`、`none` 之一。

<a id="user-email--e"></a>
#### --Namespace-角色 (-nr) {#namespace-role--nr}

指定该组应具有的 Namespace 角色。可以重复多次添加
组中有多个 Namespace 角色。值的形式为 `<Namespaceid>-<role>`，其中 Namespace ID 是 Namespace 的完整 ID，角色为 `admin`、`read` 或 `write` 之一。示例：`myNamespace.abc123-read` 为 `myNamespace.abc123` Namespace 添加读取角色。

<a id="set-access"></a>
#### --追加 (-a) {#append--a}

将追加 Namespace 角色，而不是替换已分配的所有现有角色。这允许在不知道已分配给组的角色的情况下添加 Namespace 角色。

<a id="group-id--id"></a>
#### --删除 (-r) {#remove--r}

将删除给定的 Namespace 角色，而不是替换已分配的所有现有角色。这允许在不知道已分配给组的角色的情况下删除 Namespace 角色。
