<a id="configure-audit-log"></a>
## 使用 AWS Kinesis {#configure-audit-log} 配置审核日志
要设置审核日志，您必须拥有 Amazon Web Services (AWS) 账户并设置 Kinesis Data Streams。

1. 如果您没有 AWS 账户，请按照 [创建并激活 AWS 账户](https://aws.amazon.com/premiumsupport/knowledge-center/create-and-activate-aws-account/) 中 AWS 的说明进行操作。
2. 要设置 Kinesis Data Streams，请打开 [AWS 管理控制台](https://aws.amazon.com/console/)，搜索 Kinesis，然后开始设置过程。

您可以使用 [此 AWS CloudFormation 模板](https://temporal-auditlogs-config.s3.us-west-2.amazonaws.com/cloudformation/iam-role-for-temporal-audit-logs.yaml) 创建一个 IAM 角色，该角色可以访问您账户中的 Kinesis 流。

请注意，Kinesis 的速率限制为每秒 1,000 条消息，并且对写入的记录数量和记录大小都有配额。
有关详细信息，请参阅 [为什么我的 Kinesis 数据流受到限制？](https://aws.amazon.com/premiumsupport/knowledge-center/kinesis-data-stream-throttling/)

<a id="create-an-audit-log-sink"></a>
### 创建审核日志接收器 {#create-an-audit-log-sink}

1. 在 Temporal Cloud 用户界面中，选择“**设置**”。
1. 在**设置**页面上，选择**审核日志**。
1. 在 **审核日志集成** 卡中，选择 **设置**。
1. 在 **审核日志集成** 页面上，选择您的 **访问方法**（**自动** 或 **手动**）。
   - **自动：** 从云 UI 在您的 AWS 账户中配置 AWS CloudFormation 堆栈。
   - **手动：** 使用生成的 AWS CloudFormation 模板手动设置 Kinesis。
1. 在 **Kinesis ARN** 中，粘贴您的 AWS 账户中的 Kinesis ARN。
1. 在 **角色名称** 中，提供新 IAM 角色的名称。
1. 在 **选择 AWS 区域** 中，为您的 Kinesis 流选择适当的区域。

如果您选择 **自动** 访问方法，请继续执行以下步骤：

1. 选择 **保存并启动堆栈**。
1. 在 AWS CloudFormation 控制台的 **堆栈名称** 中，指定堆栈的名称。
1. 在页面右下角，选择“**创建堆栈**”。

如果您选择 **手动** 访问方法，请继续执行以下步骤：

1. 选择**保存并下载模板**。
1. 打开[AWS CloudFormation 控制台](https://console.aws.amazon.com/cloudformation/)。
1. 选择**创建堆栈**。
1. 在 **创建堆栈** 页面上，选择 **模板已准备好** 和 **Update 模板文件**。
1. 选择 **选择文件** 并指定您在步骤 1 中生成的模板。
1. 在本页和接下来的两页上选择“**下一步**”。
1. 在**查看**页面上，选择**创建堆栈**。

为了确保审核日志可以流入 Kinesis 流，您可以使用 **验证** 按钮来确认其设置正确。这验证了 Temporal 可以成功写入您的流。
如果一切配置正确，您将看到 `Success` 状态，指示 Temporal 已写入 kinesis 流。

<a id="consume-an-audit-log"></a>
## 使用审核日志 {#consume-an-audit-log}
**如何使用审核日志**

创建审核日志接收器后，等待日志流入 Kinesis 流。
配置接收器后 10 分钟内您将看到第一个日志。

:::note

您必须配置并实现您自己的 Kinesis 流使用者。
有关示例，请参见 [使用审核日志的示例](#example-of-consuming-an-audit-log)。

:::

<a id="example-of-consuming-an-audit-log"></a>
### 使用审核日志的示例 {#example-of-consuming-an-audit-log}

以下 Go 代码是使用 Kinesis 流中的审核日志并将其传送到 S3 存储桶的示例。

```go
func main() {
   fmt.Println("print audit log from S3")
   cfg, err := config.LoadDefaultConfig(context.TODO(),
      config.WithSharedConfigProfile("your_profile"),
   )
   if err != nil {
      fmt.Println(err)
   }
   s3Client := s3.NewFromConfig(cfg)
   response, err := s3Client.GetObject(
      context.Background(),
      &s3.GetObjectInput{
         Bucket: aws.String("your_bucket_name"),
         Key:    aws.String("your_s3_file_path")})
   if err != nil {
      fmt.Println(err)
   }
   defer response.Body.Close()

   content, err := io.ReadAll(response.Body)

   fmt.Println(string(content))
}
```

前面的代码还在终端中打印日志。
