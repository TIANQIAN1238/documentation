## Namespace 的专用网络连接 {#private-network-connectivity-for-namespaces}

除默认互联网终端节点外，Temporal Cloud 还支持通过 AWS PrivateLink 或 GCP Private Services Connect 与 Namespace 进行私有连接。

无论您选择如何连接，Namespace 访问始终通过 [API 密钥](/cloud/api-keys#overview) 或 [mTLS](/cloud/certificates) 进行安全身份验证。

有关 IP 地址稳定性和白名单的信息，请参阅 [IP地址](/cloud/connectivity/ip-addresses)。

<a id="required-steps"></a>
### 所需步骤 {#required-steps}

要使用 Temporal Cloud 的专用连接：

1. 设置您的VPC到Temporal Namespace所在区域的私有连接。
1. Update 您的私有 DNS 和/或 Worker 配置以使用私有连接。
1.（完成 Google PSC 设置所需，如果使用 AWS PrivateLink，则可选）：为专用连接创建连接规则并将其附加到目标 Namespace。这将阻止所有不通过专用连接对 Namespace 的访问，但您也可以添加公共规则以允许 Internet 连接。

对于步骤 1 和 2，请遵循我们针对目标 Namespace 云提供商的指南：
- [AWS 私有链接](/cloud/connectivity/aws-connectivity) 创建和私有 DNS 设置
- [Google 云私有服务连接](/cloud/connectivity/gcp-connectivity) 创建和私有 DNS 设置

:::caution Finish client setup (complete step 2)

创建专用连接后，您必须设置专用 DNS 或 Update 要使用专用连接的所有客户端的配置。

我们建议使用私有 DNS。

如果不执行此步骤，您的客户端如果之前使用公共连接，则可以通过互联网连接到 Namespace，否则将根本无法连接。

如果这不适合您，请参阅 [我们更新客户端上的服务器和 TLS 设置的指南](/cloud/connectivity#update-dns-or-clients-to-use-private-connectivity)。

:::

对于步骤 3，请继续阅读 [连接规则](/cloud/connectivity#connectivity-rules) 的详细信息。

<a id="connectivity-rules"></a>
## 连接规则 {#connectivity-rules}

:::tip Support, stability, and dependency info

连接规则当前位于 [公开预览](/evaluate/development-production-features/release-stages#public-preview) 中。

:::

:::info Web UI Connectivity

Temporal Cloud Web UI 当前不受连接规则强制执行的约束。
即使 Namespace 配置了专用连接规则，该 Namespace 的 Web UI 仍然可以通过公共 Internet 进行访问。

:::

＃＃＃ 定义

连接规则是 Temporal Cloud 的机制，用于限制可用于访问 Namespace 的网络访问路径。

默认情况下，Namespace 的连接规则为零，并且可以从 1. 公共互联网和 2. 您配置到包含 Namespace 的区域的所有专用连接进行访问。无论连接规则如何，Namespace 访问始终通过 [API 密钥](/cloud/api-keys#overview) 或 [mTLS](/cloud/certificates) 进行安全身份验证。

当您将一个或多个连接规则附加到 Namespace 时，Temporal Cloud 将立即阻止所有没有相应连接规则的流量访问 Namespace。一个 Namespace 可以有多个连接规则，并且可以混合公共和私有规则。

每个连接规则指定通用公共（即互联网）访问或特定的专用连接。

公共连接规则不带任何参数。

AWS PrivateLink (PL) 专用连接规则需要以下参数：

- `connection-id`：PL 连接的 VPC 终端节点 ID（例如：`vpce-00939a7ed9EXAMPLE`）
- `region`：PL 连接的区域，以 aws 为前缀（例如：`aws-us-east-1`）。必须与 Namespace 位于同一区域。请参阅 [Temporal Cloud 地区列表](/cloud/regions) 了解支持的区域。

GCP Private Service Connect (PSC) 专用连接规则需要以下参数：

- `connection-id`：PSC 连接的 ID（例如：`1234567890123456789`）
- `region`：PSC 连接的区域，以 gcp 为前缀（例如：`gcp-us-east1`）。必须与 Namespace 位于同一区域。请参阅 [Temporal Cloud 地区列表](/cloud/regions) 了解支持的区域。
- `gcp-project-id`：您在其中创建 PSC 连接的 GCP 项目的 ID（例如：`my-example-project-123`）

可以使用 [tcld](https://docs.temporal.io/cloud/tcld/)、[地形](https://github.com/temporalio/terraform-provider-temporalcloud/) 或 [云运营 API](/ops) 创建和管理连接规则

<a id="definition"></a>
### 权限和限制 {#definition}

只有 [帐户管理员和帐户所有者](/cloud/manage-access/roles-and-permissions#account-level-roles) 可以创建和管理连接规则。连接规则对帐户开发人员、帐户管理员和帐户所有者可见。

默认情况下，每个 Namespace 仅限 5 条专用连接规则，每个帐户仅限 50 条专用连接规则。您可以 [联系支持人员](/cloud/support#support-ticket) 请求更高的限制。

每个帐户只允许使用一条公共规则，因为它是通用的，并且可以重复用于您希望在 Internet 上可用的所有 Namespace。尝试创建多个公共规则将引发错误。

<a id="permissions-and-limits"></a>
## 创建连接规则 {#permissions-and-limits}

<a id="creating-a-connectivity-rule"></a>
### Temporal Cloud CLI (tcld) {#creating-a-connectivity-rule}

创建专用连接规则 (AWS)：

```bash
tcld connectivity-rule create --connectivity-type private --connection-id "vpce-abcde" --region "aws-us-east-1"
```

创建专用连接规则 (GCP)：

```bash
tcld connectivity-rule create --connectivity-type private --connection-id "1234567890" --region "gcp-us-central1" --gcp-project-id "my-project-123"
```

创建公共连接规则（您只需在帐户中执行此操作一次）：

```bash
tcld connectivity-rule create --connectivity-type public
```

`cr` 别名的工作方式相同：

专用连接规则：

```bash
tcld cr create --connectivity-type private --connection-id "vpce-abcde" --region "aws-us-east-1"
```

```bash
tcld cr create --connectivity-type public
```

<a id="terraform"></a>
## 将连接规则附加到 Namespace {#terraform}

当心！当在 Namespace 上设置任何连接规则时，该 Namespace 只能通过这些规则中定义的连接进行访问。如果您删除 Worker 正在使用的连接规则，您的流量将会中断。

如果您已经有使用 Namespace 的 Worker，同时添加公共规则和任何私有规则可以帮助您避免意外的访问权限丢失。然后，您可以确保所有 Worker 都使用专用连接，然后删除公共规则。

<a id="attach-connectivity-rules-to-a-namespace"></a>
### Temporal Cloud CLI (tcld) {#attach-connectivity-rules-to-a-namespace}

在Namespace上设置连接规则：

```bash
tcld namespace set-connectivity-rules --namespace "my-namespace.abc123" --connectivity-rule-ids "rule-id-1" --connectivity-rule-ids "rule-id-2"
```

或者使用别名：

```bash
tcld n scrs -n "my-namespace.abc123" --ids "rule-id-1" --ids "rule-id-2"
```

连接规则作为一组附加，因此如果规则 `rule-a`、`rule-b` 和 `rule-c` 附加到 Namespace，并且您只想分离 `rule-c`，则可以进行一次附加 `rule-a` 和 `rule-b` 的调用：

```bash
tcld namespace set-connectivity-rules --namespace "my-namespace.abc123 --ids rule-a --ids rule-b
```

删除所有连接规则（这将使 Namespace 公开）：

```bash
tcld namespace set-connectivity-rules --namespace "my-namespace.abc123" --remove-all
```

## 查看 Namespace 的连接规则 {#terraform}

您可以通过两种方式查看附加到特定 Namespace 的连接规则。

<a id="view-the-connectivity-rules-for-a-namespace"></a>
### 获取 Namespace {#view-the-connectivity-rules-for-a-namespace}

连接规则包含在 `Namespace get` 命令返回的 Namespace 详细信息中。

```bash
tcld namespace get -n "my-namespace.abc123"
```

<a id="get-namespace"></a>
### 按 Namespace 列出连接规则 {#get-namespace}

要仅查看特定 Namespace 的连接规则（没有其他 Namespace 详细信息），请使用带有 Namespace 参数的 `connectivity-rule list` 命令。

```bash
tcld connectivity-rule list -n "my-namespace.abc123"
```

<a id="list-connectivity-rules-by-namespace"></a>
## Update DNS 或客户端使用专用连接 {#list-connectivity-rules-by-namespace}

我们强烈建议使用私有 DNS，而不是更新客户端服务器和 TLS 设置：

- [如何在AWS中设置私有DNS](/cloud/connectivity/aws-connectivity#configuring-private-dns-for-aws-privatelink) 
- [如何在 GCP 中设置私有 DNS](/cloud/connectivity/gcp-connectivity#configuring-private-dns-for-gcp-private-service-connect)

如果您无法配置私有 DNS，则必须在 Temporal CLIents 中进行两项设置：

1. 将端点服务器地址设置为 PrivateLink 或 Private Services Connect 端点（例如 `vpce-0123456789abcdef-abc.us-east-1.vpce.amazonaws.com:7233` 或 `<GCP PSC IP address>:7233`）
2. 设置 TLS 配置以覆盖 TLS 服务器名称（例如 my-Namespace.my-account.tmprl.cloud）

更新这些设置取决于您使用的客户端。

<a id="update-dns-or-clients-to-use-private-connectivity"></a>

<a id="temporal-sdks"></a>
## 控制平面连接 {#temporal-sdks}

使用 Temporal Cloud [网页用户界面](/web-ui)、[地形提供商](/cloud/terraform-provider)、[`tcld` 命令行](/cloud/tcld) 或 [云运营 API](/ops) 需要对 Temporal Cloud 控制平面进行网络访问。

<a id="control-plane-connectivity"></a>
### 控制平面主机名 {#control-plane-connectivity}

不同的主机名用于服务的不同部分。

- `saas-api.tmprl.cloud`（Terraform、tcld 和 Cloud Ops API 需要） 
- `web.onboarding.tmprl.cloud`（Web UI 需要）
- `web.saas-api.tmprl.cloud`（Web UI 需要）

<a id="control-plane-hostnames"></a>
### AWS PrivateLink 与 Temporal Cloud 控制平面的连接 {#control-plane-hostnames}

Temporal Cloud 支持与控制平面的 [AWS 私有链接](https://aws.amazon.com/privatelink/) 连接，这允许从无法传出公共互联网的 VPC 中运行的应用程序进行访问。 Temporal Cloud **不**支持限制帐户，以便专用连接是控制平面的唯一连接方法；控制平面始终可以通过公共互联网访问。

无论您选择如何连接，控制平面访问始终通过 [API 密钥](/cloud/api-keys#overview) 或 JWT 令牌进行安全身份验证。

要设置与 Temporal Cloud 控制平面的 PrivateLink 连接，请遵循 [这些说明](/cloud/connectivity/aws-connectivity)，但使用以下控制平面端点信息：

| 主机名               | 地区      | 控制平面 PrivateLink 服务名称                    |
| ---------------------- | ----------- | --------------------------------------------------------- |
| `saas-api.tmprl.cloud` | `us-west-2` | `com.amazonaws.vpce.us-west-2.vpce-svc-0c57a5930b6f6be0e` |

控制平面 PrivateLink 端点包括 [私有 DNS 名称](https://docs.aws.amazon.com/vpc/latest/privatelink/manage-dns-names.html)，它允许您的客户端使用 PrivateLink 连接，而无需设置私有 DNS 或覆盖客户端配置。要使用 DNS 名称，请确保您的 VPC 启用了 `Enable DNS hostnames` 和 `Enable DNS support` 选项。如果无法使用 DNS 名称，也可以手动设置 [设置私有 DNS](/cloud/connectivity/aws-connectivity#configuring-private-dns-for-aws-privatelink) 或 [覆盖客户端上的服务器和 TLS 设置](/cloud/connectivity#update-dns-or-clients-to-use-private-connectivity)。

:::caution Finish client setup

创建专用连接后，您必须使用提供的 DNS 名称、设置专用 DNS 或 Update 配置您想要使用专用连接的所有客户端。

如果没有此步骤，如果您的客户端之前使用公共连接，则他们可能会通过互联网连接到控制平面，否则他们将根本无法连接。
:::

还可以通过 AWS us-west-2 中的 PrivateLink 终端节点访问控制平面，该终端节点可用于 Namespace 流量，但我们强烈建议对控制平面流量使用控制平面特定终端节点。
