:::danger PromQL endpoint deprecated

PromQL 端点及其 `Temporal_cloud_v0_*` 指标已于 2026 年 4 月 2 日弃用，并且不再接受新用户。
将于 **2026 年 10 月 5 日**对所有用户禁用 PromQL 端点。

迁移到 [OpenMetrics 端点](/cloud/metrics/openmetrics)，它提供具有更高基数、准确百分位数和简化身份验证的 `Temporal_cloud_v1_*` 指标。
有关完整的 v0 到 v1 指标映射，请参阅 [迁移指南](/cloud/metrics/openmetrics/migration-guide)。

:::

指标是一种测量或数据点，可提供对系统性能和运行状况的洞察。
本文档介绍了已弃用的 Temporal Cloud PromQL 端点中可用的 `Temporal_cloud_v0_*` 指标。
有关当前指标参考，请参阅 [OpenMetrics 指标参考](/cloud/metrics/openmetrics/metrics-reference)。

本文档描述：

- **[可用的 Temporal Cloud 指标](#available-metrics)**：
  Temporal Cloud 发出的指标包括 gRPC 错误、请求、与轮询器匹配的成功任务的计数等。
- **[指标标签](#metrics-labels)**：
  Temporal Cloud 指标标签可以过滤指标并帮助对结果进行分类和区分。
- **[运营](#metrics-operations)**：
  操作是一种特殊类型的标签，用于对收集指标时执行的操作类型进行分类。

:::info SDK METRICS

本文档讨论 [Temporal Cloud](/cloud) 发出的指标。
Temporal SDK 还发出源自 Temporal CLIents 和 Worker 进程的指标。
您可以在其 [专用页面](/references/sdk-metrics) 上找到有关 Temporal SDK 指标的信息。

请注意：

- SDK 指标以短语 `Temporal_` 开头。
- Temporal Cloud 指标以 `Temporal_cloud_` 开头。

:::

## 可用的 Temporal Cloud 指标 {#available-metrics}

**Temporal Cloud 发出哪些指标？**

针对您的 Namespace 发出以下指标：

### 前端服务指标 {#frontend}

#### Temporal_cloud_v0_frontend_service_error_count

这是按操作聚合返回的 gRPC 错误计数。
标签： Temporal_account、Temporal_Namespace、操作、Temporal_service_type

#### Temporal_cloud_v0_frontend_service_request_count

这是按操作聚合的收到的 gRPC 请求的计数。
标签： Temporal_account、Temporal_Namespace、操作、Temporal_service_type

#### Temporal_cloud_v0_resource_exhausted_error_count

收到的 gRPC 请求受 Temporal Cloud 速率限制，按原因聚合。
标签： Temporal_account、Temporal_Namespace、resource_exhausted_cause

#### Temporal_cloud_v0_state_transition_count

每个 Namespace 的状态转换计数。

#### Temporal_cloud_v0_total_action_count

Temporal Cloud 操作的大致计数。
标签： Temporal_account、Temporal_Namespace、is_background、Namespace_mode

### 民意调查指标 {#poll}

#### Temporal_cloud_v0_poll_success_count

与轮询器成功匹配的任务。
标签： Temporal_account、Temporal_Namespace、操作、task_type、Temporal_service_type

#### Temporal_cloud_v0_poll_success_sync_count

成功同步与轮询器匹配的任务。
标签： Temporal_account、Temporal_Namespace、操作、task_type、Temporal_service_type

#### Temporal_cloud_v0_poll_timeout_count

当轮询器在超时之前没有可用的任务时。
标签： Temporal_account、Temporal_Namespace、操作、task_type、Temporal_service_type

### 复制延迟指标 {#replication-lag}

#### Temporal_cloud_v0_replication_lag_bucket

具有高可用性的 Namespace 在特定时间间隔内的 [复制滞后](/cloud/metrics/openmetrics/metrics-reference#temporal_cloud_v1_replication_lag_p99) 直方图。
标签: Temporal_account, Temporal_Namespace, le

#### Temporal_cloud_v0_replication_lag_count

具有高可用性的 Namespace 在特定时间间隔内的 [复制滞后](/cloud/metrics/openmetrics/metrics-reference#temporal_cloud_v1_replication_lag_p99) 计数。
标签： Temporal_帐户、Temporal_Namespace

#### Temporal_cloud_v0_replication_lag_sum

具有高可用性的 Namespace 在特定时间间隔内 [复制滞后](/cloud/metrics/openmetrics/metrics-reference#temporal_cloud_v1_replication_lag_p99) 的总和。
标签： Temporal_帐户、Temporal_Namespace

<a id="schedule"></a>
### Schedule 指标 {#schedule}

#### Temporal_cloud_v0_Schedule_action_success_count

成功执行 Scheduled Workflow。
标签： Temporal_帐户、Temporal_Namespace

#### Temporal_cloud_v0_Schedule_buffer_overruns_count

当配置 `buffer_all` 重叠策略时，平均 Schedule 游程长度大于平均 Schedule 间隔。
标签： Temporal_帐户、Temporal_Namespace

#### Temporal_cloud_v0_Schedule_missed_catchup_window_count

当 Workflow 延迟时间长于追赶窗口时，跳过 Scheduled 执行。
标签： Temporal_帐户、Temporal_Namespace

#### Temporal_cloud_v0_Schedule_rate_limited_count

Workflow 由于超出速率限制而延迟。
标签： Temporal_帐户、Temporal_Namespace

### 服务延迟指标 {#service-latency}

#### Temporal_cloud_v0_service_latency_bucket

`SignalWithStartWorkflowExecution`、`SignalWorkflowExecution`、`StartWorkflowExecution` 操作的延迟。
标签： Temporal_account、Temporal_Namespace、le、操作、Temporal_service_type

#### Temporal_cloud_v0_service_latency_count

`SignalWithStartWorkflowExecution`、`SignalWorkflowExecution`、`StartWorkflowExecution` 操作的延迟观察计数。
标签： Temporal_account、Temporal_Namespace、操作、Temporal_service_type

#### Temporal_cloud_v0_service_latency_sum

`SignalWithStartWorkflowExecution`、`SignalWorkflowExecution`、`StartWorkflowExecution` 操作的延迟观察时间总和。
标签： Temporal_account、Temporal_Namespace、操作、Temporal_service_type

<a id="workflow"></a>
### Workflow 指标 {#workflow}

#### Temporal_cloud_v0_Workflow_cancel_count

Workflows 在完成执行之前被取消。
标签： Temporal_account、Temporal_Namespace、操作、Temporal_service_type

#### Temporal_cloud_v0_Workflow_continued_as_new_count

Workflow Execution 从过去的执行中延续为新。
标签： Temporal_account、Temporal_Namespace、操作、Temporal_service_type

#### Temporal_cloud_v0_Workflow_failed_count

Workflow 在完成之前失败。
标签： Temporal_account、Temporal_Namespace、操作、Temporal_service_type

#### Temporal_cloud_v0_Workflow_success_count

Workflow 已成功完成。
标签： Temporal_account、Temporal_Namespace、操作、Temporal_service_type

#### Temporal_cloud_v0_Workflow_terminate_count

Workflows 在完成执行之前终止。
标签： Temporal_account、Temporal_Namespace、操作、Temporal_service_type

#### Temporal_cloud_v0_Workflow_timeout_count

Workflow 在完成执行之前超时。
标签： Temporal_account、Temporal_Namespace、操作、Temporal_service_type

## 指标标签 {#metrics-labels}

**您可以使用哪些标签来过滤指标？**

Temporal Cloud 指标在其关联元数据中包含称为标签的键值对。
标签可帮助您对指标进行分类和区分，以进行精确筛选、Querying 和聚合。
使用标签来过滤特定属性或比较值，例如直方图中的数字桶。
添加的上下文增强了监控和分析功能，提供对数据的更深入的见解。

使用以下标签来过滤指标：

| 标签                      | 解释                                                                                                                                                                                                                                                                                |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `le`                       | 小于或等于 (`le`) 在直方图中用于根据观测值小于或等于预定义上限将观测值分类到桶中。                                                                                                                      |
| `operation`                | 这包括 gRPC 操作和一般云操作，例如如：<ul><li>SignalWorkflowExecution</li><li>StartBatchOperation</li><li>StartWorkflowExecution</li><li>Tas kQueueMgr</li><li>终止Workflow执行</li><li>ZZT ERM031ZZNamespace</li><li>UpdateSchedule</li></ul>请参阅：[公制运算](#metrics-operations) 和 [Temporal Cloud 操作参考](/references/operation-list)|
| `resource_exhausted_cause` | 资源耗尽的原因。                                                                                                                                                                                                                                                             |
| `task_type`                | Activity、Workflow 或 Nexus。                                                                                                                                                                                                                                                              |
| `Temporal_account`         | Temporal 帐户。                                                                                                                                                                                                                                                                          |
| `Temporal_Namespace`       | Temporal Namespace。                                                                                                                                                                                                                                                                        |
| `Temporal_service_type`    | 前端或匹配或历史或 Worker。                                                                                                                                                                                                                                                 |
| `is_background`            | `Temporal_cloud_v0_total_action_count` 上的此标签指示 Temporal 后台作业何时生成操作，例如：每小时 Workflow 导出。                                                                                                                                |
| `Namespace_mode`           | `Temporal_cloud_v0_total_action_count` 上的此标签指示操作是由活动 Namespace 还是由备用 Namespace 产生。对于常规 Namespace，`Namespace_mode` 将始终处于“活动”状态。                                                                                             |

以下是如何使用标签过滤指标的示例：

```text
temporal_cloud_v0_poll_success_count{__rollup__="true", operation="TaskQueueMgr", task_type="Activity", temporal_account="12345", temporal_namespace="your_namespace.12345", temporal_service_type="matching"}
```
