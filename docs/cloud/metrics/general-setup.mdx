<a id="configure-via-ui"></a>
## 使用 UI {#configure-via-ui} 配置
**如何使用 Temporal Cloud UI 配置指标端点**

:::note

要查看和管理第三方集成设置，您的用户帐户必须具有帐户所有者或全局管理员 [角色](/cloud/manage-access/roles-and-permissions#account-level-roles)。

:::

要分配证书并生成指标端点，请按照以下步骤操作：

1. 使用帐户所有者或全局管理员 [角色](/cloud/manage-access/roles-and-permissions#account-level-roles) 登录到 Temporal Cloud UI。
2. 转至 **设置** 并选择 **可观察性**。
4. 添加您的根 CA 证书 (.pem) 并保存。
   请注意，如果已设置可观察性端点，您可以在此处附加根 CA 证书，以在可观察性工具中使用生成的可观察性端点。
5. 要测试您的端点，请在主机上运行以下命令：
   ```
   curl -v --cert <path to your client-cert.pem> --key <path to your client-cert.key> "<your generated Temporal Cloud prometheus_endpoint>/api/v1/query?query=temporal_cloud_v0_state_transition_count"
   ```
如果您在 Temporal Cloud 实例中的 Namespace 上运行 Workflow，则运行此命令后您应该会看到一些数据。

页面刷新后，新的指标端点将显示在 **Endpoint** 下方，格式为 `https://<account-id>.tmprl.cloud/prometheus`。
使用端点来配置您的可观察性工具。
例如，如果您使用 Grafana，请参阅 [Grafana数据源配置](/cloud/metrics/prometheus-grafana#grafana-data-source-configuration)。

您还可以通过 [普罗米修斯 HTTP API](https://prometheus.io/docs/prometheus/latest/querying/api/) 访问 Query，网址如下：

```
https://<account-id>.tmprl.cloud/prometheus/api/v1/query?query=temporal_cloud_v0_state_transition_count
```

例如：

```
$ curl --cert client.pem --key client-key.pem "https://<account-id>.tmprl.cloud/prometheus/api/v1/query?query=temporal_cloud_v0_state_transition_count" | jq .
```

<a id="configure-via-cli-tcld"></a>
## 使用 tcld 配置端点 {#configure-via-cli-tcld}
**如何使用 tcld CLI 配置指标端点。**

要将证书添加到指标端点，请使用 [`tcld account metrics accepted-client-ca add`](/cloud/tcld/account#add)。

要启用指标端点，请使用 [`tcld account metrics enable`](/cloud/tcld/account#enable)。

要禁用指标端点，请使用 [`tcld account metrics disable`](/cloud/tcld/account#disable)。

有关详细信息，请参阅 [tcld 帐户指标命令](/cloud/tcld/account#metrics)。

<a id="query-promql"></a>
## Query 用于具有 PromQL 端点的指标 {#query-promql}
Temporal Cloud 以 Prometheus 支持的格式发出指标。
Prometheus 是一个用于警报和监控的开源工具包。
Temporal 服务通过 [普罗米修斯 HTTP API 端点](https://prometheus.io/docs/prometheus/latest/querying/api/) 公开云指标。
Temporal Cloud 指标为可视化、监控和可观察性平台提供兼容的数据源。

您可以使用 [速度](https://prometheus.io/docs/prometheus/latest/querying/functions/#rate) 或 [增加](https://prometheus.io/docs/prometheus/latest/querying/functions/#increase) 等函数来计算 Temporal Cloud 指标的增长率：

```
rate(temporal_cloud_v0_frontend_service_request_count[$__rate_interval])
```

或者您可以使用 Prometheus 计算平均延迟或直方图四分位数：

```
# Average latency
rate(temporal_cloud_v0_service_latency_sum[$__rate_interval])
/ rate(temporal_cloud_v0_service_latency_count[$__rate_interval])

# Approximate 99th percentile latency broken down by operation
histogram_quantile(0.99, sum(rate(temporal_cloud_v0_service_latency_bucket[$__rate_interval])) by (le, operation))
```

每 30 秒抓取一次指标，并以 1 分钟的滞后时间暴露给指标端点。\
端点以 15 秒的分辨率返回数据，这会导致相同的值显示两次。

设置具有 Temporal Cloud 可观测性的 Grafana，通过创建或获取 Temporal Cloud 指标的 Prometheus 端点并启用 SDK 指标来查看指标。
