Temporal Cloud 的 [开放指标](https://openmetrics.io/) 端点以行业标准 Prometheus 格式为您的 Temporal Cloud 工作负载提供操作指标，从而利用现有的可观测性堆栈实现跨 Namespace、Workflow 和 Task Queue 的全面监控。

<a id="quick-links"></a>

* [集成](/cloud/metrics/openmetrics/metrics-integrations) - 开始通过常见集成导出指标
* [API文档](/cloud/metrics/openmetrics/api-reference) - 端点规范和高级配置
* [指标参考](/cloud/metrics/openmetrics/metrics-reference) - 所有指标的完整目录以及描述和标签

<a id="overview"></a>
## 概述 {#overview}
Temporal Cloud OpenMetrics 公开了 50 多个指标，涵盖 Workflow 生命周期、Task Queue 操作、服务性能和系统限制。所有指标均在一分钟窗口内聚合，并可在三分钟内进行抓取。

* [设置身份验证和抓取](/cloud/metrics/openmetrics/api-reference#authentication) 以及 API 文档。 
* 浏览 [完整的指标目录](/cloud/metrics/openmetrics/metrics-reference) 的说明和标签。 
* 使用 Query 端点的团队应查看 [迁移指南](/cloud/metrics/openmetrics/migration-guide)。

<a id="api-key-authentication"></a>
## API密钥认证 {#api-key-authentication}
创建具有“Metrics Read-Only”角色的 [服务帐户](/cloud/metrics/openmetrics/migration-guide#create-an-api-key)，生成 API 密钥，并立即开始抓取 - 无需证书轮换或分发。

<a id="global-endpoint"></a>
## 全局端点 {#global-endpoint}
这是 `metrics.Temporal.io` 的单个端点，它通过 API 密钥身份验证和标准 HTTPS 为您的整个帐户中的所有指标提供服务。

<a id="namespace-and-metric-filtering"></a>
## Namespace 和指标过滤 {#namespace-and-metric-filtering}
您可以使用 Query 参数启用选择性抓取来管理数据量和成本，支持通配符以实现灵活的 Namespace 选择和特定指标过滤。

<a id="dashboard-templates"></a>
## 仪表板模板 {#dashboard-templates}
生产就绪的 [Grafana 仪表板](https://github.com/grafana/jsonnet-libs/blob/master/temporal-mixin/dashboards/temporal-overview.json) 为 Visibility 提供即时的预构建查询和可视化。
