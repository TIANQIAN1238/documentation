本文档描述了 Temporal Cloud OpenMetrics 端点可用的所有指标。


<a id="metric-conventions"></a>
## 公制约定 {#metric-conventions}

<a id="metric-types"></a>
### 指标类型 {#metric-types}

所有指标都作为 OpenMetrics 仪表公开，但代表不同的测量类型：

* *速率指标*：每秒聚合值的速率  
* *价值指标*：回溯窗口内的最新聚合值（例如积压、限制）  
* *百分位指标*：预先计算的聚合延迟百分位（以秒为单位）

:::note

所有指标均存储为 1 分钟聚合。

:::

<a id="common-labels"></a>
### 常用标签 {#common-labels}

所有指标都包含这些基本标签：

| 标签 | 描述 |
| ----- | ----- |
| `Temporal_Namespace` | Temporal Namespace |
| `Temporal_account` | Temporal 帐户标识符 |
| `region` | 指标源自的云区域 |

<a id="opt-in-labels"></a>
### 选择加入标签 {#opt-in-labels}

某些标签由于其高基数而被**选择加入**。
当您抓取 OpenMetrics 端点时，默认情况下不包含这些标签。
要启用选择加入标签，请将其添加到抓取 URL 上的 `labels` Query 参数中。
启用选择加入标签后，它会填充在支持它的**所有指标**上。

| 标签 | 可用于 | 描述 |
| ----- | ----- | ----- |
| `Temporal_Activity_type` | Activity 指标 | Activity 类型名称 |
| `Worker_version` | `Temporal_cloud_v1_approximate_backlog_count` | Worker版本 |

例如，要在抓取结果中包含 `Temporal_Activity_type`：

```
/v1/metrics?labels=temporal_activity_type
```

<a id="task-queue-metrics"></a>
### Task Queue 指标 {#task-queue-metrics}

:::caution High Cardinality

这些指标可能具有高基数，具体取决于存在的 Task Queue 数量。

:::

#### Temporal\_cloud\_v1\_approximate\_backlog\_count

Task Queue 中待处理任务的大致数量。已启动的活动不包含在计数中，因为它们已从 Task Queue 中出列。

| 标签 | 描述 |
| ----- | ----- |
| `Temporal_task_queue` | Task Queue 名称 |
| `task_type` | 任务类型：`Workflow` 或 `Activity` |
| `task_priority` | 任务优先级 |
| `Worker_version` | Worker 版本（选择加入） |

**类型**：值

#### Temporal\_cloud\_v1\_poll\_success\_count

每秒成功匹配的任务数。

| 标签 | 描述 |
| ----- | ----- |
| `operation` | 轮询操作名称 |
| `task_type` | 任务类型：`Workflow` 或 `Activity` |
| `Temporal_task_queue` | Task Queue 名称 |

**类型**：费率

#### Temporal\_cloud\_v1\_poll\_success\_sync\_count

每秒同步匹配的任务（无轮询等待）。

| 标签 | 描述 |
| ----- | ----- |
| `operation` | 轮询操作名称 |
| `task_type` | 任务类型：`Workflow` 或 `Activity` |
| `Temporal_task_queue` | Task Queue 名称 |

**类型**：费率

#### Temporal\_cloud\_v1\_poll\_timeout\_count

未收到任务而超时的轮询请求的比率。

| 标签 | 描述 |
| ----- | ----- |
| `operation` | 轮询操作名称 |
| `task_type` | 任务类型：`Workflow` 或 `Activity` |
| `Temporal_task_queue` | Task Queue 名称 |

**类型**：费率

#### Temporal\_cloud\_v1\_no\_poller\_tasks\_count

添加到没有活动轮询器的队列中的任务的速率。

| 标签 | 描述 |
| ----- | ----- |
| `Temporal_task_queue` | Task Queue 名称 |
| `task_type` | 任务类型：`Workflow` 或 `Activity` |

**类型**：费率

<a id="namespace-metrics"></a>
### Namespace 指标 {#namespace-metrics}

#### Temporal\_cloud\_v1\_Namespace\_open\_Workflows

Namespace 中当前打开的 Workflow 数量。

**类型**：值

#### Temporal\_cloud\_v1\_total\_action\_count

每秒执行的操作总数。 `is_background=false` 的操作计入“`Temporal_cloud_v1_action_limit`”。

| 标签 | 描述 |
| ----- | ----- |
| `is_background` | 操作是否为背景：`true` 或 `false`。后台操作（例如历史记录导出）不计入操作速率限制 |
| `Namespace_mode` | 指示操作是由 `active` 还是 `standby` Namespace 产生 |

:::note

不包括 `region` 标签。操作的范围为 Namespace 级别。

:::

**类型**：费率

#### Temporal\_cloud\_v1\_total\_action\_throttled\_count

每秒限制的操作总数。

**类型**：费率

<a id="temporalcloudv1operationscount"></a>

每秒执行的操作数。

| 标签 | 描述 |
| ----- | ----- |

| `is_background` | 操作是否为后台：`true` 或 `false`。后台操作不计入操作速率限制 |
| `Namespace_mode` | 指示操作是由 `active` 还是 `standby` Namespace 生成 |

**类型**：费率

#### Temporal\_cloud\_v1\_operations\_throttled\_count

由于每秒速率限制，操作受到限制。

| 标签 | 描述 |
| ----- | ----- |
| `operation` | 操作名称 |
| `is_background` | 操作是否为后台：`true` 或 `false`。后台操作不计入操作速率限制 |
| `Namespace_mode` | 指示 `active` 或 `standby` Namespace 中的操作是否受到限制 |

**类型**：费率

<a id="limit-metrics"></a>
### 限制指标 {#limit-metrics}

<a id="temporalcloudv1operationslimit"></a>
#### Temporal\_cloud\_v1\_操作\_limit {#temporalcloudv1operationslimit}

Namespace 当前配置的每秒操作数限制。

**类型**：值

#### Temporal\_cloud\_v1\_action\_limit

当前为 Namespace 配置的每秒操作限制。使用“`Temporal_cloud_v1_total_action_count`` and `is_background=false”跟踪此限制的利用率。

**类型**：值

#### Temporal\_cloud\_v1\_service\_request\_limit

Namespace 当前配置的前端服务 RPS 限制。使用“`Temporal_cloud_v1_service_request_count`”跟踪此限制的利用率

**类型**：值

#### Temporal\_cloud\_v1\_poller\_limit

Namespace 当前配置的轮询器限制。使用“`Temporal_cloud_v1_service_pending_requests`”跟踪此限制的利用率。

**类型**：值

#### Temporal\_cloud\_v1\_action\_on\_demand\_envelope\_limit

每秒操作的按需信封限制。对于处于预配置容量模式的 Namespace，这显示了在按需模式下运行时的操作限制。对于已处于按需模式的 Namespace，该值跟踪与 `Temporal_cloud_v1_action_limit` 相同的值。

:::note

不包括 `region` 标签。限制范围为 Namespace 级别。

:::

**类型**：值

#### Temporal\_cloud\_v1\_operations\_on\_demand\_envelope\_limit

每秒操作数的按需包络限制。对于处于预配置容量模式的 Namespace，这显示了在按需模式下运行时的操作限制。对于已处于按需模式的 Namespace，该值跟踪与 `Temporal_cloud_v1_operations_limit` 相同的值。

:::note

不包括 `region` 标签。限制范围为 Namespace 级别。

:::

**类型**：值

#### Temporal\_cloud\_v1\_service\_request\_on\_demand\_envelope\_limit

每秒服务请求的按需信封限制。对于处于预配置容量模式的 Namespace，这显示了在按需模式下运行时的服务请求限制。对于已处于按需模式的 Namespace，该值跟踪与 `Temporal_cloud_v1_service_request_limit` 相同的值。

:::note

不包括 `region` 标签。限制范围为 Namespace 级别。

:::

**类型**：值
