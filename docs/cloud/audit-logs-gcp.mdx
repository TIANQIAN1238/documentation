<a id="manual-setup-prerequisites"></a>
## 手动设置先决条件 {#manual-setup-prerequisites}

:::note

仅手动设置需要这些步骤。
如果您使用 Terraform 进行部署，则无需满足这些先决条件。

:::

在配置手动审核日志接收器之前，请在 Google Cloud 中完成以下步骤：

1. 创建一个 Pub/Sub 主题并记下其主题名称，例如 `test-auditlog`。
1. 在 Google Cloud 中的同一项目中设置服务帐号，然后按照
   Temporal Cloud 用于配置该服务帐户权限的 UI。

<a id="create-an-audit-log-sink"></a>
## 创建审计日志接收器 {#create-an-audit-log-sink}

1. 在 Temporal Cloud 用户界面中，选择“**设置**”。
1. 在**设置**页面上，选择**审核日志**。
1. 在 **审核日志集成** 卡中，选择 **设置**。
1. 在 **审核日志集成** 页面上，选择 **Pub/Sub**。
1. 在 **服务帐户电子邮件** 字段中，输入您在先决条件中创建的服务帐户的电子邮件。
1. 在 **主题名称** 字段中，输入您在先决条件中创建的 Pub/Sub 主题的主题名称。
1. 有两种方法可以配置服务帐户以写入 Pub/Sub 接收器：选择 **Manual** 手动配置帐户，或 **Deploy with Terraform** 使用 Terraform。
   如果您使用 Terraform，则无需完成上述先决步骤。
1. 按照您选择的方法的 Temporal Cloud UI 中的说明进行操作。
1. 为确保审核日志能够到达您的 Pub/Sub 主题，您可以使用 **验证** 按钮来确认其设置是否正确。这验证了 Temporal 可以成功写入您的主题。
如果一切配置正确，您将看到 `Success` 状态，指示 Temporal 已写入 Pub/Sub 主题。
1. 点击**创建**配置审核日志。
   审核日志将在 10 分钟内开始显示在 Pub/Sub 中
