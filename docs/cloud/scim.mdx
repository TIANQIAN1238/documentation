[SCIM](https://scim.cloud/) 允许您将身份提供商 (IdP) 与 Temporal Cloud 集成，以自动执行用户配置和访问。配置 SCIM 后，IdP 中的更改将自动反映在 Temporal Cloud 中，包括：

- 用户创建/入职
- 用户删除/下线
- 组中的用户成员资格

您可以将 SCIM 组映射到 Temporal Cloud [角色和权限](/cloud/manage-access/roles-and-permissions)，以便用户根据所属组自动获得所需的 Temporal Cloud 访问权限。

:::info

SCIM 是一项付费功能。有关详细信息，请参阅 [定价页面](/cloud/pricing)。

:::

<a id="supported-idp-vendors"></a>
## 支持的 IdP 供应商 {#supported-idp-vendors}

支持的上游 IdP 供应商包括：
* [奥克塔](#configure-scim-with-okta)
* 微软 Entra ID (Azure AD)
* 谷歌工作区
* 一次登录
* 赛博方舟
* 跳云
* PingFederate
* 任何符合 SCIM 2.0 的提供商

<a id="preparing-for-scim"></a>
## SCIM 准备 {#preparing-for-scim}

在开始使用 SCIM 之前，您需要完成此清单：

1. 配置 [SAML](/cloud/saml) 单点登录。
1. 确定您组织的 **IdP 管理员**，他负责配置和管理您的 SCIM 集成。
   当您在此过程的下一阶段寻求支持时，请指定他们的联系方式。

完成这些步骤后，您就可以提交 [支持票](/cloud/support#support-ticket) 以启用 SCIM。

:::tip Adding and removing users

启用 SCIM 进行用户管理后，您仍然可以使用 Temporal Cloud 界面在 SCIM 之外添加和删除用户，直到禁用用户生命周期管理。
您始终可以从 Temporal Cloud 界面更改用户或组的帐户角色。

:::

## 使用 SCIM 和 Okta {#configure-scim-with-okta}

1. Temporal 支持在您的帐户上启用 SCIM 集成。
   启用集成会自动通过电子邮件将配置链接发送给您的 Okta 管理员。
   这授权他们设置集成。
1. 您的 Okta 管理员打开提供的链接。
   该链接提供了配置集成的分步说明。
1. 在 Okta 中配置后，Temporal Cloud 将开始接收 SCIM 消息并自动加入和退出 Okta 中配置的用户和组。

需要注意的几点：

- 用户和组更改事件在 Okta 中发生后 10 分钟内应用。
- SCIM 的用户生命周期管理还允许从组成员身份派生用户角色。
- 在 Temporal Cloud 中同步群组后，您可以使用 `tcld` 为该群组分配角色。
  有关说明，请参阅 [用户组管理](https://github.com/temporalio/tcld?tab=readme-ov-file#user-group-management) 页面。
