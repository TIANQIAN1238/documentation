这些指南概述了使用 Temporal Cloud 的基本原则和最佳实践。它的存在是为了提供一个
**经过验证的、固执己见的**框架，可以帮助没有启用计划或想要评估的团队
并完善他们对 Temporal 的使用。

＃＃ 概述

如果没有明确定义的 Temporal 标准，组织经常会因不一致的 Workflow 实施而苦苦挣扎，
最佳实践分散，开发方法不一致。该文档框架可以帮助开发人员
通过提供以下内容建立强大的 Temporal 标准：

- **经过验证的基础原则**已在不同的用例中得到验证
- **标准化实施模式**供团队在项目中一致采用
- **有信心**符合 Temporal 的架构原则和推荐实践

通过遵循本指南，开发人员可以定义全面的 Temporal 标准，以确保他们的 Workflow
编排实施是可维护的、可扩展的，并且从一开始就与平台最佳实践保持一致。

## 目标受众

本节适用于：

- 负责在其组织内构建 Temporal Cloud 实践的开发人员。
- 任何构建教程、课程、入门路径或文档的人
- 创建 Temporal 相关学习材料的合作伙伴或供应商
