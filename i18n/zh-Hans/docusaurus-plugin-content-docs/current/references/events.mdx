[事件](/Workflow-execution/event#event) 由 [Temporal 服务](/Temporal-service) 创建以响应外部事件，[命令](/Workflow-execution#command) 由 [Workflow Execution](/Workflow-execution) 生成。
下面列出了 Workflow Execution [Event History](/Workflow-execution/event#event-history) 中可能出现的所有事件。

### WorkflowExecutionStarted 开始

这始终是 Workflow Execution Event History 中的第一个 [事件](/Workflow-execution/event#event)。
它表明 Temporal 服务收到了生成 Workflow Execution 的请求。

|领域|描述 |
| ---------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Workflow_类型 |已启动的 [Workflow](/Workflows) 的 [名称](/Workflow-definition#Workflow-type)。                                                                   |
|父级_Workflow_Namespace |父级 [Workflow Execution](/Workflow-execution) 的 [Namespace](/Namespaces)（如果适用）。                                                           |
|父_Workflow_执行 |标识父 Workflow 和执行运行。                                                                                                          |
|父发起的事件 ID |此事件对应的 [StartWorkflowExecutionInitiated](#startchildWorkflowexecutioninitied) 事件的 ID。                                            |
|任务队列 |此 [Workflow Task](/tasks#Workflow-task) 已排入队列的 [Task Queue](/task-queue)。
|输入 |由 SDK 反序列化以向 Workflow 提供参数的信息。                                                                              |
| Workflow_execution_timeout | Workflow_execution_timeout | [Workflow Execution](/Workflow-执行) 的总超时时间，包括重试和 Continue-As-New。                                               |
| Workflow_run_timeout | Workflow_run_timeout |单次 Workflow 运行超时。                                                                                                                              |
| Workflow_task_timeout | Workflow_task_timeout |单个Workflow Task超时。                                                                                                                             |
|继续执行运行 ID先前 Workflow 的 [运行 ID](/Workflow-execution/Workflowid-runid#run-id) 继续作为新的、重试或由 Cron 执行到此 Workflow 中。     |
|发起人|允许 Workflow 继续作为新的 Workflow Execution。                                                                                                   |
|持续失败 |失败的序列化结果。                                                                                                                                |
|最后完成结果 |来自先前完成的[任务](/tasks#task) 的信息（如果适用）。                                                                                  |
|原始执行运行 ID原始 Workflow 的 [Run Id](/Workflow-execution/Workflowid-runid#run-id) 已启动。                                                                    |
|身份|请求启动此 Workflow 的 [Client](/self-hosted-guide/security#client-connections) 或父 Workflow [Worker](/Workers#Worker) 的 ID。 |
|第一个执行运行 ID沿着 [Continue-As-New](/Workflow-execution/Continue-As-New) 链的第一个 [运行 Id](/Workflow-execution/Workflowid-runid#run-id) 运行并重置。     |
|重试策略 |由服务的动态配置确定的重试次数。将进行重试，直到达到“Schedule_to_close_timeout”。                  |
|尝试|为完成此任务而尝试的次数。                                                                                              |
| Workflow_execution_expiration_time | Workflow_execution_expiration_time | Workflow Execution 将[超时]的绝对时间(/encyclopedia/Detecting-Workflow-failures#Workflow-execution-timeout)。                       |
| cron_Schedule | cron_Schedule |显示 Workflow 的 [Cron Schedule](/cron-job)（如果适用）。                                                                                             |
|第一_Workflow_task_backoff |包含 Workflow 的本次迭代为 Scheduled 与下次运行之间的时间量。适用于 Cron 调度。                |
|备忘录 | Workflow 中显示的非索引信息。                                                                                                               |
|搜索属性 |提供用于设置 Workflow 的 [Search Attributes](/search-attribute) 的数据。                                                                              |
| prev_auto_reset_points |                                                                                                                                                                |
|标题| [Signal](/sending-messages#sending-Signals) 的发送者传递的信息被复制到 [Workflow Task](/tasks#Workflow-task) 中。             |
|完成回调 | Workflow 启动时附加完成回调。                                                                                                  |

### Workflow执行完成

这表明 [Workflow Execution](/Workflow-执行) 已成功完成。 [事件](/Workflow-execution/event#event) 包含 Workflow Execution 结果。

|领域|描述 |
| -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
|结果|已完成的 [Workflow](/Workflows) 的序列化结果。                                                                                       |
| Workflow_task_completed_event_id | Workflow_task_completed_event_id |报告事件的 [WorkflowTaskCompleted](#Workflowtaskcompleted) 的 ID。                                              |
| new_execution_run_id | 新执行运行id新 Workflow Execution 的 [运行 ID](/Workflow-execution/Workflowid-runid#run-id) 是由于 [Cron Schedule](/cron-job) 启动的。 |

### WorkflowExecutionFailed

此 [事件](/Workflow-execution/event#event) 指示 [Workflow Execution](/Workflow-execution) 未成功完成并包含 Workflow Execution 错误。

|领域|描述 |
| -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
|失败| [Workflow](/Workflows) 故障的序列化结果。                                                                                             |
|重试状态|提供是否应该重试[任务](/tasks#task) 的原因。                                                            |
| Workflow_task_completed_event_id | Workflow_task_completed_event_id |报告事件的 [WorkflowTaskCompleted](#Workflowtaskcompleted) 的 [运行 ID](/Workflow-execution/Workflowid-runid#run-id)。 |
| new_execution_run_id | 新执行运行id由 Cron 或 [重试](/encyclopedia/retry-policies) 启动的新 Workflow 的 [运行 ID](/Workflow-execution/Workflowid-runid#run-id)。            |

### WorkflowExecutionTimedOut

此 [Event](/Workflow-execution/event#event) 类型表示 [Workflow Execution](/Workflow-execution) 已由 [Temporal Server](/Temporal-service/Temporal-server) 超时，因为 [Workflow](/Workflows) 尚未在以下时间内完成[超时]（/百科全书/检测-Workflow-failures#Workflow-执行超时）设置。

|领域|描述 |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- |
|重试状态|提供是否应该重试[任务](/tasks#task) 的原因。                                                 |
| new_execution_run_id | 新执行运行id由 Cron 或 [重试](/encyclopedia/retry-policies) 启动的新 Workflow 的 [运行 ID](/Workflow-execution/Workflowid-runid#run-id)。 |

### WorkflowExecutionCancelRequested

此 [Event](/Workflow-execution/event#event) 类型指示已发出取消 [Workflow Execution](/Workflow-execution) 的请求。

|领域 |描述 |
| ------------------------ | | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
|原因 |用户提供的取消请求原因。                                                                                           |
|外部_发起_事件_id |请求取消的 [Workflow](/Workflows) 中事件的 [运行 ID](/Workflow-execution/Workflowid-runid#run-id)（如果适用）。 |
|外部_Workflow_执行 |识别外部 Workflow 及其执行的运行。                                                                              |
|身份|请求取消的 [Worker](/Workers#Worker) 的 ID。                                                                                 |

### WorkflowExecutionCanceled

此 [Event](/Workflow-execution/event#event) 类型表示客户端已确认取消请求，并且 [Workflow Execution](/Workflow-execution) 已被取消。

|领域|描述 |
| -------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| Workflow_task_completed_event_id | Workflow_task_completed_event_id |报告事件的 [WorkflowTaskCompleted](#Workflowtaskcompleted) 的 ID。 |
|详情 |取消时 [Workflow](/Workflows) 报告的附加信息。                 |

### WorkflowExecutionSignaled

此 [Event](/Workflow-execution/event#event) 类型表示 [Workflow](/Workflows) 已收到 [Signal](/sending-messages#sending-Signals) 事件。
事件类型包含 Signal 名称和 Signal 负载。

|领域|描述 |
| ----------- | -------------------------------------------------------------------------------------------------------------------------- |
| Signal_名称 |要触发的 Signal 的名称/类型。                                                                          |
|输入|由 SDK 反序列化以向 Workflow 函数提供参数的信息。                    |
|身份|标识 Signal 连接到 Workflow 的 [Worker](/Workers#Worker)。                                       |
|标题| Signal 的发送者传递的信息被复制到 [Workflow Task](/tasks#Workflow-task) 中。 |

### WorkflowExecutionTermated

此 [Event](/Workflow-execution/event#event) 类型指示 [Workflow Execution](/Workflow-execution) 已被强制终止，并且可能调用了终止 Workflow API。

|领域 |描述 |
| -------- | -------------------------------------------------------------------------------- |
|原因|用户或客户为 Workflow 终止提供的信息。 |
|详情 | Workflow 在终止时报告的附加信息。    |
|身份|标识请求终止的 Worker。                    |

### WorkflowExecutionContinuedAsNew

此 [Event](/Workflow-execution/event#event) 类型表示 Workflow 已成功完成，并且新的 Workflow 已在同一事务中启动。
此事件类型包含最后的 [Workflow Execution](/Workflow-执行) 结果以及新的 Workflow Execution 输入。

|领域|描述 |
| -------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| new_execution_run_id | 新执行运行id此 Continue-As-New 事件启动的新 Workflow 的 [运行 ID](/Workflow-execution/Workflowid-runid#run-id)。 |
| Workflow_类型 |由此事件启动的 Workflow 的名称/类型。                                                            |
|任务队列 |此 [Workflow Task](/tasks#Workflow-task) 已排入队列的 [Task Queue](/task-queue)。
|输入 |由 SDK 反序列化以向 Workflow 提供参数的信息。                                    |
| Workflow_run_timeout | Workflow_run_timeout |单次 Workflow 运行超时。                                                                                    |
| Workflow_task_timeout | Workflow_task_timeout |单个Workflow Task超时。                                                                                   |
| Workflow_task_completed_event_id | Workflow_task_completed_event_id |报告事件命令所用的 [WorkflowTaskCompleted](#Workflowtaskcompleted) 的 ID。              |
|退避开始间隔| [ContinuedAsNew](#Workflowexecutioncontinuedasnew) Workflow 开始延迟的时间量。        |
|发起人|允许 Workflow 作为新的执行继续。                                                                  |
|最后完成结果 |先前完成的任务传递给正在进行的执行的信息。                                        |
|标题| Signal 发送方传递的信息被复制到 Workflow Task 中。                                |
|备忘录 | Workflow 中显示的非索引信息。                                                                     |
|搜索属性 |提供用于设置 Workflow 的 [Search Attributes](/search-attribute) 的数据。                                    |

### WorkflowExecutionOptionsUpdated

此 [Event](/Workflow-execution/event#event) 类型表示 Workflow 选项已是 Updated。
事件类型包含 Updated 选项，例如版本控制覆盖或附加的完成回调。

|领域|描述 |
| -------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- |
|版本控制_覆盖 |在此事件中更新插入了版本控制覆盖。如果 nil 或 unset_versioning_override 为 true，则忽略。                    |
|取消设置版本覆盖在此事件中删除了版本控制覆盖。                                                                             |
|附加请求 ID |请求 ID 附加到正在运行的 Workflow Execution，因此具有相同请求 ID 的后续请求将被重复数据删除。 |
|附加完成回调 |附加到正在运行的 Workflow Execution 的完成回调。                                                       |

### WorkflowTaskScheduled

此 [Event](/Workflow-execution/event#event) 类型表示 [Workflow Task](/tasks#Workflow-task) 已是 Scheduled。
SDK 客户端现在应该能够处理任何新的历史事件。

|领域 |描述 |
| ---------------------- | ------------------------------------------------------------------------------------------ |
|任务队列 |此 Workflow Task 已排入队列的 [Task Queue](/task-queue)。
|开始到关闭超时 | [Worker](/Workers#Worker) 收到此任务后处理该任务所花费的时间。 |
|尝试|为完成此任务而尝试的次数。                          |

### WorkflowTask开始

此 [Event](/Workflow-execution/event#event) 类型指示 [Workflow Task](/tasks#Workflow-task) 已启动。
SDK 客户端已获取 Workflow Task 并正在处理新的历史事件。

|领域|描述 |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------- |
| Scheduled_event_id | Scheduled_event_id |此 Workflow Task 对应的 [WorkflowTaskScheduled](#WorkflowtaskScheduled) 事件的 ID。 |
|身份|标识启动此任务的 [Worker](/Workers#Worker)。                                            |
|请求 ID |标识 Workflow Task 请求。                                                                       |

### Workflow任务已完成

此 [Event](/Workflow-execution/event#event) 类型指示 [Workflow Task](/tasks#Workflow-task) 已完成。

|领域|描述 |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------- |
| Scheduled_event_id | Scheduled_event_id |此 Workflow Task 对应的 [WorkflowTaskScheduled](#WorkflowtaskScheduled) 事件的 ID。 |
|开始事件 ID |此任务对应的 [WorkflowTaskStarted](#Workflowtaskstarted) 事件的 ID。              |
|身份|完成此任务的 [Worker](/Workers#Worker) 的身份。                                         |
|二进制校验和 |完成此任务的 Worker 的二进制 ID。                                                           |

此 [Event](/Workflow-execution/event#event) 类型指示 [Workflow Task](/tasks#Workflow-task) 遇到 [超时](/encyclopedia/Detecting-Workflow-failures#Workflow-task-timeout)。
要么是带有本地缓存​​的 SDK 客户端当时不可用，要么是 SDK 客户端处理任务的时间太长。

|领域|描述 |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------- |
| Scheduled_event_id | Scheduled_event_id |此 Workflow Task 对应的 [WorkflowTaskScheduled](#WorkflowtaskScheduled) 事件的 ID。 |

### WorkflowTask失败

此 [Event](/Workflow-execution/event#event) 类型指示 [Workflow Task](/tasks#Workflow-task) 遇到故障。
通常这意味着 Workflow 是不确定的。
但是，Workflow 重置功能也使用此事件。

|领域|描述 |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Scheduled_event_id | Scheduled_event_id |此 Workflow Task 对应的 [WorkflowTaskScheduled](#WorkflowtaskScheduled) 事件的 ID。                                  |
|开始事件 ID |此 Workflow Task 对应的 [WorkflowTaskStarted](#Workflowtaskstarted) 事件的 ID。                                      |
|失败| Workflow Task 故障的详细信息。                                                                                                     |
|身份|失败此任务的 [Worker](/Workers#Worker) 的身份。必须显式定义 Worker 才能返回该字段的值。 |
|基本运行 ID | Workflow 的原始 [运行 ID](/Workflow-execution/Workflowid-runid#run-id)。                                                          |
|新的运行 ID |重置 Workflow 的运行 ID。                                                                                                            |
| fork_event_version |标识分叉到重置 Workflow 的事件版本。                                                                      |
|二进制校验和 |失败此任务的 Worker 的二进制 ID。必须显式定义 Worker 才能返回该字段的值。                   |

### ActivityTaskScheduled

此 [事件](/Workflow-execution/event#event) 类型指示 [Activity 任务](/tasks#Activity-task) 是 Scheduled。
SDK 客户端应获取此 Activity 任务并执行。
该事件类型包含 Activity 输入以及 Activity 超时配置。

|领域|描述 |
| -------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Activity_id |由 [Worker](/Workers#Worker) 或用户分配给此 Activity 的标识符。                                                                   |
| Activity_类型 | [Activity的类型](/Activity-definition#Activity-type)是Scheduled。                                                                     |
| Namespace | [Activity](/activities) 所在的 Workflow 的 Namespace。
|任务队列 |此 Activity 任务已排入队列的 [Task Queue](/task-queue)。
|标题| [Signal](/sending-messages#sending-Signals) 的发送者传递的信息被复制到 [Workflow Task](/tasks#Workflow-task) 中。 |
|输入 |由 SDK 反序列化以向 [Workflow](/Workflows) 函数提供参数的信息。                                           |
| Schedule_to_close_timeout | Schedule_to_close_timeout |调用者等待 Activity 完成的时间。限制此 Activity 尝试重试的时间。    |
| Schedule_to_start_timeout | Schedule_to_start_timeout |限制 Activity 任务可以在 Task Queue 中停留的时间。此超时无法重试。                                                    |
|开始到关闭超时 | Activity 被 Worker 拾取后允许的最大执行时间。此超时是可重试的。                         |
|心跳超时 |成功的 Worker 心跳之间允许的最长时间。                                                                               |
| Workflow_task_completed_event_id | Workflow_task_completed_event_id |报告事件的 [WorkflowTaskCompleted](#Workflowtaskcompleted) 的 ID。                                                    |
|重试策略 |由服务的动态配置确定的重试次数。将进行重试，直到达到“Schedule_to_close_timeout”。      |

### ActivityTask开始

此 [Event](/Workflow-execution/event#event) 类型表示 [Activity 任务执行](/tasks#Activity-task-execution) 已启动。 SDK Worker 获取 Activity 任务并开始处理 [Activity](/activities) 调用。 “ActivityTaskStarted”是在任务被分派到 Worker 时由服务器生成的，而不是在 Worker 开始执行任务时生成的。

但请注意，直到终端事件（例如 [ActivityTaskCompleted](#Activitytaskcompleted) 或 [ActivityTaskFailed](#Activitytaskfailed)）发生时，此事件才会写入历史记录。

|领域|描述 |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------- |
| Scheduled_event_id | Scheduled_event_id |此任务对应的 [ActivityTaskScheduled](#ActivitytaskScheduled) 事件的 ID。                   |
|身份|标识启动任务的 [Worker](/Workers#Worker)。                                                      |
|请求 ID |标识 Activity 任务请求。                                                                                |
|尝试|为完成此任务而尝试的次数。                                                    |
|最后一次失败 |最近失败事件的详细信息。仅当任务先前失败并重试时才分配值。 |

此事件类型包含 [Activity 执行](/Activity-执行) 结果。

|领域|描述 |

| Scheduled_event_id | Scheduled_event_id |此完成事件对应的 [ActivityTaskScheduled](#ActivitytaskScheduled) 事件的 ID。 |
|开始事件 ID |此任务对应的 [ActivityTaskStarted](#Activitytaskstarted) 事件的 ID。                 |
|身份|完成此任务的 [Worker](/Workers#Worker) 的身份。                                            |

### ActivityTask失败

此 [事件](/Workflow-execution/event#event) 类型指示 [Activity 任务](/tasks#Activity-task) 失败。
SDK 客户端获取了 Activity 任务，但未成功完成。
此事件类型包含 [Activity 执行](/Activity-执行) 错误。

|领域|描述 |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------- |
|失败| [Workflow](/Workflows) 故障的序列化结果。                                                      |
| Scheduled_event_id | Scheduled_event_id |此失败事件对应的 [ActivityTaskScheduled](#ActivitytaskScheduled) 事件的 ID。 |

|重试状态|提供是否应该重试任务的原因。                                    |

### ActivityTaskTimedOut

此 [Event](/Workflow-execution/event#event) 类型表示 Activity 已根据 [Temporal Server](/Temporal-service/Temporal-server) 超时，原因是以下 [Activity](/activities) 超时之一：[Schedule-to-Close超时]（/百科全书/检测-Activity-failures#Schedule-关闭超时）和[Schedule-启动超时]（/百科全书/检测-Activity-failures#Schedule-启动超时）。

|领域|描述 |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------- |
|失败| [Workflow](/Workflows) 故障的序列化结果。                                                      |
| Scheduled_event_id | Scheduled_event_id |此超时事件对应的 [ActivityTaskScheduled](#ActivitytaskScheduled) 事件的 ID。 |
|开始事件 ID |此超时对应的 [ActivityTaskStarted](#Activitytaskstarted) 事件的 ID。           |
|重试状态|提供是否应该重试任务的原因。                                    |
|超时类型 |导致此事件的超时类型，例如开始到关闭、Schedule 到关闭、Schedule 到开始。     |

您可以运行包含 Activity 执行的 Workflow，该执行的时间比您设置的开始到关闭超时更长，并使用将 MaxAttempts 设置为 1 的 RetryPolicy，以便它不会无限期地重试。
当 Activity 超时时，您将观察到“ActivityTaskTimedOut”事件包含文档中缺少的其他属性，包括导致该事件的超时类型。

### Timer开始

此 [Event](/Workflow-execution/event#event) 类型表示 Timer 已启动。
