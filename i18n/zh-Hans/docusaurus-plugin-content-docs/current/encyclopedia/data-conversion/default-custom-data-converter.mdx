## 什么是默认 Data Converter？ {#默认数据转换器}

每个 Temporal SDK 都包含并使用默认的 Data Converter。
默认 Data Converter 使用一系列有效负载转换器将对象转换为字节，并支持二进制、Protobuf 和 JSON 格式。
它按以下顺序对值进行编码：

- 空
- 字节数组
- Protobuf JSON
- JSON

在无法在运行时确定参数类型的 SDK（例如 TypeScript）中，Protobuf 不包含在默认转换器中。

例如：

- 如果值是 Protobuf 消息的实例，则使用 [proto3 JSON](https://developers.google.com/protocol-buffers/docs/proto3#json) 进行编码。
- 如果值不为 null、二进制或 Protobuf，则它会编码为 JSON。最常见的输入类型（包括字符串、整数、浮点数和布尔值）可序列化为 JSON。如果它的任何部分不能序列化为 JSON，{/*（例如，日期 - 请参阅 JSON 数据类型）*/}，则会引发错误。

默认 Data Converter 根据对象的根类型而不是嵌套类型来序列化对象。
某些 SDK 的 JSON 序列化程序无法在不实现 [自定义 Data Converter](#custom-data-converter) 的情况下处理包含 Protobuf 子对象的列表。

## 什么是定制 Data Converter？ {#自定义数据转换器}

自定义 Data Converter 使用用于 [Payload](/dataconversion#payload) 转换或编码的自定义逻辑扩展了默认 Data Converter。

您可以创建自定义 Data Converter 来更改格式（例如，使用 [MessagePack](https://msgpack.org/) 而不是 JSON）或添加压缩和加密。

有效负载编解码器通过字节到字节的转换对[有效负载](/dataconversion#payload)进行编码和解码。
要使用自定义加密或压缩逻辑，请使用“encode”函数中的加密/压缩逻辑和“decode”函数中的解密/解压缩逻辑创建自定义有效负载编解码器。
要实现自定义有效负载编解码器，您可以覆盖默认的 Data Converter，或创建定义其自己的有效负载转换器的自定义 Data Converter。

定制 Data Converter 并不适用于所有数据；例如，[Search Attributes](/search-attribute) 保持未编码状态，因此可以对其进行索引以进行搜索。

定制的Data Converter可以具有以下三个组件：

- [有效负载转换器](/有效负载转换器)
- [有效负载编解码器](/有效负载编解码器)
- [故障转换器](/故障转换器)
