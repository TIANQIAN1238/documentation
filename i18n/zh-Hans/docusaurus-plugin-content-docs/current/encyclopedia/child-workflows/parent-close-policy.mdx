## 什么是父关闭政策？ {#父关闭策略}

父级关闭策略确定如果子级 Workflow Execution 的父级更改为关闭状态（已完成、失败或超时），子级 Workflow Execution 会发生什么情况。

- [如何使用 Go SDK 设置父关闭策略](/develop/go/Workflows/child-Workflows#parent-close-policy)
- [如何使用 Java SDK 设置父关闭策略](/develop/java/Workflows/child-Workflows#parent-close-policy)
- [如何使用 PHP SDK 设置父关闭策略](/develop/php/Workflows/child-Workflows#parent-close-policy)
- [如何使用 Python SDK 设置父关闭策略](/develop/python/Workflows/child-Workflows#parent-close-policy)
- [如何使用 TypeScript SDK 设置父关闭策略](/develop/typescript/Workflows/child-Workflows#parent-close-policy)
- [如何使用 .NET SDK 设置父关闭策略](/develop/dotnet/Workflows/child-Workflows#parent-close-policy)

有三个可能的值：

- **放弃：** 子级 Workflow Execution 不受影响。
- **请求取消：** 取消请求发送到子 Workflow Execution。
- **终止**（默认）：子级 Workflow Execution 被强制终止。

[`ParentClosePolicy`](https://github.com/Temporalio/api/blob/c1f04d0856a3ba2995e92717607f83536b5a44f5/Temporal/api/enums/v1/Workflow.proto#L44) 原型定义。

每个子 Workflow Execution 可能有自己的父关闭策略。
此策略仅适用于儿童 Workflow Execution，其他情况下无效。

您可以为每个孩子设置策略，这意味着您可以选择不以每个孩子为基础传播终止/取消。
这对于异步启动 Child Workflow 非常有用（请参阅[此处的相关问题](https://community.Temporal.io/t/best-way-to-create-an-async-child-Workflow/114) 或相应的 SDK 文档）。
