You pay only for what you use.
Our pricing reflects your use of [_Actions_](#action), [_Storage_](#storage), and [_Support_](/cloud/support#support).

Your total invoice each calendar month is the combination of Temporal Cloud consumption ([Actions](#action) and [Storage](#storage)), and a [Temporal Cloud Plan](#base_plans) that includes [Support](/cloud/support#support).

### Actions {#action}

**What are Temporal Actions?**

Actions are the primary unit of consumption-based pricing for Temporal Cloud.
They track billable operations within the Temporal Cloud Service, such as starting Workflows, recording a Heartbeat or sending messages.
**Specific Billable Actions are discussed on the [Actions](/cloud/actions) page.**

[Reach out](https://pages.Temporal.io/contact-us) to our team for more information or to help size your number of Actions.

### Storage {#storage}

**How Workflow Storage works**

A Workflow's execution might exist for a few seconds, a day, month, or even forever.
The Temporal Service stores the Workflow Execution's [Event History](/Workflow-execution/event#event-history).
Under this framework, a Workflow Execution has only two states, open (Active Storage) or closed (Retained Storage).

- _Active Storage_ measures the amount of storage used by active Workflows.

- When the execution of a Workflow finishes, Temporal Cloud stores Event History for a defined [Retention Period](/Temporal-service/Temporal-server#retention-period), which is set by the user per Namespace.
  This is _Retained Storage_.
  Typical uses of Retained Storage include compliance, debugging, workload refresh, and business analytics.
  When closed Workflow Histories need to be retained for more than the 90-day maximum period on Temporal Cloud, we recommend using our [**Export**](/cloud/export) feature.

Storage costs are measured in gigabyte-hours (GBh).

### Pricing options {#pricing-options}

**How to Pay for Temporal Cloud**

After you exceed your Actions and Storage allocations in your base tier, Temporal Cloud offers two payment options: Pay-As-You-Go and Commitments.
Both models meter and bill for three primary components: [Actions](#action), [Storage](#storage), and [your Temporal Cloud Plan](/cloud/support#support).

- With Pay-As-You-Go, you are invoiced each calendar month based on your consumption.
  Pay-As-You-Go pricing automatically applies volume prices as your Actions scale.
- With Commitments, you pre-purchase your Temporal Cloud spend with Temporal Credits.
  Temporal Credits pay for your Temporal Cloud consumption, including Temporal Cloud Plan charges.

作为这一承诺的交换，您将获得额外的折扣。

- 账户操作量超过 200M 操作

此价格用于在您的承诺中指定的时间范围内对您帐户中的操作和活动存储进行计费。

承诺必须用 Temporal 积分支付。
Temporal 积分用于支付您的 Temporal Cloud 消费，包括 Temporal Cloud 计划费用。
Temporal 积分相当于 1 美元。
例如，购买 20,000 积分可获得 20,000 Temporal 积分。
最低信用购买量需要相当于您承诺的第一年。
对于多年优惠，请联系[销售](mailto:sales@Temporal.io)以获得最准确的定价。

### 承诺定价问答

**多年承诺如何发挥作用？**

我们的销售团队与您合作，根据您的预期支出匹配年度信用购买。
这使您的付款符合年度条款，而不是一笔预付费用。

**如果我在任期结束前用完基于承诺的 Temporal 积分，会发生什么情况？**

在剩余期限内，您将继续获得协商的折扣价格。
您将根据您最近日历月的支出收到另一笔信用购买的发票。
该金额乘以您任期的年度部分中剩余的月份。
如果您上个月的支出为 5,000 美元，并且年度期限已满 10 个月，您将收到 10,000 Temporal 积分的发票，以支付剩余两个月的费用。

**如果我在学期结束时还有未使用的 Temporal 积分，会发生什么情况？**

承诺可能很难估计。
Temporal Cloud 提供两种方式来滚动未使用的积分：

- 当您续订相同或更大金额的承诺时，Temporal Cloud 会将任何未使用的积分转入您的新承诺。
- 如果您需要缩小承诺规模，Temporal Cloud 将最多将初始信用购买金额的 10% 转入新承诺。

**我如何做出承诺并购买 Temporal 积分？**

请通过 [sales@Temporal.io](mailto:sales@Temporal.io) 联系我们的团队或联系您的专属客户经理。
您还可以通过 [AWS Marketplace](https://aws.amazon.com/marketplace/pp/prodview-xx2x66m6fp2lo) 购买 Temporal Cloud 承诺积分。

### 容量模式 {#capacity-modes-pricing}

**什么是容量模式？**

Temporal 提供按需和预配置容量模式。 
按需容量会根据过去的使用情况自动调整。 
预配置容量模式允许您定义 Workflow 所需的容量，并且对于处理超出标准按需限制的流量非常有用。
有关如何设置容量以及相关限制的详细信息，请参阅[容量模式](/cloud/capacity-modes)。

:::tip Support, stability, and dependency info

预配置容量当前处于[公共预览](/evaluate/development-product-features/release-stages#public-preview)。

:::

**容量模式的定价如何运作？**

累积的操作数可能会受到容量模式的影响。 
对于按需容量模式，操作会照常累积。 
对于预配置容量，每个日历小时必须为预配置的每个 [Temporal 资源单元 (TRU)](/cloud/capacity-modes) 使用最少数量的操作。
如果一个日历小时内的 Action 数量没有超过最小分配量，子类型 ns_capacity:tru 的 Action 将被记录到所需的数量。 
第一个 TRU 的最低要求是 0 个操作，因为它与 Temporal 上的默认速率限制一致。 
对于每个额外的 TRU，每小时的最低要求为 360,000 次操作。 
每小时 360,000 个操作表示所请求资源的利用率为 20%。 
这可以计算为每个 TRU 500 APS * 3600 秒（1 小时）* 20%= 360,000 个操作。

例如：

如果您的 Namespace 请求 4 个 TRU（即 2,000 APS）4 小时，并且使用情况如下：
* 第 1 小时：2,000,000 次操作
* 第 2 小时：5,000,000 次操作
* 第 3 小时：4,000,000 个操作
* 第 4 小时：500,000 次操作
那么预配置容量的成本将计算如下：

请求 4 个 TRU - 包含 1 个默认 TRU = 具有最低使用要求的 3 个 TRU
3 个 TRU * 每小时 360,000 个操作= 每小时最少 1,080,000 个操作
* 第 1 小时：2,000,000 次操作 > 最少 1,080,000 次操作 = 不产生额外操作
* 第 2 小时：5,000,000 次操作 > 最少 1,080,000 次操作 = 不产生额外操作
* 第 3 小时：4,000,000 次操作 > 最少 1,080,000 次操作 = 不产生额外操作
* 第 4 小时：500,000 个操作 < 1,080,000 个最小操作，因此 1,080,000-500,000 个操作 = 580,000 个额外操作将添加到该小时中
4 小时总计：2,000,000+5,000,000+4,000,000+1,080,000=12,080,000 次操作

如果 TRU 在一个小时内多次更改，则该小时内的最高值将用于计算所需的最少操作。

Temporal 的预配置容量定价方法符合我们的目标，即仅按您使用的量收费。 
如果您使用的请求容量超出默认值 20% 或更多，则最低每小时分配不会对您的价格产生影响，并且仅在未使用请求的容量时才会产生额外费用。 
为了避免对预配置容量收取费用，建议将容量“返回”到池中，并在使用情况稳定时切换回按需模式。



### 高可用性功能定价 {#high-availability-features}

**高可用性 (HA) 功能的定价如何运作？**

## 配置 XPROT026X 服务健康检查 {#health-checks}

高可用性功能的定价与您的工作负载量一致。
Namespace 中的操作和存储会影响您的操作和存储消耗。
要估算此部署模型的成本，请将 2 倍乘数应用于您正在复制的 Namespace 中的操作和存储，并将此扩展包含在您的帐户消耗中。
对于添加副本的现有 Namespace，2 倍乘数仅适用于副本处于活动状态后的操作和存储。
在创建副本之前或在创建副本但尚未处于“活动”状态时使用的存储和完成的操作不会产生额外费用。

:::tip Future Pricing Update

未来，对于需要大量跨云数据传输的Namespace，Temporal将收取额外费用，以覆盖云提供商收取的出口费用。
所有 Namespace 将免费获得大量网络传输，因此绝大多数 Namespace 将免费。

:::

升级现有 Namespace 时，需要考虑以下几点：

- Temporal 不会对升级到具有高可用性功能的 Namespace 之前完成的历史操作收费。
  只有正在进行的（飞行中）和新的 Workflow Execution 才会产生消耗。
- Temporal 从添加副本开始对现有（正在进行的）和新 Workflow 的所有操作进行收费。
- Temporal 从添加新副本的角度对保留（历史）、运行（持续）和新 Workflow Execution 的复制存储进行收费。

### 公平定价{#fairness-pricing}

**公平定价如何运作？**

当在 Namespace 上启用 [公平性](/develop/task-queue-priority-fairness#task-queue-fairness) 时，无论单个 Workflow 或活动是否使用公平性密钥，该 Namespace 中的每个操作在功能开启的每个小时都会额外收取“0.1”操作费用。

下面的示例使用 Namespace，“your-Namespace”，通常每小时生成 10,000 个操作。

**启用公平性，但未使用公平性密钥**

如果启用了公平性，但没有 Workflow 或活动发送“fairness_key”元数据，则所有任务将继续像以前一样进行路由。公平操作仍然适用：计费取决于 Namespace 上是否启用公平性。

|小时 | Namespace 中的操作 |启用公平性？ |公平行动|总计费操作 |
|--------------|----------------------|--------------------------------|--------------------|------------------------|
|下午 2:00–3:00 | 10,000 |是的 | +1,000 | 11,000 |

**带有和不带有公平密钥的 Workflow 的混合**

假设“your-Namespace”有两个 Task Queue：
- `支付队列`，其中 Workflow 传递 `fairness_key = "customer-id"`
-“通知队列”，其中 Workflow 没有公平键，并且行为与以前完全相同

如果每个队列在给定小时内生成 5,000 个操作（总共 10,000 个），则公平操作适用于全部 10,000 个操作，而不仅仅是“付款队列”中的 5,000 个操作。

|小时 | Actions from payment-queue |通知队列中的操作 | Namespace 总操作 |公平|总计费操作 |
|--------------|----------------------------------------|----------------------------------|------------------------------------|------------------|------------------------|
|下午 2:00–3:00 | 5,000 | 5,000 | 10,000 | +1,000 | 11,000 |

## 计费问题常见问题解答 {#pricing-faq}

**Temporal 接受哪些付款方式？**

您可以使用信用卡、ACH 或电汇付款。
要使用 AWS 账户支付 Temporal Cloud，请在 AWS Marketplace 上注册 [Temporal Cloud 即用即付](https://aws.amazon.com/marketplace/pp/prodview-xx2x66m6fp2lo)。

**我多久需要付费一次？**

Temporal Cloud 开具上个月的使用情况和费用的发票。
上个月的发票于当月 3 号开具。
例如，5 月份的发票将于 6 月 3 日午夜（世界标准时间）开具。

**我在哪里可以查看我的使用情况和账单信息？**

账户所有者和财务管理员可以随时查看其详细的账单数据。
访问 Temporal Cloud 中的[使用情况和计费仪表板](/cloud/billing-and-cost)。

**如何购买 Temporal Cloud 积分？**

您可以通过 [sales@Temporal.io](mailto:sales@Temporal.io) 联系我们的团队来购买 Temporal Cloud 积分。

**运行 Temporal Cloud 的最低成本是多少？**

Essentials 计划起价为 100 美元/月。
超出计划分配的消耗按消耗量计费。

**我可以通过 Amazon、Azure 或 Google Cloud Platform Marketplace 购买 Temporal Cloud 吗？**

通过 AWS Marketplace 购买 Temporal Cloud 有两种方法：

- 按量付费可用[此处](https://aws.amazon.com/marketplace/pp/prodview-xx2x66m6fp2lo)
- 积分：通过私人优惠提供，请联系我们的团队 [sales@Temporal.io](mailto:sales@Temporal.io)

要在 Google Cloud Marketplace 上购买 Temporal Cloud，请通过 [sales@Temporal.io](mailto:sales@Temporal.io) 联系我们的团队。

**如何查看剩余的 Temporal Cloud 积分？**

要查看剩余的 Temporal Cloud 积分，账户所有者和财务管理员可以登录 Temporal Cloud 并转至“设置”>“账单”。
您需要适当的管理权限才能访问此部分。

**如果我超出了启动计划等促销活动中的可用积分，会发生什么情况？**

对于从启动计划或促销活动中获得免费积分的客户，一旦其积分余额在该月底用完，就会向其开具发票。

**促销积分会过期吗？**

通过启动计划或优惠获得的积分有一个到期日。
该日期是注册过程的一部分。

**如何 Update 我的付款信息？**

账户所有者和财务管理员可以随时在计划选项卡下的 Temporal Cloud [账单](https://cloud.Temporal.io/billing) 页面上查看 Update 付款信息。
您需要适当的管理权限才能访问此部分。
选择“管理付款方式”按钮。

有关更多详细信息，请参阅[此概述](/cloud/billing-and-cost)。

**如果我的付款失败会怎样？**

Temporal将定期向您发送电子邮件提醒您完成付款。

**如何查看我的发票和账单历史记录？**

发票通过电子邮件发送给帐户所有者或指定的帐单联系人。
账户所有者和财务管理员可以随时查看他们的[详细账单信息](https://cloud.Temporal.io/billing)。
有关详细信息，请参阅我们的[计费和成本](/cloud/billing-and-cost) 页面。
您需要适当的管理权限才能访问此部分。
或者，要查看发票和账单历史记录，请通过 [ar@Temporal.io]（邮件至：ar@Temporal.io）联系 Temporal 财务。

**Temporal 收取销售税/增值税吗？**

我们根据要求在美国司法管辖区收取适用的销售税。

**我如何取消我的帐户？**

帐户所有者可以在计费中心的“计划”选项卡中删除其帐户并取消其订阅。
有关如何访问计费中心的详细信息，请参阅[计费和成本](/cloud/billing-and-cost)页面。

**如果我取消帐户，我会立即失去访问权限吗？**

一旦 Temporal 完成卸载过程，客户将失去对 Temporal Cloud 的访问权限。
计费独立于此过程。

**取消后我可以重新激活我的帐户吗？**

不。
当您的帐户被取消时，您的帐户数据将被删除且无法恢复。
要返回 Temporal Cloud，您必须重新注册。
我们将为您分配一个新的 Temporal 帐户，并将您视为新客户。
