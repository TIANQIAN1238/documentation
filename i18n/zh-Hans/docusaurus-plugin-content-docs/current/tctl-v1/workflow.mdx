＃＃ 取消

`tctl Workflow cancel --Query` 命令取消 [Workflow Execution](/Workflow-执行)。

取消正在运行的 Workflow Execution 会在历史记录中记录“WorkflowExecutionCancelRequested”事件。
新的 [Workflow Task](/tasks#Workflow-task) 将是 Scheduled。
取消后，Workflow Execution可以执行清理工作。

另请参见 [`tctl Workflow 终止 --Query`](#terminate)。

`tctl Workflow 取消 --Query <Query> <修饰符>`

以下修饰符控制命令的行为。

### --Workflow_id

指定 [Workflow Id](/Workflow-execution/Workflowid-runid#Workflow-id)。

别名：`-w`

**例子**

```bash
tctl workflow cancel --workflow_id <id>
```

### --run_id

指定 [运行 ID](/Workflow-execution/Workflowid-runid#run-id)。

别名：`-r`

**例子**

```bash
tctl workflow cancel --run_id <id>
```

＃＃ 数数

`tctl Workflow count` 命令对 [Workflow Executions](/Workflow-执行) 进行计数。
此命令需要启用 Elasticsearch。

`tctl Workflow 计数 <修饰符>`

以下修饰符控制命令的行为。

### --Query

_所需修饰符_

指定 [Search Attributes](/search-attribute) 的类似 SQL 的 Query。

别名：`-q`

**例子**

要对所有打开的 [Workflow Executions](/Workflow-execution) 进行计数：

```bash
tctl workflow count --query 'ExecutionStatus="Running"'
```

### --最早时间

指定列出的最早开始时间。
支持的格式如下：

- `<年>-<月>-<日>T<小时>:<分钟>:<秒><+|-><偏移小时>:<偏移分钟>`
- 原始 Unix 纪元时间（自 1970 年 1 月 1 日 0000 UTC 以来的毫秒数）。
- `<n><duration`，其中 `<n>` 是 0 到 1000000 之间的值，而 `<duration>` 是以下值之一：
  - “第二个”或“s”
  - ‘分钟’或‘米’
  - `hour` 或 `h`
  - `day` 或 `d`
  - `week` 或 `w`
  - “月份”或“M”
  - “年”或“y”

**示例**

要指定 2022 年 1 月 2 日印度标准时间下午 3:04:05：

```bash
tctl workflow list --earliest-time '2022-01-02T15:04:05+05:30'
```

要指定当前时间之前 15 分钟：

```bash
tctl workflow list --earliest-time '15minute'
```

### --Query

**如何使用 tctl 通过 [列表过滤器](/list-filter) 列出和过滤 Workflow Execution。**

仅当使用集群配置了 [高级 Visibility](/Visibility#advanced-Visibility) 时，才支持“--Query”标志。

使用“--Query”选项会导致 tctl 忽略所有其他过滤器选项，包括“open”、“earliest_time”、“latest_time”、“Workflow_id”和“Workflow_type”。

别名：`-q`

**例子**

```bashbash
tctl workflow list --query "WorkflowId=<your-workflow-id>"
```

更多示例：

```bashbash
tctl workflow list \
  --query "WorkflowType='main.SampleParentWorkflow' AND ExecutionStatus='Running'"
```

```bashbash
tctl workflow list \
  --query '(CustomKeywordField = "keyword1" and CustomIntField >= 5) or CustomKeywordField = "keyword2"' \
  --print_search_attr
```

```bashbash
tctl workflow list \
  --query 'CustomKeywordField in ("keyword2", "keyword1") and CustomIntField >= 5 and CloseTime between "2018-06-07T16:16:36-08:00" and "2019-06-07T16:46:34-08:00" order by CustomDatetimeField desc' \
  --print_search_attr
```

```bashbash
tctl workflow list \
  --query 'WorkflowType = "main.Workflow" and (WorkflowId = "1645a588-4772-4dab-b276-5f9db108b3a8" or RunId = "be66519b-5f09-40cd-b2e8-20e4106244dc")'
```

```bashbash
tctl workflow list \
  --query 'WorkflowType = "main.Workflow" StartTime > "2019-06-07T16:46:34-08:00" and ExecutionStatus = "Running"'
```

列出不止一页。


## Query

别名：`q`

`tctl Workflow Query` 命令将 [Query](/sending-messages#sending-queries) 发送到 [Workflow Execution](/Workflow-execution)。

查询可用于检索具有给定参数的全部或部分 Workflow 状态。

```bash
$ tctl workflow query --workflow_id "HelloQuery" --query_type "getCount"
Query result as JSON:
3
```

查询也可用于已完成的 Workflow。
让我们通过更新其问候语来完成 Workflow，然后更新 Query 现在完成的 Workflow。

```bash
$ tctl workflow signal --workflow_id "HelloQuery" --name "updateGreeting" --input \"Bye\"
Signal workflow succeeded.
$ tctl workflow query --workflow_id "HelloQuery" --query_type "getCount"
Query result as JSON:
4
```

查询写法如下：

`tctl Workflow Query --Workflow_id [修饰符]`

以下修饰符控制命令的行为。
执行此命令时始终包含所需的修饰符。

### --Workflow_id

指定 [Workflow Id](/Workflow-execution/Workflowid-runid#Workflow-id)。 **此修饰符是必需的。**

别名：`-w`

**例子**

```bash
tctl workflow query --workflow_id <id>
```

### --run_id

指定 [运行 ID](/Workflow-execution/Workflowid-runid#run-id)。

别名：`-r`

**例子**

```bash
tctl workflow query --run_id <id>
```

### --Query_type

指定要运行的 Query 的类型。

**例子**

```bash

```

＃＃＃  - 输入

## Signal

`tctl Workflow Signal` 命令 [Signals](/sending-messages#sending-Signals) 和 [Workflow Execution](/Workflow-execution)。

Workflow 通过 Signal 名称侦听 Signal，并且可以设置侦听一个或多个 Signal 名称。
Workflow 还可以侦听 SQL 查询。

下面的 Workflow 侦听“HelloSignal”的实例：

```bash
tctl workflow start  --workflow_id "HelloSignal" --taskqueue HelloWorldTaskQueue --workflow_type HelloWorld --execution_timeout 3600 --input \"World\"
```

Worker 在收到 Signal 后将返回此输出：

```text
13:57:44.258 [workflow-method] INFO  c.t.s.javaquickstart.GettingStarted - 1: Hello World!
```

Signals 还可用于更改变量值。

```bash
tctl workflow signal --workflow_id "HelloSignal" --name "updateGreeting" --input \"Hi\"
```

输出将与收到的第一个 Signal 发生变化。

```text
13:57:44.258 [workflow-method] INFO  c.t.s.javaquickstart.GettingStarted - 1: Hello World!
13:58:22.352 [workflow-method] INFO  c.t.s.javaquickstart.GettingStarted - 2: Hi World!
```

当发送 Signal 时，会产生一个等待条件来阻止任何包含相同输入值的 Signal。
但是，在我们的示例中更改问候语可以解除阻止：

```bash
tctl workflow signal --workflow_id "HelloSignal" --name "updateGreeting" --input \"Welcome\"
```

Worker 输出：

```text
13:57:44.258 [workflow-method] INFO  c.t.s.javaquickstart.GettingStarted - 1: Hello World!
13:58:22.352 [workflow-method] INFO  c.t.s.javaquickstart.GettingStarted - 2: Hi World!
13:59:29.097 [workflow-method] INFO  c.t.s.javaquickstart.GettingStarted - 3: Welcome World!
```

发送 Signal 不需要运行 Worker。

```bash
tctl workflow signal --workflow_id "HelloSignal" --name "updateGreeting" --input \"Welcome\"
```

CLI 输出：

```text
Signal workflow succeeded.
```

Signal 请求在 Temporal Server 内部排队，直到 Worker 重新启动。
如果给定的 Signal 包含与之前相同的输入，则排队的 Signal 将被忽略。

通过发送带有“再见”问候语的 Signal 来完成 Workflow：

```bash
tctl workflow signal --workflow_id "HelloSignal" --name "updateGreeting" --input \"Bye\"
```

检查Workflow Execution是否已完成。

```bash
tctl workflow showid HelloSignal
```

Signal的写法如下：

```bash
tctl workflow signal --workflow_id <id> <modifiers>
```

或者

```bash
tctl workflow signal --query <query> <modifiers>
```

传递 [Signal](/sending-messages#sending-Signals) 的输入。
输入必须为 JSON 格式。

别名：`-i`

**例子**

```bash
tctl workflow signal --query <query> --input <json>
```

### --输入文件

从 JSON 文件传递​​ [Signal](/sending-messages#sending-Signals) 的输入。

**例子**

```bash
tctl workflow signal --query <query> --input_file <filename>
```

＃＃ 堆

`tctl Workflow stack` 命令使用 `__stack_trace` 作为 Query 类型查询 [Workflow Execution](/Workflow-execution)。

此命令可用于定位 [Workflow Definition](/Workflow-definition) 中的错误和块。

`tctl Workflow 堆栈 <修饰符>`

以下修饰符控制命令的行为。

### --Workflow_id

**这是必需的修饰符。**

指定 [Workflow Id](/Workflow-execution/Workflowid-runid#Workflow-id)。

别名：`-w`

**例子**

```bash
tctl workflow stack --workflow_id <id>
```

### --run_id

Specify a [Run Id](/Workflow-execution/Workflowid-runid#run-id).

Alias: `-r`

**Example**

```bash
tctl workflow stack --run_id <id>
```

### --input

Pass input for the Query.
Input must be in JSON format.
For multiple JSON objects, concatenate them and use spaces as separators.

Alias: `-i`

**Example**

```bash
tctl workflow stack --input <json>
```

### --input_file

从 JSON 文件传递​​ Query 的输入。
对于多个 JSON 对象，将它们连接起来并使用空格或换行符作为分隔符。
来自命令行的输入将覆盖来自文件的输入。

```bash
tctl workflow stack --input_file <filename>
```

### --Query_reject_condition

＃＃ 开始

`tctl Workflow start` 命令启动一个新的 [Workflow Execution](/Workflow-execution)。
与“run”不同，此命令在启动 Workflow 后立即返回 Workflow Id 和运行 ID。

`tctl Workflow 开始 <修饰符>`

以下修饰符控制命令的行为。
执行此命令时始终包含所需的修饰符。

### --任务队列

指定 [Task Queue](/任务队列)。

别名：`--t`

**例子**

```bash
tctl workflow start --taskqueue <name>
```

### --Workflow_id

**这是必需的修饰符。**

指定 [Workflow Id](/Workflow-execution/Workflowid-runid#Workflow-id)。

别名：`-w`

**例子**

```bash
tctl workflow start --workflow_id <id>
```

如果启动 Workflow 时未提供 Id，则客户端会生成 UUID 形式的 Id。
Temporal 建议使用业务 ID，而不是客户端生成的 UUID。

**例子**

```bash
tctl workflow start  --workflow_id "HelloTemporal1" --taskqueue HelloWorldTaskQueue --workflow_type HelloWorld --execution_timeout 3600 --input \"Temporal\"
```

### --Workflow_type

指定 [Workflow Type](/Workflow-definition#Workflow-type) 的名称。

**例子**

```bash
tctl workflow start --workflow_type <name>
```

### --执行超时

指定 [Workflow Execution](/Workflow-execution) 的 [Start-To-Close Timeout](/encyclopedia/Detecting-Activity-failures#start-to-close-timeout)（以秒为单位）。
默认值为 0。

**例子**

```bash
tctl workflow start --execution_timeout <seconds>
```

### --Workflow_task_timeout

指定 [Workflow Task](/tasks#Workflow-task) 的 [Start-To-Close Timeout](/encyclopedia/Detecting-Activity-failures#start-to-close-timeout)（以秒为单位）。
默认值为 10。

**例子**

```bash
tctl workflow start --workflow_task_timeout <seconds>
```

### --cron

指定 [Cron Schedule](/cron-job#cron-Schedules)。

**例子**

```bash
tctl workflow start --cron <string>
```

### --Workflowid重用策略

指定 [Workflow Id 重用策略](/Workflow-execution/Workflowid-runid#Workflow-id-reuse-policy)。
配置是否允许在新的 [Workflow Execution](/Workflow-execution) 中使用相同的 [Workflow Id](/Workflow-execution/Workflowid-runid#Workflow-id)。

共有三个允许值：

- [AllowDuplicateFailedOnly](/Workflow-execution/Workflowid-runid#Workflow-id-reuse-policy)
- [允许重复](/Workflow-执行/Workflowid-runid#Workflow-id-重用-策略)
- [拒绝重复](/Workflow-执行/Workflowid-runid#Workflow-id-重用-策略)

**示例**

```bash
tctl workflow start --workflowidreusepolicy AllowDuplicate
tctl workflow start --workflowidreusepolicy AllowDuplicateFailedOnly
tctl workflow start --workflowidreusepolicy RejectDuplicate
```

:::note

多个具有相同 ID 的 Workflow 不能同时运行

:::

＃＃＃  - 输入

传递 Workflow 的输入。
输入必须为 JSON 格式。
对于多个 JSON 对象，请在单独的“--input”选项中传递每个对象。
使用“null”表示空值。

别名：`-i`

**例子**

```bash
tctl workflow start --input <json>
```

### --输入文件

从 JSON 文件传递​​ Workflow 的输入。
对于多个 JSON 对象，将它们连接起来并使用空格或换行符作为分隔符。
来自命令行的输入将覆盖来自文件的输入。

**例子**

```bash
tctl workflow start --input_file <filename>
```

### --memo_key

传递备忘录的密钥。
对于多个键，将它们连接起来并使用空格作为分隔符。

**例子**

```bash
tctl workflow start --memo_key <key>
```

### --备忘录

从 JSON 文件传递​​ [memo](/Workflow-execution#memo) 的信息。

备忘录是不可变的键/值对，可以在启动 Workflow 时附加到 Workflow 运行。
列出 Workflow 时，备注可见。

对于多个备忘录，请将它们连接起来并使用空格作为分隔符。
该顺序必须与“--memo_key”中键的顺序匹配。

**例子**

```bash
tctl workflow start \
  -tq your-task-queue \
  -wt your-workflow \
  -et 60 \
  -i '"temporal"' \
  -memo_key '<key values>' \
  -memo '<value>'
```

### --memo_file

从 JSON 文件传递​​备忘录信息。
对于多个 JSON 对象，将它们连接起来并使用空格或换行符作为分隔符。
该顺序必须与“--memo_key”中键的顺序匹配。

**例子**

```bash
tctl workflow start --memo_file <filename>
```

### --search_attr_key

指定 [Search Attribute](/search-attribute) 名称。
对于多个名称，将它们连接起来并使用竖线（`|`）作为分隔符。

要列出有效的 Search Attribute，请使用“tctl cluster get-search-attributes”命令。

**例子**

```bash
tctl workflow start --search_attr_key <key>
```

### --搜索属性值

指定 [Search Attribute](/search-attribute) 值。
对于多个值，将它们连接起来并使用管道（`|`）作为分隔符。
如果值是数组，请使用 JSON 格式，例如 `["a","b"]`、`[1,2]`、`["true","false"]` 或 `["2022-06-07T17:16:34-08:00","2022-06-07T18:16:34-08:00"]`。

要列出有效的 Search Attribute 和值类型，请使用“tctl cluster get-search-attributes”命令。

**例子**

```bash
tctl workflow start --search_attr_value <value>
```

## 终止

“tctl Workflow 终止”命令终止 [Workflow Execution](/Workflow-执行)。

终止正在运行的 Workflow Execution 将“WorkflowExecutionTermulated”事件记录为历史记录中的关闭事件。
Scheduled 不再是 [Workflow Task](/tasks#Workflow-task)。

另请参阅[`tctl Workflow 取消`](#cancel)。

`tctl Workflow 终止 --Query <修饰符>`

以下修饰符控制命令的行为。

### --Workflow_id

_所需修饰符_

指定 [Workflow Id](/Workflow-execution/Workflowid-runid#Workflow-id)。

别名：`-w`

**例子**

```bash
tctl workflow terminate --workflow_id <id>
```

### --run_id

指定 [运行 ID](/Workflow-execution/Workflowid-runid#run-id)。

如果未指定“run_id”，“tctl”将终止指定“Workflow_id”的最后一个 Workflow Execution。

别名：`-r`

**例子**

```bash
tctl workflow terminate --run_id <id>
```

＃＃＃  - 原因

指定终止 [Workflow Execution](/Workflow-执行) 的原因。

**例子**

```bash
tctl workflow terminate --workflow_id --reason <string>
```
