## 添加区域

使用 `tcld Namespace add-region` 将 <ToolTipTerm term="replica" /> 区域添加到现有 Temporal Cloud [Namespace](/Namespaces)，将其升级以支持 [高可用性](/cloud/high-availability)。

请参阅[区域](/cloud/regions) 了解可用区域及其支持的复制选项。

以下修饰符控制命令的行为。

#### --请求 ID

用于异步操作的请求标识符。如果未设置，服务器会分配一个标识符。

别名：`-r`

#### --Namespace

**必需。** 指定 Temporal Cloud 上托管的 Namespace。如果未指定，则使用环境变量“$Temporal_CLOUD_Namespace”的值。

别名：`-n`

＃＃＃＃  - 地区

**必需。** 要添加到现有 Namespace 的区域。有关受支持区域的列表，请参阅[区域](/cloud/regions)。

:::tip Choosing Replica Regions

[Grafana 仪表板示例](https://github.com/grafana/jsonnet-libs/blob/master/XPROT026X-mixin/dashboards/XPROT026X-overview.json) 包含“使用情况和配额”部分
分别为这些限制和计数指标创建演示图表。

:::

别名：`--re`

**例子**

```bash
tcld namespace add-region \
    --namespace <namespace_id> \
    --region <replica_region_name>
```

指定要在其中创建副本的区域的区域名称（例如“us-east-1”）作为“--region”标志的参数。
有关同区域、多区域和多云复制选项的详细信息，请参阅[高可用性](/cloud/high-availability)。

一旦您的 Namespace 可供使用，Temporal Cloud 就会发送电子邮件警报。

#### --云提供商

该地区的云提供商。 [`aws`、`gcp`] 之一。

默认值：`aws`

＃＃ 创造

`tcld Namespace create` 命令在 Temporal Cloud 中创建 Temporal [Namespace](/Namespaces)。

别名：`c`

以下修饰符控制命令的行为。

#### --Namespace

**必填。** 新 Namespace 的名称。这将成为 Namespace ID 的一部分 (`<Namespace_name>.<account_suffix>`)。

别名：`-n`

＃＃＃＃  - 地区

**必需。** 用于创建 Namespace 的云提供商区域。为标准 Namespace 提供一个“--region”，或为具有 [高可用性](/cloud/high-availability) 的 Namespace 提供两个。

请参阅[区域](/cloud/regions) 了解可用区域及其支持的复制选项。

别名：`--re`

#### --auth-方法

Namespace 的身份验证方法。 [`mTLS`、`api_key`] 之一。

- `mTLS`（默认）：需要 `--ca-certificate` 或 `--ca-certificate-file`
- `api_key`：没有其他修饰符

**例子**

```bash
tcld namespace create \
    --namespace test-namespace.a1b2c \
    --region us-east-1 \
    --auth-method api_key
```

#### --ca 证书

Base64 编码的 [CA 证书](/cloud/certificates)。如果同时指定了“--ca-certificate”和“--ca-certificate-file”，则仅使用“--ca-certificate”。

别名：`-c`

#### --ca 证书文件

[CA 证书](/cloud/certificates) PEM 文件的路径。如果指定了两个选项，则仅使用“--ca-certificate”。

别名：`--cf`

#### --证书过滤器文件

定义要应用于 Namespace 的[证书过滤器](/cloud/certificates#manage-certificate-filters) 的 JSON 文件的路径。

示例 JSON: `{ "filters": [ { "commonName": "test1" } ] }`

如果同时指定了“--certificate-filter-file”和“--certificate-filter-input”，则该命令将返回错误。

别名：`--cff`

#### --证书过滤器输入

定义要应用于 Namespace 的[证书过滤器](/cloud/certificates#manage-certificate-filters) 的 JSON 字符串。

示例 JSON: `{ "filters": [ { "commonName": "test1" } ] }`

如果同时指定了“--certificate-filter-input”和“--certificate-filter-file”，该命令将返回错误。

别名：`--cfi`

#### --云提供商

该地区的云提供商。 [`aws`、`gcp`] 之一。

默认值：`aws`

别名：`--cp`

#### --连接规则 ID

应用到 Namespace 的 [连接规则](/cloud/connectivity#connectivity-rules) ID 列表。可以指定多次。

别名：`--ids`

**例子**

```bash
tcld namespace create \
    --namespace test-namespace.a1b2c \
    --region us-east-1 \
    --auth-method api_key \
    --connectivity-rule-ids <rule_id1> \
    --connectivity-rule-ids <rule_id2>
```

#### --启用删除保护

在 Namespace 上启用[删除保护](/cloud/Namespaces#delete-protection)。

默认值：“假”

别名：`--edp`

#### --端点

[Codec Server](/生产部署/数据加密)端点用于解码与此 Namespace 交互的所有用户的有效负载。必须是 HTTPS。

别名：`-e`

#### --包含凭证

调用 [Codec Server](/生产部署/数据加密) 时包含跨域凭据。

默认值：“假”

别名：`--ic`

#### --pass-access-token

将用户访问令牌传递到 [Codec Server](/生产部署/数据加密)端点。

默认值：“假”

别名：`--pat`

- 您的身份验证凭据。
  - 如果您使用 API 密钥，请提供 API 密钥值。
  - 如果您使用 XPROT043X，请提供 XPROT043X CA 证书和 XPROT043X 私钥。
- 您的 _XPROT025X 和帐户 ID_ 组合，遵循格式“<XPROT025X_id>.<account_id>”。
- 建议的端点是 XPROT042X XPROT025X 端点：`<XPROT025X>.<account>.tmprl.cloud:7233`。
  该端点适用于所有 XPROT025X，并自动将流量定向到具有 [高可用性](/cloud/high-availability) 的 XPROT025X 的活动区域。
  有关端点选项的更多信息，请参阅[访问 XPROT025Xs](/cloud/XPROT025Xs#access-XPROT025Xs)。

用于异步操作的请求标识符。如果未设置，服务器会分配一个标识符。

别名：`-r`

#### --保留天数

关闭的 Workflow Execution 的[保留期](/Temporal-service/Temporal-server#retention-period)（以天为单位）。

默认值：`30`

别名：`--rd`

#### --搜索属性

格式为“_name_=_type_”的自定义 [Search Attribute](/search-attribute)。可以指定多次。

_type_ 的有效值：`Bool` | `日期时间` | “双”| `Int` | `关键字` | `文本`

别名：`--sa`

**例子**

```bash
tcld namespace create \
    --namespace test-namespace.a1b2c \
    --region us-east-1 \
    --auth-method api_key \
    --search-attribute "customer_id=Int" \
    --search-attribute "customer_name=Text"
```

＃＃＃＃  - 标签

格式为“_key_=_value_”的[标签](/cloud/Namespaces#tag-a-Namespace)。可以指定多次。

请参阅[标签结构和限制](/cloud/Namespaces#tag-struct-and-limits)。

别名：`--t`

**例子**

```bash
tcld namespace create \
    --namespace test-namespace.a1b2c \
    --region us-east-1 \
    --auth-method api_key \
    --tag "key=value" \
    --tag "key2=value2"
```

#### --用户-Namespace-权限

用户的 [Namespace 级别权限](/cloud/manage-access/roles-and-permissions#Namespace-level-permissions)，格式为“_email_=_permission_”。可以指定多次。

_permission_ 的有效值：`Admin` | `写` | `阅读`

别名：`-p`

**例子**

```bash
tcld namespace create \
    --namespace test-namespace.a1b2c \
    --region us-east-1 \
    --auth-method api_key \
    --user-namespace-permission "user@example.com=Admin" \
    --user-namespace-permission "user2@example.com=Write"
```

## 删除区域

使用 `tcld Namespace delete-region` 删除现有 Temporal Cloud 的 <ToolTipTerm term="replica" />
[Namespace](/Namespaces)。删除副本会禁用[高可用性功能](/cloud/high-availability) 和结果
您必须经过 7 天的等待期，然后才能在同一位置重新启用高可用性功能。参考
[启用高可用性](/cloud/high-availability/enable) 了解更多信息。

以下修饰符控制命令的行为。

#### --请求 ID

用于异步操作的请求标识符。如果未设置，服务器会分配一个标识符。

别名：`-r`

#### --Namespace

**必需。** 指定 Temporal Cloud 上托管的 Namespace。如果未指定，则使用环境变量“$Temporal_CLOUD_Namespace”的值。

别名：`-n`

＃＃＃＃  - 地区

**必需。** 要从 Namespace 中删除的区域。指定区域名称，例如“us-east-1”。移除后，Temporal 停止复制，Namespace 成为标准 Namespace。删除 Namespace 区域后 7 天内无法重新添加区域或添加新区域。

别名：`--re`

**例子**

```bash
tcld namespace delete-region \
    --namespace <namespace_id> \
    --region <region_name>
```

使用 API 密钥身份验证时，请在按 Enter 之前添加您的 API 凭据：

```bash
tcld --api-key <your_api_key> \
    delete-region \
    --namespace <namespace_id> \
    --region <region_name>
```

#### --云提供商

要故障转移到的区域的云提供商。 [aws、gcp] 之一。

默认值：aws（默认值：“aws”）

## 故障转移

使用[高可用性功能](/cloud/high-availability) 对 Temporal Namespace 进行故障转移。故障转移切换
从主 Namespace 到其副本的 Namespace 区域。

**例子**

```bash
tcld namespace failover \
    --namespace <namespace_id> \
    --region <target_region>
```

使用 API 密钥身份验证时，请在按 Enter 之前添加您的 API 凭据：

```bash
tcld --api-key <your_api_key> \
    namespace failover \
    --namespace <namespace_id> \
    --region <target_region>
```

#### --请求 ID

指定用于异步操作的请求标识符。如果没有指定，服务器会分配一个请求
标识符。

别名：`-r`

#### --Namespace

**必需。** 指定 Temporal Cloud 上托管的 Namespace。如果未指定，则使用环境变量“$Temporal_CLOUD_Namespace”的值。

别名：`-n`

＃＃＃＃  - 地区

**必需。** 故障转移_to_的区域。指定区域名称，例如“us-east-1”。

有关受支持区域的列表，请参阅[区域](/cloud/regions)。

别名：`--re`

**例子**

```bash
tcld namespace failover \
    --namespace <namespace_id> \
    --region <region_name>
```

#### --ca 证书

_必需的修饰符，除非指定了“--ca-certificate-file”_。

A base64-encoded CA certificate.

If both `--ca-certificate` and `--ca-certificate-file` are specified, only `--ca-certificate` is used.

Alias: `-c`

#### --cloud-provider

The cloud provider of the region to failover to. One of [aws, gcp].

Default: aws (default: "aws")

Provide a name for the export sink.

_Required modifier_

_Required modifier_

## accepted-client-ca

The `tcld Namespace accepted-client-ca` commands manage the client CA certificates of the specified
[Namespace](/Namespaces) in Temporal Cloud. The certificates are used to verify client connections.

:::note

Base64 versions of the CA certificate files are accepted by these commands.

:::

Alias: `ca`

- [tcld Namespace accepted-client-ca add](#add)
- [tcld Namespace accepted-client-ca list](#list)
- [tcld Namespace accepted-client-ca set](#set)
- [tcld Namespace accepted-client-ca remove](#remove)

:::important

Do not use a CA certificate that is signed with an insecure signature algorithm, such as SHA-1. Such signatures will be
rejected. Existing CA certificates that use SHA-1 can stop working without warning.

For more information about the vulnerabilities of SHA-1, see [SHAttered](https://shattered.io/).

:::

### add

The `tcld Namespace accepted-client-ca add` command adds client CA certificates to a [Namespace](/Namespaces) in
Temporal Cloud.

`tcld Namespace accepted-client-ca add --ca-certificate <value>`

Alias: `a`

The following modifiers control the behavior of the command.

#### --Namespace

Specify a Namespace hosted on Temporal Cloud. If not specified, the value of the environment variable
$Temporal_CLOUD_Namespace is used.

Alias: `-n`

**Example**

```bash
tcld namespace accepted-client-ca add \
    --namespace <namespace_id> \
    --ca-certificate <encoded_certificate>
```

#### --request-id

Specify a request identifier to use for the asynchronous operation. If not specified, the server assigns a request
identifier.

Alias: `-r`

**Example**

```bash
tcld namespace accepted-client-ca add \
    --request-id <request_id> \
    --ca-certificate <encoded_certificate>
```

#### --resource-version

Specify a resource version (ETag) to Update from. If not specified, the latest version is used.

Alias: `-v`

**Example**

```bash
tcld namespace accepted-client-ca add \
    --resource-version <etag> \
    --ca-certificate <encoded_certificate>
```

#### --ca-certificate

_Required modifier unless `--ca-certificate-file` is specified_

Specify a base64-encoded string of a CA certificate PEM file.

If both `--ca-certificate` and `--ca-certificate-file` are specified, only `--ca-certificate` is used.

Alias: `-c`

**Example**

```bash
tcld namespace accepted-client-ca add \
    --ca-certificate <encoded_certificate>
```

#### --ca-certificate-file

_Required modifier unless `--ca-certificate` is specified_

Specify a path to a CA certificate PEM file.

If both `--ca-certificate` and `--ca-certificate-file` are specified, only `--ca-certificate` is used.

Alias: `-f`

**Example**

```bash
tcld namespace accepted-client-ca add \
    --ca-certificate-file <path>
```

### list

The `tcld Namespace accepted-client-ca list` command lists the client CA certificates that are currently configured for
a [Namespace](/Namespaces) in Temporal Cloud.

The following modifiers control the behavior of the command.

#### --Namespace


#### --resource-version

Specify a resource version (ETag) to Update from. If not specified, the latest version is used.

Alias: `-v`

**Example**

```bash
tcld namespace accepted-client-ca remove \
    --resource-version <etag> \
    --ca-certificate <encoded_certificate>
```

#### --ca-certificate

_Required modifier unless `--ca-certificate-fingerprint` or `--ca-certificate-file` is specified_

Specify the base64-encoded string of a CA certificate PEM file.

If `--ca-certificate-fingerprint` is also specified, both `--ca-certificate` and `--ca-certificate-file` are ignored.

If `--ca-certificate-file` is also specified but `--ca-certificate-fingerprint` is not, only `--ca-certificate` is used.

Alias: `-c`

**Example**

```bash
tcld namespace accepted-client-ca remove \
    --ca-certificate <encoded_certificate>
```

#### --ca-certificate-file

_Required modifier unless `--ca-certificate-fingerprint` or `--ca-certificate` is specified_

Specify a path to a CA certificate PEM file.

If `--ca-certificate-fingerprint` is also specified, both `--ca-certificate-file` and `--ca-certificate` are ignored.

If `--ca-certificate` is also specified but `--ca-certificate-fingerprint` is not, only `--ca-certificate` is used.

Alias: `-f`

**Example**

```bash
tcld namespace accepted-client-ca remove \
    --ca-certificate-file <path>
```

#### --ca-certificate-fingerprint

_Required modifier unless `--ca-certificate` or `--ca-certificate-file` is specified_

Specify the fingerprint of a CA certificate.

If `--ca-certificate`, `--ca-certificate-file`, or both are also specified, they are ignored.

Alias: `--fp`

**Example**

```bash
tcld namespace accepted-client-ca remove \
    --ca-certificate-fingerprint <fingerprint>
```

### set

The `tcld Namespace accepted-client-ca set` command sets the client CA certificates for a [Namespace](/Namespaces) in
Temporal Cloud.

`tcld Namespace 接受客户端 ca 设置 --ca-证书 <值>`

别名：`s`

{/* 如何使用 tcld 在 Temporal Cloud 中滚动接受的客户端 CA 证书 */}

更新 CA 证书时，遵循滚动更新过程非常重要。这样做可以使您的 Namespace 能够服务
两个 CA 证书都会保留一段时间，直到旧 CA 证书的流量停止。

1. 创建一个包含新旧 CA 证书 PEM 块的单个文件。只需连接 PEM 块即可
   在相邻的线上。

   ```
   -----BEGIN CERTIFICATE-----
   ... old CA cert ...
   -----END CERTIFICATE-----
   -----BEGIN CERTIFICATE-----
   ... new CA cert ...
   -----END CERTIFICATE-----
   ```

1. 使用 CA 证书捆绑文件运行“tcld Namespace Accepted-client-ca set”命令。

   ```bash
   tcld namespace accepted-client-ca set \
       --ca-certificate-file <path>
   ```

1. 监控旧证书的流量直至其停止。

1. 创建另一个仅包含新 CA 证书的文件。

1. 使用 Updated CA 证书捆绑文件再次运行“tcld Namespace Accepted-client-ca set”命令。

## 标签

“tcld Namespace 标签”命令管理指定的 [标签](/cloud/Namespaces#tag-a-Namespace)
Temporal Cloud 中的 [Namespace](/Namespaces)。

别名：`t`

- [tcld Namespace 标签更新插入](#upsert)
- [tcld Namespace 标签删除](#remove)

### 更新插入

为指定的 Namespace 添加新标签或 Update 现有标签值。

别名：`u`

以下修饰符控制命令的行为。

#### --Namespace

_所需修饰符_

指定 Temporal Cloud 上托管的 Namespace。如果没有指定，则为环境变量的值
使用 $Temporal_CLOUD_Namespace。

别名：`-n`

#### --请求 ID

用于异步操作的请求标识符。如果未设置，服务器会分配一个标识符。

别名：`-r`

＃＃＃＃  - 标签

_必需的修饰符；可以指定多次_

格式为“_key_=_value_”的标签。

[标签结构和限制](/cloud/Namespaces#tag-struct-and-limits)。

别名：`--t`

**例子**

```bash
tcld namespace tags upsert \
    --namespace <namespace_id> \
    --tag "key1=value1" \
    --tag "key2=updated"
```

＃＃＃ 消除

使用密钥删除指定 Namespace 的现有标签。

别名：`rm`

以下修饰符控制命令的行为。

#### --Namespace

_所需修饰符_

指定 Temporal Cloud 上托管的 Namespace。如果没有指定，则为环境变量的值
使用 $Temporal_CLOUD_Namespace。

别名：`-n`

#### --请求 ID

用于异步操作的请求标识符。如果未设置，服务器会分配一个标识符。

别名：`-r`

#### --标签键

_必需的修饰符；可以指定多次_

标签键字符串。

[标签密钥结构和限制](/cloud/Namespaces#tag-struct-and-limits)。

别名：`--tk`

**例子**

```bash
tcld namespace tags remove \
    --namespace <namespace_id> \
    --tag-key "key1" \
    --tag-key "key2"
```
