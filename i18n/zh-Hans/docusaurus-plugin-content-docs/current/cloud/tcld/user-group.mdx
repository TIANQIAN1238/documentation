## 删除用户

删除作为组成员的一名或多名用户。

别名：`ru`

以下标志控制命令的行为。

#### --组 ID (-id)

指定要从中删除用户的组的 ID。

#### --用户电子邮件 (-e)

要从组中删除的用户的电子邮件地址。可以多次指定此标志，以便使用一个命令删除多个用户。

## 设置访问

此命令设置组的访问角色。它遵循与 [create](#create) 命令相同的约定，指定一个可选帐户角色和 0 个或多个 Namespace 角色。

别名：`萨`

#### --组 ID (-id)

指定要设置访问权限的组 ID。

#### --账户角色

该组应具有的帐户角色。 “admin”、“read”、“developer”、“owner”、“financeadmin”、“none”之一。

#### --Namespace-角色 (-nr)

指定该组应具有的 Namespace 角色。可以重复多次添加
组中的多个 Namespace 角色。值的形式为“<Namespaceid>-<role>”，其中 Namespace ID 是 Namespace 的完整 ID，角色是“admin”、“read”或“write”之一。示例：“myNamespace.abc123-read”为“myNamespace.abc123”Namespace 添加读取角色。

#### --追加 (-a)

将追加 Namespace 角色，而不是替换已分配的所有现有角色。这允许在不知道已分配给组的角色的情况下添加 Namespace 角色。

#### --删除 (-r)

将删除给定的 Namespace 角色，而不是替换已分配的所有现有角色。这允许在不知道已分配给组的角色的情况下删除 Namespace 角色。
