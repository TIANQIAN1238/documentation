通过安全断言标记与 Temporal Cloud 与您组织的身份提供商 (IdP) 集成
语言 (SAML) 2.0 允许您使用组织的 IdP 对 Temporal Cloud 帐户的用户进行身份验证。这个
允许您强制执行企业身份策略，例如多重身份验证 (MFA)、密码复杂性、
并降低凭证被盗的风险。

SAML 包含在商业、企业和关键任务计划中。欲了解更多详情，请参阅
[Temporal Cloud 定价](/cloud/pricing#base_plans)。

## 将 SAML 与您的 Temporal Cloud 帐户集成

1. 找到您的 [Temporal Cloud 帐户 ID](/cloud/Namespaces#Temporal-cloud-account-id)。可以查看您的帐户 ID
   并从右上角的 Temporal Cloud 用户配置文件下拉菜单复制。或者，找到您的
   [Namespace ID](/cloud/Namespaces#Temporal-cloud-Namespace-id)。帐户 ID 是以下五或六个字符
   句点 (.)，例如“f45a2”。您将需要帐户 ID 来构建您的回调 URL 和您的实体
   标识符。
1. 按照以下一组说明为您的 IdP 配置 SAML：
   - [Microsoft Entra ID](#configure-saml-with-azure-ad)
   - [Okta](#configure-saml-with-okta)
1. [与我们分享您的连接信息并测试您的连接。](#finish-saml-configuration)

## 如何使用 Microsoft Entra ID 配置 SAML {#configure-saml-with-azure-ad}

如果您想使用通用的 Microsoft 登录机制，则无需使用 Entra ID 设置 SAML。只需选择
**在 Temporal Cloud 登录页面上继续使用 Microsoft**。

要将 Entra ID 用作 SAML IdP，请创建 Microsoft Entra ID Enterprise 应用程序。

1. 登录 [Microsoft Entra ID](https://portal.azure.com/)。
1. 在主页的“**管理 Microsoft Entra ID**”下，选择“**查看**”。
1. 在顶部附近的**概述**页面上，选择**添加 > 企业应用程序**。
1. 在顶部附近的 **浏览 Microsoft Entra ID Gallery** 页面上，选择 **创建您自己的应用程序**。
1. 在 **创建您自己的应用程序** 窗格中，提供您的应用程序的名称（例如“Temporal-cloud”）并
   选择**集成您在库中找不到的任何其他应用程序**。
1. 选择**保存**。
1. 在 **入门** 部分中，选择 **2。设置单点登录**。
1. 在**单点登录**页面上，选择**SAML**。
1. 在“基于 SAML 的登录”页面的“基本 SAML 配置”部分中，选择“编辑”。
1. 在 **标识符（实体 ID）** 中，输入以下实体标识符，包括您的帐户 ID（如注明）：

   ```bash
   urn:auth0:prod-tmprl:ACCOUNT_ID-saml
   ```

正确形成的实体标识符如下所示：

   ```bash
   urn:auth0:prod-tmprl:f45a2-saml
   ```

1. 在**回复 URL（断言消费者服务 URL）**中，输入以下回调 URL，包括您的帐户 ID，其中
   表示：

   ```bash
   https://login.tmprl.cloud/login/callback?connection=ACCOUNT_ID-saml
   ```

正确形成的回调 URL 如下所示：

   ```bash
   https://login.tmprl.cloud/login/callback?connection=f45a2-saml
   ```

1. 在 **登录 URL** 中，输入以下登录 URL，包括您的帐户 ID（如所示）：

   ```bash
   https://cloud.temporal.io/login/saml?connection=ACCOUNT_ID-saml
   ```

正确形成的登录 URL 如下所示：

   ```bash
   https://cloud.temporal.io/login/saml?connection=f45a2-saml
   ```

1. 其他字段可以留空。在窗格顶部附近，选择“**保存**”。
1. 在 **属性和声明** 部分中，选择 **编辑**。配置以下设置。在**所需索赔**下：
   - 将**唯一用户标识符 (NameID)** 设置为 `user.userprincipalname`
   - 将 **NameID 格式** 设置为 `emailAddress`

这些是 Microsoft Entra ID 的默认设置。然后在**其他索赔**下，确保**电子邮件**和
   **姓名** 存在。

1. 收集您需要发送给我们的信息：
   - 在 **基于 SAML 的登录** 页面的 **SAML 证书** 部分中，选择 **证书的下载链接
     (Base64)**。
   - 在 **基于 SAML 的登录** 页面的 **设置 _APPLICATION_NAME_** 部分中，复制 **登录 URL** 的值。

要完成将 Microsoft Entra ID 设置为 SAML IdP，请参阅[完成 SAML 配置](#finish-saml-configuration)。

## 如何使用 Okta 配置 SAML {#configure-saml-with-okta}

要将 Okta 用作 SAML IdP，请配置新的 Okta 应用程序集成。

1. 登录[Okta 管理控制台](https://www.okta.com/login/)。
1. 在左侧导航窗格中，选择“**应用程序 > 应用程序**”。
1. 在 **应用程序** 页面上，选择 **创建应用程序集成**。
1. 在 **创建新的应用程序集成** 对话框中，选择 **SAML 2.0**，然后选择 **下一步**。
1. 在 **创建 SAML 集成** 页面的 **常规设置** 部分中，提供应用程序的名称
   （例如`Temporal-cloud`），然后选择**下一步**。
1. 在 **单点登录 URL** 的 **配置 SAML** 部分中，输入以下回调 URL，包括您的帐户
   标明 ID 的位置：

   ```bash
   https://login.tmprl.cloud/login/callback?connection=ACCOUNT_ID-saml
   ```

正确形成的回调 URL 如下所示：

   ```bash
   https://login.tmprl.cloud/login/callback?connection=f45a2-saml
   ```

1. 在 **受众 URI（SP 实体 ID）** 中，输入以下实体标识符，包括您的帐户 ID（如注明）：

   ```bash
   urn:auth0:prod-tmprl:ACCOUNT_ID-saml
   ```

正确形成的实体标识符如下所示：

   ```bash
   urn:auth0:prod-tmprl:f45a2-saml
   ```

1. 连接 Temporal 时，我们需要用户的完整电子邮件地址。
   - 在**姓名 ID 格式**中，选择“电子邮件地址”。
   - 在**属性语句**中，设置**电子邮件**和**姓名**。
1. 选择**下一步**。
1. 在 **反馈** 部分中，选择 **完成**。
1. 在 **应用程序** 页面上，选择您刚刚创建的应用程序集成的名称。
1. 在应用程序集成页面上，选择“**登录**”选项卡。
1. 在 **SAML 设置**下，选择 **查看 SAML 设置说明**。
1. 收集您需要发送给我们的信息：
   - 复制 IdP 设置。
   - 下载有效证书。

要完成将 Okta 设置为 SAML IdP，请参阅下一部分：
[完成 SAML 配置](#finish-saml-configuration)。

## 如何完成 SAML 配置 {#finish-saml-configuration}

在您使用 IdP 配置 SAML 后，我们就可以完成我们这边的配置。
[创建支持票证](/cloud/support#support-ticket)，其中包含以下信息：

- 您的应用程序中的登录 URL
- PEM 格式的 X.509 SAML 登录证书
- 一个或多个 IdP 域映射到 SAML 连接

通常，提供的 IdP 域与您的电子邮件地址的域相同。您可以提供多个 IdP
域。

当您收到我们确认我们已完成配置后，请登录 Temporal Cloud。不过这一次，
输入您的电子邮件地址，然后单击“**继续**”。您将被引导至 IdP 的身份验证页面。
