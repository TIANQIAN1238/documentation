:::tip Namespaces with High Availability features and AWS PrivateLink

使用 PrivateLink 时，需要正确的网络配置才能使故障转移对客户端和 Worker 透明。
本页面介绍如何在 AWS PrivateLink 上为具有高可用性功能的 Namespace 配置路由。

:::

要将 AWS PrivateLink 与高可用性功能结合使用，您可能需要：

- 覆盖区域 DNS 区域。
- 确保两个区域之间的网络连接。

这些说明假设您已经具备 PrivateLink 连接。如果没有，请按照我们的[创建 AWS PrivateLink 连接和配置私有 DNS 指南](/cloud/connectivity/aws-connectivity)。

## 客户端解决方案

使用 PrivateLink 时，您通过 VPC 终端节点连接到 Temporal Cloud，该终端节点使用您网络的本地地址。
Temporal 将每个“region.<tmprl_domain>”视为单独的区域。
此设置允许您覆盖默认区域，确保流量在您正在使用的区域内部路由。

Namespace 的活动区域反映在 CNAME 记录的目标中。
例如，如果 Namespace 的活动区域是 AWS us-west-2，则 DNS 配置将如下所示：

| ha-Namespace.account-id.tmprl.cloud | ha-Namespace.account-id.tmprl.cloud |别名 | aws-us-west-2.region.tmprl.cloud |
| ----------------------------------- | -----| -------------------------------- |

故障转移后，CNAME 记录将为 Updated 以指向故障转移区域，例如：

| ha-Namespace.account-id.tmprl.cloud | ha-Namespace.account-id.tmprl.cloud |别名 | aws-us-east-1.region.tmprl.cloud |
| ----------------------------------- | -----| -------------------------------- |

Temporal 域没有更改，但 CNAME Updated 从 us-west-2 更改为 us-east-1。

## 设置 DNS 覆盖

:::caution

尚未为 GCP 多区域 Namespace 提供专用连接。

:::

- `XPROT045X 服务帐户 XPROT037X`：XPROT037X 服务帐户的名称或描述字段
- `XPROT045X service-account set-account-role`：到 XPROT037X 服务帐户的帐户角色
- `XPROT045X service-account set-XPROT025X-permissions`：XPROT037X 服务帐户的 XPROT025X 权限

当您的 Worker 连接到 Namespace 时，它们首先解析 `<ns>.<acct>.<tmprl_domain>` 记录。
这指向“<aws-active-region>.region.tmprl.cloud”，然后解析为您的内部 IP 地址。

考虑如何为此设置配置 Worker。
您可以让 Worker 在两个区域中持续运行，也可以使用 Transit Gateway 或 VPC 对等互连在区域之间建立连接。
这样，一旦发生故障转移，Worker 就可以访问新激活的区域。

使用具有高可用性功能的 Namespace 时，Namespace 的 DNS 记录“<ns>.<acct>.<tmprl_domain>”指向格式为“<region>.region.<tmprl_domain>”的区域 DNS 记录。
这里，“<region>”是 Namespace 当前的活动区域。

在故障转移期间，Temporal Cloud 将 Namespace DNS 记录的目标从一个区域更改为另一区域。
Namespace DNS 记录配置为 15 秒 <a href="https://en.wikipedia.org/wiki/Time_to_live">TTL</a>。
任何 DNS 缓存都应在此时间内重新解析记录。
根据经验，接收 Updated DNS 记录大约需要 TTL 的两倍 (2x)。
客户端应在最多 30 秒的延迟内汇聚到新的目标区域。
