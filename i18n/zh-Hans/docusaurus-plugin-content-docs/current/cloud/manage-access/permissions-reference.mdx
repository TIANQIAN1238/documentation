Temporal Cloud 访问控制分为两个范围：

- 账户级角色权限
- Namespace级权限

在每个范围内，权限适用于公开记录的 [Temporal Cloud Ops API](https://docs.Temporal.io/ops)
端点以及其他非云运营功能，例如 Temporal Cloud UI 和内部自动化行为。

## 帐户级别访问权限 {#account-level-access}

通过为用户和服务帐户分配帐户级角色来授予帐户级访问权限。 Temporal Cloud
支持以下帐户级别角色：

- 账户所有者
- 全局管理员
- 开发商
- 财务管理员
- 只读

## Namespace级权限

通过为用户和服务帐户分配 Namespace 级别的权限，可以向用户和服务帐户授予 Namespace 级别的权限。
Temporal Cloud 支持以下 Namespace 级别权限：

- Namespace 管理员
- 写
- 阅读

具有全局管理员和帐户所有者角色的用户自动拥有对以下位置的所有 Namespace 的 Namespace 管理员权限：
帐户。

## 额外的授权行为

某些 API 被授予所有帐户级角色，但在运行时强制执行额外的授权规则。行动组
授予调用 API 的权限，但调用者可以交互的范围取决于他们的角色。

### API密钥授权行为

所有角色都可以创建和管理自己的**自己的** API 密钥。 API 密钥继承其所有者的权限 - 它不能
授予超出所属用户或服务帐户已有权限的访问权限。

|行为 |只读 |开发商|财务管理|全局管理|帐户所有者 |
| --------------------------------------------------- | :-----: | :-----: | :------------: | :----------: | :------------: |
|创建、查看、Update 和删除自己的 API 密钥 |     ✔ |     ✔ |       ✔ |      ✔ |       ✔ |
|查看、Update 和删除帐户中的任何 API 密钥 |           |           |               |      ✔ |       ✔ |

**受影响的 API：** CreateApiKey、GetApiKey、GetApiKeys、UpdateApiKey、DeleteApiKey

### 服务帐户授权行为

所有角色都可以在其帐户中列出服务帐户。但是，创建、Update 和删除服务的能力
帐户取决于服务帐户的范围和调用者的角色。

|行为 |只读 |开发商|财务管理|全局管理|帐户所有者 |
| ------------------------------------------------ | :-----: | :-----: | :------------: | :----------: | :------------: |
|列出帐户中的所有服务帐户 |     ✔ |     ✔ |       ✔ |      ✔ |       ✔ |
|管理无范围（帐户级）服务帐户 |           |           |               |      ✔ |       ✔ |
|管理 Namespace 范围的服务帐户 |     §|     §|       §|      ✔ |       ✔ |

§ 需要目标 Namespace 的 Namespace 管理员权限。任何角色都可以管理 Namespace 范围的服务帐户，如果
他们在 Namespace 上持有 Namespace 管理员。

**受影响的 API：** CreateServiceAccount、GetServiceAccount、GetServiceAccounts、UpdateServiceAccount、
删除服务帐户
