Temporal Nexus 现已[普遍可用](/evaluate/development-product-features/release-stages#general-availability)。
了解为什么应在[评估指南](/evaluate/Nexus) 中使用 Nexus。

:::

Temporal Cloud 建立在[核心 Nexus 体验](/Nexus) 的基础上，具有：

- **全局 [Nexus 注册表](/Nexus/registry)** - 范围涵盖所有 Namespace 的整个帐户。任何 Namespace 中的 Worker 都可以托管 Nexus Service 供其他人使用。
- **内置[访问控制](/Nexus/registry#configure-runtime-access-controls)** - 限制哪个调用方 Namespace 可以在运行时使用 Nexus 端点。
- **[审核日志记录](/cloud/audit-logs)** - 将 Nexus 注册表操作（创建、Update、删除端点）流式传输到审核日志集成。
- **多区域连接** - Nexus 请求使用受全球 mTLS 保护的 Envoy 网格在 AWS 和 GCP 内部以及之间的 Namespace 之间进行路由。与以 [高可用性](/cloud/high-availability) 作为端点目标的 Namespace 兼容。
- **[Terraform 支持](/cloud/terraform-provider#manage-Temporal-cloud-Nexus-endpoints-with-terraform)** - 使用 Temporal Cloud Terraform 提供程序管理 Nexus 端点。
