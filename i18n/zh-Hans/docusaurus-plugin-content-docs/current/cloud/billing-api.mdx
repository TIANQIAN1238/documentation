Temporal Cloud 计费 API 通过按需计费报告提供 Namespace 级别的成本归因。 
报告以 CSV 格式提供，可以通过 API 访问或直接下载以在 FinOps 工具和成本管理平台中使用。

此 API 是 [云操作 API](/ops) 的一部分。

:::tip Support, stability, and dependency info

Temporal Cloud 计费 API 处于[预发布](/evaluate/development-product-features/release-stages#pre-release) 状态。

:::

计费 API 允许您：

- 生成指定发票月份的账单报告  
- 检索报告状态和元数据  
- 下载可输入内部分析工具或云成本管理平台的 CSV 报告

帐单报告包含：

- 精准的Namespace级成本归属  
- 每小时粒度  
- [FOCUS](https://focus.finops.org/) 友好的数据格式

有关完整的请求和响应架构，请参阅下面的架构。

计费报告生成是**异步**。您启动报告创建，然后轮询是否完成。

## 报告数据限制

对于预发布，只能为当前帐单月份和上一个帐单月份生成每小时粒度的报告。

## 允许的日期范围

日期范围必须使用帐单月份边界 (MM/YYYY)。 
请求可能包括当前的帐单月份。 
最终报告中的数据包括截至“current_time”\- 24 小时的使用情况（四舍五入到最近的小时）。

## 速率限制和并发数

速率限制适用于 API 使用。

### 每账户并发数

在单个帐户内：

- 每个帐户一次仅生成一份账单报告  
- 其他请求已接受但已排队

### 报告生成延迟

报告生成时间各不相同，并且无法保证。影响它的因素包括请求的日期范围的大小和整体平台负载。

## 最佳实践

重试请求时提供幂等键（`async_operation_id`）。

使用指数退避轮询“GetBillingReport”。

生成后立即下载报告（URL 过期）。

避免在当前计费周期内频繁生成大的重叠范围。

## 生成报告

要生成报告，请执行以下步骤：

1. 使用“CreateBillingReport”创建账单报告。该响应包括“billing_report_id”和“async_operation_id”。
1. 使用 `billing_report_id` 轮询 `GetBillingReport`  
1. 当报告状态变为`BILLING_REPORT_STATE_GENERATED`时，检索下载URL  
1. 在URL过期之前下载报告

### 关键标识符

|标识符 |目的|
| -----| -----|
| `billing_report_id` |标识计费报告并用于检索元数据和下载 URL |
| `async_operation_id` |标识负责生成报告的后台操作 |

异步操作遵循标准云操作异步模型（请参阅[异步操作](/ops#per-identity-rate-limits)）。
