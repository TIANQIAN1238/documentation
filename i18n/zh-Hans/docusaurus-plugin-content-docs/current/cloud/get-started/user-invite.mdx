<TabItem value="invite-webui" label="Web UI">

要使用 Temporal Cloud UI 邀请用户：

1. 在 Temporal Web UI 中，选择窗口左侧的 **设置**。
1. 在**设置**页面上，选择窗口右上角的**创建用户**。
1. 在 **创建用户** 页面的 **电子邮件地址** 框中，键入或粘贴一个或多个电子邮件地址。
1. 在**账户级角色**中，选择一个[角色](/cloud/manage-access/roles-and-permissions#account-level-roles)。角色
   适用于电子邮件地址出现在**电子邮件地址**中的所有用户。
1. 如果帐户有任何 Namespace，它们会列在 **授予对 Namespace 的访问权限**下。要添加权限，请选择
   Namespace 旁边的复选框，然后选择
   [权限](/cloud/manage-access/roles-and-permissions#Namespace-level-permissions)。根据需要重复。
1. 分配所有权限后，选择 **发送邀请**。

</TabItem>

<TabItem value="invite-tcld" label="tcld">

使用 [`tcld 用户邀请`](/cloud/tcld/user/#invite) 命令。指定用户的电子邮件、帐户级角色，以及
可选一项或多项 Namespace 权限。

可用的帐户角色：`admin` | `开发人员` | ‘读’。

可用的 Namespace 权限：`Admin` | `写` | ‘读’。

```command
tcld user invite \
  --user-email <user@example.com> \
  --account-role <role> \
  --namespace-permission <namespace>=<permission>
```

您可以在单个命令中邀请多个用户并分配多个 Namespace 权限：


```command
tcld user invite \
  --user-email user1@example.com \
  --user-email user2@example.com \
  --account-role developer \
  --namespace-permission ns1=Admin \
  --namespace-permission ns2=Write
```

</TabItem>

<TabItem value="invite-ops-api" label="Cloud Ops API">

使用 [CreateUser](https://saas-api.tmprl.cloud/docs/httpapi.html#tag/users) 端点邀请用户。

```
POST /cloud/users
```

请求正文包含一个包含以下字段的“spec”：

- `spec.email` — 要邀请的用户的电子邮件地址。
- `spec.access.account_access.role` — 要分配的帐户级别角色。
- `spec.access.Namespace_accesses` — Namespace 名称到权限的映射。

可用角色：`ROLE_ADMIN` | `ROLE_DEVELOPER` | `ROLE_READ` | `ROLE_OWNER` | `ROLE_FINANCE_ADMIN`。

可用的 Namespace 权限：`PERMISSION_ADMIN` | `PERMISSION_WRITE` | `PERMISSION_READ`。

</TabItem>

</Tabs>

新用户会收到一封电子邮件，其中包含接受邀请并完成设置的链接。新用户必须使用此
注册链接将添加到您的帐户，除非该帐户具有 SAML 配置。如果您的帐户有 SAML
配置中，新用户可以使用其现有的 SAML 凭据登录并包含在帐户中
自动。

:::caution

新用户必须使用最初注册时使用的相同身份验证方法来登录 Temporal Cloud。如果
他们使用单点登录 (SSO)，因此必须使用相同的 SSO 提供商才能登录 Temporal Cloud。如果他们使用电子邮件并且
密码身份验证，必须使用相同的电子邮件和密码登录 Temporal Cloud，并且不能使用 SSO，
即使底层电子邮件地址相同。

:::
