## 回填

批量执行在指定时间间隔内运行的操作。
使用此命令填写 Workflow 从 Schedule 暂停时开始运行的信息，
在创建 Schedule 之前，从未来开始，或者重新处理以前的
执行间隔。

回填需要 Schedule ID 和请求涵盖的时间段。
最好使用 `BufferAll` 或 `AllowAll` 策略以避免冲突
并确保没有跳过 Workflow Execution。

例如：

```
temporal schedule backfill \
    --schedule-id "YourScheduleId" \
    --start-time "2022-05-01T00:00:00Z" \
    --end-time "2022-05-31T23:59:59Z" \
    --overlap-policy BufferAll
```

这些政策包括：

* **AllowAll**：允许无限并发 Workflow Execution。这个
  显着加快支持系统的回填过程
  并发。您必须确保运行 Workflow Execution 不会干扰
  与彼此。
* **BufferAll**：在等待时缓冲所有传入的 Workflow Execution
  运行Workflow Execution来完成。
* **跳过**：如果之前的 Workflow Execution 仍在运行，则丢弃新的
  Workflow Executions。
* **BufferOne**：与“跳过”相同，但缓冲单个 Workflow Execution
  在上一个执行完成后运行。丢弃其他 Workflow
  处决。
* **CancelOther**：取消正在运行的Workflow Execution，并将其替换为
  即将推出的新 Workflow Execution。
* **TerminateOther**：终止正在运行的Workflow Execution并替换
  它与即将推出的新 Workflow Execution 一起使用。

＃＃ 创造

在 Temporal 服务上创建新的 Schedule。 Schedule 自动启动
在您指定的时间购买新的 Workflow Execution。

例如：

```
  temporal schedule create \
    --schedule-id "YourScheduleId" \
    --calendar '{"dayOfWeek":"Fri","hour":"3","minute":"30"}' \
    --workflow-id YourBaseWorkflowIdName \
    --task-queue YourTaskQueue \
    --type YourWorkflowType
```

Schedules 支持 `--calendar`、`--interval` 和 `--cron` 的任意组合：

* 简写 `--interval` 字符串。
  例如：45m（每45分钟一班）或6h/5h（每6小时一班，在顶部
  第5个小时）。
* JSON `--calendar`，如前面的示例所示。
* Unix 风格的 `--cron` 字符串和 robfig 声明
  （@每日/@每周/@每X/等）。
  例如，每周五中午 12:30：“30 12 * * Fri”。

＃＃ 删除


## 切换

通过传递带有您所需状态的标志来暂停或取消暂停 Schedule：

```
temporal schedule toggle \
    --schedule-id "YourScheduleId" \
    --pause \
    --reason "YourReason"
```

和

```
temporal schedule toggle
    --schedule-id "YourScheduleId" \
    --unpause \
    --reason "YourReason"
```

Update 的“--reason”文本与 Schedule 的操作“notes”字段相同
沟通。如果省略，则默认为“（未提供原因）”。该字段是
在服务 Web UI 上也可见。

使用以下选项更改此命令的行为。您还可以使用任何适用于所有子命令的[全局标志](#global-flags)。

|旗帜|必填 |描述 |
|------|----------|-------------|
| `--暂停` |没有 | **布尔** 暂停 Schedule。 |
| `--原因` |没有 | **字符串** 暂停或取消暂停 Schedule 的原因。 |
| `--Schedule-id`、`-s` |是的 | **字符串** Schedule ID。 |
| `--取消暂停` |没有 | **布尔** 取消暂停 Schedule。 |

## Update

Update 是现有的 Schedule，具有新的配置详细信息，包括时间
规范、行动和政策：

```
temporal schedule update \
    --schedule-id "YourScheduleId" \
    --workflow-type "NewWorkflowType"
```

此命令执行 Schedule 的完全替换
配置。任何未提供的选项将重置为默认值
价值观。您必须重新指定所有选项，而不仅仅是您想要的选项
改变。要查看 Schedule 的当前配置，请使用
更新前的“Temporal Schedule 描述”。

Schedule 备忘录和 Search Attributes 不能与此成为 Updated
命令。它们仅在 Schedule 创建期间设置，不受影响
由 Updates 提供。
