```

```
temporal task-queue config get \

```
temporal task-queue describe \
  --task-queue YourTaskQueue
```

此命令提供给定 Task Queue 的轮询器信息。
Workflow 和 Activity 轮询使用单独的 Task Queue：

```
temporal task-queue describe \
    --task-queue YourTaskQueue \
    --task-queue-type "activity"
```

此命令提供以下 Task Queue 统计信息：
- `ApproximateBacklogCount`：此中积压的任务的大致数量
  Task Queue。可能会计算过期的任务，但最终会收敛到右侧
  值。
- `ApproximateBacklogAge`：待办事项中最旧任务的大致年龄，
  基于其创建时间（以秒为单位）。
- `TasksAddRate`：将任务添加到任务的大致速率
  队列，以每秒任务数为单位，在过去 30 秒内取平均值。
  包括立即分派的任务，无需进入待办事项列表
  （同步匹配的任务），以及添加到待办事项中的任务。 （参见下面的注释。）
- `TasksDispatchRate`：任务调度的大致速率
  Task Queue，以每秒任务数衡量，过去 30 次的平均值
  秒。  包括立即分派的任务，无需进入待办事项列表
  （同步匹配的任务），以及添加到待办事项中的任务。 （参见下面的注释。）
- `BacklogIncreaseRate`：积压大小的大致速率
  增加（如果为正）或减少（如果为负），以每个任务的任务来衡量
  第二，过去 30 秒的平均值。  这大致相当于：
  `TasksAddRate` - `TasksDispatchRate`。

注意：“TasksAddRate”和“TasksDispatchRate”指标可能与
添加/分派的实际速率，因为任务可能会急切地分派到
可用的 Worker，或者可能仅适用于特定的 Worker（它们是“粘性的”）。
这些指标不计算此类任务。尽管不准确
这两个指标，“BacklogIncreaseRate”的派生指标是准确的
对于超过几秒的积压。

通过检查可访问性，安全地停用分配有构建 ID 的 Worker
所有任务类型。使用标志 `--report-reachability`：

```
temporal task-queue describe \
    --task-queue YourTaskQueue \
    --select-build-id "YourBuildId" \
    --report-reachability
```

返回所请求版本和所有任务的可达性信息
任务类型，可用于安全淘汰旧代码版本的 Worker，
前提是他们被分配了一个构建 ID。

请注意，任务可达性状态已被弃用，取而代之的是排水状态
（即 Drained 或 Draining Worker 部署版本）并将被删除 
在未来的版本中。此外，确定任务的可达性会导致一个不平凡的问题 
计算成本。

任务可达性状态按构建 ID 报告。国家可能是其中之一
以下：

- `Reachable`：使用当前版本控制规则，可以使用构建 ID
  通过新的 Workflow Execution 或活动或当前开放
  分配给队列的 Workflow 或积压的 Activity 任务。
- `ClosedWorkflowsOnly`：构建 ID 没有打开的 Workflow Executions
  新的 Workflow Execution 无法访问。它可能已经关闭
  Workflow Execution 在 Namespace 保留期内。
- `Unreachable`：此构建 ID 不用于新的 Workflow Execution，并且
  在保留期内不被任何现有的 Workflow Execution 使用。

任务可达性最终是一致的。您可能会遇到延迟，直到
可达性收敛到最准确的值。这是为了行动而设计的
以最保守的方式直到收敛。例如，“可达”是
比“ClosedWorkflowsOnly”更保守。

### 获取规则

检索所有 Worker 构建 ID 分配和关联的重定向规则
使用 Task Queue：

```
temporal task-queue versioning get-rules \
    --task-queue YourTaskQueue
```

Task Queue 支持以下版本控制规则：

- 分配规则：管理如何分配新的执行以在特定的上运行
  Worker 构建 ID。每个 Task Queue 存储一个有序分配规则列表，
  从头到尾进行评估。分配规则还允许
  通过设置斜坡百分比逐步推出新的构建 ID。
- 重定向规则：自动将源构建 ID 的工作分配给目标
  构建 ID。您最多可以为每个源构建 ID 添加一条重定向规则。
  重定向规则要求目标构建 ID 完全兼容
  源构建 ID。

### 插入赋值规则

temporal task-queue versioning insert-assignment-rule \
    --task-queue YourTaskQueue \
    --build-id "YourBuildId"
```

### 替换分配规则

无条件规则（无提示过滤器或百分比）。否则，操作
会被拒绝。将“force”设置为 true 以绕过此验证。

```
temporal task-queue versioning replace-assignment-rule \
    --task-queue YourTaskQueue \
    --rule-index AnIntegerIndex \
    --build-id "YourBuildId"

'插入赋值规则'。

```
temporal task-queue versioning replace-assignment-rule \
    --task-queue YourTaskQueue \
    --rule-index AnIntegerIndex \
    --build-id "YourBuildId" \
    --percentage AnIntegerPercent
```

百分比可能在 0 到 100（默认）之间变化。
