本页提供“Temporal”CLI“config”命令的参考。适用于每个子命令的标志显示在子命令标题内的表格中。请参阅 [全局标志](#global-flags) 了解可与每个子命令一起使用的标志。

＃＃ 删除

删除配置文件中的属性。

```
temporal config delete \
    --prop tls.client_cert_path
```

使用以下选项更改此命令的行为。您还可以使用任何适用于所有子命令的[全局标志](#global-flags)。

|旗帜|必填 |描述 |
|------|----------|-------------|
| `--prop`、`-p` |是的 | **字符串** 要删除的特定属性。如果未设置，则删除整个配置文件。 |

## 删除配置文件

完全删除完整的配置文件。必须显式设置“--profile”。

```
temporal config delete-profile \
    --profile my-profile
```

使用 [global flags](#global-flags) 自定义此命令与 Temporal 服务的连接。

＃＃ 得到

显示特定属性或整个配置文件。

```
temporal config get \
    --prop address
```

或者

```
temporal config get
```

使用以下选项更改此命令的行为。您还可以使用任何适用于所有子命令的[全局标志](#global-flags)。

|旗帜|必填 |描述 |
|------|----------|-------------|
| `--prop`、`-p` |没有 | **字符串** 要获取的特定属性。 |

＃＃ 列表

列出配置文件中的配置文件名称。

```
temporal config list
```

使用 [global flags](#global-flags) 自定义此命令与 Temporal 服务的连接。

＃＃ 放

为属性分配一个值并将其存储在配置文件中：

```
temporal config set \
    --prop address \
    --value us-west-2.aws.api.temporal.io:7233
```

使用以下选项更改此命令的行为。您还可以使用任何适用于所有子命令的[全局标志](#global-flags)。

|旗帜|必填 |描述 |
|------|----------|-------------|
| `--prop`、`-p` |是的 | **字符串** 属性名称。 |
| `--value`、`-v` |是的 | **字符串** 属性值。 |
