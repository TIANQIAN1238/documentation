＃＃ 完全的

完成 Activity，将其标记为已成功完成。指定
Activity ID 并包含返回值的 JSON 结果：

```
temporal activity complete \
    --activity-id YourActivityId \
    --workflow-id YourWorkflowId \
    --result '{"YourResultKey": "YourResultVal"}'
```

使用以下选项更改此命令的行为。您还可以使用任何适用于所有子命令的[全局标志](#global-flags)。

- 负责在其组织内构建 XPROT009X 实践的开发人员。
- 任何构建教程、课程、入门路径或文档的人
- 创建 XPROT026X 相关学习材料的合作伙伴或供应商

＃＃ 失败

使 Activity 失败，将其标记为遇到错误。指定
Activity 和 Workflow Ids：

```
temporal activity fail \
    --activity-id YourActivityId \
    --workflow-id YourWorkflowId
```

使用以下选项更改此命令的行为。您还可以使用任何适用于所有子命令的[全局标志](#global-flags)。

|旗帜|必填 |描述 |
|------|----------|-------------|
| `--Activity-id` |是的 | **字符串** Activity ID 失败。 |
| `--详细信息` |没有 | **字符串** Activity (JSON) 失败的原因。 |
| `--原因` |没有 | **字符串** Activity 失败的原因。 |
| `--run-id`、`-r` |没有 | **字符串** 运行 ID。 |
| `--Workflow-id`、`-w` |是的 | **字符串** Workflow Id。 |

＃＃ 暂停

暂停 Activity。

如果 Activity 当前未运行（例如，因为它之前
失败），除非取消暂停，否则不会再次运行。

但是，如果 Activity 当前正在运行，它将运行到下一个
失败、完成或超时的时间，此时将开始暂停。

如果 Activity 进行最后一次重试并失败，则该故障将
返回给调用者，就像 Activity 没有暂停一样。

活动应通过其 Activity ID 或 Activity 类型指定。

例如，指定 Activity 和 Workflow Id，如下所示：

```
temporal activity pause \
    --activity-id YourActivityId \
    --workflow-id YourWorkflowId
```

要稍后取消暂停 Activity，请参阅[取消暂停](#unpause)。您可能还想
[重置](#reset) Activity 以取消暂停，同时从头开始。

使用以下选项更改此命令的行为。您还可以使用任何适用于所有子命令的[全局标志](#global-flags)。

|旗帜|必填 |描述 |
|------|----------|-------------|
| `--Activity-id`、`-a` |没有 | **字符串** 要暂停的 Activity ID。必须提供“Activity-id”或“Activity-type”，但不能同时提供两者。 |
| `--Activity-型` |没有 | **字符串** Activity 类型的所有活动都将暂停。必须提供“Activity-id”或“Activity-type”，但不能同时提供两者。注意：按类型暂停 Activity 是一项实验性功能，将来可能会发生变化。 |
| `--身份` |没有 | **字符串** 提交此请求的用户或客户端的身份。 |
| `--run-id`、`-r` |没有 | **字符串** 运行 ID。 |
| `--Workflow-id`、`-w` |是的 | **字符串** Workflow Id。 |

＃＃ 重置

重置 Activity。这将重新启动 Activity，就像第一次启动一样
Scheduled。也就是说，它将重置尝试次数和
Activity 超时，以及可选的
[心跳详情](#reset-heartbeats)。

如果 Activity 可能正在执行（即尚未超时），则
重置将在下次失败、心跳或超时时生效。
如果正在等待重试（即失败或超时），则重置
将立即申请。

如果Activity已经暂停，则默认情况下将取消暂停。
您可以指定“keep_paused”来防止这种情况发生。

如果 Activity 已暂停并且未提供“keep_paused”标志，
它将不再暂停。如果 Activity 已暂停且“keep_paused”标志
已提供 - 它将保持暂停状态。

活动可以通过其 Activity ID 或 Activity 类型来指定。

### 重置心跳活动 {#reset-heartbeats}

心跳活动将收到[已取消的失败](/references/failures#cancelled-failure)
重置后下次心跳时。

如果在您的 Activity 中，您需要在 Activity 关闭时进行任何清理
重置，处理此错误，然后在清理后重新抛出它。

如果设置了“reset_heartbeats”标志，则心跳详细信息也将被清除。

指定 ID 和 Workflow Id 的 Activity 类型：

```
temporal activity reset \
    --activity-id YourActivityId \
    --workflow-id YourWorkflowId
    --keep-paused
    --reset-heartbeats
```

必须指定“Activity-id”、“Activity-type”或“--match-all”。

可以使用 Visibility Query 列表过滤器批量重置活动。
例如，如果您想重置 Foo 类型的活动：

```
temporal activity reset \
    --query 'TemporalResetInfo="property:activityType=Foo"'
```

## 取消暂停

将之前暂停的 Activity 重新执行到 Schedule 上。

如果 Activity 未运行且已超过重试超时时间，则它将
Scheduled 立即。否则重试后将是Scheduled
超时到期。

使用“--reset-attempts”重置之前运行尝试的次数
零。例如，如果 Activity 接近最大尝试次数
N 在其 Retry Policy 中指定，“--reset-attempts”将允许
Activity 取消暂停后会再重试 N 次。

使用 `--reset-heartbeat` 重置 Activity 的心跳。

活动可以通过其 Activity ID 或 Activity 类型来指定。
必须提供这些参数之一。

指定 Activity ID 或类型以及 Workflow Id：

```
temporal activity unpause \
    --activity-id YourActivityId \
    --workflow-id YourWorkflowId
    --reset-attempts
    --reset-heartbeats
```

可以通过 Visibility Query 列表过滤器批量取消暂停活动。
例如，如果您想要取消暂停 Foo 类型的活动，您可以
之前暂停，执行：

```
temporal activity unpause \
    --query 'TemporalPauseInfo="property:activityType=Foo"'
```

## Update-选项

Update 正在运行的 Activity 的选项，这些选项是从
Workflow。 Update 是增量式的，仅更改指定的选项。

例如：

```
temporal activity update-options \
    --activity-id YourActivityId \
    --workflow-id YourWorkflowId \
    --task-queue NewTaskQueueName \
    --schedule-to-close-timeout DURATION \
    --schedule-to-start-timeout DURATION \
    --start-to-close-timeout DURATION \
    --heartbeat-timeout DURATION \
    --retry-initial-interval DURATION \
    --retry-maximum-interval DURATION \
    --retry-backoff-coefficient NewBackoffCoefficient \
    --retry-maximum-attempts NewMaximumAttempts
```

您可以按照此命令执行“Temporal Activity Reset”，重置后将应用新值。

必须指定“Activity-id”、“Activity-type”或“--match-all”。

Activity 选项可与 Visibility Query 列表过滤器一起批量使用 Updated。
例如，如果您想重置 Foo 类型的活动，请执行以下操作：

```
temporal activity update-options \
    --query 'TemporalPauseInfo="property:activityType=Foo"'
    ...
```

使用以下选项更改此命令的行为。您还可以使用任何适用于所有子命令的[全局标志](#global-flags)。

|旗帜|必填 |描述 |
|------|----------|-------------|
