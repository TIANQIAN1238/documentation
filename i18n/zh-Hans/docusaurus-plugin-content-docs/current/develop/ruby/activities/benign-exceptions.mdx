**如何使用 Temporal Ruby SDK 将 Activity 错误标记为良性**

当活动抛出预期或不严重的错误时，它们可能会在日志、指标和 OpenTelemetry 跟踪中产生噪音，从而使识别真正问题变得更加困难。
通过将这些错误标记为良性，您可以将它们从可观测性数据中排除，同时仍然在 Workflow 逻辑中处理它们。

要将错误标记为良性，请在引发“ApplicationError”时将“category”参数设置为“Temporalio::Error::ApplicationError::Category::BENIGN”。

良性错误：
- 将 Activity 故障日志降级至 DEBUG 级别
- 不发出 Activity 故障指标
- 不要将 OpenTelemetry 失败状态设置为 ERROR

```ruby
require 'temporalio/activity'

class MyActivity < Temporalio::Activity::Definition
  def execute
    begin
      call_external_service
    rescue StandardError => e
      # Mark this error as benign since it's expected
      raise Temporalio::Error::ApplicationError.new(
        e.message,
        category: Temporalio::Error::ApplicationError::Category::BENIGN
      )
    end
  end
end
```

对正常操作中定期发生的 Activity 错误使用良性异常，例如轮询尚未准备就绪的外部服务，或处理将重试的预期瞬态故障。
