[Temporal CLIent](/encyclopedia/Temporal-sdks#Temporal-client) 使您能够与 Temporal 服务进行通信。
与 Temporal 服务的通信使您可以执行诸如启动 Workflow Execution、发送 Signal 和
查询 Workflow Execution、获取 Workflow 结果等。

本页向您展示如何使用 Ruby SDK 和 Temporal CLIent 执行以下操作：

- [连接到本地开发 Temporal 服务](#connect-to-development-service)
- [连接到 Temporal Cloud](#connect-to-Temporal-cloud)
- [启动 Workflow Execution](#start-Workflow)
- [获取 Workflow 结果](#get-Workflow-结果)

Temporal CLIent 无法在 Workflow 内部初始化和使用。然而，使用
Temporal CLI 位于 Activity 内部，与 Temporal 服务进行通信。

## 连接到开发 Temporal 服务 {#connect-to-development-service}

使用 [`Client.connect`](https://ruby.Temporal.io/Temporalio/Client.html#connect-class_method) 创建客户端。
连接选项包括 Temporal Server 地址、Namespace 和（可选）TLS 配置。您可以提供
这些选项直接在代码中，从 **环境变量** 或 **TOML 配置文件** 加载它们，使用
[`EnvConfig`](https://ruby.Temporal.io/Temporalio/EnvConfig.html) 帮助程序。我们推荐环境变量或
配置文件用于安全、可重复的配置。

当您在本地运行 Temporal 服务时（例如使用
[Temporal CLI 开发服务器](https://docs.Temporal.io/cli/server#start-dev))，所需的选项很少。如果你
不指定主机/端口，大多数连接默认为“127.0.0.1:7233”和“默认”Namespace。

您可以使用 TOML 配置文件来设置 Temporal CLIent 的连接选项。配置文件可以让你
配置多个配置文件，每个配置文件都有自己的一组连接选项。然后，您可以指定在以下情况下使用哪个配置文件：
创建 Temporal CLIent。您可以使用环境变量“Temporal_CONFIG_FILE”来指定
TOML 文件或直接在代码中提供文件的路径。如果不提供配置文件路径，SDK
在路径“~/.config/Temporalio/Temporal.toml”或操作系统上的等效路径中查找它。参考
[环境配置](/references/client-environment-configuration) 了解更多配置细节
文件和配置文件。

:::info

配置文件中设置的连接选项的优先级低于环境变量。这意味着如果
您在配置文件和环境变量中设置相同的选项，即环境变量值
覆盖配置文件中设置的选项。

:::

例如，以下 TOML 配置文件定义了两个配置文件：“default”和“prod”。每个配置文件都有自己的
连接选项集。

```toml title="config.toml"
# Default profile for local development
[profile.default]
address = "localhost:7233"
namespace = "default"

# Optional: Add custom gRPC headers
[profile.default.grpc_meta]
my-custom-header = "development-value"
trace-id = "dev-trace-123"

# Production profile for Temporal Cloud
[profile.prod]
address = "your-namespace.a1b2c.tmprl.cloud:7233"
namespace = "your-namespace"
api_key = "your-api-key-here"

# TLS configuration for production
[profile.prod.tls]
# TLS auto-enables when TLS config or an API key is present
# disabled = false
client_cert_path = "/etc/temporal/certs/client.pem"
client_key_path  = "/etc/temporal/certs/client.key"

# Custom headers for production
[profile.prod.grpc_meta]
environment     = "production"
service-version = "v1.2.3"
```

您可以使用配置文件中的配置文件创建 Temporal CLIent
`ClientConfig.load_client_connect_options` 功能如下。在此示例中，您加载“默认”配置文件
当地发展：

```ruby
require 'temporalio/client'
require 'temporalio/env_config'

def main
  puts '--- Loading default profile from config.toml ---'

  # For this sample to be self-contained, we explicitly provide the path to
  # the config.toml file included in this directory.
  # By default though, the config.toml file will be loaded from
  # ~/.config/temporalio/temporal.toml (or the equivalent standard config directory on your OS).
  config_file = File.join(__dir__, 'config.toml')

  # load_client_connect_options is a helper that loads a profile and prepares
  # the configuration for Client.connect. By default, it loads the
  # "default" profile.
  args, kwargs = Temporalio::EnvConfig::ClientConfig.load_client_connect_options(
    config_source: Pathname.new(config_file)
  )

  puts "Loaded 'default' profile from #{config_file}."
  puts "  Address: #{args[0]}"
  puts "  Namespace: #{args[1]}"
  puts "  gRPC Metadata: #{kwargs[:rpc_metadata]}"

  puts "\nAttempting to connect to client..."
  begin
    client = Temporalio::Client.connect(*args, **kwargs)
    puts '✅ Client connected successfully!'
    sys_info = client.workflow_service.get_system_info(Temporalio::Api::WorkflowService::V1::GetSystemInfoRequest.new)
    puts "✅ Successfully verified connection to Temporal server!\n#{sys_info}"
  rescue StandardError => e
    puts "❌ Failed to connect: #{e}"
  end
end

main if $PROGRAM_NAME == __FILE__
```

## 连接到 Temporal Cloud {#connect-to-Temporal-cloud}

您可以使用 [API 密钥](/cloud/api-keys) 或通过 mTLS 连接到 Temporal Cloud。连接至 Temporal
与连接到不安全的云或任何安全的 Temporal 服务相比，云或任何安全的 Temporal 服务需要额外的连接选项
本地开发实例：

- 您的身份验证凭据。
  - 如果您使用 API 密钥，请提供 API 密钥值。
  - 如果您使用 mTLS，请提供 mTLS CA 证书和 mTLS 私钥。
- 您的 _Namespace 和帐户 ID_ 组合，遵循格式“<Namespace_id>.<account_id>”。
- 建议的端点是 gRPC Namespace 端点：`<Namespace>.<account>.tmprl.cloud:7233`。
  该端点适用于所有 Namespace，并自动将流量定向到具有 [高可用性](/cloud/high-availability) 的 Namespace 的活动区域。
  有关端点选项的更多信息，请参阅[访问 Namespaces](/cloud/Namespaces#access-Namespaces)。

您可以在 Namespaces 选项卡上找到 Namespace 和帐户 ID 以及端点：

![左侧为 Namespace 和账户 ID 组合，右侧为区域终端节点](/img/cloud/apikeys/Namespaces-and-regional-endpoints.png)

有关管理和生成 Temporal Cloud 客户端证书的更多信息，请参阅
[如何在Temporal Cloud中管理证书](/cloud/certificates)。

您可以使用环境变量、配置文件或直接在代码中提供这些连接选项。

您可以使用环境变量“Temporal_CONFIG_FILE”指定TOML文件的位置或提供路径
直接在代码中写入文件。如果您不提供配置文件的路径，SDK 将在
默认路径`~/.config/Temporalio/Temporal.toml`。

:::info

配置文件中设置的连接选项的优先级低于环境变量。这意味着如果
您在配置文件和环境变量中设置相同的选项，即环境变量值
覆盖配置文件中设置的选项。

:::

例如，以下 TOML 配置文件定义了一个“staging”配置文件，其中包含必要的连接选项
通过 API 密钥连接到 Temporal Cloud：

```toml
# Cloud profile for Temporal Cloud
[profile.staging]
address = "your-namespace.a1b2c.tmprl.cloud:7233"
namespace = "your-namespace"
api_key = "your-api-key-here"
```

如果您想使用 mTLS 身份验证而不是 API 密钥，请将“api_key”字段替换为您的 mTLS 证书，然后
私钥：

```toml
# Cloud profile for Temporal Cloud
[profile.staging]
address = "your-namespace.a1b2c.tmprl.cloud:7233"
namespace = "your-namespace"
tls_client_cert_data = "your-tls-client-cert-data"
tls_client_key_path = "your-tls-client-key-path"
```

通过配置文件中定义的连接选项，使用
[`Client.connect` 方法](https://ruby.Temporal.io/Temporalio/Client.html#connect-class_method) 创建 Temporal
客户端使用“staging”配置文件，如下所示。加载配置文件后，您还可以通过编程方式覆盖
创建客户端之前的特定连接选项。

```ruby {8,14-16}
require 'temporalio/client'
require 'temporalio/env_config'

def main
  puts "--- Loading 'staging' profile with programmatic overrides ---"

  config_file = File.join(__dir__, 'config.toml')
  profile_name = 'staging'

  puts "The 'staging' profile in config.toml has an incorrect address (localhost:9999)."
  puts "We'll programmatically override it to the correct address."

  # Load the 'staging' profile.
  args, kwargs = Temporalio::EnvConfig::ClientConfig.load_client_connect_options(
    profile: profile_name,
    config_source: Pathname.new(config_file)
  )

  # Override the target host to the correct address.
  # This is the recommended way to override configuration values.
  args[0] = 'localhost:7233'

  puts "\nLoaded '#{profile_name}' profile from #{config_file} with overrides."
  puts "  Address: #{args[0]} (overridden from localhost:9999)"
  puts "  Namespace: #{args[1]}"

  puts "\nAttempting to connect to client..."
  begin
    client = Temporalio::Client.connect(*args, **kwargs)
    puts '✅ Client connected successfully!'
    sys_info = client.workflow_service.get_system_info(Temporalio::Api::WorkflowService::V1::GetSystemInfoRequest.new)
    puts "✅ Successfully verified connection to Temporal server!\n#{sys_info}"
  rescue StandardError => e
    puts "❌ Failed to connect: #{e}"
  end
end

main if $PROGRAM_NAME == __FILE__
```

连接 Temporal Cloud 需要以下环境变量：

- `Temporal_Namespace`：您的 Namespace 和帐户 ID 组合，格式为 `<Namespace_id>.<account_id>`。
- `Temporal_ADDRESS`：Temporal Cloud Namespace 的 gRPC 端点。
- `Temporal_API_KEY`：您的 API 密钥值。如果您使用 API 密钥身份验证，则为必需。
- `Temporal_TLS_CLIENT_CERT_DATA` 或 `Temporal_TLS_CLIENT_CERT_PATH`：您的 mTLS 客户端证书数据或文件路径。
  如果您使用 mTLS 身份验证，则为必需。
- `Temporal_TLS_CLIENT_KEY_DATA` 或 `Temporal_TLS_CLIENT_KEY_PATH`：您的 mTLS 客户端私钥数据或文件路径。
  如果您使用 mTLS 身份验证，则为必需。

在运行应用程序之前，请确保您的环境中存在这些环境变量。

## 启动 Workflow {#start-Workflow}

要启动 Workflow Execution，请提供：

- Task Queue
- Workflow Type
- 输入参数
- Workflow 选项，例如 Workflow Id

要在 Ruby 中启动 Workflow Execution，请在客户端中使用“start_Workflow”或“execute_Workflow”方法。你
必须在
赋予该方法的参数。

```ruby
result = my_client.execute_workflow(
  MyWorkflow, 'some-input',
  id: 'my-workflow-id', task_queue: 'my-task-queue'
)
puts "Result: #{result}"
```

## 获取 Workflow 结果 {#get-Workflow-results}

Workflow Execution 启动后，Workflow Id 和运行 ID 可用于唯一标识它。

您可以阻塞直到结果可用，或者稍后使用句柄检索它。

您还可以在 Workflow 运行时使用查询来访问 Workflow 状态和结果。

在客户端上使用“start_Workflow”或“Workflow_handle”返回Workflow句柄。然后使用`result`方法
等待 Workflow 的结果。

```ruby
handle = my_client.workflow_handle('my-workflow-id')
result = handle.result
puts "Result: #{result}"
```
