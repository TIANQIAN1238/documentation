## 运行 Worker 进程 {#run-a-dev-Worker}

**如何使用 Temporal Python SDK 运行 Worker 进程。**

[Worker Process](/Workers#Worker-process) 是执行 Workflow 函数和 Activity 函数的位置。

- Worker 进程中的每个 [Worker 实体](/Workers#Worker-entity) 必须注册确切的 Workflow Types 和 Activity
  它可以执行的类型。
- 每个 Worker 实体还必须将自己与一个 [Task Queue](/task-queue) 关联。
- 轮询相同 Task Queue 的每个 Worker 实体必须使用相同的 Workflow Type 和 Activity 类型进行注册。

[Worker 实体](/Workers#Worker-entity) 是 Worker 进程中侦听特定任务的组件
队列。

尽管多个 Worker 实体可以位于单个 Worker 进程中，但单个 Worker 实体 Worker 进程可能是
完全足够了。有关更多信息，请参阅 [Worker 调优指南](/develop/Worker-performance)。

Worker 实体包含 Workflow、Worker 和/或 Activity、Worker，这在 Workflow Execution 和
Activity 分别执行。

要开发 Worker，请使用“Worker()”构造函数并将您的客户端、Task Queue、Workflow 和活动添加为
论据。以下代码示例创建一个 Worker，该 Worker 轮询来自 Task Queue 的任务并执行
Workflow。创建 Worker 时，它接受 Workflows 参数中的 Workflow 列表，即 Workflow 中的活动列表
活动参数，或两者。

```python

from temporalio.worker import Worker
# ...
# ...
async def main():
    client = await Client.connect("localhost:7233")
    worker = Worker(
        client,
        task_queue="your-task-queue",
        workflows=[YourWorkflow],
        activities=[your_activity],
    )
    await worker.run()

if __name__ == "__main__":
    asyncio.run(main())
```

### 寄存器类型 {#register-types}

**如何使用 Temporal Python SDK 注册类型。**

所有监听相同 Task Queue 名称的 Worker 必须注册才能处理完全相同的 Workflow 类型和
Activity 类型。

如果 Worker 轮询任务以查找它不知道的 Workflow Type 或 Activity 类型，则该任务将失败。然而，
任务失败不会导致关联的 Workflow Execution 失败。

创建“Worker”时，它接受“Workflows”参数中的 Workflow 列表，即
`activities` 参数，或两者。
