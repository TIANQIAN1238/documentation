[Temporal CLIent](/encyclopedia/Temporal-sdks#Temporal-client) 使您能够与 Temporal 服务进行通信。
与 Temporal 服务的通信使您可以执行诸如启动 Workflow Execution、发送 Signal 和
查询 Workflow Execution、获取 Workflow 结果等。

本页向您展示如何使用 Python SDK 和 Temporal CLIent 执行以下操作：

- [连接到本地开发 Temporal 服务](#connect-to-development-service)
- [连接到 Temporal Cloud](#connect-to-Temporal-cloud)
- [启动 Workflow Execution](#start-Workflow-执行)
- [获取 Workflow 结果](#get-Workflow-结果)

Temporal CLIent 无法在 Workflow 内部初始化和使用。然而，使用
Temporal CLI 位于 Activity 内部，与 Temporal 服务进行通信。

## 连接到开发 Temporal 服务 {#connect-to-development-service}

使用 [`Client.connect`](https://python.Temporal.io/Temporalio.client.Client.html#connect) 创建客户端。连接方式
选项包括 Temporal Server 地址、Namespace 和（可选）TLS 配置。您可以提供这些
直接在代码中使用选项，从 **环境变量** 或 **TOML 配置文件** 加载它们，使用
[`envconfig`](https://python.Temporal.io/Temporalio.envconfig.html) 帮助程序。我们推荐环境变量或
配置文件用于安全、可重复的配置。

当您在本地运行 Temporal 服务时（例如使用
[Temporal CLI 开发服务器](https://docs.Temporal.io/cli/server#start-dev))，所需的选项很少。如果你
不指定主机/端口，大多数连接默认为“127.0.0.1:7233”和“默认”Namespace。

TOML 文件或直接在代码中提供文件的路径。如果不提供配置文件路径，SDK

覆盖配置文件中设置的选项。

例如，以下 TOML 配置文件定义了两个配置文件：“default”和“prod”。每个配置文件都有自己的
连接选项集。

```toml title="config.toml"
# Default profile for local development
[profile.default]
address = "localhost:7233"
namespace = "default"

# Optional: Add custom gRPC headers
[profile.default.grpc_meta]
my-custom-header = "development-value"
trace-id = "dev-trace-123"

# Production profile for Temporal Cloud
[profile.prod]
address = "your-namespace.a1b2c.tmprl.cloud:7233"
namespace = "your-namespace"
api_key = "your-api-key-here"

# TLS configuration for production
[profile.prod.tls]
# TLS auto-enables when TLS config or an API key is present
# disabled = false
client_cert_path = "/etc/temporal/certs/client.pem"
client_key_path  = "/etc/temporal/certs/client.key"

# Custom headers for production
[profile.prod.grpc_meta]
environment     = "production"
service-version = "v1.2.3"
```

您可以使用配置文件中的配置文件创建 Temporal CLIent
`ClientConfig.load_client_connect_config` 函数如下。在此示例中，您加载本地的“默认”配置文件
发展：

```python {23-25}
import asyncio
from pathlib import Path

from temporalio.client import Client
from temporalio.envconfig import ClientConfig


async def main():
    """
    Loads the default profile from the config.toml file in this directory.
    """
    print("--- Loading default profile from config.toml ---")

    # For this sample to be self-contained, we explicitly provide the path to
    # the config.toml file included in this directory.
    # By default though, the config.toml file will be loaded from
    # ~/.config/temporalio/temporal.toml (or the equivalent standard config directory on your OS).
    config_file = Path(__file__).parent / "config.toml"

    # load_client_connect_config is a helper that loads a profile and prepares
    # the config dictionary for Client.connect. By default, it loads the
    # "default" profile.
    connect_config = ClientConfig.load_client_connect_config(
        config_file=str(config_file)
    )

    print(f"Loaded 'default' profile from {config_file}.")
    print(f"  Address: {connect_config.get('target_host')}")
    print(f"  Namespace: {connect_config.get('namespace')}")
    print(f"  gRPC Metadata: {connect_config.get('rpc_metadata')}")

    print("\nAttempting to connect to client...")
    try:
        await Client.connect(**connect_config)  # type: ignore
        print("✅ Client connected successfully!")
    except Exception as e:
        print(f"❌ Failed to connect: {e}")


if __name__ == "__main__":
    asyncio.run(main())
```

## 连接到 Temporal Cloud {#connect-to-Temporal-cloud}

您可以使用 [API 密钥](/cloud/api-keys) 或通过 mTLS 连接到 Temporal Cloud。连接至 Temporal
与连接到不安全的云或任何安全的 Temporal 服务相比，云或任何安全的 Temporal 服务需要额外的连接选项
本地开发实例：

- 您的身份验证凭据。
  - 如果您使用 API 密钥，请提供 API 密钥值。
  - 如果您使用 mTLS，请提供 mTLS CA 证书和 mTLS 私钥。
- 您的 _Namespace 和帐户 ID_ 组合，遵循格式“<Namespace_id>.<account_id>”。
- 建议的端点是 gRPC Namespace 端点：`<Namespace>.<account>.tmprl.cloud:7233`。
  该端点适用于所有 Namespace，并自动将流量定向到具有 [高可用性](/cloud/high-availability) 的 Namespace 的活动区域。
  有关端点选项的更多信息，请参阅[访问 Namespaces](/cloud/Namespaces#access-Namespaces)。

您可以在 Namespaces 选项卡上找到 Namespace 和帐户 ID 以及端点：

![左侧为 Namespace 和账户 ID 组合，右侧为区域终端节点](/img/cloud/apikeys/Namespaces-and-regional-endpoints.png)

有关管理和生成 Temporal Cloud 客户端证书的更多信息，请参阅
[如何在Temporal Cloud中管理证书](/cloud/certificates)。

您可以使用环境变量、配置文件或直接在代码中提供这些连接选项。

创建 Temporal CLIent。有关可在 TOML 文件中设置的所有可用配置选项的列表，请参阅
[环境配置](/references/client-environment-configuration)。

您可以使用环境变量“Temporal_CONFIG_FILE”指定TOML文件的位置或提供路径

:::info

配置文件中设置的连接选项的优先级低于环境变量。这意味着如果
您在配置文件和环境变量中设置相同的选项，即环境变量值
覆盖配置文件中设置的选项。

:::

例如，以下 TOML 配置文件定义了一个“staging”配置文件，其中包含必要的连接选项
通过 API 密钥连接到 Temporal Cloud：

```toml
# Cloud profile for Temporal Cloud
[profile.staging]
address = "your-namespace.a1b2c.tmprl.cloud:7233"
namespace = "your-namespace"
api_key = "your-api-key-here"
```

如果您想使用 mTLS 身份验证而不是 API 密钥，请将“api_key”字段替换为您的 mTLS 证书，然后
私钥：

```toml
# Cloud profile for Temporal Cloud
[profile.staging]
address = "your-namespace.a1b2c.tmprl.cloud:7233"
namespace = "your-namespace"
tls_client_cert_data = "your-tls-client-cert-data"
tls_client_key_path = "your-tls-client-key-path"
```

通过配置文件中定义的连接选项，使用
`Client` 类上的 [`connect` 方法](https://python.Temporal.io/Temporalio.client.Client.html#connect) 创建一个
Temporal CLIent 使用“staging”配置文件，如下所示。加载配置文件后，您还可以通过编程方式
在创建客户端之前覆盖特定的连接选项。

```python {14,23-25}
import asyncio
from pathlib import Path

from temporalio.client import Client
from temporalio.envconfig import ClientConfig


async def main():
    """
    Demonstrates loading a named profile and overriding values programmatically.
    """
    print("--- Loading 'staging' profile with programmatic overrides ---")

    config_file = Path(__file__).parent / "config.toml"
    profile_name = "staging"

    print(
        "The 'staging' profile in config.toml has an incorrect address (localhost:9999)."
    )
    print("We'll programmatically override it to the correct address.")

    # Load the 'staging' profile.
    connect_config = ClientConfig.load_client_connect_config(
        profile=profile_name,
        config_file=str(config_file),
    )

    # Override the target host to the correct address.
    # This is the recommended way to override configuration values.
    connect_config["target_host"] = "localhost:7233"

    print(f"\nLoaded '{profile_name}' profile from {config_file} with overrides.")
    print(
        f"  Address: {connect_config.get('target_host')} (overridden from localhost:9999)"
    )
    print(f"  Namespace: {connect_config.get('namespace')}")

    print("\nAttempting to connect to client...")
    try:
        await Client.connect(**connect_config)  # type: ignore
        print("✅ Client connected successfully!")
    except Exception as e:
        print(f"❌ Failed to connect: {e}")


if __name__ == "__main__":
    asyncio.run(main())
```

## 启动 Workflow Execution {#start-Workflow-execution}

**如何使用 Python SDK 启动 Workflow Execution**

[Workflow Execution](/Workflow-execution) 语义依赖于多个参数，即启动 Workflow Execution
您必须提供将用于任务的 Task Queue（Worker 正在轮询的任务）、Workflow Type、
特定于语言的上下文数据和 Workflow 函数参数。

在下面的示例中，所有 Workflow Execution 均使用 Temporal CLIent 启动。从中生成 Workflow Execution
在另一个 Workflow Execution 中，使用 [Child Workflow](/develop/python/Workflows/child-Workflows) 或外部 Workflow
API。

请参阅 [自定义 Workflow Type](/develop/python/Workflows/basics#Workflow-type) 部分以了解如何自定义
Workflow Type 的名称。

生成 Workflow Execution 的请求会导致 Temporal 服务创建第一个事件
([WorkflowExecutionStarted](/references/events#Workflowexecutionstarted)) 在 Workflow Execution Event History 中。的
Temporal 服务然后创建第一个 Workflow Task，从而产生第一个
[WorkflowTaskScheduled](/references/events#WorkflowtaskScheduled) 事件。

要在 Python 中启动 Workflow Execution，请使用
[`start_Workflow()`](https://python.Temporal.io/Temporalio.client.Client.html#start_Workflow) 或
[`execute_Workflow()`](https://python.Temporal.io/Temporalio.client.Client.html#execute_Workflow) 异步方法
在客户端中。

    print(f"Result: {result}")


if __name__ == "__main__":

### 获取 Workflow Execution 的结果 {#get-Workflow-results}

**如何使用 Python SDK 获取 Workflow Execution 的结果**

如果启动 Workflow Execution 的调用成功，您将获得对 Workflow Execution 运行 ID 的访问权限。

Workflow Id、Run Id 和 Namespace 可用于唯一标识系统中的 Workflow Execution 并获取其
结果。

可以阻止结果的进展（同步执行）或在其他时间点获取结果
（异步执行）。

在 Temporal Platform 中，使用查询作为访问状态和数据的首选方法也是可以接受的。
Workflow Execution 的结果。

使用 [`start_Workflow()`](https://python.Temporal.io/Temporalio.client.Client.html#start_Workflow) 或
[`get_Workflow_handle()`](https://python.Temporal.io/Temporalio.client.Client.html#get_Workflow_handle) 返回一个
Workflow 手柄。然后使用 [`result`](https://python.Temporal.io/Temporalio.client.WorkflowHandle.html#result) 方法
等待 Workflow 的结果。

要通过 ID 获取现有 Workflow 的句柄，您可以使用
[`get_Workflow_handle()`](https://python.Temporal.io/Temporalio.client.Client.html#get_Workflow_handle)，或使用
[`get_Workflow_handle_for()`](https://python.Temporal.io/Temporalio.client.Client.html#get_Workflow_handle_for) 类型
安全。

然后使用 [`describe()`](https://python.Temporal.io/Temporalio.client.WorkflowHandle.html#describe) 获取当前
Workflow 的状态。如果Workflow不存在，则调用失败。
