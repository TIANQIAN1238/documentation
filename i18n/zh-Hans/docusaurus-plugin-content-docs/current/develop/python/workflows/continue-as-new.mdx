## Continue-As-New 是什么？ {＃什么}

[Continue-As-New](/Workflow-execution/Continue-As-New) 让 Workflow Execution 成功关闭并创建新的 Workflow Execution。
当您的 Workflow 变得太长或接近某些缩放限制时，您可以将其视为检查点。

新的Workflow Execution位于同一个[链](/Workflow-execution#Workflow-execution-chain)；它保留了相同的 Workflow Id，但获得了新的运行 ID 和新的 Event History。
它还接收 Workflow 的常用参数。

## 如何使用 Python SDK 进行 Continue-As-New {#how}

首先，设计您的 Workflow 参数，以便您可以在 Continue-As-New 进入下一次 Workflow 运行时传递“当前状态”。
对于 Workflow 的原始调用者，此状态通常设置为“无”。

@dataclass
class ClusterManagerInput:
    state: Optional[ClusterManagerState] = None
    test_continue_as_new: bool = False

@workflow.run
async def run(self, input: ClusterManagerInput) -> ClusterManagerResult:

````
上面代码片段中的测试钩子已在[下面](#how-to-test)中介绍。

在 Workflow 内部，调用相同类型的 [`continue_as_new()`](https://python.Temporal.io/Temporalio.Workflow.html#continue_as_new) 函数。
这会立即停止 Workflow 并启动新的 Workflow。

workflow.continue_as_new(
  ClusterManagerInput(
      state=self.state,
      test_continue_as_new=input.test_continue_as_new,
  )
)
````

### 带消息处理程序的 Workflow 的注意事项 {#with-message-handlers}

如果您使用 Update 或 Signal，请勿从处理程序调用 Continue-As-New。
相反，请等待主 Workflow 中的处理程序完成，然后再运行 `continue_as_new`。
请参阅 [`all_handlers_finished`](message-passing#wait-for-message-handlers) 示例以获取指导。

如果启动 XPROT002X 的调用成功，您将获得对 XPROT002X 运行 ID 的访问权限。

当您的 Workflow 可能会遇到性能下降或 [Event History 限制](/Workflow-execution/event#event-history) 时，请使用 Continue-As-New。

Temporal 跟踪 Workflow 针对这些限制的进度，让您知道何时应该使用 Continue-As-New。
调用 `Workflow.info().is_continue_as_new_suggested()` 检查是否到了时间。

## 如何使用 Python SDK 测试 Continue-As-New {#how-to-test}

测试 Workflow 自然是 Continue-As-New 可能非常耗时且占用资源。
相反，添加一个测试挂钩以在自动化测试中更快地检查 Workflow 的 Continue-As-New 行为。

例如，当“test_continue_as_new == True”时，此示例创建一个名为“self.max_history_length”的仅测试变量，并将其设置为一个较小的值。
Workflow 中的辅助方法在每次考虑使用 Continue-As-New 时都会对其进行检查：

<div className="copycode-notice-container">
  <a href="https://github.com/Temporalio/samples-python/blob/main/message_passing/safe_message_handlers/Workflow.py">
    查看源代码
  </a>{' '}
在应用程序代码的其余部分的上下文中。
</div>

```python
def should_continue_as_new(self) -> bool:
    if workflow.info().is_continue_as_new_suggested():
        return True
    # For testing
    if (
        self.max_history_length
        and workflow.info().get_current_history_length() > self.max_history_length
    ):
        return True
    return False
```
