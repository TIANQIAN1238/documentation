## 开发基本的 Activity {#develop-activities}

**如何使用 Temporal Python SDK 开发基本的 Activity。**

Workflow 所做的主要事情之一是协调活动的执行。 Activity 是正常功能
或旨在执行单个、明确定义的操作（短期或长期运行）的方法执行，例如
XPROT040修复数据库、调用第三方API或转码媒体文件。 Activity可以与外界互动
Temporal Platform 或使用 Temporal CLIent 与 Temporal 服务交互。为了使 Workflow 能够
执行Activity，我们必须定义[Activity Definition](/Activity-definition)。

您可以使用“@Activity.defn”装饰器开发Activity Definition。将函数注册为 Activity
通过装饰器参数使用自定义名称，例如 `@Activity.defn(name="your_Activity")`。

:::note

Temporal Python SDK 支持多种实现 Activity 的方法：

- 异步使用[`asyncio`](https://docs.python.org/3/library/asyncio.html)
- 同步多线程使用
  [`concurrent.futures.ThreadPoolExecutor`](https://docs.python.org/3/library/concurrent.futures.html#threadpoolexecutor)
- 同步多进程使用
  [`concurrent.futures.ProcessPoolExecutor`](https://docs.python.org/3/library/concurrent.futures.html#processpoolexecutor)
  和
  [`multiprocessing.managers.SyncManager`](https://docs.python.org/3/library/multiprocessing.html#multiprocessing.managers.SyncManager)

在 Python 中阻止异步事件循环会将异步程序变成执行的同步程序
连续地，违背了使用“asyncio”的全部目的。这也可能导致潜在的僵局，并且不可预测
导致任务无法执行的行为。调试这些问题可能很困难且耗时，因为
定位阻塞调用的来源可能并不总是显而易见的。

因此，请考虑不要从异步 Activity 内进行阻塞调用，或者使用异步安全库来
执行这些操作。如果必须使用阻塞库，请考虑使用同步 Activity。

:::

### 开发Activity参数{#Activity-参数}

**如何使用 Temporal Python SDK 开发 Activity 参数。**

[Activity Definition](/Activity-定义) 可以使用的参数总数没有明确的限制
支持。但是，最终编码到 gRPC 消息有效负载中的数据的总大小是有限制的。

单个参数的最大大小限制为 2 MB。 gRPC 消息的总大小，其中包括所有
参数，最大限制为 4 MB。

另外，请记住，所有有效负载数据都记录在
[Workflow Execution Event History](/Workflow-execution/event#event-history) 和大型事件历史记录可能会影响 Worker
性能。这是因为整个 Event History 可以通过以下方式传输到 Worker 进程：
[Workflow Task](/tasks#Workflow-任务)。

{/* TODO 链接到 gRPC 限制部分（如果可用）*/}

某些 SDK 要求您传递上下文对象，而其他 SDK 则不需要。当涉及到您的应用程序数据时，即数据
被序列化并编码到有效负载中 - 我们建议您使用单个对象作为包装的参数
传递给活动的应用程序数据。这样您就可以更改传递到 Activity 的数据，而无需
破坏函数或方法签名。

Activity参数是用`@Activity.defn`修饰的函数的函数参数。这些可以是任何数据
Temporal 类型可以转换，包括正确类型注释的数据类。从技术上讲，这可以是多个
参数，但 Temporal 强烈鼓励使用包含所有输入字段的单个数据类参数。

克隆 [samples-python](https://github.com/Temporalio/samples-python) 存储库以继续操作：

在应用程序代码的其余部分的上下文中。

```python
from temporalio import activity
from your_dataobject_dacx import YourParams

# ...
# ...
@activity.defn(name="your_activity")

    return f"{input.greeting}, {input.name}!"
```

### 定义 Activity 返回值 {#Activity-return-values}

**如何使用 Temporal Python SDK 定义 Activity 返回值。**

从 Activity 返回的所有数据都必须是可序列化的。

Activity 返回值受 Temporal 中的有效负载大小限制。默认负载大小限制为 2MB，并且
Event History 事务中任何 gRPC 消息大小的硬限制为 4MB
（[请参阅此处的云限制](https://docs.Temporal.io/cloud/limits#per-message-gRPC-limit)）。请记住，所有返回
值记录在 [Workflow Execution Event History](/Workflow-execution/event#event-history) 中。

Activity 执行可以返回输入和其他 Activity 值。

以下示例定义了一个 Activity，它将一个对象作为输入并返回一个字符串。

### 自定义您的 Activity 类型 {#Activity-type}

**如何定制您的 Activity 类型**

活动有一个称为 Activity 名称的类型。以下示例演示了如何设置自定义
Activity 类型的名称。

您可以使用装饰器参数中的自定义名称来自定义 Activity 名称。例如，
`@Activity.defn(name="your-Activity")`。如果未指定 name 参数，Activity 名称默认为
函数名称。
