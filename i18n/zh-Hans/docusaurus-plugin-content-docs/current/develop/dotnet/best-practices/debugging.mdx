## 调试 {#debug}

此页面显示如何执行以下操作：

- [在开发环境中调试](#debug-in-a-development-environment)
- [在生产环境中调试](#debug-in-a-development-product)

### 在开发环境中调试 {#debug-in-a-development-environment}

在开发 Workflow 时，您可以使用常规开发工具（日志记录和调试器）来查看 Workflow 中发生的情况。
除了日志记录和调试器等常规开发工具外，您还可以使用 [Web UI](/web-ui) 或 [Temporal CLI](/cli) 查看 Workflow 中发生的情况。
Web UI 可让您深入了解 Workflow，从而更轻松地识别问题并实时监控 Workflow 的状态。

### 在生产环境中调试 {#debug-in-a-development-product}

**如何使用 Temporal .NET SDK 在生产环境中进行调试**

您可以使用以下方法调试生产版 Workflow：

- [网页用户界面](/web-ui)
- [Temporal CLI](/cli)
- [重播](/develop/dotnet/best-practices/testing-suite#replay)
- [跟踪](/develop/dotnet/platform/observability#tracing)
- [日志记录](/develop/dotnet/platform/observability#logging)

您可以使用指标和 [Worker 性能指南](/develop/Worker-performance) 来调试和调整 Worker 性能。
有关更多信息，请参阅 [Observability ▶️ Metrics](/develop/dotnet/platform/observability#metrics) 设置 SDK 指标。

使用[云指标](/cloud/metrics/) 或[自托管服务器指标](/self-hosted-guide/product-checklist#scaling-and-metrics) 调试服务器性能。
