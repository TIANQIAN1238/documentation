Temporal 支持使用元数据向 Workflow 和事件添加上下文。 
这有助于用户识别和了解 Workflow 及其操作。

## 添加摘要和详细信息到 Workflows

### 启动 Workflow

启动 Workflow 时，您可以提供静态摘要和详细信息，以帮助在 UI 中识别 Workflow：

```csharp
using Temporalio.Client;

// Create client
var client = await TemporalClient.ConnectAsync(new("localhost:7233"));

// Start a Workflow with static summary and details
var handle = await client.StartWorkflowAsync(
    (YourWorkflow wf) => wf.RunAsync("Workflow input"),
    new WorkflowOptions
    {
        Id = "your-Workflow-id",
        TaskQueue = "your-task-queue",
        StaticSummary = "Order processing for customer #12345",
        StaticDetails = "Processing premium order with expedited shipping"
    });
```

`StaticSummary` 是出现在 Workflow 列表视图中的单行描述，限制为 200 字节。
`StaticDetails` 可以是多行的，并提供 Workflow 详细信息视图中显示的更全面的信息，最大限制为 20K 字节。

输入格式是标准 Markdown，不包括图像、HTML 和脚本。

您还可以使用具有相同参数的“ExecuteWorkflowAsync”方法：

```csharp
var result = await client.ExecuteWorkflowAsync(
    (YourWorkflow wf) => wf.RunAsync("Workflow input"),
    new WorkflowOptions
    {
        Id = "your-Workflow-id",
        TaskQueue = "your-task-queue",
        StaticSummary = "Order processing for customer #12345",
        StaticDetails = "Processing premium order with expedited shipping"
    });
```

### Workflow 内部

在 Workflow 中，您可以获取并设置_当前 Workflow 详细信息_。 
与 Workflow 启动时设置的静态摘要/详细信息不同，该值在 Workflow 的整个生命周期中可以是 Updated。 
当前的 Workflow 详细信息也采用 Markdown 格式（不包括图像、HTML 和脚本），并且可以跨多行。

```csharp
using Temporalio.Workflows;

[Workflow]
public class YourWorkflow
{
    [WorkflowRun]
    public async Task<string> RunAsync(string input)
    {
        // Get the current details
        var currentDetails = Workflow.CurrentDetails;
        Workflow.Logger.LogInformation($"Current details: {currentDetails}");
        
        // Set/update the current details
        Workflow.CurrentDetails = "Updated Workflow details with new status";
        
        return "Workflow completed";
    }
}
```

### 添加摘要到活动和 Timer

当从 Workflow 中启动活动时，您可以将元数据参数“Summary”附加到活动：

```csharp
using Temporalio.Activities;
using Temporalio.Workflows;

[Workflow]
public class YourWorkflow
{
    [WorkflowRun]
    public async Task<string> RunAsync(string input)
    {
        // Execute an activity with a summary
        var result = await Workflow.ExecuteActivityAsync(
            (YourActivities act) => act.YourActivityAsync(input),
            new ActivityOptions
            {
                StartToCloseTimeout = TimeSpan.FromSeconds(10),
                Summary = "Processing user data"
            });
        
        return result;
    }
}
```

同样，您可以将“摘要”附加到 Workflow 内的 Timer：

```csharp
using Temporalio.Workflows;

[Workflow]
public class YourWorkflow
{
    [WorkflowRun]
    public async Task<string> RunAsync(string input)
    {
        // Create a timer with a summary
        await Workflow.DelayWithOptionsAsync(new DelayOptions(TimeSpan.FromMinutes(5))
        {
            Summary = "Waiting for payment confirmation"
        });
        
        return "Timer completed";
    }
}
```

“Summary”的输入格式是字符串，且限制为 200 个字节。

## 在 UI 中查看摘要和详细信息

将摘要和详细信息添加到 Workflow、活动和 Timer 后，您可以在 Temporal Web UI 中查看这些丰富的信息。 
导航到 Workflow 的详细信息页面，查看两个关键位置显示的元数据：

### Workflow 概述部分

- **摘要和详细信息** - 显示启动 Workflow 时设置的静态摘要和静态详细信息
- **当前详细信息** - 显示 Workflow Execution 期间可进行 Updated 的动态详细信息

Workflow 的所有详细信息都支持标准 Markdown 格式（不包括图像、HTML 和脚本），允许您创建丰富的结构化信息显示。

### Event History

Workflow 和 Event History 中的各个事件会显示其关联的摘要（如果有）。

Workflow、Activity 和 Timer 摘要以紫色文本显示在其相应事件旁边，提供即时上下文，无需您展开事件详细信息。 
当您展开事件时，摘要也会在详细视图中突出显示。
