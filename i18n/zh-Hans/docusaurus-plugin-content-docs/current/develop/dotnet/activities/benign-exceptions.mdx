**如何使用 Temporal .NET SDK 将 Activity 错误标记为良性**

当活动抛出预期或不严重的错误时，它们可能会在日志、指标和 OpenTelemetry 跟踪中产生噪音，从而使识别真正问题变得更加困难。
通过将这些错误标记为良性，您可以将它们从可观测性数据中排除，同时仍然在 Workflow 逻辑中处理它们。

要将错误标记为良性，请在引发 [`ApplicationFailureException`](https://dotnet.Temporal.io/api/Temporalio.Exceptions.ApplicationFailureException.html) 时将 `category` 参数设置为 `ApplicationErrorCategory.Benign`。

良性错误：
- 将 Activity 故障日志降级至 DEBUG 级别
- 不发出 Activity 故障指标
- 不要将 OpenTelemetry 失败状态设置为 ERROR

```csharp
using Temporalio.Activities;
using Temporalio.Api.Enums.V1;
using Temporalio.Exceptions;

public class MyActivities
{
    [Activity]
    public async Task<string> MyActivityAsync()
    {
        try
        {
            return await CallExternalServiceAsync();
        }
        catch (Exception e)
        {
            // Mark this error as benign since it's expected
            throw new ApplicationFailureException(
                "Service is down",
                inner: e,
                category: ApplicationErrorCategory.Benign);
        }
    }
}
```

对正常操作中定期发生的 Activity 错误使用良性异常，例如轮询尚未准备就绪的外部服务，或处理将重试的预期瞬态故障。
