## Workflow 超时 {#Workflow-超时}

每个 Workflow 超时控制 Workflow Execution 不同方面的最大持续时间。

Workflow 超时在[启动 Workflow Execution](#Workflow-超时)时设置。

- **[Workflow Execution Timeout](/encyclopedia/Detecting-Workflow-failures#Workflow-execution-timeout)** - 限制单个 Workflow Execution 可以执行的最长时间。
- **[Workflow 运行超时](/encyclopedia/Detecting-Workflow-failures#Workflow-run-timeout)：**限制单次 Workflow 运行可以持续的最长时间。
- **[Workflow Task 超时](/encyclopedia/Detecting-Workflow-failures#Workflow-task-timeout)：** 限制 Worker 可以执行 Workflow Task 的最长时间。

当调用“StartWorkflowAsync”或“ExecuteWorkflowAsync”时，可以在“WorkflowOptions”中设置这些值。

可用的超时有：

- 执行超时
- 运行超时
- 任务超时

```csharp
var result = await client.ExecuteWorkflowAsync(
    (MyWorkflow wf) => wf.RunAsync(),
    new(id: "my-workflow-id", taskQueue: "my-task-queue")
    {
        WorkflowExecutionTimeout = TimeSpan.FromMinutes(5),
    });
```

### 设置 Workflow 重试 {#Workflow-retries}

Retry Policy 可以与超时配合，提供精细控制以优化执行体验。

如果发生故障，请使用 [Retry Policy](/encyclopedia/retry-policies) 重试 Workflow Execution。

默认情况下，Workflow Execution 不会重试，并且仅在某些情况下才应将重试策略与 Workflow Execution 一起使用。

当调用“StartWorkflowAsync”或“ExecuteWorkflowAsync”时，可以在“WorkflowOptions”中设置“RetryPolicy”。

```csharp
var result = await client.ExecuteWorkflowAsync(
    (MyWorkflow wf) => wf.RunAsync(),
    new(id: "my-workflow-id", taskQueue: "my-task-queue")
    {
        RetryPolicy = new() { MaximumInterval = TimeSpan.FromSeconds(10) },
    });
```
